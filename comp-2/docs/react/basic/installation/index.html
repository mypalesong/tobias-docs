설치 및 환경 설정

React 개발 환경을 설정하는 방법을 알아봅니다.

사전 요구사항​

React를 시작하기 전에 다음이 설치되어 있어야 합니다:

  • Node.js (버전 18.0 이상)
  • npm 또는 yarn (패키지 매니저)
  • 코드 에디터 (VS Code 권장)

Node.js 설치 확인​

node --version
npm --version

1. Create React App (CRA)​

가장 간단한 시작 방법은 Create React App을 사용하는 것입니다.

npx create-react-app my-app
cd my-app
npm start

프로젝트 구조​

my-app/
├── node_modules/
├── public/
│   ├── index.html
│   └── favicon.ico
├── src/
│   ├── App.js
│   ├── App.css
│   ├── index.js
│   └── index.css
├── package.json
└── README.md

2. Vite (권장)​

Vite는 더 빠른 개발 서버와 빌드를 제공합니다.

npm create vite@latest my-app -- --template react
cd my-app
npm install
npm run dev

Vite 장점​

  • ⚡ 빠른 시작 속도
  • 🔥 즉각적인 HMR (Hot Module Replacement)
  • 📦 최적화된 빌드

3. Next.js​

서버 사이드 렌더링이 필요한 경우:

npx create-next-app@latest my-app
cd my-app
npm run dev

개발 도구 설치​

VS Code 확장​

  1. ES7+ React/Redux/React-Native snippets

    • React 코드 스니펫 제공

  2. Prettier - Code formatter

    • 코드 자동 포맷팅

  3. ESLint

    • 코드 품질 검사

React Developer Tools​

브라우저 확장 프로그램 설치:

  • Chrome
  • Firefox

첫 번째 React 앱 실행​

1. 개발 서버 시작​

npm start
# 또는
npm run dev

2. 브라우저에서 확인​

기본적으로 http://localhost:3000 (CRA) 또는 http://localhost:5173 (Vite)에서 실행됩니다.

3. 코드 수정해보기​

src/App.js 파일을 열고 수정해보세요:

function App() {
  return (
    <div className="App">
      <h1>안녕하세요, React!</h1>
      <p>첫 번째 React 앱입니다.</p>
    </div>
  );
}

export default App;

저장하면 브라우저가 자동으로 새로고침됩니다.

빌드 및 배포​

프로덕션 빌드​

npm run build

빌드된 파일은 build/ 또는 dist/ 폴더에 생성됩니다.

배포 옵션​

  • Vercel: Next.js 개발사, 무료 호스팅
  • Netlify: 정적 사이트 호스팅
  • GitHub Pages: GitHub를 통한 무료 호스팅
  • AWS S3: Amazon 클라우드 호스팅

패키지 매니저 비교​

기능npmyarnpnpm
속도보통빠름매우 빠름
디스크 공간많음많음적음
학습 곡선쉬움쉬움쉬움

문제 해결​

포트가 이미 사용 중인 경우​

# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# macOS/Linux
lsof -ti:3000 | xargs kill

캐시 문제​

# npm
npm cache clean --force

# yarn
yarn cache clean

node_modules 재설치​

rm -rf node_modules package-lock.json
npm install
