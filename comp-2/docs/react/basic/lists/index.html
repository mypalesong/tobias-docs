리스트와 Key

배열 데이터를 렌더링하고 각 항목을 고유하게 식별하는 방법을 알아봅니다.

리스트 렌더링​

JavaScript의 map() 함수를 사용하여 배열을 컴포넌트 목록으로 변환합니다.

기본 사용법​

function NumberList() {
  const numbers = [1, 2, 3, 4, 5];

  return (
    <ul>
      {numbers.map(number => (
        <li key={number}>{number}</li>
      ))}
    </ul>
  );
}

객체 배열 렌더링​

function UserList() {
  const users = [
    { id: 1, name: '홍길동', email: 'hong@example.com' },
    { id: 2, name: '김철수', email: 'kim@example.com' },
    { id: 3, name: '이영희', email: 'lee@example.com' }
  ];

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </li>
      ))}
    </ul>
  );
}

Key란?​

Key는 React가 어떤 항목이 변경, 추가, 삭제되었는지 식별하는 데 도움을 줍니다.

Key의 중요성​

// ❌ Key가 없으면 경고 발생
{items.map(item => <li>{item}</li>)}

// ✅ Key 제공
{items.map(item => <li key={item.id}>{item.name}</li>)}

Key 선택하기​

1. 고유 ID 사용 (권장)​

function TodoList({ todos }) {
  return (
    <ul>
      {todos.map(todo => (
        <li key={todo.id}>
          {todo.text}
        </li>
      ))}
    </ul>
  );
}

2. Index 사용 (최후의 수단)​

// ⚠️ 항목의 순서가 바뀔 수 있다면 사용하지 마세요
function List({ items }) {
  return (
    <ul>
      {items.map((item, index) => (
        <li key={index}>{item}</li>
      ))}
    </ul>
  );
}

Index를 Key로 사용하면 안 되는 경우:

  • 항목이 추가/삭제될 수 있는 경우
  • 항목의 순서가 바뀔 수 있는 경우
  • 항목이 입력을 포함하는 경우

3. 고유 문자열 생성​

import { nanoid } from 'nanoid';

function addTodo(text) {
  const newTodo = {
    id: nanoid(),  // 고유 ID 생성
    text: text
  };
  // ...
}

Key 규칙​

1. 형제 사이에서 고유해야 함​

// ✅ 서로 다른 리스트에서는 같은 Key 사용 가능
function Page() {
  const users = [{ id: 1, name: '홍길동' }];
  const products = [{ id: 1, name: '상품' }];

  return (
    <div>
      <ul>
        {users.map(user => <li key={user.id}>{user.name}</li>)}
      </ul>
      <ul>
        {products.map(product => <li key={product.id}>{product.name}</li>)}
      </ul>
    </div>
  );
}

2. Key는 변경되지 않아야 함​

// ❌ 잘못된 예 - 렌더링마다 새 Key 생성
{items.map(item => <li key={Math.random()}>{item}</li>)}

// ✅ 올바른 예 - 안정적인 Key 사용
{items.map(item => <li key={item.id}>{item}</li>)}

3. Key는 Props로 전달되지 않음​

function ListItem({ key, value }) {
  console.log(key);  // undefined
  return <li>{value}</li>;
}

// key는 자동으로 전달되지 않음
// 필요하면 다른 prop으로 전달
function List({ items }) {
  return (
    <ul>
      {items.map(item => (
        <ListItem
          key={item.id}
          id={item.id}  // 명시적으로 전달
          value={item.name}
        />
      ))}
    </ul>
  );
}

컴포넌트로 추출​

리스트 항목을 별도의 컴포넌트로 추출할 수 있습니다.

인라인 렌더링​

function TodoList({ todos }) {
  return (
    <ul>
      {todos.map(todo => (
        <li key={todo.id}>
          <input type="checkbox" checked={todo.completed} />
          <span>{todo.text}</span>
        </li>
      ))}
    </ul>
  );
}

컴포넌트 추출​

function TodoItem({ todo }) {
  return (
    <li>
      <input type="checkbox" checked={todo.completed} />
      <span>{todo.text}</span>
    </li>
  );
}

function TodoList({ todos }) {
  return (
    <ul>
      {todos.map(todo => (
        <TodoItem key={todo.id} todo={todo} />
      ))}
    </ul>
  );
}

주의: Key는 map() 호출 내부에서 지정해야 합니다.

필터링과 정렬​

필터링​

function TodoList({ todos }) {
  const activeTodos = todos.filter(todo => !todo.completed);

  return (
    <ul>
      {activeTodos.map(todo => (
        <li key={todo.id}>{todo.text}</li>
      ))}
    </ul>
  );
}

정렬​

function UserList({ users }) {
  const sortedUsers = [...users].sort((a, b) =>
    a.name.localeCompare(b.name)
  );

  return (
    <ul>
      {sortedUsers.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

필터링 + 정렬​

function ProductList({ products, category }) {
  const filteredAndSorted = products
    .filter(product => product.category === category)
    .sort((a, b) => a.price - b.price);

  return (
    <ul>
      {filteredAndSorted.map(product => (
        <li key={product.id}>
          {product.name} - ${product.price}
        </li>
      ))}
    </ul>
  );
}

실전 예제​

Todo 리스트​

function TodoApp() {
  const [todos, setTodos] = useState([
    { id: 1, text: '공부하기', completed: false },
    { id: 2, text: '운동하기', completed: true },
    { id: 3, text: '독서하기', completed: false }
  ]);

  const toggleTodo = (id) => {
    setTodos(todos.map(todo =>
      todo.id === id
        ? { ...todo, completed: !todo.completed }
        : todo
    ));
  };

  const deleteTodo = (id) => {
    setTodos(todos.filter(todo => todo.id !== id));
  };

  return (
    <ul>
      {todos.map(todo => (
        <li
          key={todo.id}
          style={{
            textDecoration: todo.completed ? 'line-through' : 'none'
          }}
        >
          <input
            type="checkbox"
            checked={todo.completed}
            onChange={() => toggleTodo(todo.id)}
          />
          <span>{todo.text}</span>
          <button onClick={() => deleteTodo(todo.id)}>삭제</button>
        </li>
      ))}
    </ul>
  );
}

검색 가능한 리스트​

function SearchableList({ items }) {
  const [query, setQuery] = useState('');

  const filteredItems = items.filter(item =>
    item.name.toLowerCase().includes(query.toLowerCase())
  );

  return (
    <div>
      <input
        type="search"
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="검색..."
      />
      <ul>
        {filteredItems.map(item => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
      {filteredItems.length === 0 && (
        <p>검색 결과가 없습니다.</p>
      )}
    </div>
  );
}

페이지네이션​

function PaginatedList({ items, itemsPerPage = 10 }) {
  const [currentPage, setCurrentPage] = useState(1);

  const totalPages = Math.ceil(items.length / itemsPerPage);
  const startIndex = (currentPage - 1) * itemsPerPage;
  const endIndex = startIndex + itemsPerPage;
  const currentItems = items.slice(startIndex, endIndex);

  return (
    <div>
      <ul>
        {currentItems.map(item => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>

      <div className="pagination">
        <button
          onClick={() => setCurrentPage(prev => prev - 1)}
          disabled={currentPage === 1}
        >
          이전
        </button>

        <span>페이지 {currentPage} / {totalPages}</span>

        <button
          onClick={() => setCurrentPage(prev => prev + 1)}
          disabled={currentPage === totalPages}
        >
          다음
        </button>
      </div>
    </div>
  );
}

성능 최적화​

useMemo로 필터링 최적화​

function FilteredList({ items, filter }) {
  const filteredItems = useMemo(() => {
    return items.filter(item => item.category === filter);
  }, [items, filter]);

  return (
    <ul>
      {filteredItems.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}

React.memo로 컴포넌트 최적화​

const ListItem = React.memo(function ListItem({ item, onDelete }) {
  return (
    <li>
      {item.name}
      <button onClick={() => onDelete(item.id)}>삭제</button>
    </li>
  );
});

function List({ items, onDelete }) {
  return (
    <ul>
      {items.map(item => (
        <ListItem
          key={item.id}
          item={item}
          onDelete={onDelete}
        />
      ))}
    </ul>
  );
}

베스트 프랙티스​

1. 안정적인 Key 사용​

// ❌ 불안정한 Key
{items.map((item, index) => <Item key={index} />)}
{items.map(item => <Item key={Math.random()} />)}

// ✅ 안정적인 Key
{items.map(item => <Item key={item.id} />)}

2. Key를 다른 용도로 사용하지 않기​

// ❌
{items.map(item => <Item key={item.id} key2={item.id} />)}

// ✅
{items.map(item => <Item key={item.id} id={item.id} />)}

3. 복잡한 로직은 추출하기​

// ❌ JSX에 복잡한 로직
return (
  <ul>
    {items
      .filter(/* 복잡한 조건 */)
      .sort(/* 복잡한 정렬 */)
      .map(item => <Item key={item.id} />)}
  </ul>
);

// ✅ 로직 추출
const processedItems = useMemo(() => {
  return items
    .filter(/* 복잡한 조건 */)
    .sort(/* 복잡한 정렬 */);
}, [items]);

return (
  <ul>
    {processedItems.map(item => <Item key={item.id} />)}
  </ul>
);
