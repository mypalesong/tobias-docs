보안 (Security)

DevOps 환경에서의 보안은 개발부터 운영까지 전체 라이프사이클에 걸쳐 통합되어야 합니다. 이 문서에서는 DevSecOps 개념과 실무에서 활용할 수 있는 보안 도구 및 모범 사례를 다룹니다.

1. DevSecOps 개요​

1.1 DevSecOps란?​

DevSecOps는 개발(Development), 보안(Security), 운영(Operations)을 통합하여 소프트웨어 개발 프로세스의 모든 단계에 보안을 내재화하는 접근 방식입니다.

graph LR
    A[Plan] --> B[Code]
    B --> C[Build]
    C --> D[Test]
    D --> E[Release]
    E --> F[Deploy]
    F --> G[Operate]
    G --> H[Monitor]
    H --> A

    I[Security] -.-> A
    I -.-> B
    I -.-> C
    I -.-> D
    I -.-> E
    I -.-> F
    I -.-> G
    I -.-> H
팁

DevSecOps의 핵심 원칙은 "Shift Left"입니다. 보안을 개발 초기 단계부터 고려하여 문제를 조기에 발견하고 수정 비용을 절감할 수 있습니다.

1.2 보안 자동화​

CI/CD 파이프라인에 보안 검사를 자동화하는 예시:

# .gitlab-ci.yml
stages:
  - code-analysis
  - security-scan
  - build
  - test
  - deploy

# SAST (Static Application Security Testing)
sast:
  stage: code-analysis
  image: returntocorp/semgrep
  script:
    - semgrep --config=auto --json --output=semgrep-report.json
  artifacts:
    reports:
      sast: semgrep-report.json

# Secret Detection
secret-detection:
  stage: code-analysis
  image: trufflesecurity/trufflehog:latest
  script:
    - trufflehog filesystem . --json > trufflehog-report.json
  artifacts:
    paths:
      - trufflehog-report.json

# Dependency Scanning
dependency-scanning:
  stage: security-scan
  image: aquasec/trivy:latest
  script:
    - trivy fs --security-checks vuln --format json --output dependency-report.json .
  artifacts:
    reports:
      dependency_scanning: dependency-report.json

# Container Scanning
container-scanning:
  stage: security-scan
  image: aquasec/trivy:latest
  script:
    - trivy image --format json --output container-report.json $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA
  dependencies:
    - build
  artifacts:
    reports:
      container_scanning: container-report.json

2.2 Injection 방어​

SQL Injection 방어 예시:

# ❌ 취약한 코드
def get_user(username):
    query = f"SELECT * FROM users WHERE username = '{username}'"
    return db.execute(query)

# ✅ 안전한 코드 (Parameterized Query)
def get_user(username):
    query = "SELECT * FROM users WHERE username = ?"
    return db.execute(query, (username,))

# ✅ ORM 사용
from sqlalchemy import select
def get_user(username):
    stmt = select(User).where(User.username == username)
    return session.execute(stmt).scalar_one_or_none()
경고

사용자 입력을 직접 쿼리에 삽입하지 마세요. 항상 파라미터화된 쿼리나 ORM을 사용하여 Injection 공격을 방어해야 합니다.

2.3 Access Control 구현​

Role-Based Access Control (RBAC) 예시:

// Express.js middleware
const checkPermission = (requiredRole) => {
  return (req, res, next) => {
    if (!req.user) {
      return res.status(401).json({ error: 'Unauthorized' });
    }

    const userRoles = req.user.roles || [];

    if (!userRoles.includes(requiredRole)) {
      return res.status(403).json({ error: 'Forbidden' });
    }

    next();
  };
};

// 사용 예시
app.delete('/api/users/:id',
  checkPermission('admin'),
  async (req, res) => {
    // 관리자만 사용자 삭제 가능
    await deleteUser(req.params.id);
    res.json({ success: true });
  }
);

app.get('/api/users/me',
  checkPermission('user'),
  async (req, res) => {
    // 인증된 모든 사용자 접근 가능
    res.json(req.user);
  }
);

3. 시크릿 관리​

3.1 HashiCorp Vault​

Vault를 사용한 시크릿 관리:

# Vault 서버 시작 (개발 모드)
vault server -dev

# 환경 변수 설정
export VAULT_ADDR='http://127.0.0.1:8200'
export VAULT_TOKEN='root-token'

# 시크릿 저장
vault kv put secret/myapp/config \
  db_password='super-secret-password' \
  api_key='my-api-key-12345'

# 시크릿 조회
vault kv get secret/myapp/config

# JSON 형식으로 출력
vault kv get -format=json secret/myapp/config | jq -r '.data.data.db_password'

애플리케이션에서 Vault 사용:

import hvac

# Vault 클라이언트 초기화
client = hvac.Client(
    url='http://127.0.0.1:8200',
    token='root-token'
)

# 시크릿 읽기
secret = client.secrets.kv.v2.read_secret_version(
    path='myapp/config',
    mount_point='secret'
)

db_password = secret['data']['data']['db_password']
api_key = secret['data']['data']['api_key']

# 동적 시크릿 생성 (Database)
db_creds = client.secrets.database.generate_credentials(
    name='my-role',
    mount_point='database'
)

username = db_creds['data']['username']
password = db_creds['data']['password']

3.2 Kubernetes Secrets​

Kubernetes에서 시크릿 관리:

# 시크릿 생성
kubectl create secret generic db-credentials \
  --from-literal=username=admin \
  --from-literal=password='P@ssw0rd123'

# 파일에서 시크릿 생성
kubectl create secret generic tls-certs \
  --from-file=tls.crt=./server.crt \
  --from-file=tls.key=./server.key

# YAML로 시크릿 정의 (Base64 인코딩 필요)
echo -n 'admin' | base64  # YWRtaW4=
echo -n 'P@ssw0rd123' | base64  # UEBzc3cwcmQxMjM=
# secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: db-credentials
type: Opaque
data:
  username: YWRtaW4=
  password: UEBzc3cwcmQxMjM=
---
# Pod에서 시크릿 사용
apiVersion: v1
kind: Pod
metadata:
  name: myapp
spec:
  containers:
  - name: app
    image: myapp:latest
    env:
    # 환경 변수로 주입
    - name: DB_USERNAME
      valueFrom:
        secretKeyRef:
          name: db-credentials
          key: username
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: db-credentials
          key: password
    # 볼륨으로 마운트
    volumeMounts:
    - name: tls-certs
      mountPath: "/etc/tls"
      readOnly: true
  volumes:
  - name: tls-certs
    secret:
      secretName: tls-certs
팁

Kubernetes Secrets는 기본적으로 Base64 인코딩만 되어 있어 암호화되지 않습니다. 프로덕션 환경에서는 Sealed Secrets나 External Secrets Operator를 사용하여 외부 시크릿 저장소와 통합하세요.

4.1 Docker 이미지 보안​

보안을 고려한 Dockerfile 작성:

# ✅ 보안 모범 사례를 적용한 Dockerfile

# 1. 최소한의 베이스 이미지 사용
FROM node:18-alpine AS builder

# 2. 보안 업데이트 적용
RUN apk update && apk upgrade && apk add --no-cache dumb-init

# 3. 비-root 사용자 생성
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001

# 4. 작업 디렉토리 설정
WORKDIR /app

# 5. 의존성 파일만 먼저 복사 (레이어 캐싱 최적화)
COPY package*.json ./

# 6. 프로덕션 의존성만 설치
RUN npm ci --only=production && \
    npm cache clean --force

# 7. 애플리케이션 파일 복사
COPY --chown=nodejs:nodejs . .

# 8. 빌드 실행
RUN npm run build

# 멀티스테이지 빌드로 최종 이미지 크기 최소화
FROM node:18-alpine

# 보안 업데이트
RUN apk update && apk upgrade && apk add --no-cache dumb-init

# 비-root 사용자
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001

WORKDIR /app

# 빌드 아티팩트만 복사
COPY --from=builder --chown=nodejs:nodejs /app/dist ./dist
COPY --from=builder --chown=nodejs:nodejs /app/node_modules ./node_modules
COPY --from=builder --chown=nodejs:nodejs /app/package.json ./

# 비-root 사용자로 전환
USER nodejs

# 불필요한 포트 노출 최소화
EXPOSE 3000

# 헬스체크 추가
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:3000/health', (r) => {process.exit(r.statusCode === 200 ? 0 : 1)})"

# dumb-init으로 실행 (PID 1 문제 해결)
ENTRYPOINT ["dumb-init", "--"]
CMD ["node", "dist/server.js"]

4.2 Trivy를 사용한 취약점 스캐닝​

Trivy로 이미지 스캔:

on: [push, pull_request]

    runs-on: ubuntu-latest

      - name: Run Trivy vulnerability scanner

        with:
          image-ref: myapp:${{ github.sha }}
          format: 'sarif'
          output: 'trivy-results.sarif'
          severity: 'CRITICAL,HIGH'

      - name: Upload Trivy results to GitHub Security
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: 'trivy-results.sarif'

      - name: Fail on high vulnerabilities
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: myapp:${{ github.sha }}
          exit-code: '1'
          severity: 'CRITICAL,HIGH'

  name: default-deny-all
  namespace: production

  podSelector: {}
  policyTypes:
  - Ingress
  - Egress

  - Ingress

        matchLabels:
          app: frontend
    ports:
    - protocol: TCP
      port: 8080
---

  namespace: production

      app: backend
  policyTypes:
  - Egress
  egress:

    - podSelector:
        matchLabels:

5.2 TLS/SSL 설정​

Nginx에서 TLS 설정:

# /etc/nginx/conf.d/ssl.conf
server {
    listen 80;
    server_name example.com;

    # HTTP -> HTTPS 리다이렉트
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name example.com;

    # TLS 인증서
    ssl_certificate /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;

    # TLS 프로토콜 (TLS 1.2, 1.3만 허용)
    ssl_protocols TLSv1.2 TLSv1.3;

    # 강력한 암호화 스위트
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384';
    ssl_prefer_server_ciphers off;

    # OCSP Stapling
    ssl_stapling on;
    ssl_stapling_verify on;
    ssl_trusted_certificate /etc/nginx/ssl/chain.pem;

    # Diffie-Hellman 파라미터
    ssl_dhparam /etc/nginx/ssl/dhparam.pem;

    # 보안 헤더
    add_header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';" always;

    # SSL 세션 캐시
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    location / {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Express.js에서 Rate Limiting:

const rateLimit = require('express-rate-limit');
const RedisStore = require('rate-limit-redis');
const redis = require('redis');

const redisClient = redis.createClient({
  host: process.env.REDIS_HOST,
  port: process.env.REDIS_PORT
});

// IP 기반 Rate Limiting
const limiter = rateLimit({
  store: new RedisStore({
    client: redisClient,
    prefix: 'rl:'
  }),
  windowMs: 15 * 60 * 1000, // 15분

  standardHeaders: true,
  legacyHeaders: false

  store: new RedisStore({
    client: redisClient,
    prefix: 'rl:api:'
  }),

    // API 키 티어에 따라 다른 제한
    const tier = req.apiKey?.tier || 'free';
    return tier === 'premium' ? 10000 : 1000;
  },
  keyGenerator: (req) => req.apiKey?.id || req.ip
});

import bcrypt
from typing import Tuple

def hash_password(password: str) -> str:
    """비밀번호를 안전하게 해싱"""
    # bcrypt는 자동으로 salt를 생성
    salt = bcrypt.gensalt(rounds=12)
    hashed = bcrypt.hashpw(password.encode('utf-8'), salt)
    return hashed.decode('utf-8')

def verify_password(password: str, hashed: str) -> bool:
    """비밀번호 검증"""
    return bcrypt.checkpw(
        password.encode('utf-8'),
        hashed.encode('utf-8')
    )

# 사용 예시
def register_user(username: str, password: str):
    # ❌ 평문 저장 - 절대 하지 말 것!
    # db.save(username, password)

    hashed_password = hash_password(password)
    db.save(username, hashed_password)

def login_user(username: str, password: str) -> bool:
    user = db.get(username)
    if not user:
        return False

    return verify_password(password, user.hashed_password)

감사 로깅 구현:

import logging
import json
from datetime import datetime
from typing import Dict, Any

class AuditLogger:
    """규정 준수를 위한 감사 로거"""

    def __init__(self, log_file: str = 'audit.log'):
        self.logger = logging.getLogger('audit')
        self.logger.setLevel(logging.INFO)

        handler = logging.FileHandler(log_file)
        handler.setFormatter(logging.Formatter('%(message)s'))
        self.logger.addHandler(handler)

    def log_event(
        self,
        event_type: str,
        user_id: str,
        action: str,
        resource: str,
        status: str,
        metadata: Dict[str, Any] = None
    ):
        """감사 이벤트 로깅"""
        audit_entry = {
            'timestamp': datetime.utcnow().isoformat(),
            'event_type': event_type,
            'user_id': user_id,
            'action': action,
            'resource': resource,
            'status': status,
            'metadata': metadata or {}
        }

        self.logger.info(json.dumps(audit_entry))

    def log_access(self, user_id: str, resource: str, granted: bool):
        """접근 시도 로깅"""
        self.log_event(
            event_type='access_attempt',
            user_id=user_id,
            action='access',
            resource=resource,
            status='granted' if granted else 'denied'
        )

    def log_data_change(self, user_id: str, resource: str, operation: str):
        """데이터 변경 로깅"""
        self.log_event(
            event_type='data_change',
            user_id=user_id,
            action=operation,
            resource=resource,
            status='success'
        )

# 사용 예시

def delete_user(admin_id: str, user_id: str):
    # 권한 확인
    if not has_admin_permission(admin_id):
        audit.log_access(admin_id, f'/users/{user_id}', granted=False)
        raise PermissionError('Admin permission required')

    # 사용자 삭제
    db.delete_user(user_id)

    # 감사 로그 기록
    audit.log_data_change(
        user_id=admin_id,
        resource=f'/users/{user_id}',
        operation='delete'
    )
