Infrastructure as Code (IaC)

Infrastructure as Code는 인프라를 코드로 관리하고 프로비저닝하는 현대적인 DevOps 실천 방법입니다.

1. IaC 개념과 이점​

1.1 Infrastructure as Code란?​

Infrastructure as Code(IaC)는 인프라 구성을 코드 형태로 정의하고 관리하는 방법론입니다. 수동 프로세스 대신 코드를 통해 인프라를 자동으로 프로비저닝하고 관리합니다.

핵심 특징:

  • 선언적 방식: 원하는 최종 상태를 정의
  • 명령적 방식: 목표 달성을 위한 단계별 명령 정의
  • 버전 관리: Git을 통한 인프라 변경 이력 추적
  • 재사용성: 모듈화를 통한 코드 재사용

1.2 IaC의 이점​

일관성과 표준화

# 동일한 환경을 반복적으로 생성
environments:
  - development
  - staging
  - production
# 모든 환경이 동일한 구성 템플릿 사용

주요 이점:

  1. 반복 가능성: 동일한 환경을 여러 번 생성
  2. 속도: 수동 작업 대비 빠른 프로비저닝
  3. 비용 절감: 자동화를 통한 인건비 감소
  4. 위험 감소: 수동 오류 제거
  5. 문서화: 코드 자체가 문서 역할
팁

IaC를 도입하면 인프라 프로비저닝 시간을 몇 시간에서 몇 분으로 단축할 수 있습니다.

2. Terraform 기초​

2.1 Terraform 소개​

Terraform은 HashiCorp에서 개발한 오픈소스 IaC 도구로, 선언적 구성 파일을 사용하여 인프라를 관리합니다.

Terraform 설치:

# Windows (Chocolatey)
choco install terraform

# macOS (Homebrew)
brew install terraform

# Linux (Ubuntu/Debian)
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform

# 설치 확인
terraform version

2.2 Terraform 기본 구조​

기본 구성 파일 (main.tf):

# Provider 설정
terraform {
  required_version = ">= 1.0"
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = var.aws_region

  default_tags {
    tags = {
      Environment = var.environment
      ManagedBy   = "Terraform"
    }
  }
}

# 변수 정의
variable "aws_region" {
  description = "AWS 리전"
  type        = string
  default     = "ap-northeast-2"
}

variable "environment" {
  description = "환경 이름"
  type        = string
  default     = "development"
}

# 리소스 정의
resource "aws_vpc" "main" {
  cidr_block           = "10.0.0.0/16"
  enable_dns_hostnames = true
  enable_dns_support   = true

  tags = {
    Name = "${var.environment}-vpc"
  }
}

resource "aws_subnet" "public" {
  count             = 2
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.${count.index}.0/24"
  availability_zone = data.aws_availability_zones.available.names[count.index]

  tags = {
    Name = "${var.environment}-public-subnet-${count.index + 1}"
  }
}

# 데이터 소스
data "aws_availability_zones" "available" {
  state = "available"
}

# 출력
output "vpc_id" {
  description = "VPC ID"
  value       = aws_vpc.main.id
}

output "public_subnet_ids" {
  description = "공용 서브넷 ID 목록"
  value       = aws_subnet.public[*].id
}

2.3 Terraform 워크플로우​

기본 명령어:

# 1. 초기화 - Provider 및 모듈 다운로드
terraform init

# 2. 형식 검증 및 자동 포맷팅
terraform fmt

# 3. 구성 유효성 검증
terraform validate

# 4. 실행 계획 확인
terraform plan

# 5. 변경 사항 적용
terraform apply

# 6. 리소스 정보 조회
terraform show

# 7. 출력 값 확인
terraform output

# 8. 리소스 제거
terraform destroy

# 9. 상태 확인
terraform state list
terraform state show <resource_name>
경고

terraform apply 실행 전 반드시 terraform plan으로 변경 사항을 확인하세요. 프로덕션 환경에서는 자동 승인(-auto-approve)을 사용하지 마세요.

3. AWS 프로비저닝​

3.1 EC2 인스턴스 생성​

완전한 EC2 예제:

# variables.tf
variable "instance_type" {
  description = "EC2 인스턴스 타입"
  type        = string
  default     = "t3.micro"
}

variable "key_name" {
  description = "SSH 키 페어 이름"
  type        = string
}

# main.tf
# AMI 조회
data "aws_ami" "ubuntu" {
  most_recent = true
  owners      = ["099720109477"] # Canonical

  filter {
    name   = "name"
    values = ["ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64-server-*"]
  }

  filter {
    name   = "virtualization-type"
    values = ["hvm"]
  }
}

# 보안 그룹
resource "aws_security_group" "web" {
  name        = "${var.environment}-web-sg"
  description = "웹 서버 보안 그룹"
  vpc_id      = aws_vpc.main.id

  ingress {
    description = "HTTP"
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    description = "HTTPS"
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    description = "SSH"
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"] # 실제 환경에서는 특정 IP로 제한
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = {
    Name = "${var.environment}-web-sg"
  }
}

# EC2 인스턴스
resource "aws_instance" "web" {
  ami           = data.aws_ami.ubuntu.id
  instance_type = var.instance_type
  subnet_id     = aws_subnet.public[0].id

  vpc_security_group_ids = [aws_security_group.web.id]
  key_name              = var.key_name

  user_data = <<-EOF
              #!/bin/bash
              apt-get update
              apt-get install -y nginx
              systemctl start nginx
              systemctl enable nginx
              echo "<h1>Hello from Terraform!</h1>" > /var/www/html/index.html
              EOF

  root_block_device {
    volume_size = 20
    volume_type = "gp3"
    encrypted   = true
  }

  metadata_options {
    http_endpoint = "enabled"
    http_tokens   = "required" # IMDSv2 강제
  }

  tags = {
    Name = "${var.environment}-web-server"
  }
}

# Elastic IP
resource "aws_eip" "web" {
  instance = aws_instance.web.id
  domain   = "vpc"

  tags = {
    Name = "${var.environment}-web-eip"
  }
}

output "instance_public_ip" {
  description = "EC2 인스턴스 공인 IP"
  value       = aws_eip.web.public_ip
}

3.2 S3 및 CloudFront 배포​

정적 웹사이트 호스팅:

# S3 버킷
resource "aws_s3_bucket" "website" {
  bucket = "${var.environment}-my-website-bucket"

  tags = {
    Name = "${var.environment}-website"
  }
}

# 버킷 버저닝
resource "aws_s3_bucket_versioning" "website" {
  bucket = aws_s3_bucket.website.id

  versioning_configuration {
    status = "Enabled"
  }
}

# 버킷 암호화
resource "aws_s3_bucket_server_side_encryption_configuration" "website" {
  bucket = aws_s3_bucket.website.id

  rule {
    apply_server_side_encryption_by_default {
      sse_algorithm = "AES256"
    }
  }
}

# CloudFront OAI
resource "aws_cloudfront_origin_access_identity" "website" {
  comment = "${var.environment} website OAI"
}

# S3 버킷 정책
resource "aws_s3_bucket_policy" "website" {
  bucket = aws_s3_bucket.website.id

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Sid    = "AllowCloudFrontOAI"
        Effect = "Allow"
        Principal = {
          AWS = aws_cloudfront_origin_access_identity.website.iam_arn
        }
        Action   = "s3:GetObject"
        Resource = "${aws_s3_bucket.website.arn}/*"
      }
    ]
  })
}

# CloudFront 배포
resource "aws_cloudfront_distribution" "website" {
  enabled             = true
  is_ipv6_enabled     = true
  default_root_object = "index.html"
  price_class         = "PriceClass_200"

  origin {
    domain_name = aws_s3_bucket.website.bucket_regional_domain_name
    origin_id   = "S3-${aws_s3_bucket.website.id}"

    s3_origin_config {
      origin_access_identity = aws_cloudfront_origin_access_identity.website.cloudfront_access_identity_path
    }
  }

  default_cache_behavior {
    allowed_methods  = ["GET", "HEAD", "OPTIONS"]
    cached_methods   = ["GET", "HEAD"]
    target_origin_id = "S3-${aws_s3_bucket.website.id}"

    forwarded_values {
      query_string = false
      cookies {
        forward = "none"
      }
    }

    viewer_protocol_policy = "redirect-to-https"
    min_ttl                = 0
    default_ttl            = 3600
    max_ttl                = 86400
    compress               = true
  }

  restrictions {
    geo_restriction {
      restriction_type = "none"
    }
  }

  viewer_certificate {
    cloudfront_default_certificate = true
  }

  tags = {
    Name = "${var.environment}-website-cdn"
  }
}

output "cloudfront_domain_name" {
  description = "CloudFront 배포 도메인"
  value       = aws_cloudfront_distribution.website.domain_name
}

  backup_window           = "03:00-04:00"
  maintenance_window      = "mon:04:00-mon:05:00"

  skip_final_snapshot       = var.environment != "production"
  final_snapshot_identifier = var.environment == "production" ? "${var.environment}-final-snapshot-${formatdate("YYYY-MM-DD-hhmm", timestamp())}" : null

  enabled_cloudwatch_logs_exports = ["postgresql", "upgrade"]

  tags = {
    Name = "${var.environment}-postgres-db"
  }
}

output "db_endpoint" {
  description = "데이터베이스 엔드포인트"
  value       = aws_db_instance.main.endpoint
  sensitive   = true

  source_ranges = ["0.0.0.0/0"]

  name    = "${var.environment}-allow-ssh"

  source_ranges = ["0.0.0.0/0"] # 실제 환경에서는 제한 필요
  target_tags   = ["web-server"]

# 고정 IP
resource "google_compute_address" "static" {
  name   = "${var.environment}-web-ip"

resource "google_compute_instance" "web" {

  tags = ["web-server"]

    }
  }

    subnetwork = google_compute_subnetwork.subnet.id

    access_config {
      nat_ip = google_compute_address.static.address
    }
  }

  metadata_startup_script = <<-EOF
    #!/bin/bash
    apt-get update
    apt-get install -y nginx
    systemctl start nginx
    echo "<h1>Hello from GCP!</h1>" > /var/www/html/index.html

  service_account {
    scopes = ["cloud-platform"]
  }
}

output "instance_ip" {
  description = "인스턴스 공인 IP"
  value       = google_compute_address.static.address
}

6.3 Playbook 작성​

웹 서버 구성 (playbooks/webserver.yml):

---
- name: 웹 서버 구성
  hosts: web
  become: yes
  vars:
    nginx_port: 80
    app_user: www-data
    document_root: /var/www/html

  tasks:
    - name: 시스템 업데이트
      apt:
        update_cache: yes
        cache_valid_time: 3600

    - name: Nginx 설치
      apt:
        name: nginx
        state: present

    - name: UFW 방화벽 설정
      ufw:
        rule: allow
        port: "{{ item }}"
        proto: tcp
      loop:
        - "22"
        - "80"
        - "443"

    - name: Nginx 설정 파일 배포
      template:
        src: templates/nginx.conf.j2
        dest: /etc/nginx/sites-available/default
        owner: root
        group: root
        mode: '0644'
      notify: Reload Nginx

    - name: 웹사이트 디렉토리 생성
      file:
        path: "{{ document_root }}"
        state: directory
        owner: "{{ app_user }}"
        group: "{{ app_user }}"
        mode: '0755'

    - name: index.html 배포
      copy:
        src: files/index.html
        dest: "{{ document_root }}/index.html"
        owner: "{{ app_user }}"
        group: "{{ app_user }}"
        mode: '0644'

    - name: Nginx 서비스 시작 및 활성화
      systemd:
        name: nginx
        state: started
        enabled: yes

    - name: 서비스 상태 확인
      systemd:
        name: nginx
        state: started
      register: nginx_status

    - name: 서비스 상태 출력
      debug:
        msg: "Nginx is {{ nginx_status.status.ActiveState }}"

  handlers:
    - name: Reload Nginx
      systemd:
        name: nginx
        state: reloaded

템플릿 파일 (templates/nginx.conf.j2):

server {
    listen {{ nginx_port }} default_server;
    listen [::]:{{ nginx_port }} default_server;

    root {{ document_root }};

    server_name {{ ansible_fqdn }};

    location / {
        try_files $uri $uri/ =404;
    }

    # 보안 헤더
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # 로그 설정
    access_log /var/log/nginx/{{ ansible_hostname }}_access.log;
    error_log /var/log/nginx/{{ ansible_hostname }}_error.log;
}

import * as pulumi from "@pulumi/pulumi";
import * as aws from "@pulumi/aws";

// 구성 값 가져오기
const config = new pulumi.Config();
const environment = config.get("environment") || "dev";
const instanceType = config.get("instanceType") || "t3.micro";

// VPC 생성
const vpc = new aws.ec2.Vpc(`${environment}-vpc`, {
    cidrBlock: "10.0.0.0/16",
    enableDnsHostnames: true,
    enableDnsSupport: true,
    tags: {
        Name: `${environment}-vpc`,
        Environment: environment,
    },
});

// 인터넷 게이트웨이
const igw = new aws.ec2.InternetGateway(`${environment}-igw`, {
    vpcId: vpc.id,
    tags: {
        Name: `${environment}-igw`,
    },
});

// 공용 서브넷
const publicSubnet = new aws.ec2.Subnet(`${environment}-public-subnet`, {
    vpcId: vpc.id,
    cidrBlock: "10.0.1.0/24",
    availabilityZone: "ap-northeast-2a",
    mapPublicIpOnLaunch: true,
    tags: {
        Name: `${environment}-public-subnet`,
    },
});

// 라우팅 테이블
const routeTable = new aws.ec2.RouteTable(`${environment}-rt`, {
    vpcId: vpc.id,
    routes: [
        {
            cidrBlock: "0.0.0.0/0",
            gatewayId: igw.id,
        },
    ],
    tags: {
        Name: `${environment}-public-rt`,
    },
});

// 라우팅 테이블 연결
const routeTableAssociation = new aws.ec2.RouteTableAssociation(
    `${environment}-rta`,
    {
        subnetId: publicSubnet.id,
        routeTableId: routeTable.id,
    }
);

// 보안 그룹
const securityGroup = new aws.ec2.SecurityGroup(`${environment}-web-sg`, {
    vpcId: vpc.id,
    description: "웹 서버 보안 그룹",
    ingress: [
        {
            protocol: "tcp",
            fromPort: 80,
            toPort: 80,
            cidrBlocks: ["0.0.0.0/0"],
            description: "HTTP",
        },
        {
            protocol: "tcp",
            fromPort: 443,
            toPort: 443,
            cidrBlocks: ["0.0.0.0/0"],
            description: "HTTPS",
        },
        {
            protocol: "tcp",
            fromPort: 22,
            toPort: 22,
            cidrBlocks: ["0.0.0.0/0"],
            description: "SSH",
        },
    ],
    egress: [
        {
            protocol: "-1",
            fromPort: 0,
            toPort: 0,
            cidrBlocks: ["0.0.0.0/0"],
        },
    ],
    tags: {
        Name: `${environment}-web-sg`,
    },
});

// AMI 조회
const ami = aws.ec2.getAmi({
    mostRecent: true,
    owners: ["099720109477"], // Canonical
    filters: [
        {
            name: "name",
            values: ["ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64-server-*"],
        },
        {
            name: "virtualization-type",
            values: ["hvm"],
        },
    ],
});

// 사용자 데이터 스크립트
const userData = `#!/bin/bash
apt-get update
apt-get install -y nginx
systemctl start nginx
systemctl enable nginx
echo "<h1>Hello from Pulumi on ${environment}!</h1>" > /var/www/html/index.html
`;

// EC2 인스턴스
const server = new aws.ec2.Instance(`${environment}-web-server`, {
    instanceType: instanceType,
    ami: ami.then(ami => ami.id),
    subnetId: publicSubnet.id,
    vpcSecurityGroupIds: [securityGroup.id],
    userData: userData,
    tags: {
        Name: `${environment}-web-server`,
    },
});

// Elastic IP
const eip = new aws.ec2.Eip(`${environment}-eip`, {
    instance: server.id,
    domain: "vpc",
    tags: {
        Name: `${environment}-web-eip`,
    },
});

// 출력
export const vpcId = vpc.id;
export const publicIp = eip.publicIp;
export const serverUrl = pulumi.interpolate`http://${eip.publicIp}`;

environment = config.get("environment") or "dev"

# VPC 네트워크
network = gcp.compute.Network(
    f"{environment}-network",
    auto_create_subnetworks=False,
    description=f"{environment} VPC network",
)

# 서브넷
subnet = gcp.compute.Subnetwork(
    f"{environment}-subnet",
    ip_cidr_range="10.0.1.0/24",
    region=region,
    network=network.id,
)

# 방화벽 규칙
firewall = gcp.compute.Firewall(
    f"{environment}-firewall",
    network=network.self_link,
    allows=[
        gcp.compute.FirewallAllowArgs(
            protocol="tcp",
            ports=["22", "80", "443"],
        )
    ],
    source_ranges=["0.0.0.0/0"],
    target_tags=["web-server"],
)

# 고정 IP
static_ip = gcp.compute.Address(
    f"{environment}-ip",
    region=region,
)

# 시작 스크립트
startup_script = """#!/bin/bash
apt-get update
apt-get install -y nginx
systemctl start nginx
echo '<h1>Hello from Pulumi on GCP!</h1>' > /var/www/html/index.html
"""

# 인스턴스
instance = gcp.compute.Instance(
    f"{environment}-instance",
    machine_type="e2-micro",
    zone=f"{region}-a",
    tags=["web-server"],
    boot_disk=gcp.compute.InstanceBootDiskArgs(
        initialize_params=gcp.compute.InstanceBootDiskInitializeParamsArgs(
            image="ubuntu-os-cloud/ubuntu-2204-lts",
            size=20,
        ),
    ),
    network_interfaces=[
        gcp.compute.InstanceNetworkInterfaceArgs(
            network=network.id,
            subnetwork=subnet.id,
            access_configs=[
                gcp.compute.InstanceNetworkInterfaceAccessConfigArgs(
                    nat_ip=static_ip.address,
                )
            ],
        )
    ],
    metadata_startup_script=startup_script,
    service_account=gcp.compute.InstanceServiceAccountArgs(
        scopes=["https://www.googleapis.com/auth/cloud-platform"],
    ),
)

# 출력
pulumi.export("instance_name", instance.name)
pulumi.export("instance_ip", static_ip.address)
pulumi.export("instance_url", static_ip.address.apply(lambda ip: f"http://{ip}"))

9. 상태 관리​

9.1 Terraform 상태 파일​

Terraform 상태 파일(terraform.tfstate)은 관리되는 인프라의 현재 상태를 추적합니다.

로컬 상태 백업:

# 상태 백업
cp terraform.tfstate terraform.tfstate.backup

# 상태 목록 보기
terraform state list

# 특정 리소스 상태 보기
terraform state show aws_instance.web

# 상태에서 리소스 제거 (실제 리소스는 유지)
terraform state rm aws_instance.web

# 리소스 이동/이름 변경
terraform state mv aws_instance.web aws_instance.web_server

9.2 원격 상태 관리​

S3 백엔드 설정:

terraform {
  backend "s3" {
    bucket         = "my-terraform-state-bucket"
    key            = "production/terraform.tfstate"
    region         = "ap-northeast-2"
    encrypt        = true
    dynamodb_table = "terraform-state-lock"

    # 상태 파일 버저닝 활성화
    versioning = true
  }
}

# DynamoDB 테이블 생성 (상태 잠금용)
resource "aws_dynamodb_table" "terraform_state_lock" {
  name           = "terraform-state-lock"
  billing_mode   = "PAY_PER_REQUEST"
  hash_key       = "LockID"

  attribute {
    name = "LockID"
    type = "S"
  }

  tags = {
    Name = "Terraform State Lock Table"
  }
}

Terraform Cloud 백엔드:

terraform {
  cloud {
    organization = "my-organization"

    workspaces {
      name = "production-infrastructure"
    }
  }
}

GCS 백엔드 (Google Cloud Storage):

terraform {
  backend "gcs" {
    bucket  = "my-terraform-state"
    prefix  = "production/state"
  }
}

Azure Blob Storage 백엔드:

terraform {
  backend "azurerm" {
    resource_group_name  = "terraform-state-rg"
    storage_account_name = "terraformstate"
    container_name       = "tfstate"
    key                  = "production.terraform.tfstate"
  }
}
팁

원격 상태 백엔드를 사용하면 팀 협업이 가능하고, 상태 파일이 안전하게 암호화되며, 동시 실행을 방지하는 잠금 기능을 사용할 수 있습니다.

9.3 상태 잠금​

동시 실행 방지:

# Terraform은 자동으로 상태를 잠급니다
terraform apply

# 강제 잠금 해제 (주의해서 사용)
terraform force-unlock <lock-id>

10. 모듈화​

10.1 Terraform 모듈​

모듈 구조:

modules/
└── vpc/
    ├── main.tf
    ├── variables.tf
    ├── outputs.tf
    └── README.md

모듈 정의 (modules/vpc/main.tf):

variable "vpc_cidr" {
  description = "VPC CIDR 블록"
  type        = string
}

variable "environment" {
  description = "환경 이름"
  type        = string
}

variable "availability_zones" {
  description = "가용 영역 목록"
  type        = list(string)
}

resource "aws_vpc" "main" {
  cidr_block           = var.vpc_cidr
  enable_dns_hostnames = true
  enable_dns_support   = true

  tags = {
    Name        = "${var.environment}-vpc"
    Environment = var.environment
  }
}

resource "aws_subnet" "public" {
  count             = length(var.availability_zones)
  vpc_id            = aws_vpc.main.id
  cidr_block        = cidrsubnet(var.vpc_cidr, 8, count.index)
  availability_zone = var.availability_zones[count.index]

  tags = {
    Name = "${var.environment}-public-${count.index + 1}"
  }
}

resource "aws_internet_gateway" "main" {
  vpc_id = aws_vpc.main.id

  tags = {
    Name = "${var.environment}-igw"
  }
}

output "vpc_id" {
  description = "VPC ID"
  value       = aws_vpc.main.id
}

output "public_subnet_ids" {
  description = "공용 서브넷 ID 목록"
  value       = aws_subnet.public[*].id
}

모듈 사용:

module "vpc" {
  source = "./modules/vpc"

  vpc_cidr           = "10.0.0.0/16"
  environment        = "production"
  availability_zones = ["ap-northeast-2a", "ap-northeast-2b", "ap-northeast-2c"]
}

# 모듈 출력 참조
resource "aws_instance" "web" {
  subnet_id = module.vpc.public_subnet_ids[0]
  # ...
}

output "vpc_id" {
  value = module.vpc.vpc_id
}

10.2 재사용 가능한 모듈 설계​

입력 변수 검증:

variable "instance_type" {
  description = "EC2 인스턴스 타입"
  type        = string

  validation {
    condition     = can(regex("^t3\\.", var.instance_type))
    error_message = "인스턴스 타입은 t3 계열이어야 합니다."
  }
}

variable "environment" {
  description = "환경 이름"
  type        = string

  validation {
    condition     = contains(["dev", "staging", "production"], var.environment)
    error_message = "환경은 dev, staging, production 중 하나여야 합니다."
  }
}

동적 블록:

variable "ingress_rules" {
  description = "인그레스 규칙 목록"
  type = list(object({
    from_port   = number
    to_port     = number
    protocol    = string
    cidr_blocks = list(string)
    description = string
  }))
}

resource "aws_security_group" "main" {
  name   = "${var.environment}-sg"
  vpc_id = var.vpc_id

  dynamic "ingress" {
    for_each = var.ingress_rules
    content {
      from_port   = ingress.value.from_port
      to_port     = ingress.value.to_port
      protocol    = ingress.value.protocol
      cidr_blocks = ingress.value.cidr_blocks
      description = ingress.value.description
    }
  }
}

  secret_id = "prod/db/password"

  username = "admin"
  password = jsondecode(data.aws_secretsmanager_secret_version.db_password.secret_string)["password"]

resource "random_password" "db_password" {
  length  = 32
  special = true

  secret_id     = aws_secretsmanager_secret.db_password.id
  secret_string = random_password.db_password.result
}

출력 값 민감도 설정:

output "db_password" {

11.7 CI/CD 통합​

GitHub Actions 예제 (.github/workflows/terraform.yml):

name: Terraform

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  terraform:
    name: Terraform
    runs-on: ubuntu-latest

    steps:
      - name: 코드 체크아웃
        uses: actions/checkout@v3

      - name: Terraform 설정
        uses: hashicorp/setup-terraform@v2
        with:
          terraform_version: 1.5.0

      - name: Terraform 초기화
        run: terraform init

      - name: Terraform 형식 검사
        run: terraform fmt -check

      - name: Terraform 검증
        run: terraform validate

      - name: Terraform 계획
        run: terraform plan -no-color
        env:
          AWS_ACCESS_KEY_ID: ${{ secrets.AWS_ACCESS_KEY_ID }}
          AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_ACCESS_KEY }}

      - name: Terraform 적용
        if: github.ref == 'refs/heads/main' && github.event_name == 'push'
        run: terraform apply -auto-approve
        env:
          AWS_ACCESS_KEY_ID: ${{ secrets.AWS_ACCESS_KEY_ID }}
          AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_ACCESS_KEY }}

GitLab CI 예제 (.gitlab-ci.yml):

image:
  name: hashicorp/terraform:1.5
  entrypoint: [""]

stages:
  - validate
  - plan
  - apply

before_script:
  - terraform init

validate:
  stage: validate
  script:
    - terraform fmt -check
    - terraform validate

plan:
  stage: plan
  script:
    - terraform plan -out=tfplan
  artifacts:
    paths:
      - tfplan

apply:
  stage: apply
  script:
    - terraform apply -auto-approve tfplan
  when: manual
  only:
    - main

12. 요약​

Infrastructure as Code는 현대 클라우드 인프라 관리의 핵심입니다:

  1. 도구 선택: Terraform(멀티 클라우드), CloudFormation(AWS 네이티브), Pulumi(범용 언어)
  2. 상태 관리: 원격 백엔드를 사용하여 팀 협업 지원
  3. 모듈화: 재사용 가능한 모듈로 코드 중복 제거
  4. 보안: 민감한 데이터를 시크릿 관리 서비스에 저장
  5. 버전 관리: Git을 통한 인프라 변경 이력 추적
  6. 자동화: CI/CD 파이프라인에 IaC 통합
  7. 테스트: 프로덕션 배포 전 검증 및 테스트
  8. 문서화: 코드 자체가 문서이지만 README와 주석 추가
경고

IaC를 처음 도입할 때는 작은 프로젝트나 개발 환경부터 시작하여 팀이 익숙해진 후 프로덕션에 적용하세요.
