APScheduler 가이드

Python에서 작업을 스케줄링하는 강력하고 유연한 라이브러리 APScheduler를 배워보세요.

APScheduler란?​

**APScheduler (Advanced Python Scheduler)**는 Python 애플리케이션에서 작업을 나중에 실행하거나 주기적으로 실행하기 위한 작업 스케줄링 라이브러리입니다.

✅ 다양한 스케줄러 - Background, Blocking, AsyncIO, Tornado 등
✅ 유연한 트리거 - Date, Interval, Cron 기반 스케줄링
✅ 영구 저장 - SQLAlchemy, MongoDB, Redis 지원
✅ 여러 실행자 - Thread, Process, AsyncIO 기반 실행
✅ 간단한 API - 직관적인 사용법
✅ 이벤트 시스템 - Job 실행 모니터링

APScheduler vs 다른 도구​

vs Celery Beat​

**Celery Beat**
- Celery 필수
- 메시지 브로커 필요 (Redis, RabbitMQ)
- 분산 작업에 최적화
- 복잡한 설정

**APScheduler**
- 독립 실행 가능
- 메시지 브로커 불필요
- 단일 애플리케이션에 최적화
- 간단한 설정

vs Cron​

**Cron**
- OS 수준
- 외부 스크립트 실행
- 제한적인 스케줄링
- Python 컨텍스트 없음

**APScheduler**
- Python 애플리케이션 내부
- Python 함수 직접 실행
- 유연한 스케줄링 (초 단위까지)
- 전체 Python 컨텍스트 사용

핵심 구성 요소​

1. Schedulers (스케줄러)​

작업 스케줄링을 관리하는 메인 인터페이스입니다.

from apscheduler.schedulers.background import BackgroundScheduler

scheduler = BackgroundScheduler()
scheduler.start()

스케줄러 타입:

  • BackgroundScheduler: 별도 스레드에서 실행
  • BlockingScheduler: 메인 스레드에서 블로킹
  • AsyncIOScheduler: asyncio 이벤트 루프
  • TornadoScheduler: Tornado 애플리케이션
  • TwistedScheduler: Twisted 애플리케이션
  • QtScheduler: Qt 애플리케이션

2. Triggers (트리거)​

작업 실행 시점을 결정합니다.

from apscheduler.triggers.cron import CronTrigger

# Cron 표현식
scheduler.add_job(
    my_function,
    CronTrigger(hour=9, minute=30)  # 매일 09:30
)

# Interval
scheduler.add_job(
    my_function,
    'interval',
    minutes=30  # 30분마다
)

# Date
scheduler.add_job(
    my_function,
    'date',
    run_date='2024-12-31 23:59:59'  # 특정 시간
)

3. Job Stores (작업 저장소)​

스케줄된 작업을 저장합니다.

from apscheduler.jobstores.sqlalchemy import SQLAlchemyJobStore

jobstores = {
    'default': SQLAlchemyJobStore(url='sqlite:///jobs.sqlite')
}

scheduler = BackgroundScheduler(jobstores=jobstores)

Job Store 타입:

  • MemoryJobStore: 메모리 (기본값, 재시작 시 손실)
  • SQLAlchemyJobStore: 관계형 DB (PostgreSQL, MySQL, SQLite)
  • MongoDBJobStore: MongoDB
  • RedisJobStore: Redis

4. Executors (실행자)​

작업을 실제로 실행합니다.

from apscheduler.executors.pool import ThreadPoolExecutor, ProcessPoolExecutor

executors = {
    'default': ThreadPoolExecutor(20),
    'processpool': ProcessPoolExecutor(5)
}

scheduler = BackgroundScheduler(executors=executors)

기본 사용법​

간단한 예제​

from apscheduler.schedulers.background import BackgroundScheduler
from datetime import datetime
import time

def my_job():
    print(f"Job executed at {datetime.now()}")

# 스케줄러 생성
scheduler = BackgroundScheduler()

# Job 추가 - 10초마다 실행
scheduler.add_job(my_job, 'interval', seconds=10)

# 스케줄러 시작
scheduler.start()

try:
    # 메인 프로그램 유지
    while True:
        time.sleep(2)
except (KeyboardInterrupt, SystemExit):
    # 종료 시 스케줄러 정리
    scheduler.shutdown()

데코레이터 방식​

from apscheduler.schedulers.blocking import BlockingScheduler

scheduler = BlockingScheduler()

@scheduler.scheduled_job('interval', minutes=5)
def my_job():
    print("This job runs every 5 minutes")

@scheduler.scheduled_job('cron', hour=9, minute=30)
def morning_job():
    print("Good morning! (runs at 9:30 AM)")

scheduler.start()

첫 번째 스케줄러​

from apscheduler.schedulers.background import BackgroundScheduler
from datetime import datetime
import time

def tick():
    """매 초마다 현재 시간 출력"""
    print(f'Tick! The time is: {datetime.now()}')

def main():
    # 스케줄러 생성
    scheduler = BackgroundScheduler()

    # Job 추가 - 3초마다 실행
    scheduler.add_job(tick, 'interval', seconds=3)

    # 스케줄러 시작
    scheduler.start()
    print('Scheduler started. Press Ctrl+C to exit.')

    try:
        # 메인 프로그램 계속 실행
        while True:
            time.sleep(1)
    except (KeyboardInterrupt, SystemExit):
        # 정리
        scheduler.shutdown()
        print('Scheduler shut down.')

if __name__ == '__main__':
    main()

간단한 비교​

언제 APScheduler를 사용하나?​

# ✅ APScheduler가 적합한 경우
- 단일 Python 애플리케이션 내 스케줄링
- Flask/Django 앱에 스케줄링 기능 추가
- 간단한 주기적 작업
- 빠른 프로토타이핑

# ❌ APScheduler가 부적합한 경우
- 분산 작업 큐 (→ Celery 사용)
- 복잡한 워크플로우 (→ Airflow/Dagster 사용)
- OS 수준 크론잡 (→ crontab 사용)

일반적인 패턴​

웹 앱 통합​

# Flask 예제
from flask import Flask
from apscheduler.schedulers.background import BackgroundScheduler

app = Flask(__name__)
scheduler = BackgroundScheduler()

def scheduled_task():
    with app.app_context():
        # Flask 컨텍스트에서 작업 수행
        print("Running scheduled task")

scheduler.add_job(scheduled_task, 'interval', hours=1)
scheduler.start()

@app.route('/')
def index():
    return "Hello World"

if __name__ == '__main__':
    app.run()

데이터베이스 백업​

import subprocess
from datetime import datetime

def backup_database():
    """PostgreSQL 백업"""
    timestamp = datetime.now().strftime('%Y%m%d_%H%M%S')
    filename = f'backup_{timestamp}.sql'

    subprocess.run([
        'pg_dump',
        '-U', 'username',
        '-d', 'database',
        '-f', filename
    ])

    print(f'Backup created: {filename}')

# 매일 새벽 2시 백업
scheduler.add_job(
    backup_database,
    'cron',
    hour=2,
    minute=0
)
