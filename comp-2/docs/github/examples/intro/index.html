실전 예제편 소개

실제 개발 현장에서 마주치는 상황을 시나리오로 구성했습니다. 각 스토리를 따라가면서 자연스럽게 Git과 GitHub의 핵심 기능을 익힐 수 있습니다.

🎭 등장인물​

민지 (신입 개발자)​

  • 경력: 6개월
  • 역할: 프론트엔드 개발자
  • 특징: 열정적이지만 Git이 아직 서툴다

준호 (시니어 개발자)​

  • 경력: 5년
  • 역할: 백엔드 개발자, 테크 리드
  • 특징: 코드 리뷰를 꼼꼼히 하며 팀원들을 잘 가르쳐준다

수현 (데브옵스 엔지니어)​

  • 경력: 3년
  • 역할: CI/CD 및 인프라 관리
  • 특징: 자동화를 좋아하고 보안에 민감하다

현우 (프로젝트 매니저)​

  • 경력: 4년
  • 역할: 프로젝트 관리, 릴리스 담당
  • 특징: 일정 관리에 철저하고 문서화를 중요시한다

📖 시나리오 목록​

🌱 초급 시나리오​

시나리오 1: 민지의 첫 출근​

상황: 신입 개발자 민지가 첫 출근해서 프로젝트에 투입됩니다. 배우는 내용:

  • 저장소 클론
  • 브랜치 생성
  • 첫 커밋과 푸시
  • Pull Request 생성
  • 코드 리뷰 대응

시나리오 2: 긴급 버그 발생!​

상황: 프로덕션에서 치명적인 버그가 발견되어 긴급 수정이 필요합니다. 배우는 내용:

  • Hotfix 브랜치 생성
  • Stash 활용
  • 긴급 배포 프로세스
  • 여러 브랜치에 동일 수정 적용 (Cherry-pick)

시나리오 3: 실수했을 때​

상황: 민지가 실수로 잘못된 브랜치에 커밋하고, 중요한 파일을 삭제했습니다. 배우는 내용:

  • 커밋 취소
  • 삭제된 파일 복구
  • Reset vs Revert
  • Reflog 활용

🚀 중급 시나리오​

시나리오 4: 대규모 기능 개발​

상황: 새로운 결제 시스템을 개발하는 2주짜리 프로젝트입니다. 배우는 내용:

  • Feature 브랜치 전략
  • 정기적인 Rebase
  • 충돌 해결
  • PR 분할 전략

시나리오 5: 오픈소스 기여하기​

상황: 민지가 처음으로 오픈소스 프로젝트에 기여합니다. 배우는 내용:

  • Fork & Clone
  • Upstream 동기화
  • 컨트리뷰션 가이드라인 따르기
  • 메인테이너와 소통하기

시나리오 6: 팀 협업의 충돌​

상황: 3명의 개발자가 같은 파일을 동시에 수정하면서 충돌이 발생합니다. 배우는 내용:

  • 복잡한 Merge 충돌 해결
  • 3-way merge 이해
  • 팀원과 커뮤니케이션
  • 충돌 예방 전략

🎯 고급 시나리오​

시나리오 7: 릴리스 데이​

상황: 현우가 v2.0.0 릴리스를 준비하고 배포합니다. 배우는 내용:

  • Release 브랜치 관리
  • Semantic Versioning
  • Changelog 자동 생성
  • GitHub Actions 배포 파이프라인

시나리오 8: 보안 사고 대응​

상황: 수현이 API 키가 커밋된 것을 발견하고 긴급 조치합니다. 배우는 내용:

  • Git 히스토리에서 Secrets 제거
  • BFG Repo-Cleaner 사용
  • 보안 스캐닝 자동화
  • 사고 대응 프로세스

시나리오 9: 레거시 프로젝트 리팩토링​

상황: 준호가 오래된 코드베이스를 현대적인 구조로 개선합니다. 배우는 내용:

  • 점진적 리팩토링 전략
  • Interactive Rebase로 커밋 정리
  • Git Bisect로 버그 추적
  • 안전한 대규모 변경

시나리오 10: Monorepo 관리​

상황: 여러 프로젝트를 하나의 저장소로 통합 관리합니다. 배우는 내용:

  • Monorepo 구조 설계
  • Sparse Checkout
  • Submodules vs Subtree
  • 효율적인 CI/CD 파이프라인

시나리오 11: Code Review 문화 만들기​

상황: 팀에 건강한 코드 리뷰 문화를 정착시킵니다. 배우는 내용:

  • 효과적인 코드 리뷰 방법
  • PR 템플릿과 체크리스트
  • CODEOWNERS 설정
  • 리뷰 자동화 도구

시나리오 12: 여러 환경 배포 관리​

상황: Dev, Staging, Production 환경을 Git 브랜치로 관리합니다. 배우는 내용:

  • 환경별 브랜치 전략
  • 자동화된 배포 파이프라인
  • 환경별 설정 관리
  • Rollback 전략

시나리오 13: Git Worktree 활용​

상황: 여러 브랜치에서 동시에 작업하기 위해 Worktree를 활용합니다. 배우는 내용:

  • Git Worktree 이해
  • 여러 브랜치 동시 작업
  • Worktree 관리 방법
  • Stash vs Worktree 비교

시나리오 14: Git 성능 최적화​

상황: 저장소가 커지면서 느려진 Git 성능을 개선합니다. 배우는 내용:

  • 저장소 크기 분석 및 최적화
  • Git LFS로 대용량 파일 관리
  • Shallow Clone & Partial Clone
  • Git GC 및 유지보수

시나리오 15: 고급 GitHub Actions 활용​

상황: 복잡한 CI/CD 파이프라인을 GitHub Actions로 구축합니다. 배우는 내용:

  • Matrix Build로 여러 환경 동시 테스트
  • Reusable Workflow 작성
  • Custom Action 개발
  • 조건부 실행 및 의존성 관리

시나리오 16: Git 내부 동작 이해​

상황: Git의 내부 구조를 파헤쳐 진정한 Git 마스터가 됩니다. 배우는 내용:

  • .git 폴더 구조 이해
  • Git Objects (blob, tree, commit, tag)
  • Refs와 HEAD의 역할
  • Packfile과 Delta 압축
  • Plumbing vs Porcelain 명령어

🎓 학습 방법​

1. 순서대로 학습​

초급 → 중급 → 고급 순서로 진행하는 것을 추천합니다.

2. 직접 실습​

각 시나리오의 명령어를 직접 따라해보세요. 실습 저장소를 만들어서 실험해도 좋습니다.

3. 변형 시도​

시나리오를 조금씩 변형해서 다양한 상황을 연습해보세요.

4. 팀과 공유​

배운 내용을 팀원들과 공유하고 함께 연습하세요.

💡 실습 저장소 준비​

# 연습용 저장소 생성
mkdir git-practice
cd git-practice
git init
echo "# Git Practice" > README.md
git add README.md
git commit -m "Initial commit"

# GitHub에 저장소 생성 및 푸시
gh repo create git-practice --public --source=. --remote=origin --push

📝 시나리오 읽는 법​

각 시나리오는 다음 구조로 구성됩니다:

  1. 📌 상황 설명: 현재 무슨 일이 일어났는지
  2. 🎯 목표: 이번 시나리오에서 해결할 문제
  3. 💬 대화: 팀원들의 대화로 상황 이해
  4. ⚡ 실습: 단계별 명령어와 설명
  5. ✅ 검증: 올바르게 수행했는지 확인
  6. 🎓 배운 내용: 핵심 개념 정리
  7. 💡 실전 팁: 실무에서 유용한 추가 팁
  8. 🔗 관련 문서: 더 자세한 내용을 볼 수 있는 링크
