Git Stash

작업 중인 변경사항을 임시로 저장하는 Git Stash 사용법을 배웁니다.

Stash란?​

Stash는 작업 디렉토리의 변경사항을 임시로 저장하고, 나중에 다시 적용할 수 있게 해주는 기능입니다.

언제 사용하나요?​

  • 🔄 긴급하게 다른 브랜치로 전환해야 할 때
  • 🧪 실험적인 코드를 임시 저장하고 싶을 때
  • 🔀 Pull 전에 로컬 변경사항을 저장해야 할 때
  • 🎯 작업 중인 내용을 다른 브랜치에 적용하고 싶을 때

기본 사용법​

Stash 저장​

# 현재 변경사항을 stash에 저장
git stash

# 메시지와 함께 저장
git stash save "WIP: working on login feature"

# 또는 (최신 방식)
git stash push -m "WIP: working on login feature"

Stash 목록 보기​

# 저장된 stash 목록
git stash list

# 출력 예시:
# stash@{0}: WIP on main: 1234567 commit message
# stash@{1}: WIP on feature: 89abcde another commit
# stash@{2}: On main: saved work

Stash 적용​

# 가장 최근 stash 적용 (stash는 유지)
git stash apply

# 특정 stash 적용
git stash apply stash@{1}

# 적용 후 stash 삭제
git stash pop

# 특정 stash를 pop
git stash pop stash@{1}

Stash 삭제​

# 가장 최근 stash 삭제
git stash drop

# 특정 stash 삭제
git stash drop stash@{1}

# 모든 stash 삭제
git stash clear

고급 사용법​

Tracked 파일만 Stash​

# 추적 중인 파일만 stash
git stash push

# Untracked 파일도 포함
git stash push -u
# 또는
git stash push --include-untracked

특정 파일만 Stash​

# 특정 파일만 stash
git stash push -m "Stash specific files" -- file1.txt file2.js

# 특정 패턴의 파일만
git stash push -m "Stash all JS files" -- "*.js"

Stash 내용 확인​

# 가장 최근 stash의 변경사항 보기
git stash show

# 상세 내용 보기
git stash show -p

# 특정 stash 보기
git stash show stash@{1} -p

새 브랜치로 Stash 적용​

# stash를 새 브랜치에 적용
git stash branch new-feature-branch

# 특정 stash를 새 브랜치로
git stash branch bug-fix stash@{2}

실습 예제​

예제 1: 긴급 작업 전환​

# 1. feature 브랜치에서 작업 중
git checkout feature/user-profile
echo "work in progress" >> profile.js

# 2. 긴급 버그 수정 요청
git status  # 변경사항 확인

# 3. 현재 작업 stash
git stash save "WIP: user profile implementation"

# 4. main으로 전환하여 버그 수정
git checkout main
git checkout -b hotfix/login-bug
# ... 버그 수정 ...
git commit -am "fix: resolve login bug"
git push

# 5. 원래 작업으로 복귀
git checkout feature/user-profile
git stash pop

# 6. 작업 계속

예제 2: Pull 전 변경사항 저장​

# 1. 로컬에서 작업 중
echo "local changes" >> file.txt

# 2. Pull 시도
git pull
# error: Your local changes would be overwritten

# 3. Stash 후 pull
git stash
git pull

# 4. 변경사항 다시 적용
git stash pop

# 5. 충돌 있으면 해결
# ... 충돌 해결 ...
git add .

예제 3: 실험적 변경사항 저장​

# 1. 실험적 코드 작성
echo "experimental feature" >> experiment.js

# 2. 임시 저장
git stash save "Experiment: new algorithm"

# 3. 다른 작업 수행
# ... 작업 ...

# 4. 나중에 실험 재개
git stash list
git stash apply stash@{0}

# 5. 실험 성공 시 커밋
git add experiment.js
git commit -m "feat: implement new algorithm"

# 6. stash 삭제
git stash drop

예제 4: 잘못된 브랜치에서 작업한 경우​

# 1. main에서 실수로 작업함
git checkout main
# ... 많은 작업 ...

# 2. 변경사항 stash
git stash save "Work done on wrong branch"

# 3. 올바른 브랜치 생성
git checkout -b feature/correct-branch

# 4. stash 적용
git stash pop

# 5. 커밋
git add .
git commit -m "feat: add new feature"

Stash 패턴​

패턴 1: Daily Stash​

# 퇴근 전
git stash save "EOD: $(date '+%Y-%m-%d') work in progress"

# 다음날 출근 후
git stash list
git stash apply stash@{0}

패턴 2: Context Switching​

# 작업 A에서 작업 B로
git stash save "Context: Feature A - authentication"
git checkout feature-b
# ... 작업 B ...
git checkout feature-a
git stash pop

패턴 3: Multiple Stashes​

# 여러 작업 상태 저장
git stash save "Component A: styling"
git stash save "Component B: logic"
git stash save "Component C: tests"

# 필요한 stash만 적용
git stash list
git stash apply stash@{1}  # Component B

고급 기능​

Patch 모드로 Stash​

일부 변경사항만 선택적으로 stash:

# 대화형 모드
git stash save -p "Partial stash"

# 각 hunk마다 선택
# y - 이 hunk stash
# n - 이 hunk 유지
# q - 종료
# a - 이 파일의 모든 hunk stash
# d - 이 파일의 모든 hunk 유지

Stash를 커밋으로 변환​

# Stash의 변경사항을 커밋으로
git stash show -p stash@{0} | git apply
git add .
git commit -m "Applied stash as commit"
git stash drop stash@{0}

Stash 간 비교​

# 두 stash 비교
git diff stash@{0} stash@{1}

# Stash와 현재 작업 디렉토리 비교
git diff stash@{0}

Stash 관리 팁​

의미 있는 메시지 사용​

# ❌ 나쁜 예
git stash
git stash save "work"
git stash save "temp"

# ✅ 좋은 예
git stash save "WIP: user authentication - missing tests"
git stash save "Experiment: Redis caching implementation"
git stash save "Backup before risky refactor"

정기적인 정리​

# 오래된 stash 확인
git stash list

# 불필요한 stash 삭제
git stash drop stash@{5}

# 모두 정리 (주의!)
git stash clear

Stash 백업​

# Stash를 브랜치로 백업
git stash branch backup-stash-$(date +%Y%m%d)

일반적인 문제 해결​

문제 1: Stash Pop 충돌​

git stash pop
# CONFLICT (content): Merge conflict in file.txt

# 충돌 해결
code file.txt
git add file.txt

# stash는 자동으로 삭제되지 않으므로 수동 삭제
git stash drop

문제 2: Stash 복구​

# 실수로 삭제한 stash 복구

# 1. fsck로 찾기
git fsck --unreachable | grep commit

# 2. 각 커밋 확인
git show <commit-hash>

# 3. 찾은 커밋을 stash로 복구
git update-ref refs/stash <commit-hash> -m "Recovered stash"

문제 3: Untracked 파일 포함 안 됨​

# 기본 stash는 untracked 파일 무시
git stash

# Untracked 파일 포함
git stash save -u "Include untracked files"

# 모든 파일 포함 (ignored 제외)
git stash save -u "All files"

# Ignored 파일까지 포함
git stash save -a "Everything"

Stash vs 다른 방법​

Stash vs Commit​

# Stash: 임시 저장, 히스토리에 안 남음
git stash

# WIP Commit: 히스토리에 남음
git add .
git commit -m "WIP: temporary commit"
# 나중에 커밋 수정
git reset HEAD~1

Stash vs Branch​

# Stash: 빠른 전환
git stash
git checkout other-branch

# Branch: 영구 보존
git checkout -b wip-branch
git add .
git commit -m "WIP"
git checkout other-branch

유용한 Alias​

# 편리한 alias 설정
git config --global alias.st 'stash'
git config --global alias.stp 'stash pop'
git config --global alias.stl 'stash list'
git config --global alias.sts 'stash show -p'

# 사용
git st "Quick save"
git stp
git stl
