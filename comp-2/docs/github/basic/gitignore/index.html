.gitignore 사용법

.gitignore 파일을 사용하여 Git이 추적하지 않을 파일을 지정하는 방법을 배웁니다.

.gitignore란?​

.gitignore 파일은 Git이 무시할 파일과 디렉토리를 지정합니다. 불필요한 파일(로그, 빌드 결과, 임시 파일 등)을 버전 관리에서 제외할 수 있습니다.

기본 문법​

파일 무시​

# 특정 파일
secret.txt
config.local.js

# 특정 확장자
*.log
*.tmp

# 모든 디렉토리의 특정 파일
**/secrets.txt

디렉토리 무시​

# 특정 디렉토리
node_modules/
dist/
build/

# 모든 하위 디렉토리
**/temp/

패턴 매칭​

# 모든 .log 파일
*.log

# test로 시작하는 모든 파일
test*

# test로 끝나는 모든 파일
*test

# 한 문자 매칭
test?.txt

# 대괄호 패턴
test[0-9].txt
test[abc].txt

예외 처리​

# 모든 .log 파일 무시
*.log

# 하지만 important.log는 추적
!important.log

주석​

# 이것은 주석입니다
*.log  # 라인 끝 주석

고급 사용법​

전역 .gitignore​

모든 Git 저장소에 적용되는 전역 .gitignore를 설정할 수 있습니다.

# 전역 gitignore 파일 생성
touch ~/.gitignore_global

# 전역 gitignore 설정
git config --global core.excludesfile ~/.gitignore_global

~/.gitignore_global 내용 예시:

# OS
.DS_Store
Thumbs.db

# IDE
.vscode/
.idea/
*.swp

# 기타
.env

이미 추적 중인 파일 무시하기​

# 1. .gitignore에 파일 추가
echo "config.txt" >> .gitignore

# 2. Git 추적에서 제거 (파일은 유지)
git rm --cached config.txt

# 3. 커밋
git add .gitignore
git commit -m "chore: ignore config.txt"
팁

--cached 옵션을 사용하면 파일은 삭제되지 않고 Git 추적만 중단됩니다.

디렉토리만 무시하기 (파일은 제외)​

# logs 디렉토리 무시
logs/

# 하지만 .gitkeep 파일은 유지
!logs/.gitkeep

특정 깊이의 파일만 무시​

# 루트의 TODO 파일만 무시
/TODO

# 모든 위치의 TODO 파일 무시
TODO

# src 디렉토리 바로 아래의 파일만
/src/*.js

# src의 모든 하위 디렉토리 포함
/src/**/*.js

실습 예제​

예제 1: 새 프로젝트에 .gitignore 추가​

# 1. .gitignore 파일 생성
touch .gitignore

# 2. 패턴 추가
cat > .gitignore << EOF
node_modules/
.env
*.log
dist/
EOF

# 3. Git에 추가
git add .gitignore
git commit -m "chore: add .gitignore"

예제 2: 이미 커밋된 파일 무시하기​

# 1. 현재 상태 확인
git status

# 2. .env 파일을 무시 목록에 추가
echo ".env" >> .gitignore

# 3. Git 추적에서 제거
git rm --cached .env

# 4. 커밋
git add .gitignore
git commit -m "chore: ignore .env file"

# 5. 푸시
git push

예제 3: 무시된 파일 확인​

# 무시된 파일 확인
git status --ignored

# 특정 파일이 무시되는지 확인
git check-ignore -v filename.txt

유용한 팁​

.gitkeep으로 빈 디렉토리 추적​

Git은 빈 디렉토리를 추적하지 않습니다. 빈 디렉토리를 유지하려면:

# 빈 디렉토리 생성
mkdir logs

# .gitkeep 파일 생성
touch logs/.gitkeep

# 커밋
git add logs/.gitkeep
git commit -m "chore: add logs directory"

민감한 정보 관리​

# 환경 변수
.env
.env.local
.env.*.local

# 비밀 키
*.key
*.pem
secrets/

# 설정 파일
config.local.js
config.production.js

.env.example 파일로 필요한 환경 변수 목록 제공:

# .env.example
DATABASE_URL=
API_KEY=
SECRET_KEY=

디버깅​

파일이 무시되지 않을 때​

# 1. Git 캐시 초기화
git rm -r --cached .

# 2. 모든 파일 다시 추가
git add .

# 3. 커밋
git commit -m "chore: fix .gitignore"

특정 파일이 왜 무시되는지 확인​

git check-ignore -v path/to/file

출력 예시:

.gitignore:3:*.log    path/to/debug.log

일반적인 실수​

1. 이미 커밋된 파일을 .gitignore에 추가​

❌ 잘못된 방법:

echo "secret.txt" >> .gitignore
# secret.txt는 여전히 추적됨

✅ 올바른 방법:

echo "secret.txt" >> .gitignore
git rm --cached secret.txt
git commit -m "chore: stop tracking secret.txt"

2. 전역 패턴 사용 시 주의​

# ❌ 모든 test 디렉토리 무시 (의도하지 않은 파일도 무시될 수 있음)
test/

# ✅ 특정 위치만 무시
/test/
