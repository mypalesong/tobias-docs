브랜치 기초

브랜치는 독립적인 작업 공간을 만들어 동시에 여러 기능을 개발할 수 있게 해줍니다.

브랜치란?​

브랜치는 커밋의 포인터입니다. 새로운 기능을 개발하거나 버그를 수정할 때 메인 코드에 영향을 주지 않고 작업할 수 있습니다.

기본 명령어​

브랜치 목록 보기​

# 로컬 브랜치
git branch

# 모든 브랜치 (원격 포함)
git branch -a

# 원격 브랜치만
git branch -r

브랜치 생성​

git branch feature-login

브랜치 전환​

# 기존 방식
git checkout feature-login

# 최신 방식 (Git 2.23+)
git switch feature-login

브랜치 생성과 전환을 동시에​

# 기존 방식
git checkout -b feature-signup

# 최신 방식
git switch -c feature-payment

브랜치 삭제​

# 일반 삭제
git branch -d feature-login

# 강제 삭제 (병합되지 않은 브랜치)
git branch -D feature-login

# 원격 브랜치 삭제
git push origin --delete feature-login

브랜치 병합​

Fast-forward 병합​

git checkout main
git merge feature-login

3-way 병합​

git checkout main
git merge --no-ff feature-login

병합 충돌 해결​

# 1. 병합 시도
git merge feature-branch

# 2. 충돌 발생 시 파일 수정
# <<<<<<< HEAD
# 현재 브랜치의 내용
# =======
# 병합하려는 브랜치의 내용
# >>>>>>> feature-branch

# 3. 충돌 해결 후 스테이징
git add conflicted-file.txt

# 4. 병합 커밋 생성
git commit

브랜치 전략​

Git Flow​

main          # 프로덕션 코드
├─ develop    # 개발 브랜치
   ├─ feature/login      # 기능 브랜치
   ├─ feature/signup     # 기능 브랜치
   └─ hotfix/bug-123    # 핫픽스 브랜치

GitHub Flow​

main          # 항상 배포 가능한 상태
├─ feature/add-user-profile
├─ feature/improve-performance
└─ fix/login-bug

실습 예제​

# 1. 새 기능 브랜치 생성 및 전환
git switch -c feature/user-profile

# 2. 작업 수행
echo "User profile page" > profile.html
git add profile.html
git commit -m "feat: add user profile page"

# 3. main 브랜치로 전환
git switch main

# 4. 최신 코드 가져오기
git pull

# 5. feature 브랜치 병합
git merge feature/user-profile

# 6. 푸시
git push

# 7. feature 브랜치 삭제
git branch -d feature/user-profile
