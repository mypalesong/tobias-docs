변경사항 취소하기

Git에서 실수를 되돌리는 다양한 방법을 배웁니다.

작업 디렉토리의 변경사항 취소​

git restore (Git 2.23+)​

# 특정 파일의 변경사항 취소
git restore filename.txt

# 모든 파일의 변경사항 취소
git restore .

# 특정 디렉토리의 변경사항 취소
git restore src/

git checkout (구버전)​

# 특정 파일 복원
git checkout -- filename.txt

# 모든 파일 복원
git checkout -- .
팁

Git 2.23 이상에서는 git restore를 사용하는 것이 권장됩니다.

스테이징 영역에서 제거​

git restore --staged​

# 특정 파일을 스테이징에서 제거
git restore --staged filename.txt

# 모든 파일을 스테이징에서 제거
git restore --staged .

git reset (구버전)​

# 특정 파일 unstage
git reset HEAD filename.txt

# 모든 파일 unstage
git reset HEAD

커밋 취소​

git reset​

Git reset은 3가지 모드가 있습니다:

--soft: 커밋만 취소 (변경사항은 스테이징 상태로 유지)​

# 마지막 커밋 취소
git reset --soft HEAD~1

# 마지막 2개 커밋 취소
git reset --soft HEAD~2

# 특정 커밋으로 이동
git reset --soft commit-hash

사용 사례: 커밋 메시지를 다시 작성하고 싶을 때

--mixed (기본값): 커밋 취소 + 스테이징 해제​

# 마지막 커밋 취소
git reset HEAD~1
# 또는
git reset --mixed HEAD~1

사용 사례: 커밋을 취소하고 파일을 다시 선택해서 커밋하고 싶을 때

--hard: 커밋 취소 + 모든 변경사항 삭제​

# 마지막 커밋을 완전히 삭제
git reset --hard HEAD~1

# 특정 커밋 상태로 완전히 돌아가기
git reset --hard commit-hash
위험

--hard 옵션은 모든 변경사항을 삭제합니다! 주의해서 사용하세요.

git revert​

커밋을 취소하는 새로운 커밋을 만듭니다 (이력이 남음).

# 마지막 커밋 되돌리기
git revert HEAD

# 특정 커밋 되돌리기
git revert commit-hash

# 여러 커밋 되돌리기
git revert commit1 commit2 commit3

# 커밋 메시지 편집 없이
git revert --no-edit HEAD

# 자동 커밋 없이 (수동으로 커밋)
git revert --no-commit HEAD
팁

이미 푸시한 커밋은 reset 대신 revert를 사용하세요!

reset vs revert 비교​

git reset​

  • 이력을 삭제함
  • 로컬에서만 사용 권장
  • 푸시된 커밋에는 사용하지 말 것
  • 협업 시 문제 발생 가능

git revert​

  • 새로운 커밋을 생성함
  • 원격 저장소에도 안전하게 사용 가능
  • 이력이 명확하게 남음
  • 협업에 안전

실수 복구하기​

삭제한 커밋 복구​

# reflog로 이전 커밋 해시 찾기
git reflog

# 원하는 커밋으로 복구
git reset --hard commit-hash

삭제한 브랜치 복구​

# reflog에서 브랜치의 마지막 커밋 찾기
git reflog

# 브랜치 재생성
git branch recovered-branch commit-hash

실수로 hard reset한 경우​

# 1. reflog 확인
git reflog

# 2. 원하는 시점 찾기
# 예: HEAD@{2}: commit: important changes

# 3. 해당 시점으로 복구
git reset --hard HEAD@{2}

파일 복원​

이전 커밋의 파일 가져오기​

# 특정 커밋의 파일 복원
git restore --source=commit-hash filename.txt

# HEAD~2 시점의 파일 복원
git restore --source=HEAD~2 filename.txt

삭제된 파일 복구​

# 1. 파일이 삭제된 커밋 찾기
git log -- deleted-file.txt

# 2. 삭제되기 전 커밋에서 복원
git restore --source=commit-hash deleted-file.txt

실습 예제​

예제 1: 잘못된 변경사항 취소​

# 1. 파일 수정
echo "wrong content" >> README.md

# 2. 상태 확인
git status

# 3. 변경사항 취소
git restore README.md

# 4. 확인
git status

예제 2: 잘못 스테이징한 파일 제거​

# 1. 실수로 모든 파일 추가
git add .

# 2. 특정 파일만 unstage
git restore --staged secret.txt

# 3. 확인
git status

예제 3: 마지막 커밋 수정​

# 1. 잘못된 커밋 생성
git commit -m "wrong message"

# 2. 커밋 취소 (변경사항은 유지)
git reset --soft HEAD~1

# 3. 올바른 커밋 생성
git commit -m "correct message"

예제 4: 푸시한 커밋 되돌리기​

# 1. 되돌릴 커밋 확인
git log --oneline

# 2. revert로 되돌리기
git revert HEAD

# 3. 푸시
git push

고급 활용​

특정 커밋의 특정 파일만 복원​

git show commit-hash:path/to/file.txt > file.txt

여러 커밋을 하나로 합치기 (Squash)​

# 1. 마지막 3개 커밋을 soft reset
git reset --soft HEAD~3

# 2. 하나의 커밋으로 생성
git commit -m "Combined commit message"

커밋 메시지만 수정​

git commit --amend -m "New commit message"
경고

이미 푸시한 커밋은 --amend하지 마세요!

커밋에 파일 추가​

# 1. 파일 스테이징
git add forgotten-file.txt

# 2. 마지막 커밋에 추가
git commit --amend --no-edit

Clean - 추적되지 않는 파일 삭제​

미리 보기​

# 삭제될 파일 목록 확인
git clean -n

# 디렉토리 포함
git clean -n -d

실제 삭제​

# 추적되지 않는 파일 삭제
git clean -f

# 디렉토리도 삭제
git clean -f -d

# .gitignore 파일도 삭제
git clean -f -x
위험

git clean은 복구할 수 없습니다! 반드시 -n으로 먼저 확인하세요.

정리 표​

상황명령어
작업 디렉토리 변경 취소git restore file.txt
스테이징 취소git restore --staged file.txt
마지막 커밋 취소 (로컬)git reset --soft HEAD~1
마지막 커밋 완전 삭제git reset --hard HEAD~1
푸시한 커밋 되돌리기git revert HEAD
커밋 메시지 수정git commit --amend
삭제한 커밋 복구git reflog → git reset --hard
추적 안 되는 파일 삭제git clean -f -d
