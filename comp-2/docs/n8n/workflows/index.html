3. 워크플로우 작성

n8n에서 워크플로우를 작성하고 관리하는 방법을 상세히 알아봅니다. 기본부터 고급 기법까지 실전 예제와 함께 학습합니다.

3.1 워크플로우 생성 및 편집​

워크플로우 생성​

n8n에서 새로운 워크플로우를 만드는 방법입니다.

1. n8n 대시보드에서 **"New Workflow"** 클릭
2. 워크플로우 이름 설정 (예: "고객 데이터 동기화")
3. 첫 번째 노드 추가 (보통 Trigger 노드)
4. 노드 연결 및 설정
5. **Save** 버튼으로 저장
이름 규칙

워크플로우 이름은 명확하고 설명적으로 작성하세요.

  • Good: "Slack_신규주문_알림"
  • Bad: "workflow1"

워크플로우 편집기 인터페이스​

┌─────────────────────────────────────────────┐
│  [저장] [실행] [활성화]        [설정] [삭제] │
├─────────────────────────────────────────────┤
│                                             │
│  [노드 팔레트]     [캔버스]                  │
│                                             │
│  Trigger          ┌──────┐                  │
│  ├ Webhook        │Node 1│                  │
│  ├ Cron           └───┬──┘                  │
│  └ Manual             │                     │
│                       ▼                     │
│  Action           ┌──────┐                  │
│  ├ HTTP           │Node 2│                  │
│  ├ Email          └──────┘                  │
│  └ Database                                 │
│                                             │
└─────────────────────────────────────────────┘

노드 추가 및 연결​

노드 추가 방법:

1. **"+" 버튼 클릭** - 빈 캔버스나 기존 노드 옆
2. **노드 검색** - 이름으로 검색 (예: "http", "slack")
3. **카테고리 탐색** - Trigger, Action, Logic 등
4. **노드 클릭** - 캔버스에 추가

노드 연결:

// 노드 연결 방법
[출발 노드의 출력 점] ──드래그──> [도착 노드의 입력 점]

// 여러 노드 연결
        ┌─> [Node B]
[Node A]─┼─> [Node C]
        └─> [Node D]

워크플로우 설정​

{
  "name": "고객_데이터_동기화",
  "settings": {
    "executionOrder": "v1",
    "saveDataErrorExecution": "all",
    "saveDataSuccessExecution": "all",
    "saveManualExecutions": true,
    "callerPolicy": "workflowsFromSameOwner",
    "timezone": "Asia/Seoul"
  }
}
실행 데이터 보관

프로덕션에서는 saveDataSuccessExecution을 "none"으로 설정하여 저장공간을 절약하세요.

3.2 Trigger 노드 사용​

Trigger 노드는 워크플로우를 시작하는 노드입니다.

Manual Trigger​

수동으로 워크플로우를 실행할 때 사용합니다.

// Manual Trigger 설정
{
  "name": "Manual Trigger",
  "type": "n8n-nodes-base.manualTrigger",
  "parameters": {}
}

사용 사례:

  • 테스트 및 디버깅
  • 일회성 데이터 처리
  • 관리자가 수동으로 실행하는 작업

Webhook Trigger​

HTTP 요청으로 워크플로우를 트리거합니다.

// Webhook Trigger 설정
{
  "name": "Webhook",
  "type": "n8n-nodes-base.webhook",
  "parameters": {
    "httpMethod": "POST",
    "path": "customer-created",
    "responseMode": "responseNode",
    "options": {
      "allowedOrigins": "*"
    }
  }
}

Webhook 호출 예시:

# curl로 Webhook 호출
curl -X POST http://localhost:5678/webhook/customer-created \
  -H "Content-Type: application/json" \
  -d '{
    "name": "홍길동",
    "email": "hong@example.com",
    "plan": "premium"
  }'
# Python으로 Webhook 호출
import requests

webhook_url = "http://localhost:5678/webhook/customer-created"

data = {
    "name": "홍길동",
    "email": "hong@example.com",
    "plan": "premium"
}

response = requests.post(webhook_url, json=data)
print(response.json())
Webhook 응답

responseMode를 "responseNode"로 설정하면 워크플로우 마지막에 Respond to Webhook 노드로 응답을 반환할 수 있습니다.

Cron Trigger​

일정에 따라 워크플로우를 자동 실행합니다.

// Cron Trigger 설정
{
  "name": "Cron",
  "type": "n8n-nodes-base.cron",
  "parameters": {
    "triggerTimes": {
      "item": [
        {
          "mode": "everyX",
          "value": 1,
          "unit": "hours"
        }
      ]
    }
  }
}

Cron 표현식 예시:

// 매일 오전 9시
"0 9 * * *"

// 매주 월요일 오전 10시
"0 10 * * 1"

// 매월 1일 오전 8시
"0 8 1 * *"

// 매 15분마다
"*/15 * * * *"

// 평일 오전 9시-오후 6시, 매시간
"0 9-18 * * 1-5"

Cron 설정 옵션:

**Every X:**
- 매 X분/시간/일마다 실행
- 예: 매 30분, 매 2시간

**Custom:**
- Cron 표현식 직접 입력
- 예: "0 9 * * 1-5" (평일 오전 9시)

**Specific Times:**
- 특정 시간 선택
- 예: 09:00, 12:00, 18:00

3.3 HTTP Request 노드​

외부 API와 통신하는 가장 중요한 노드입니다.

기본 사용법​

// GET Request
{
  "method": "GET",
  "url": "https://api.github.com/users/{{$json.username}}",
  "authentication": "none",
  "options": {
    "timeout": 10000
  }
}

HTTP 메서드​

GET - 데이터 조회:

// GitHub 사용자 정보 조회
{
  "method": "GET",
  "url": "https://api.github.com/users/torvalds",
  "headers": {
    "Accept": "application/vnd.github.v3+json"
  }
}

POST - 데이터 생성:

// Slack 메시지 전송
{
  "method": "POST",
  "url": "https://slack.com/api/chat.postMessage",
  "headers": {
    "Authorization": "Bearer {{$credentials.slackToken}}",
    "Content-Type": "application/json"
  },
  "body": {
    "channel": "#general",
    "text": "새로운 고객이 등록되었습니다: {{$json.name}}"
  }
}

PUT - 데이터 업데이트:

// 고객 정보 업데이트
{
  "method": "PUT",
  "url": "https://api.example.com/customers/{{$json.id}}",
  "body": {
    "name": "{{$json.name}}",
    "status": "active",
    "updated_at": "{{$now}}"
  }
}

DELETE - 데이터 삭제:

// 리소스 삭제
{
  "method": "DELETE",
  "url": "https://api.example.com/resources/{{$json.id}}",
  "headers": {
    "Authorization": "Bearer {{$credentials.apiKey}}"
  }
}

인증 (Authentication)​

Bearer Token:

{
  "authentication": "genericCredentialType",
  "genericAuthType": "httpHeaderAuth",
  "httpHeaderAuth": {
    "name": "Authorization",
    "value": "Bearer YOUR_TOKEN_HERE"
  }
}

API Key:

// Query Parameter
{
  "url": "https://api.example.com/data",
  "sendQuery": true,
  "queryParameters": {
    "parameters": [
      {
        "name": "api_key",
        "value": "YOUR_API_KEY"
      }
    ]
  }
}

// Header
{
  "headers": {
    "X-API-Key": "YOUR_API_KEY"
  }
}

Basic Auth:

{
  "authentication": "genericCredentialType",
  "genericAuthType": "httpBasicAuth",
  "httpBasicAuth": {
    "user": "username",
    "password": "password"
  }
}

페이지네이션 처리​

// 페이지네이션 설정
{
  "url": "https://api.example.com/items",
  "options": {
    "pagination": {
      "paginationMode": "updateParameter",
      "limitPagesFetched": true,
      "maxRequests": 10,
      "parameters": {
        "pageSize": {
          "name": "limit",
          "value": 100
        },
        "offset": {
          "name": "offset",
          "value": "={{ $pageCount * 100 }}"
        }
      }
    }
  }
}

3.4 Function 노드 (JavaScript)​

JavaScript 코드로 데이터를 처리합니다.

기본 구조​

// Function 노드 기본 구조
// items는 입력 데이터 배열
for (const item of items) {
  // item.json에 데이터 있음

  // 데이터 처리
  item.json.processed = true;
  item.json.timestamp = Date.now();
}

return items;

데이터 변환​

// 데이터 변환 예시
for (const item of items) {
  // 필드 이름 변경
  item.json.fullName = `${item.json.firstName} ${item.json.lastName}`;

  // 날짜 포맷팅
  const date = new Date(item.json.createdAt);
  item.json.formattedDate = date.toLocaleDateString('ko-KR');

  // 숫자 계산
  item.json.totalPrice = item.json.price * item.json.quantity;
  item.json.taxAmount = item.json.totalPrice * 0.1;
}

return items;

필터링​

// 조건에 맞는 아이템만 반환
const filteredItems = items.filter(item => {
  // 가격이 10000원 이상인 항목만
  return item.json.price >= 10000;
});

return filteredItems;

데이터 생성​

// 새로운 아이템 생성
const newItems = [];

for (let i = 0; i < 10; i++) {
  newItems.push({
    json: {
      id: i + 1,
      name: `Item ${i + 1}`,
      created: new Date().toISOString()
    }
  });
}

return newItems;

외부 API 호출​

// Function 노드에서 axios 사용
const axios = require('axios');

const results = [];

for (const item of items) {
  try {
    const response = await axios.get(
      `https://api.example.com/users/${item.json.userId}`
    );

    results.push({
      json: {
        ...item.json,
        userData: response.data
      }
    });
  } catch (error) {
    results.push({
      json: {
        ...item.json,
        error: error.message
      }
    });
  }
}

return results;

복잡한 데이터 처리​

// 배열 그룹핑 및 집계
const grouped = {};

for (const item of items) {
  const category = item.json.category;

  if (!grouped[category]) {
    grouped[category] = {
      category: category,
      count: 0,
      totalAmount: 0,
      items: []
    };
  }

  grouped[category].count++;
  grouped[category].totalAmount += item.json.amount;
  grouped[category].items.push(item.json.name);
}

// 객체를 배열로 변환
const result = Object.values(grouped).map(group => ({
  json: group
}));

return result;
Function 노드 제한사항

Function 노드는 Node.js 환경에서 실행되므로 브라우저 전용 API(window, document 등)는 사용할 수 없습니다.

3.5 IF, Switch, Merge 노드​

조건 분기와 데이터 병합을 처리하는 노드들입니다.

IF 노드​

조건에 따라 두 경로로 분기합니다.

// IF 노드 설정
{
  "conditions": {
    "boolean": [
      {
        "value1": "={{$json.amount}}",
        "operation": "larger",
        "value2": 100000
      }
    ]
  }
}

IF 노드 예시:

                    ┌─> [true] ─> [VIP 처리]
[데이터 입력] ─> [IF]
                    └─> [false] ─> [일반 처리]

복합 조건:

// AND 조건 (모든 조건 만족)
{
  "conditions": {
    "boolean": [
      {
        "value1": "={{$json.amount}}",
        "operation": "larger",
        "value2": 100000
      },
      {
        "value1": "={{$json.status}}",
        "operation": "equal",
        "value2": "active"
      }
    ]
  },
  "combineOperation": "all"
}

// OR 조건 (하나만 만족)
{
  "combineOperation": "any"
}

Switch 노드​

여러 조건으로 다중 분기합니다.

// Switch 노드 설정
{
  "mode": "rules",
  "rules": {
    "values": [
      {
        "conditions": {
          "string": [
            {
              "value1": "={{$json.plan}}",
              "operation": "equals",
              "value2": "premium"
            }
          ]
        },
        "output": 0
      },
      {
        "conditions": {
          "string": [
            {
              "value1": "={{$json.plan}}",
              "operation": "equals",
              "value2": "basic"
            }
          ]
        },
        "output": 1
      }
    ]
  },
  "fallbackOutput": 2
}

Switch 노드 흐름:

                     ┌─> [Output 0: Premium] ─> [프리미엄 처리]
                     │
[데이터] ─> [Switch]─┼─> [Output 1: Basic] ─> [기본 처리]
                     │
                     └─> [Output 2: Default] ─> [기타 처리]

Merge 노드​

여러 경로의 데이터를 합칩니다.

// Merge 노드 모드

// 1. Append (단순 결합)
[A, B, C] + [D, E, F] = [A, B, C, D, E, F]

// 2. Merge By Index (인덱스로 병합)
[{id: 1}, {id: 2}] + [{name: 'A'}, {name: 'B'}]
= [{id: 1, name: 'A'}, {id: 2, name: 'B'}]

// 3. Merge By Key (키로 병합)
Input 1: [{id: 1, name: 'John'}]
Input 2: [{id: 1, age: 30}]
Result: [{id: 1, name: 'John', age: 30}]

Merge 노드 설정:

{
  "mode": "mergeByKey",
  "mergeByKey": {
    "key1": "id",
    "key2": "id"
  }
}

3.6 Set, Move Binary Data 노드​

데이터 구조를 변경하는 노드들입니다.

Set 노드​

데이터 필드를 설정하거나 수정합니다.

// Set 노드 설정
{
  "keepOnlySet": false,
  "values": {
    "string": [
      {
        "name": "fullName",
        "value": "={{$json.firstName}} {{$json.lastName}}"
      }
    ],
    "number": [
      {
        "name": "totalPrice",
        "value": "={{$json.price * $json.quantity}}"
      }
    ],
    "boolean": [
      {
        "name": "isVIP",
        "value": "={{$json.amount > 100000}}"
      }
    ]
  }
}

Set 노드 활용 예시:

// 1. 필드 추가
Input:  {name: "홍길동", age: 30}
Set:    timestamp = "2024-01-01"
Output: {name: "홍길동", age: 30, timestamp: "2024-01-01"}

// 2. 필드 수정
Input:  {price: "10000"}
Set:    price = 10000 (문자열 → 숫자)
Output: {price: 10000}

// 3. 계산된 필드
Input:  {price: 10000, quantity: 3}
Set:    total = price * quantity
Output: {price: 10000, quantity: 3, total: 30000}
keepOnlySet 옵션

keepOnlySet: true로 설정하면 Set 노드에서 설정한 필드만 남기고 나머지는 제거합니다.

Move Binary Data 노드​

바이너리 데이터(파일, 이미지 등)를 JSON으로 변환하거나 그 반대로 변환합니다.

// JSON → Binary
{
  "mode": "jsonToBinary",
  "options": {
    "sourceProperty": "data",
    "fileName": "={{$json.filename}}",
    "mimeType": "application/json"
  }
}

// Binary → JSON
{
  "mode": "binaryToJson",
  "options": {
    "sourceProperty": "data",
    "encoding": "utf8"
  }
}

3.7 Split/Aggregate 데이터​

데이터를 분할하거나 집계하는 방법입니다.

Split In Batches 노드​

대량의 데이터를 배치로 나누어 처리합니다.

// Split In Batches 설정
{
  "batchSize": 10,
  "options": {
    "reset": false
  }
}

Split In Batches 흐름:

[100개 아이템] ─> [Split In Batches(10)]
                       │
                       ├─> Batch 1 (10개) ─> [처리]
                       ├─> Batch 2 (10개) ─> [처리]
                       ├─> Batch 3 (10개) ─> [처리]
                       └─> ...

배치 처리 워크플로우:

// 1. Split In Batches로 데이터 분할
// 2. HTTP Request로 API 호출 (배치마다)
// 3. Wait 노드로 API 제한 회피
// 4. 모든 배치 처리 완료 후 다음 단계

[데이터(1000개)]
  ─> [Split In Batches(100)]
  ─> [HTTP Request]
  ─> [Wait 1초]
  ─> [루프]
  ─> [완료]

Aggregate 노드​

여러 실행의 데이터를 하나로 모읍니다.

// Aggregate 설정
{
  "aggregate": "aggregateAllItemData",
  "options": {}
}

Item Lists 노드​

배열 데이터를 개별 아이템으로 분할하거나 합칩니다.

// Split (배열 → 개별 아이템)
Input:  {items: [1, 2, 3, 4, 5]}
Output: {value: 1}, {value: 2}, {value: 3}, {value: 4}, {value: 5}

// Aggregate (개별 아이템 → 배열)
Input:  {name: "A"}, {name: "B"}, {name: "C"}
Output: {items: [{name: "A"}, {name: "B"}, {name: "C"}]}

3.8 루프와 반복​

워크플로우에서 루프를 구현하는 방법입니다.

Loop Over Items​

Split In Batches 노드로 루프 구현:

[시작]
  ─> [Split In Batches(1)]
  ─> [처리 로직]
  ─> [다음 배치?]
       ├─ Yes ─> [Split In Batches로 돌아감]
       └─ No ─> [종료]

조건부 루프​

IF 노드와 조합하여 조건부 루프:

// 재시도 로직
[API 호출]
  ─> [IF: 성공?]
       ├─ true ─> [완료]
       └─ false ─> [재시도 카운터 증가]
                    ─> [IF: 재시도 < 3?]
                         ├─ true ─> [API 호출로 돌아감]
                         └─ false ─> [에러 처리]

실전 루프 예시​

페이지네이션 루프:

// Function 노드에서 페이지 처리
let allData = [];
let page = 1;
let hasMore = true;

while (hasMore && page <= 10) {
  const response = await fetch(
    `https://api.example.com/items?page=${page}&limit=100`
  );

  const data = await response.json();
  allData = allData.concat(data.items);

  hasMore = data.hasMore;
  page++;
}

return [{ json: { items: allData, totalPages: page - 1 } }];

재시도 로직:

// Function 노드에서 재시도
async function fetchWithRetry(url, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch(url);
      if (response.ok) {
        return await response.json();
      }
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      // 지수 백오프
      await new Promise(r => setTimeout(r, Math.pow(2, i) * 1000));
    }
  }
}

const result = await fetchWithRetry(
  'https://api.example.com/data'
);

return [{ json: result }];

3.12 Python으로 워크플로우 관리​

Python을 사용하여 n8n 워크플로우를 프로그래밍 방식으로 관리합니다.

n8n API 클라이언트​

import requests
from typing import Dict, List, Optional
import json

class N8nClient:
    """n8n API 클라이언트"""

    def __init__(self, base_url: str, api_key: str):
        self.base_url = base_url.rstrip('/')
        self.api_key = api_key
        self.headers = {
            'X-N8N-API-KEY': api_key,
            'Content-Type': 'application/json'
        }

    def get_workflows(self) -> List[Dict]:
        """모든 워크플로우 조회"""
        response = requests.get(
            f'{self.base_url}/workflows',
            headers=self.headers
        )
        response.raise_for_status()
        return response.json()['data']

    def get_workflow(self, workflow_id: str) -> Dict:
        """특정 워크플로우 조회"""
        response = requests.get(
            f'{self.base_url}/workflows/{workflow_id}',
            headers=self.headers
        )
        response.raise_for_status()
        return response.json()

    def create_workflow(self, workflow: Dict) -> Dict:
        """워크플로우 생성"""
        response = requests.post(
            f'{self.base_url}/workflows',
            headers=self.headers,
            json=workflow
        )
        response.raise_for_status()
        return response.json()

    def update_workflow(self, workflow_id: str, workflow: Dict) -> Dict:
        """워크플로우 업데이트"""
        response = requests.put(
            f'{self.base_url}/workflows/{workflow_id}',
            headers=self.headers,
            json=workflow
        )
        response.raise_for_status()
        return response.json()

    def delete_workflow(self, workflow_id: str) -> bool:
        """워크플로우 삭제"""
        response = requests.delete(
            f'{self.base_url}/workflows/{workflow_id}',
            headers=self.headers
        )
        response.raise_for_status()
        return True

    def activate_workflow(self, workflow_id: str) -> Dict:
        """워크플로우 활성화"""
        workflow = self.get_workflow(workflow_id)
        workflow['active'] = True
        return self.update_workflow(workflow_id, workflow)

    def deactivate_workflow(self, workflow_id: str) -> Dict:
        """워크플로우 비활성화"""
        workflow = self.get_workflow(workflow_id)
        workflow['active'] = False
        return self.update_workflow(workflow_id, workflow)

    def execute_workflow(self, workflow_id: str, data: Optional[Dict] = None) -> Dict:
        """워크플로우 실행"""
        response = requests.post(
            f'{self.base_url}/workflows/{workflow_id}/execute',
            headers=self.headers,
            json={'data': data or {}}
        )
        response.raise_for_status()
        return response.json()

    def get_executions(self, workflow_id: Optional[str] = None) -> List[Dict]:
        """실행 기록 조회"""
        params = {}
        if workflow_id:
            params['workflowId'] = workflow_id

        response = requests.get(
            f'{self.base_url}/executions',
            headers=self.headers,
            params=params
        )
        response.raise_for_status()
        return response.json()['data']

# deploy_workflow.py
import os
import json
from n8n_client import N8nClient

def deploy_workflow(workflow_file: str, env: str = 'production'):
    """워크플로우를 환경에 배포"""

    # 환경별 설정
    configs = {
        'development': {
            'url': 'http://localhost:5678/api/v1',
            'api_key': os.getenv('N8N_DEV_API_KEY')
        },
        'production': {
            'url': 'https://n8n.company.com/api/v1',
            'api_key': os.getenv('N8N_PROD_API_KEY')
        }
    }

    config = configs[env]
    client = N8nClient(config['url'], config['api_key'])

    # 워크플로우 파일 로드
    with open(workflow_file, 'r', encoding='utf-8') as f:
        workflow = json.load(f)

    # 기존 워크플로우 찾기
    existing = None
    workflows = client.get_workflows()
    for wf in workflows:
        if wf['name'] == workflow['name']:
            existing = wf
            break

    if existing:
        # 업데이트
        print(f"Updating workflow: {workflow['name']}")
        result = client.update_workflow(existing['id'], workflow)
        print(f"Updated: {result['id']}")
    else:
        # 새로 생성
        print(f"Creating workflow: {workflow['name']}")
        result = client.create_workflow(workflow)
        print(f"Created: {result['id']}")

    return result

# 사용
if __name__ == '__main__':
    import sys

    if len(sys.argv) < 2:
        print("Usage: python deploy_workflow.py <workflow.json> [env]")
        sys.exit(1)

    workflow_file = sys.argv[1]
    env = sys.argv[2] if len(sys.argv) > 2 else 'production'

    deploy_workflow(workflow_file, env)

# backup_workflows.py
import os
import json
from datetime import datetime
from n8n_client import N8nClient

def backup_workflows(output_dir: str = './backups'):
    """모든 워크플로우를 백업"""

    client = N8nClient(
        base_url=os.getenv('N8N_URL'),
        api_key=os.getenv('N8N_API_KEY')
    )

    # 백업 디렉토리 생성
    timestamp = datetime.now().strftime('%Y%m%d_%H%M%S')
    backup_dir = os.path.join(output_dir, timestamp)
    os.makedirs(backup_dir, exist_ok=True)

    # 모든 워크플로우 조회
    workflows = client.get_workflows()

    print(f"Backing up {len(workflows)} workflows...")

    for workflow in workflows:
        # 워크플로우 상세 정보 조회
        workflow_detail = client.get_workflow(workflow['id'])

        # 파일명 생성 (안전한 파일명)
        safe_name = "".join(
            c if c.isalnum() or c in ('-', '_') else '_'
            for c in workflow['name']
        )
        filename = f"{workflow['id']}_{safe_name}.json"
        filepath = os.path.join(backup_dir, filename)

        # JSON으로 저장
        with open(filepath, 'w', encoding='utf-8') as f:
            json.dump(workflow_detail, f, indent=2, ensure_ascii=False)

        print(f"Backed up: {workflow['name']}")

    print(f"\nBackup completed: {backup_dir}")
    return backup_dir

if __name__ == '__main__':
    backup_workflows()

            failed = [
                ex for ex in executions
                if ex['finished'] == False or ex['stoppedAt'] is None
            ]

            if failed:
                print(f"\n[{datetime.now()}] Found {len(failed)} failed executions:")
                for ex in failed:
                    print(f"  - Workflow: {ex['workflowData']['name']}")
                    print(f"    ID: {ex['id']}")
                    print(f"    Started: {ex['startedAt']}")

                    # 여기서 Slack/이메일 알림 전송 가능

            last_check = datetime.now()
            time.sleep(check_interval)

        except Exception as e:
            print(f"Error: {e}")
            time.sleep(check_interval)

if __name__ == '__main__':
    monitor_workflows()

워크플로우 테스트 스크립트​

# test_workflow.py
import json
from n8n_client import N8nClient
import os

def test_workflow(workflow_id: str, test_cases: list):
    """워크플로우 테스트"""

    client = N8nClient(
        base_url=os.getenv('N8N_URL'),
        api_key=os.getenv('N8N_API_KEY')
    )

    workflow = client.get_workflow(workflow_id)
    print(f"Testing workflow: {workflow['name']}\n")

    results = []

    for i, test_case in enumerate(test_cases, 1):
        print(f"Test case {i}:")
        print(f"  Input: {json.dumps(test_case['input'], indent=2)}")

        try:
            # 워크플로우 실행
            result = client.execute_workflow(
                workflow_id,
                test_case['input']
            )

            # 결과 검증
            success = test_case['validate'](result)

            results.append({
                'case': i,
                'success': success,
                'result': result
            })

            print(f"  Result: {'✅ PASS' if success else '❌ FAIL'}\n")

        except Exception as e:
            print(f"  Result: ❌ ERROR - {e}\n")
            results.append({
                'case': i,
                'success': False,
                'error': str(e)
            })

    # 요약
    passed = sum(1 for r in results if r['success'])
    total = len(results)
    print(f"Summary: {passed}/{total} tests passed")

    return results

# 사용 예시
if __name__ == '__main__':
    test_cases = [
        {
            'input': {'email': 'test@example.com', 'name': 'Test User'},
            'validate': lambda result: 'error' not in result
        },
        {
            'input': {'email': 'invalid-email', 'name': 'Test'},
            'validate': lambda result: 'error' in result
        }
    ]

    test_workflow('workflow-id', test_cases)

연습 4: 에러 처리​

과제: API 호출 실패 시 3번까지 재시도하는 워크플로우를 만드세요.

해답 보기
// Function 노드: 재시도 로직
const maxRetries = 3;
const retryDelay = 2000; // 2초

async function fetchWithRetry(url, retries = 0) {
  try {
    const response = await fetch(url);

    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`);
    }

    return await response.json();

  } catch (error) {
    if (retries < maxRetries) {
      console.log(`Retry ${retries + 1}/${maxRetries}...`);

      // 지수 백오프
      await new Promise(resolve =>
        setTimeout(resolve, retryDelay * Math.pow(2, retries))
      );

      return fetchWithRetry(url, retries + 1);
    }

    throw error;
  }
}

const results = [];

for (const item of items) {
  try {
    const data = await fetchWithRetry(
      `https://api.example.com/data/${item.json.id}`
    );

    results.push({
      json: {
        ...item.json,
        data: data,
        success: true
      }
    });

  } catch (error) {
    results.push({
      json: {
        ...item.json,
        error: error.message,
        success: false
      }
    });
  }
}

return results;

과제: 주문 데이터를 받아서 다음 정보를 계산하세요:

  • 총 매출
  • 제품별 판매량
  • 상위 5개 제품
  • 시간대별 주문 수

// Function 노드: 데이터 집계
const orders = items.map(item => item.json);

// 1. 총 매출
const totalRevenue = orders.reduce(
  (sum, order) => sum + order.amount,
  0
);

// 2. 제품별 판매량
const productSales = orders.reduce((acc, order) => {
  const product = order.productName;
  if (!acc[product]) {
    acc[product] = {
      name: product,
      quantity: 0,
      revenue: 0
    };
  }
  acc[product].quantity += order.quantity;
  acc[product].revenue += order.amount;
  return acc;
}, {});

// 3. 상위 5개 제품
const topProducts = Object.values(productSales)
  .sort((a, b) => b.revenue - a.revenue)
  .slice(0, 5);

// 4. 시간대별 주문 수
const ordersByHour = orders.reduce((acc, order) => {
  const hour = new Date(order.createdAt).getHours();
  acc[hour] = (acc[hour] || 0) + 1;
  return acc;
}, {});

// 결과 반환
return [{
  json: {
    summary: {
      totalOrders: orders.length,
      totalRevenue: totalRevenue,
      averageOrderValue: totalRevenue / orders.length
    },
    productSales: Object.values(productSales),
    topProducts: topProducts,
    ordersByHour: Object.entries(ordersByHour).map(([hour, count]) => ({
      hour: parseInt(hour),
      count: count
    }))
  }
}];
