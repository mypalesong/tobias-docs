2. 기본 개념

n8n을 효과적으로 사용하기 위해 알아야 할 핵심 개념들을 학습합니다. 이 문서에서는 워크플로우의 구조부터 실제 실행까지의 전 과정을 다룹니다.

2.1 워크플로우 개념​

워크플로우(Workflow)는 n8n에서 자동화 작업을 정의하는 기본 단위입니다. 하나 이상의 노드(Node)들이 연결되어 데이터를 처리하고 전달하는 흐름을 만듭니다.

워크플로우의 구성 요소​

[Trigger Node] → [Processing Node] → [Action Node] → [Output Node]
  • 시작점(Trigger): 워크플로우를 실행시키는 이벤트
  • 처리 과정(Processing): 데이터 변환, 필터링, 조건 분기
  • 실행(Action): 외부 서비스 호출, 데이터 저장
  • 종료(Output): 결과 반환 또는 저장

워크플로우 생명주기​

  1. 설계(Design): 노드를 추가하고 연결
  2. 테스트(Test): 수동으로 실행하여 검증
  3. 활성화(Activate): 자동 실행 시작
  4. 모니터링(Monitor): 실행 로그 확인
  5. 최적화(Optimize): 성능 개선 및 에러 처리
팁

워크플로우는 재사용 가능합니다. 한 번 만든 워크플로우를 JSON으로 내보내어 다른 n8n 인스턴스에서 가져올 수 있습니다.

2.2 노드 타입​

n8n의 노드는 기능과 역할에 따라 여러 타입으로 분류됩니다.

Trigger 노드​

워크플로우를 시작하는 노드입니다. 특정 이벤트가 발생하면 워크플로우가 자동으로 실행됩니다.

주요 Trigger 노드:

노드 이름설명사용 예시
WebhookHTTP 요청 수신API 엔드포인트 생성
Schedule Trigger정해진 시간에 실행매일 오전 9시 리포트 생성
CronCron 표현식으로 스케줄링복잡한 스케줄 패턴
Manual Trigger수동 실행 버튼테스트 및 디버깅
Email Trigger (IMAP)이메일 수신 시첨부파일 자동 처리
# n8n Webhook을 Python에서 호출하는 예제
import requests
import json

webhook_url = "https://your-n8n-instance.com/webhook/test"

data = {
    "event": "user_signup",
    "user": {
        "name": "홍길동",
        "email": "hong@example.com"
    }
}

response = requests.post(
    webhook_url,
    json=data,
    headers={"Content-Type": "application/json"}
)

print(f"Status: {response.status_code}")
print(f"Response: {response.json()}")

Regular 노드 (처리 노드)​

데이터를 처리하고 변환하는 노드입니다.

카테고리별 Regular 노드:

1. 데이터 변환

  • Set: 데이터 필드 설정 및 변경
  • Function: JavaScript 코드로 자유로운 데이터 처리
  • Code: Python/JavaScript 실행 환경

2. 흐름 제어

  • IF: 조건에 따라 분기
  • Switch: 다중 조건 분기
  • Merge: 여러 경로의 데이터 병합

3. 데이터 작업

  • Item Lists: 배열 데이터 분할/병합
  • Filter: 조건에 맞는 항목만 필터링
  • Sort: 데이터 정렬

4. 외부 서비스

  • HTTP Request: REST API 호출
  • Database: SQL 쿼리 실행
  • 각종 앱 통합 노드: Slack, Google Sheets, Notion 등

Output 노드​

워크플로우의 결과를 출력하거나 저장하는 노드입니다.

  • Respond to Webhook: Webhook 호출자에게 응답 반환
  • Write to File: 파일 시스템에 저장
  • Database Insert: 데이터베이스에 저장
  • Send Email: 이메일 발송
팁

하나의 워크플로우에 여러 개의 출력 노드를 사용할 수 있습니다. 예를 들어, 데이터를 데이터베이스에 저장하면서 동시에 Slack으로 알림을 보낼 수 있습니다.

2.3 커넥션과 데이터 흐름​

커넥션 타입​

노드 간 연결은 데이터가 흐르는 경로를 정의합니다.

┌─────────────┐
│   Node A    │
└──┬──────┬───┘
   │      │
   │      └─────── Success Path (초록색)
   │
   └──────────── Error Path (빨간색)

1. Main Connection (주 연결)

  • 일반적인 데이터 흐름
  • 초록색 선으로 표시
  • 성공적으로 실행된 결과가 전달됨

2. Error Connection (에러 연결)

  • 노드 실행 실패 시 활성화
  • 빨간색 선으로 표시
  • 에러 처리 로직 구현 가능

데이터 전달 방식​

단일 항목 전달:

{
  "json": {
    "name": "홍길동",
    "age": 30
  }
}

다중 항목 전달:

[
  {
    "json": {
      "name": "홍길동",
      "age": 30
    }
  },
  {
    "json": {
      "name": "김철수",
      "age": 25
    }
  }
]

실행 순서​

n8n은 기본적으로 왼쪽에서 오른쪽, 위에서 아래 순서로 노드를 실행합니다.

# n8n API를 통해 워크플로우 실행하기
import requests

n8n_url = "https://your-n8n-instance.com"
api_key = "your-api-key"

# 워크플로우 실행
workflow_id = "123"
response = requests.post(
    f"{n8n_url}/api/v1/workflows/{workflow_id}/execute",
    headers={
        "X-N8N-API-KEY": api_key,
        "Content-Type": "application/json"
    },
    json={
        "data": {
            "input": "test data"
        }
    }
)

execution_id = response.json()["data"]["executionId"]
print(f"Execution ID: {execution_id}")

2.4 실행 모드​

n8n 워크플로우는 세 가지 방식으로 실행됩니다.

Manual (수동 실행)​

  • 사용 시기: 개발 및 테스트 단계
  • 실행 방법: 에디터에서 "Execute Workflow" 버튼 클릭
  • 특징:
    • 즉시 실행 및 결과 확인
    • 각 노드의 입출력 데이터 확인 가능
    • 실행 기록에 남지 않음
[Manual Trigger] 버튼 클릭 → 워크플로우 실행 → 즉시 결과 표시

Active (활성화 모드)​

  • 사용 시기: 프로덕션 환경
  • 실행 방법: 워크플로우 활성화 스위치 ON
  • 특징:
    • Trigger 조건이 충족되면 자동 실행
    • 백그라운드에서 계속 대기
    • 실행 기록 자동 저장
// Schedule Trigger 설정 예시
{
  "mode": "everyX",
  "value": 15,
  "unit": "minutes"
}
// → 15분마다 자동 실행

Webhook (웹훅 모드)​

  • 사용 시기: 외부 시스템과 연동
  • 실행 방법: HTTP 요청 수신
  • 특징:
    • 실시간 이벤트 처리
    • POST, GET, PUT, DELETE 지원
    • 요청자에게 응답 반환 가능
# Webhook 응답 예제
import requests

webhook_url = "https://your-n8n-instance.com/webhook/process-order"

order_data = {
    "order_id": "ORD-12345",
    "items": [
        {"product": "노트북", "quantity": 1, "price": 1500000},
        {"product": "마우스", "quantity": 2, "price": 30000}
    ],
    "total": 1560000
}

response = requests.post(webhook_url, json=order_data)

if response.status_code == 200:
    result = response.json()
    print(f"주문 처리 완료: {result['message']}")
else:
    print(f"오류 발생: {response.status_code}")
팁

개발 중에는 Manual 모드로 테스트하고, 검증이 끝나면 Active 모드로 전환하세요. Webhook은 외부 시스템과 통합할 때 사용합니다.

2.5 데이터 구조​

n8n에서 노드 간에 전달되는 데이터는 특정 구조를 가집니다.

$json​

가장 일반적인 데이터 형식으로, JSON 객체가 저장됩니다.

// Set 노드에서 데이터 설정
{
  "user": {
    "id": 1,
    "name": "홍길동",
    "email": "hong@example.com",
    "created_at": "{{ $now }}"
  },
  "metadata": {
    "source": "signup_form",
    "ip": "192.168.1.1"
  }
}

데이터 참조 방법:

// 이전 노드의 데이터 접근
{{ $json.user.name }}           // "홍길동"
{{ $json.user.email }}          // "hong@example.com"
{{ $json.metadata.source }}     // "signup_form"

// 특정 노드의 데이터 접근
{{ $node["HTTP Request"].json.result }}
{{ $node["Set User Data"].json.user.id }}

$binary​

파일, 이미지, PDF 등 바이너리 데이터를 처리할 때 사용됩니다.

// 바이너리 데이터 구조
{
  "data": "base64EncodedString...",
  "mimeType": "image/png",
  "fileExtension": "png",
  "fileName": "screenshot.png",
  "fileSize": 45678
}

사용 예시:

# n8n에서 파일 업로드 처리
import requests
import base64

# 로컬 파일을 n8n Webhook으로 전송
with open("report.pdf", "rb") as file:
    file_content = base64.b64encode(file.read()).decode()

webhook_data = {
    "fileName": "report.pdf",
    "fileData": file_content,
    "mimeType": "application/pdf"
}

response = requests.post(
    "https://your-n8n-instance.com/webhook/upload",
    json=webhook_data
)

$node​

특정 노드의 출력 데이터를 참조할 때 사용합니다.

// $node를 사용한 데이터 접근
{{ $node["Fetch Users"].json }}              // 전체 JSON 데이터
{{ $node["Fetch Users"].json.items }}        // items 배열
{{ $node["Fetch Users"].json.items[0].name }} // 첫 번째 아이템의 name

// 실행 정보 접근
{{ $node["HTTP Request"].runIndex }}         // 실행 인덱스
{{ $node["HTTP Request"].parameter }}        // 노드 파라미터

복합 데이터 처리 예제​

// Function 노드에서 복합 데이터 처리
// 여러 노드의 데이터를 조합

const userData = $node["Get User"].json;
const orderData = $node["Get Orders"].json;
const settingsData = $node["Get Settings"].json;

return {
  json: {
    user: {
      id: userData.id,
      name: userData.name,
      email: userData.email
    },
    orders: orderData.map(order => ({
      order_id: order.id,
      total: order.total,
      status: order.status
    })),
    preferences: {
      timezone: settingsData.timezone,
      language: settingsData.language
    },
    summary: {
      total_orders: orderData.length,
      total_spent: orderData.reduce((sum, o) => sum + o.total, 0)
    }
  }
};
팁

복잡한 데이터 변환이 필요할 때는 Function 노드나 Code 노드를 사용하세요. JavaScript나 Python으로 자유롭게 데이터를 처리할 수 있습니다.

2.6 Credentials 관리​

Credentials는 외부 서비스에 접근하기 위한 인증 정보를 안전하게 저장합니다.

Credential 타입​

1. OAuth2

  • Google, GitHub, Slack 등
  • 사용자 권한 위임 방식
  • 자동 토큰 갱신 지원

2. API Key

  • 단순 API 키 인증
  • Header 또는 Query Parameter로 전달

3. Username/Password

  • 기본 인증 방식
  • SMTP, Database 연결 등

4. Custom

  • 사용자 정의 인증 방식
  • 복잡한 인증 로직 구현

Credential 생성 방법​

  1. 메뉴에서 Credentials 클릭
  2. "New" 버튼 클릭
  3. 서비스 타입 선택 (예: Google Sheets API)
  4. 인증 정보 입력
  5. "Save" 버튼 클릭
# n8n API를 통한 Credential 관리 (읽기 전용)
import requests

n8n_url = "https://your-n8n-instance.com"
api_key = "your-api-key"

# Credentials 목록 조회
response = requests.get(
    f"{n8n_url}/api/v1/credentials",
    headers={"X-N8N-API-KEY": api_key}
)

credentials = response.json()["data"]

for cred in credentials:
    print(f"ID: {cred['id']}")
    print(f"Name: {cred['name']}")
    print(f"Type: {cred['type']}")
    print("---")

보안 모범 사례​

  1. 절대 하드코딩하지 않기

    // ❌ 나쁜 예
    const apiKey = "sk-1234567890abcdef";

    // ✅ 좋은 예
    const apiKey = $credentials.myApiKey;

  2. 최소 권한 원칙

    • 필요한 권한만 부여
    • 읽기 전용이 가능하면 쓰기 권한 제외

  3. 정기적인 Credential 교체

    • 중요한 서비스는 3개월마다 교체
    • 퇴사자 발생 시 즉시 교체

  4. 환경별 분리

    • 개발/스테이징/프로덕션 환경별로 다른 Credential 사용
팁

Credentials는 암호화되어 저장되며, n8n 인스턴스 외부로 노출되지 않습니다. 워크플로우를 JSON으로 내보낼 때도 Credential 정보는 제외됩니다.

환경별 설정 관리​

# Python 스크립트로 환경별 설정 관리
import os
import json

def get_n8n_config(environment):
    """환경별 n8n 설정 반환"""
    configs = {
        "development": {
            "n8n_host": "localhost",
            "n8n_port": 5678,
            "protocol": "http",
            "db_type": "sqlite",
            "log_level": "debug"
        },
        "staging": {
            "n8n_host": "staging.example.com",
            "n8n_port": 443,
            "protocol": "https",
            "db_type": "postgres",
            "log_level": "info"
        },
        "production": {
            "n8n_host": "n8n.example.com",
            "n8n_port": 443,
            "protocol": "https",
            "db_type": "postgres",
            "log_level": "warn"
        }
    }

    return configs.get(environment, configs["development"])

# 사용 예시
env = os.getenv("APP_ENV", "development")
config = get_n8n_config(env)

print(f"Environment: {env}")
print(f"n8n URL: {config['protocol']}://{config['n8n_host']}:{config['n8n_port']}")
팁

환경 변수를 사용하면 같은 워크플로우를 여러 환경(개발, 스테이징, 프로덕션)에서 다른 설정으로 실행할 수 있습니다.

2.8 워크플로우 설정​

워크플로우 수준에서 설정할 수 있는 다양한 옵션들을 알아봅니다.

Timezone (시간대)​

워크플로우의 시간대를 설정합니다. Schedule Trigger와 날짜/시간 관련 작업에 영향을 줍니다.

설정 위치: Workflow Settings → Timezone

// 시간대별 실행 시간 예시
// Timezone: Asia/Seoul
// Cron: 0 9 * * *  → 매일 오전 9시 (한국 시간)

// Timezone: America/New_York
// Cron: 0 9 * * *  → 매일 오전 9시 (뉴욕 시간)

자주 사용하는 시간대:

  • Asia/Seoul - 한국
  • America/New_York - 미국 동부
  • America/Los_Angeles - 미국 서부
  • Europe/London - 영국
  • UTC - 협정 세계시

Execution Order (실행 순서)​

여러 노드가 병렬로 실행될 수 있을 때 우선순위를 제어합니다.

설정 위치: Workflow Settings → Execution Order

옵션:

  • v0 (레거시): 위에서 아래, 왼쪽에서 오른쪽
  • v1 (최신): 더 효율적인 병렬 실행
       ┌──────────┐
       │ Start    │
       └────┬─────┘
            │
     ┌──────┴──────┐
     ↓             ↓
┌─────────┐   ┌─────────┐
│ Node A  │   │ Node B  │  ← 병렬 실행
└────┬────┘   └────┬────┘
     └──────┬──────┘
            ↓
       ┌─────────┐
       │ Merge   │
       └─────────┘

기타 워크플로우 설정​

Save Manual Executions

  • 수동 실행 결과를 저장할지 선택
  • 기본값: false

Save Execution Progress

  • 실행 중간 과정을 저장
  • 긴 워크플로우 디버깅에 유용

Timeout Workflow

  • 워크플로우 최대 실행 시간 설정
  • 무한 루프 방지
# n8n API로 워크플로우 설정 업데이트
import requests

n8n_url = "https://your-n8n-instance.com"
api_key = "your-api-key"
workflow_id = "123"

settings = {
    "settings": {
        "executionOrder": "v1",
        "saveManualExecutions": True,
        "saveExecutionProgress": True,
        "executionTimeout": 300,  # 5분
        "timezone": "Asia/Seoul"
    }
}

response = requests.patch(
    f"{n8n_url}/api/v1/workflows/{workflow_id}",
    headers={
        "X-N8N-API-KEY": api_key,
        "Content-Type": "application/json"
    },
    json=settings
)

print(f"Settings updated: {response.status_code}")

Python으로 워크플로우 내보내기/가져오기​

# 워크플로우를 JSON으로 내보내고 다른 인스턴스에 가져오기
import requests
import json

n8n_url = "https://your-n8n-instance.com"
api_key = "your-api-key"

# 워크플로우 내보내기
workflow_id = "123"
response = requests.get(
    f"{n8n_url}/api/v1/workflows/{workflow_id}",
    headers={"X-N8N-API-KEY": api_key}
)

workflow_data = response.json()["data"]

# JSON 파일로 저장
with open("weather_workflow.json", "w", encoding="utf-8") as f:
    json.dump(workflow_data, f, indent=2, ensure_ascii=False)

print("워크플로우가 weather_workflow.json 파일로 저장되었습니다.")

# 다른 인스턴스에 가져오기
new_n8n_url = "https://another-n8n-instance.com"
new_api_key = "another-api-key"

with open("weather_workflow.json", "r", encoding="utf-8") as f:
    workflow_to_import = json.load(f)

# ID 제거 (새로운 ID가 자동 생성됨)
workflow_to_import.pop("id", None)

response = requests.post(
    f"{new_n8n_url}/api/v1/workflows",
    headers={
        "X-N8N-API-KEY": new_api_key,
        "Content-Type": "application/json"
    },
    json=workflow_to_import
)

new_workflow = response.json()["data"]
print(f"워크플로우가 새 인스턴스에 생성되었습니다. ID: {new_workflow['id']}")
팁

워크플로우를 JSON 파일로 저장해두면 버전 관리, 백업, 팀 공유에 유용합니다. Git 저장소에 커밋하여 변경 이력을 추적할 수도 있습니다.

연습 1: 기본 데이터 처리​

  {"name": "김철수", "age": 17},
  {"name": "이영희", "age": 30},
  {"name": "박민수", "age": 19}
]
💡 해답 보기
[Manual Trigger]
    ↓
[Set - Input Data]
    ↓

    ↓
[View Output]

// 입력 데이터

  {"name": "홍길동", "age": 25},
  {"name": "김철수", "age": 17},
  {"name": "이영희", "age": 30},

// 성인 필터링 및 이름 대문자 변환
const adults = users
  .filter(user => user.age >= 20)
  .map(user => ({
    name: user.name.toUpperCase(),
    age: user.age
  }));

// 각 사용자를 개별 아이템으로 반환
return adults.map(user => ({ json: user }));

Switch 노드 설정:

Rule 1: {{ $json.amount < 100000 }}

  • Output: 0 (일반 처리)

Rule 2: {{ $json.amount >= 100000 && $json.amount < 500000 }}

  • Output: 1 (관리자 승인)

Rule 3: {{ $json.amount >= 500000 }}

연습 4: 에러 처리​

문제: HTTP 요청이 실패할 수 있는 상황을 처리하는 워크플로우를 만드세요. 요청이 실패하면 3번까지 재시도하고, 그래도 실패하면 에러 로그를 남기세요.

💡 해답 보기
[Manual Trigger]
    ↓
[HTTP Request] ──(success)──→ [Process Data]
    │
    └──(error)──→ [Function - Check Retry Count]
                        ├─→ [Retry] ──→ (back to HTTP Request)
                        └─→ [Log Error]

HTTP Request 노드 설정:

  • Continue On Fail: true
  • Retry On Fail: true
  • Max Retries: 3
  • Retry Interval: 1000 (1초)

Function 노드 (에러 처리):

const error = $input.first().json;

// 에러 로그 생성
const errorLog = {
  timestamp: new Date().toISOString(),
  error_message: error.message || "Unknown error",
  retry_count: error.retryCount || 0,
  max_retries: 3,
  request_url: error.request?.url || "N/A",
  status_code: error.response?.status || "N/A"
};

console.error("API Request Failed:", errorLog);

return {
  json: errorLog
};

테스트 방법:

  • 잘못된 URL로 HTTP Request 설정
  • 워크플로우 실행하여 재시도 로직 확인
