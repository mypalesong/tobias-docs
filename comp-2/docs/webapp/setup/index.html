1. 프로젝트 설정

Vue 3와 FastAPI를 사용한 풀스택 웹 애플리케이션 개발을 위한 프로젝트 설정 가이드입니다.

1.1 개발 환경 요구사항​

프로젝트 개발을 시작하기 전에 다음 도구들이 설치되어 있어야 합니다.

Python​

FastAPI 백엔드 개발을 위해 Python 3.9 이상이 필요합니다.

# Python 버전 확인
python --version
# 또는
python3 --version
Python 설치

Python이 설치되어 있지 않다면 python.org에서 다운로드하세요. Windows 사용자는 설치 시 "Add Python to PATH" 옵션을 반드시 체크해야 합니다.

Node.js​

Vue 3 프론트엔드 개발을 위해 Node.js 18 이상이 필요합니다.

# Node.js 버전 확인
node --version

# npm 버전 확인
npm --version
Node.js 설치

Node.js가 설치되어 있지 않다면 nodejs.org에서 LTS 버전을 다운로드하세요. npm은 Node.js와 함께 자동으로 설치됩니다.

Git​

버전 관리를 위해 Git이 필요합니다.

# Git 버전 확인
git --version

통합 개발 환경 (IDE)​

다음 IDE 중 하나를 권장합니다:

  • Visual Studio Code (권장)
    • Python Extension
    • Volar (Vue Language Features)
    • TypeScript Vue Plugin (Volar)
  • PyCharm Professional (Python 개발에 최적화)
  • WebStorm (JavaScript/TypeScript 개발에 최적화)
VS Code 추천 확장 프로그램
  • Python
  • Pylance
  • Vue Language Features (Volar)
  • TypeScript Vue Plugin (Volar)
  • ESLint
  • Prettier
  • GitLens

1.2 Backend 프로젝트 설정​

FastAPI 기반 백엔드 프로젝트를 설정합니다.

프로젝트 디렉토리 생성​

# 프로젝트 루트 디렉토리 생성
mkdir my-webapp
cd my-webapp

# 백엔드 디렉토리 생성
mkdir backend
cd backend

Python 가상환경 설정​

가상환경을 사용하면 프로젝트별로 독립적인 패키지 환경을 유지할 수 있습니다.

# 가상환경 생성
python -m venv venv

# 가상환경 활성화 (Windows)
venv\Scripts\activate

# 가상환경 활성화 (macOS/Linux)
source venv/bin/activate
가상환경 활성화 확인

가상환경이 활성화되면 터미널 프롬프트 앞에 (venv)가 표시됩니다.

FastAPI 및 필수 패키지 설치​

# FastAPI 및 ASGI 서버 설치
pip install fastapi uvicorn[standard]

# 추가 필수 패키지 설치
pip install python-dotenv pydantic pydantic-settings
pip install sqlalchemy alembic
pip install python-multipart
pip install httpx pytest pytest-asyncio

requirements.txt 생성​

설치된 패키지 목록을 파일로 저장합니다.

pip freeze > requirements.txt

.env 설정​

환경 변수를 관리하기 위한 .env 파일을 생성합니다.

backend/.env:

# Application Settings
APP_NAME="My WebApp"
APP_VERSION="1.0.0"
DEBUG=True

# Server Settings
HOST=0.0.0.0
PORT=8000

# Database Settings
DATABASE_URL=sqlite:///./app.db
# PostgreSQL 예시: postgresql://user:password@localhost/dbname

# Security Settings
SECRET_KEY=your-secret-key-here-change-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30

# CORS Settings

SECRET_KEY 생성

안전한 SECRET_KEY를 생성하려면 다음 명령을 사용하세요:

python -c "import secrets; print(secrets.token_urlsafe(32))"

기본 FastAPI 애플리케이션 생성​

backend/main.py:

from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from pydantic_settings import BaseSettings
from dotenv import load_dotenv
import os

# 환경 변수 로드
load_dotenv()

class Settings(BaseSettings):
    app_name: str = os.getenv("APP_NAME", "My WebApp")
    app_version: str = os.getenv("APP_VERSION", "1.0.0")
    debug: bool = os.getenv("DEBUG", "False") == "True"
    cors_origins: list = ["http://localhost:3000", "http://localhost:5173"]

    class Config:
        env_file = ".env"

settings = Settings()

# FastAPI 앱 초기화
app = FastAPI(
    title=settings.app_name,
    version=settings.app_version,
    debug=settings.debug
)

# CORS 미들웨어 설정
app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.cors_origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# 기본 라우트
@app.get("/")
async def root():
    return {
        "message": "Welcome to My WebApp API",
        "version": settings.app_version
    }

@app.get("/health")
async def health_check():
    return {"status": "healthy"}

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(
        "main:app",
        host=os.getenv("HOST", "0.0.0.0"),
        port=int(os.getenv("PORT", 8000)),
        reload=settings.debug
    )

API 클라이언트 설정​

frontend/src/api/client.ts:

import axios from 'axios'

const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
})

// 요청 인터셉터
apiClient.interceptors.request.use(
  (config) => {
    // 토큰이 있다면 헤더에 추가
    const token = localStorage.getItem('access_token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  (error) => {
    return Promise.reject(error)
  }
)

// 응답 인터셉터
apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // 인증 오류 처리
      localStorage.removeItem('access_token')
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)

export default apiClient

주요 디렉토리 설명​

Backend 디렉토리​

  • app/api: API 엔드포인트와 라우터를 관리합니다.
  • app/core: 설정, 보안, 공통 유틸리티를 포함합니다.
  • app/models: SQLAlchemy를 사용한 데이터베이스 모델을 정의합니다.
  • app/schemas: Pydantic을 사용한 요청/응답 스키마를 정의합니다.
  • app/crud: 데이터베이스 CRUD(Create, Read, Update, Delete) 작업을 처리합니다.
  • app/db: 데이터베이스 연결 및 세션 관리를 담당합니다.

Frontend 디렉토리​

  • src/api: 백엔드 API와 통신하는 클라이언트 코드입니다.
  • src/components: 재사용 가능한 Vue 컴포넌트를 저장합니다.
  • src/composables: Vue 3 Composition API 로직을 재사용 가능한 함수로 추출합니다.
  • src/stores: Pinia를 사용한 전역 상태 관리 스토어입니다.
  • src/router: Vue Router를 사용한 라우팅 설정입니다.
  • src/views: 페이지 수준의 컴포넌트입니다.
  • src/types: TypeScript 타입 정의 파일입니다.

1.5 개발 서버 실행 및 핫 리로드​

Backend 서버 실행​

# backend 디렉토리로 이동
cd backend

# 가상환경 활성화 (Windows)
venv\Scripts\activate

# 가상환경 활성화 (macOS/Linux)
source venv/bin/activate

# Uvicorn으로 서버 실행 (핫 리로드 활성화)
uvicorn main:app --reload --host 0.0.0.0 --port 8000

또는 Python으로 직접 실행:

python main.py

서버가 실행되면 다음 URL에서 확인할 수 있습니다:

  • API: http://localhost:8000
  • 자동 생성된 API 문서 (Swagger UI): http://localhost:8000/docs
  • 대체 API 문서 (ReDoc): http://localhost:8000/redoc
핫 리로드

--reload 옵션을 사용하면 코드 변경 시 서버가 자동으로 재시작됩니다. 프로덕션 환경에서는 이 옵션을 사용하지 마세요.

Frontend 서버 실행​

# frontend 디렉토리로 이동
cd frontend

# 개발 서버 실행
npm run dev

서버가 실행되면 다음 URL에서 확인할 수 있습니다:

  • 웹 애플리케이션: http://localhost:5173
Vite 핫 모듈 교체 (HMR)

Vite는 기본적으로 핫 모듈 교체(HMR)를 지원합니다. 코드를 수정하면 페이지를 새로고침하지 않고도 변경사항이 즉시 반영됩니다.

동시에 두 서버 실행​

개발 중에는 백엔드와 프론트엔드 서버를 동시에 실행해야 합니다.

방법 1: 별도의 터미널 사용

두 개의 터미널을 열어 각각 백엔드와 프론트엔드 서버를 실행합니다.

방법 2: concurrently 패키지 사용

프로젝트 루트에서:

# 전역으로 설치
npm install -g concurrently

# package.json 스크립트 추가
{
  "scripts": {
    "dev:backend": "cd backend && venv/Scripts/activate && uvicorn main:app --reload",
    "dev:frontend": "cd frontend && npm run dev",
    "dev": "concurrently \"npm run dev:backend\" \"npm run dev:frontend\""
  }
}

# 동시 실행
npm run dev

1.6 환경 변수 관리​

환경 변수를 사용하면 코드에 민감한 정보를 하드코딩하지 않고 환경별로 다른 설정을 사용할 수 있습니다.

Backend 환경 변수​

backend/.env 구조:

# 개발 환경
DEBUG=True
DATABASE_URL=sqlite:///./dev.db

# 프로덕션 환경에서는
DEBUG=False
DATABASE_URL=postgresql://user:password@localhost/prod_db
SECRET_KEY=very-secure-random-key-here

Python에서 환경 변수 사용:

import os
from dotenv import load_dotenv
from pydantic_settings import BaseSettings

load_dotenv()

class Settings(BaseSettings):
    database_url: str = os.getenv("DATABASE_URL")
    secret_key: str = os.getenv("SECRET_KEY")
    debug: bool = os.getenv("DEBUG", "False") == "True"

    class Config:
        env_file = ".env"

settings = Settings()

Frontend 환경 변수​

Vite에서는 VITE_ 접두사가 붙은 환경 변수만 클라이언트에 노출됩니다.

frontend/.env.development:

VITE_API_BASE_URL=http://localhost:8000
VITE_APP_TITLE=My WebApp (Development)
VITE_ENABLE_LOGS=true

frontend/.env.production:

VITE_API_BASE_URL=https://api.mywebapp.com
VITE_APP_TITLE=My WebApp
VITE_ENABLE_LOGS=false

TypeScript에서 환경 변수 사용:

// frontend/src/config/index.ts
export const config = {
  apiBaseUrl: import.meta.env.VITE_API_BASE_URL,
  appTitle: import.meta.env.VITE_APP_TITLE,
  enableLogs: import.meta.env.VITE_ENABLE_LOGS === 'true',
}

// 타입 안정성을 위한 타입 정의
interface ImportMetaEnv {
  readonly VITE_API_BASE_URL: string
  readonly VITE_APP_TITLE: string
  readonly VITE_ENABLE_LOGS: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}
환경 변수 보안
  • .env 파일은 절대 Git에 커밋하지 마세요.
  • .env.example 파일을 만들어 필요한 환경 변수 목록을 공유하세요.
  • 프로덕션 환경에서는 서버의 환경 변수나 비밀 관리 서비스를 사용하세요.

문제 2: Backend API 엔드포인트 추가​

/api/info 엔드포인트를 추가하여 다음 정보를 JSON으로 반환하세요:

  • app_name: 애플리케이션 이름
  • version: 버전 정보
  • environment: 개발/프로덕션 환경
  • python_version: Python 버전
해답

backend/main.py에 추가:

import sys
from fastapi import FastAPI

@app.get("/api/info")
async def api_info():
    return {
        "app_name": settings.app_name,
        "version": settings.app_version,
        "environment": "development" if settings.debug else "production",
        "python_version": f"{sys.version_info.major}.{sys.version_info.minor}.{sys.version_info.micro}"
    }

테스트:

문제 3: Frontend API 서비스 생성​

백엔드의 /api/info 엔드포인트를 호출하는 TypeScript 서비스를 작성하세요.

해답

frontend/src/api/services/info.ts:

import apiClient from '../client'

export interface ApiInfo {
  app_name: string
  version: string
  environment: string
  python_version: string
}

export const infoService = {
  async getApiInfo(): Promise<ApiInfo> {
    const response = await apiClient.get<ApiInfo>('/api/info')
    return response.data
  }
}

frontend/src/views/Home.vue에서 사용:

<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { infoService, type ApiInfo } from '@/api/services/info'

const apiInfo = ref<ApiInfo | null>(null)
const loading = ref(true)
const error = ref<string | null>(null)

onMounted(async () => {
  try {
    apiInfo.value = await infoService.getApiInfo()
  } catch (err) {
    error.value = 'API 정보를 가져오는데 실패했습니다.'
    console.error(err)
  } finally {
    loading.value = false
  }
})
</script>

<template>
  <div class="home">
    <h1>API 정보</h1>
    <div v-if="loading">로딩 중...</div>
    <div v-else-if="error">{{ error }}</div>
    <div v-else-if="apiInfo">
      <p><strong>앱 이름:</strong> {{ apiInfo.app_name }}</p>
      <p><strong>버전:</strong> {{ apiInfo.version }}</p>
      <p><strong>환경:</strong> {{ apiInfo.environment }}</p>
      <p><strong>Python 버전:</strong> {{ apiInfo.python_version }}</p>
    </div>
  </div>
</template>

문제 4: 환경별 설정 관리​

개발 환경과 프로덕션 환경에서 다른 데이터베이스 URL을 사용하도록 설정하세요.

해답

backend/.env.development:

DEBUG=True
DATABASE_URL=sqlite:///./dev.db
SECRET_KEY=dev-secret-key-not-for-production

backend/.env.production:

DEBUG=False
DATABASE_URL=postgresql://user:password@prod-server:5432/myapp_db
SECRET_KEY=very-secure-production-key-change-this

backend/main.py 수정:

import os
from dotenv import load_dotenv

# 환경에 따라 다른 .env 파일 로드
env_file = '.env.production' if os.getenv('ENV') == 'production' else '.env.development'
load_dotenv(env_file)

class Settings(BaseSettings):
    database_url: str
    secret_key: str
    debug: bool = False

    class Config:
        env_file = env_file

settings = Settings()

실행 시 환경 지정:

# 개발 환경 (기본값)
uvicorn main:app --reload

# 프로덕션 환경
ENV=production uvicorn main:app --host 0.0.0.0 --port 8000

Windows PowerShell에서:

# 개발 환경
uvicorn main:app --reload

# 프로덕션 환경
$env:ENV="production"; uvicorn main:app --host 0.0.0.0 --port 8000

문제 5: Git 워크플로우 실습​

새로운 기능을 개발하기 위한 feature 브랜치를 생성하고, 변경사항을 커밋한 후 main 브랜치에 병합하세요.

해답
# 1. 새로운 feature 브랜치 생성
git checkout -b feature/add-user-endpoint

# 2. 파일 수정 (예: 새로운 엔드포인트 추가)
# backend/main.py를 편집...

# 3. 변경사항 확인
git status
git diff

# 4. 변경사항 스테이징
git add backend/main.py

# 5. 커밋
git commit -m "feat: Add user endpoint with authentication"

# 6. main 브랜치로 전환
git checkout main

# 7. feature 브랜치 병합
git merge feature/add-user-endpoint

# 8. feature 브랜치 삭제 (옵션)
git branch -d feature/add-user-endpoint

# 9. 원격 저장소에 푸시 (옵션)
git push origin main

커밋 메시지 규칙 (Conventional Commits):

  • feat: 새로운 기능 추가
  • fix: 버그 수정
  • docs: 문서 수정
  • style: 코드 포맷팅 (기능 변경 없음)
  • refactor: 코드 리팩토링
  • test: 테스트 코드 추가/수정
  • chore: 빌드 프로세스 또는 도구 설정 변경

이제 Vue 3와 FastAPI를 사용한 풀스택 웹 애플리케이션 개발을 위한 기본 프로젝트 설정이 완료되었습니다. 다음 섹션에서는 실제 기능 구현에 대해 학습합니다.
