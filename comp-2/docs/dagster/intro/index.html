Dagster 워크플로우 가이드

Python 기반의 현대적인 데이터 오케스트레이션 프레임워크 Dagster를 배워보세요.

Dagster란?​

Dagster는 데이터 파이프라인, ETL 작업, 머신러닝 워크플로우를 구축하고 관리하기 위한 오픈소스 오케스트레이션 플랫폼입니다.

✅ Software-defined Assets - 데이터를 코드로 정의
✅ 타입 안정성 - Python 타입 힌트 활용
✅ 테스트 가능 - 유닛 테스트 및 통합 테스트
✅ 관측 가능성 - 내장 UI와 모니터링
✅ 개발자 경험 - 뛰어난 로컬 개발 환경
✅ 확장성 - 대규모 프로덕션 배포

Dagster vs 다른 도구들​

vs Airflow​

**Airflow**
- Task 중심 (어떻게 실행할지)
- DAG 정의가 복잡
- 로컬 개발이 어려움
- 의존성 관리가 암묵적

**Dagster**
- Asset 중심 (무엇을 생성할지)
- 직관적인 Python 코드
- 뛰어난 로컬 개발 환경
- 명시적 의존성 그래프

vs Prefect​

**Prefect**
- Flow 중심
- 동적 워크플로우 강점
- 클라우드 우선

**Dagster**
- Asset 중심
- 정적 워크플로우 최적화
- 로컬 우선, 클라우드 지원
- 더 강력한 타입 시스템

핵심 개념​

1. Software-defined Assets​

데이터 자산을 코드로 정의합니다.

from dagster import asset
import pandas as pd

@asset
def orders() -> pd.DataFrame:
    """주문 데이터를 CSV에서 로드"""
    return pd.read_csv("orders.csv")

@asset
def daily_sales(orders: pd.DataFrame) -> pd.DataFrame:
    """일별 매출 집계"""
    return orders.groupby("date")["amount"].sum().reset_index()

특징:

  • 데이터 자산이 일급 객체
  • 의존성이 명확히 표현됨
  • 자동 데이터 계보(lineage) 추적

3. Resources​

외부 서비스 연결을 관리합니다.

from dagster import resource, Definitions
import boto3

@resource
def s3_client():
    return boto3.client('s3')

@asset(required_resource_keys={"s3"})
def data_from_s3(context):
    s3 = context.resources.s3
    # S3에서 데이터 로드
    return s3.get_object(Bucket='my-bucket', Key='data.csv')

defs = Definitions(
    assets=[data_from_s3],
    resources={"s3": s3_client}
)

4. I/O Managers​

데이터 저장 및 로드를 담당합니다.

from dagster import IOManager, io_manager
import pickle

class PickleIOManager(IOManager):
    def handle_output(self, context, obj):
        # 데이터 저장
        with open(f"{context.asset_key}.pkl", "wb") as f:
            pickle.dump(obj, f)

    def load_input(self, context):
        # 데이터 로드
        with open(f"{context.asset_key}.pkl", "rb") as f:
            return pickle.load(f)

@io_manager
def pickle_io_manager():
    return PickleIOManager()

첫 번째 파이프라인​

1. Asset 정의​

# my_dagster_project/assets/__init__.py
from dagster import asset, Definitions
import pandas as pd
import requests

@asset
def raw_weather_data() -> pd.DataFrame:
    """OpenWeather API에서 날씨 데이터 가져오기"""
    response = requests.get(
        "https://api.openweathermap.org/data/2.5/weather",
        params={
            "q": "Seoul",
            "appid": "YOUR_API_KEY",
            "units": "metric"
        }
    )
    data = response.json()

    return pd.DataFrame([{
        "city": data["name"],
        "temp": data["main"]["temp"],
        "humidity": data["main"]["humidity"],
        "timestamp": pd.Timestamp.now()
    }])

@asset
def weather_summary(raw_weather_data: pd.DataFrame) -> dict:
    """날씨 데이터 요약"""
    return {
        "avg_temp": raw_weather_data["temp"].mean(),
        "avg_humidity": raw_weather_data["humidity"].mean()
    }

@asset
def weather_alert(weather_summary: dict) -> str:
    """날씨 알림 생성"""
    if weather_summary["avg_temp"] > 30:
        return "⚠️ 고온 주의보"
    elif weather_summary["avg_temp"] < 0:
        return "❄️ 한파 주의보"
    else:
        return "✅ 정상 날씨"

# Definitions 정의
defs = Definitions(
    assets=[raw_weather_data, weather_summary, weather_alert]
)

데이터 파이프라인 예제​

ETL 파이프라인​

from dagster import asset, AssetExecutionContext
import pandas as pd

@asset
def raw_sales_data() -> pd.DataFrame:
    """원시 매출 데이터 추출"""
    return pd.read_csv("sales.csv")

@asset
def cleaned_sales_data(
    context: AssetExecutionContext,
    raw_sales_data: pd.DataFrame
) -> pd.DataFrame:
    """데이터 정제"""
    context.log.info(f"Cleaning {len(raw_sales_data)} records")

    df = raw_sales_data.copy()

    # 결측치 제거
    df = df.dropna()

    # 날짜 변환
    df["date"] = pd.to_datetime(df["date"])

    # 중복 제거
    df = df.drop_duplicates()

    context.log.info(f"Cleaned data: {len(df)} records")
    return df

@asset
def sales_by_product(cleaned_sales_data: pd.DataFrame) -> pd.DataFrame:
    """제품별 매출 집계"""
    return cleaned_sales_data.groupby("product")["amount"].sum().reset_index()

@asset
def sales_by_date(cleaned_sales_data: pd.DataFrame) -> pd.DataFrame:
    """날짜별 매출 집계"""
    return cleaned_sales_data.groupby("date")["amount"].sum().reset_index()

@asset(deps=[sales_by_product, sales_by_date])
def sales_report() -> str:
    """매출 리포트 생성"""
    # 리포트 생성 로직
    return "sales_report_2024.pdf"

Asset Lineage​

@asset
def raw_data(): ...

@asset
def cleaned_data(raw_data): ...

@asset
def feature_1(cleaned_data): ...

@asset
def feature_2(cleaned_data): ...

@asset
def ml_model(feature_1, feature_2): ...

의존성 그래프가 자동 생성됩니다:

raw_data → cleaned_data → feature_1 → ml_model
