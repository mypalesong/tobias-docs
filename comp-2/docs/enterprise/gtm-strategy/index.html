7. Go-to-Market 전략

개요​

Go-to-Market(GTM) 전략은 제품을 시장에 출시하고, 고객을 확보하며, 성장을 가속화하는 전체적인 계획입니다. 탁월한 제품도 올바른 GTM 전략 없이는 성공할 수 없습니다.

타겟 고객 정의​

ICP (Ideal Customer Profile)​

B2B SaaS ICP 예시​

회사 특성:
  산업: IT/소프트웨어, SaaS
  규모: 10-100명
  매출: $1M-$10M/년
  성장 단계: Seed to Series A
  위치: 한국, 미국

기술 스택:
  - 클라우드 기반 (AWS, GCP)
  - 현대적 도구 사용 (Slack, Notion, Figma)
  - API 통합 중시

Pain Points:
  - 여러 도구 사용의 피로
  - 프로젝트 가시성 부족
  - 수동 보고 프로세스
  - 팀 협업 비효율

구매 행동:
  - 의사결정자: CTO, PM, VP of Engineering
  - 영향력자: 개발자, 디자이너
  - 결정 기간: 2-4주
  - 예산: $500-$5,000/월
  - 평가 기준: 사용성, 통합, 가격

신호 (Signals):
  - 채용 공고 증가 (성장 중)
  - 최근 투자 유치
  - Product Hunt, Hacker News 활동
  - 경쟁사 도구 불만 (리뷰)

ICP 검증 방법​

정량적 분석:
  - 기존 고객 분석
    → 높은 LTV, 낮은 Churn 고객 특성
  - 코호트 분석
    → 1년 유지율 >80% 고객
  - 수익 기여도
    → 상위 20% 고객 프로필

정성적 분석:
  - 고객 인터뷰 (20-30명)
  - NPS 조사 (Promoter 특성)
  - 케이스 스터디 (성공 고객)

ICP 스코어카드:
  산업 적합성: 10점
  규모 적합성: 8점
  기술 스택: 7점
  Pain Point 일치: 9점
  예산 적합성: 8점

  총점: 42/50
  → ICP 합격 (>35점)

고객 여정 (Customer Journey)​

5단계 여정​

1. 인지 (Awareness)

단계: 문제 인지, 솔루션 탐색

고객 활동:
  - 구글 검색: "best project management tool"
  - Product Hunt 탐색
  - 동료 추천
  - 블로그/YouTube 콘텐츠 소비

우리 활동:
  - SEO 콘텐츠: "10 Best PM Tools for Startups"
  - Product Hunt 런칭
  - 인플루언서 마케팅
  - 소셜 미디어 광고

목표:
  - 브랜드 인지도 구축
  - 웹사이트 트래픽 확보

측정:
  - 웹사이트 방문: 10,000/월
  - 브랜드 검색: 500/월
  - 소셜 언급: 100/월

2. 고려 (Consideration)

단계: 옵션 비교, 평가

고객 활동:
  - 경쟁사 비교
  - 리뷰 확인 (G2, Capterra)
  - 무료 체험 신청
  - 데모 요청

우리 활동:
  - 비교 페이지: "우리 vs 경쟁사"
  - 케이스 스터디
  - 무료 체험 (30일)
  - 제품 데모 (셀프/라이브)

목표:
  - 무료 체험 전환
  - 제품 가치 전달

측정:
  - 체험 신청: 500/월
  - 전환율: 5% (방문자 → 체험)
  - 데모 예약: 50/월

3. 결정 (Decision)

단계: 최종 선택, 구매

고객 활동:
  - 팀 내부 논의
  - 예산 승인
  - 경쟁사 최종 비교
  - 계약 협상 (Enterprise)

우리 활동:
  - 온보딩 지원
  - 성공 사례 공유
  - 할인 제안 (타이밍)
  - 영업팀 개입 (Enterprise)

목표:
  - 유료 전환
  - 초기 성공 보장

측정:
  - 유료 전환율: 15% (체험 → 유료)
  - 평균 전환 기간: 14일
  - ACV: $2,000

4. 사용 (Retention)

단계: 제품 사용, 가치 실현

고객 활동:
  - 일상 업무에 통합
  - 팀원 초대
  - 고급 기능 탐색

우리 활동:
  - 온보딩 이메일 시퀀스
  - 인앱 가이드
  - 사용 패턴 모니터링
  - 프로액티브 지원

목표:
  - 제품 정착 (Product Adoption)
  - 이탈 방지

측정:
  - DAU/MAU: >30%
  - 핵심 기능 사용률: >70%
  - 월 Churn: <5%

5. 확장 (Expansion)

단계: 업그레이드, 추천

고객 활동:
  - 추가 사용자 초대
  - 고급 플랜 업그레이드
  - 동료에게 추천

우리 활동:
  - 업그레이드 제안 (사용량 도달 시)
  - 크로스셀 (추가 제품)
  - 추천 프로그램
  - 고객 성공 체크인

목표:
  - MRR 증가
  - 바이럴 성장

측정:
  - NRR: >110%
  - 추천률: >30%
  - 평균 팀 크기 증가: +20%/년

마케팅 채널​

Owned Channels (소유 채널)​

1. 웹사이트 & SEO​

전략: 오가닉 트래픽 확보

핵심 페이지:
  - 홈페이지
  - 제품 페이지
  - 가격 페이지
  - 블로그
  - 케이스 스터디
  - 리소스 (가이드, 템플릿)

SEO 전략:
  키워드 타겟:
    - 브랜드: "우리회사명"
    - 카테고리: "project management software"
    - Long-tail: "best PM tool for remote teams"
    - 경쟁사: "asana alternative"

  콘텐츠 유형:
    - 가이드: "Complete Guide to Project Management"
    - 비교: "Asana vs Monday vs 우리"
    - 템플릿: "Free Project Plan Template"
    - 케이스 스터디: "How Company X increased productivity by 30%"

  기술 SEO:
    - 페이지 속도: <3초
    - 모바일 최적화
    - 구조화 데이터 (Schema.org)
    - 내부 링킹

성과:
  - 오가닉 트래픽: 10,000 → 50,000/월 (6개월)
  - 키워드 순위:
    - "project management tool": 3위
    - "team collaboration software": 5위
  - 전환율: 3% (오가닉 트래픽)

비용:
  - SEO 도구 (Ahrefs, SEMrush): $200/월
  - 콘텐츠 제작: $3,000/월
  - 기술 최적화: $5,000 (초기)
  ROI: 5:1 (6개월 후)

콘텐츠 캘린더 예시

| 주차 | 콘텐츠 유형 | 제목 | 타겟 키워드 | 예상 트래픽 |
|------|-------------|------|-------------|-------------|
| W1 | 가이드 | Project Management Basics | project management guide | 500/월 |
| W2 | 비교 | Top 10 PM Tools Compared | best pm tools | 1,000/월 |
| W3 | 케이스 스터디 | How Startup X Saved 10h/week | pm case study | 200/월 |
| W4 | 템플릿 | Free Sprint Planning Template | sprint template | 300/월 |

2. 블로그 & 콘텐츠 마케팅​

전략: 교육 중심 콘텐츠

콘텐츠 피라미드:
  Top (Hero Content): 1-2개/분기
    - 대형 가이드 (10,000+ 단어)
    - 원본 리서치/데이터
    - 예: "State of Project Management 2024"

  Middle (Hub Content): 2-4개/월
    - 심층 튜토리얼 (2,000-5,000 단어)
    - 비교/리뷰
    - 예: "Remote Team Management Guide"

  Bottom (Hygiene Content): 4-8개/월
    - How-to 가이드 (500-1,500 단어)
    - 팁/트릭
    - 예: "5 Ways to Prioritize Tasks"

배포:
  - 블로그
  - Medium, Dev.to (재발행)
  - 이메일 뉴스레터
  - 소셜 미디어 (요약)

성과:
  - 블로그 트래픽: 20,000/월
  - 리드 생성: 500/월
  - 전환율: 5% (블로그 → 체험)

투자:
  - 작가 (외주): $500/포스트
  - 편집자 (내부): $4,000/월
  - 디자인 (인포그래픽): $200/포스트

3. 이메일 마케팅​

유형:

1. 온보딩 시퀀스 (신규 가입):
   Day 0: 환영 이메일 + 퀵스타트 가이드
   Day 1: 첫 프로젝트 생성 방법
   Day 3: 팀원 초대 방법
   Day 7: 고급 기능 소개
   Day 14: 유료 전환 제안 (할인)
   Day 21: 성공 사례
   Day 28: 마지막 제안

   전환율: 15% (가입 → 유료)

2. 뉴스레터 (주간/월간):
   - 제품 업데이트
   - 사용 팁
   - 케이스 스터디
   - 산업 뉴스

   오픈율: 25%
   클릭율: 5%

3. 드립 캠페인 (세그먼트별):
   - 비활성 사용자: 재참여 유도
   - 업그레이드 후보: 기능 소개
   - 이탈 위험: 리텐션 제안

도구:
  - SendGrid, Mailchimp: $100-$500/월
  - 자동화 (Zapier, Make): $50/월

성과:
  - 이메일 리스트: 50,000
  - 뉴스레터 오픈율: 25% (업계 평균 20%)
  - 수익 기여: MRR의 15%

Paid Channels (유료 채널)​

1. Google Ads (검색 광고)​

전략: 구매 의도 높은 키워드 타겟

캠페인 구조:
  1. 브랜드 키워드:
     - "우리회사명"
     - CPC: $1-$2
     - 전환율: 10%
     - 방어 목적 (경쟁사 견제)

  2. 카테고리 키워드:
     - "project management software"
     - CPC: $10-$20
     - 전환율: 3%
     - 신규 획득

  3. 경쟁사 키워드:
     - "asana alternative"
     - CPC: $8-$15
     - 전환율: 5%
     - 전환 고객

  4. Long-tail 키워드:
     - "best pm tool for remote startups"
     - CPC: $5-$10
     - 전환율: 7%
     - 낮은 경쟁

예산 배분:
  - 브랜드: 20% ($2,000/월)
  - 카테고리: 40% ($4,000/월)
  - 경쟁사: 25% ($2,500/월)
  - Long-tail: 15% ($1,500/월)
  총 예산: $10,000/월

성과 (월):
  - 클릭: 1,000
  - 체험 신청: 100 (10% 전환)
  - 유료 고객: 15 (15% 전환)
  - CAC: $667 (=$10,000/15)
  - LTV: $2,000
  - LTV:CAC: 3:1 (목표 달성)

Google Ads 최적화

A/B 테스트:
  - 광고 카피 (5개 변형)
  - 랜딩 페이지 (3개 버전)
  - 입찰 전략 (수동 vs 자동)

품질 점수 개선:
  - 관련성: 키워드 → 광고 → 랜딩 페이지 일치
  - CTR 향상: 매력적인 광고 카피
  - 랜딩 페이지 최적화: 빠른 로딩, 명확한 CTA

네거티브 키워드:
  - "free" (무료 찾는 고객 제외)
  - "crack", "torrent" (불법 다운로드)
  - "job", "career" (채용 검색 제외)

3. 리타겟팅 (Retargeting)​

전략: 웹사이트 방문자 재타겟

세그먼트:
  1. 방문만 (미전환):

     - 예산: 30%

     - 광고: 온보딩 도움, 성공 사례

  3. 활성 사용 (미전환):
     - 광고: 유료 플랜 혜택, 할인

플랫폼:
  - Google Display Network
  - Facebook Pixel
  - LinkedIn Insight Tag

예산: $2,000/월

  - 전환율: 10% (일반 광고 3% 대비)

  - ROI: 5:1

Earned Channels (획득 채널)​

1. PR & 미디어​

전략: 브랜드 신뢰도, 인지도 구축

타겟 미디어:
  글로벌:
    - TechCrunch, The Verge (제품 런칭)
    - Forbes, Inc. (사고 리더십)
    - Product Hunt (커뮤니티)

  한국:
    - 블로터, 아웃스탠딩 (스타트업)
    - 조선일보, 한국경제 (비즈니스)
    - 디스콰이엇, GeekNews (커뮤니티)

접근 방법:
  - 보도자료: 제품 런칭, 투자 유치
  - 기자 피칭: 이메일, LinkedIn
  - 전문가 기고: Guest Post
  - 인터뷰 응답: HARO (Help a Reporter Out)

성공 사례:
  - TechCrunch 기사: 트래픽 +500% (1주)
  - Product Hunt #1: 가입 +1,000 (1일)

비용:
  - PR 에이전시: $5,000-$20,000/월 (선택적)
  - 인하우스: 무료 (시간 투자)

측정:
  - 미디어 언급: 10-20/월
  - Referral 트래픽: 5,000/월
  - 브랜드 검색 증가: +50%

2. Product Hunt​

전략: 기술 얼리어답터 확보

준비 (2-3주 전):
  - 헌터 섭외 (유명 헌터, 200+ 업보트)
  - 팀 동원 (직원, 친구, 고객)
  - 소셜 미디어 예고
  - 제품 완성도 (버그 수정, 폴리싱)

런칭 당일:
  - 시간: 오전 12:01 AM PST (최대 노출)

판매 모션 (Sales Motion)​

1. 셀프 서비스 (Self-Service)​

특징:
  - 고객이 스스로 가입, 사용, 결제
  - 영업 개입 없음

적합한 경우:
  - 낮은 가격 (<$100/월)
  - 간단한 제품
  - B2C, SMB

프로세스:
  1. 웹사이트 방문
  2. 무료 체험 가입 (이메일만)
  3. 셀프 온보딩 (인앱 가이드)
  4. 자동 업그레이드 제안
  5. 셀프 결제 (카드)

장점:
  - 낮은 CAC ($100-$300)
  - 빠른 성장 (확장성)
  - 높은 전환율 (마찰 없음)

단점:
  - 낮은 ACV (<$5,000/년)
  - 제한된 고객화

최적화:
  - 간단한 가입 (3 필드)
  - 명확한 가격
  - 즉시 가치 전달 (Aha Moment)
  - 인앱 가이드, 챗봇

예시: Slack, Dropbox, Trello

2. 인사이드 세일즈 (Inside Sales)​

특징:
  - 원격 영업 (전화, 화상)
  - 영업 사이클: 1-3개월
  - ACV: $5,000-$50,000

적합한 경우:
  - 중간 가격 ($500-$5,000/월)
  - 복잡한 제품 (설명 필요)
  - Mid-Market B2B

프로세스:
  1. 리드 생성 (마케팅)
  2. SDR(Sales Development Rep) 자격 심사
  3. AE(Account Executive) 데모
  4. 제안서, 협상
  5. 계약 체결

팀 구조:
  - SDR: 리드 자격 심사, 미팅 예약
  - AE: 데모, 클로징
  - CSM: 온보딩, 성공

비율:
  - SDR:AE = 2:1 또는 3:1
  - AE 할당량: $500K-$1M ARR/년
  - 전환율: 20-30% (데모 → 고객)

투자:
  - SDR 급여: $50K-$70K + 커미션
  - AE 급여: $80K-$120K + 커미션
  - 도구 (CRM, 영업 자동화): $100/사용자/월

예시: HubSpot, Asana (Mid-Market)

3. 필드 세일즈 (Field Sales)​

특징:
  - 대면 영업
  - 영업 사이클: 3-18개월
  - ACV: $50,000+

적합한 경우:
  - 높은 가격 ($5,000+/월)
  - 매우 복잡한 제품
  - Enterprise B2B

프로세스:
  1. 전략 계정 식별
  2. 다층 관계 구축
  3. POC (Proof of Concept)
  4. RFP (제안 요청서) 응답
  5. 협상, 법무 검토
  6. 계약 (1-3년)

팀 구조:
  - AE: 계정 소유, 관계 관리
  - SE(Sales Engineer): 기술 데모, POC
  - CSM: 온보딩, 확장

비율:
  - AE 할당량: $1M-$3M ARR/년
  - 전환율: 10-20% (POC → 고객)

투자:
  - AE 급여: $120K-$200K + 커미션 (50-100%)
  - SE 급여: $100K-$150K + 커미션
  - 출장비: $20K-$50K/년

예시: Salesforce, Oracle, SAP

영업 플레이북​

디스커버리 콜 (Discovery Call)​

목적: 고객 니즈 파악, 적합성 평가

준비:
  - 회사 리서치 (웹사이트, LinkedIn)
  - 잠재 Pain Point 가설
  - 질문 리스트

구조 (45분):
  도입 (5분):
    - 자기소개, 회사 소개
    - 어젠다 설정
    - 라포 형성

  현황 파악 (15분):
    - "현재 [업무 영역]을 어떻게 관리하시나요?"
    - "팀 규모는? 도구는?"
    - "워크플로우를 설명해주세요"

  문제 탐색 (15분):
    - "가장 큰 어려움은 무엇인가요?"
    - "왜 그것이 문제인가요?"
    - "해결되지 않으면 어떻게 되나요?"
    - "해결을 위해 시도한 것은?"

  솔루션 제시 (5분):
    - 우리 솔루션 간단 소개
    - 유사 케이스 스터디
    - 다음 단계 제안 (데모)

  마무리 (5분):
    - 질문 응답
    - 다음 미팅 예약
    - 관련 자료 발송

BANT 평가:
  - Budget: 예산 있는가?
  - Authority: 의사결정권자인가?
  - Need: 니즈가 강한가?
  - Timeline: 구매 시기는?

결과:
  - 적합: 데모 예약
  - 부적합: 정중히 거절 (시간 절약)

제품 데모 (Product Demo)​

원칙: "Show, Don't Tell"

준비:
  - 고객 니즈 기반 데모 시나리오
  - 고객 데이터 사용 (가능 시)
  - 백업 플랜 (기술 문제 대비)

구조 (60분):
  도입 (5분):
    - 디스커버리 요약 확인
    - 데모 어젠다 합의
    - 질문 시간 안내

  문제 재확인 (5분):
    - "지난번 말씀하신 [문제]가 맞나요?"
    - "오늘 가장 보고 싶은 부분은?"

  솔루션 데모 (35분):
    - 고객 시나리오 기반 시연
    - 3-5개 핵심 기능 집중
    - 인터랙티브 (고객도 조작)
    - 가치 강조 (기능 → 혜택)

  질문 & 토론 (10분):
    - 고객 우려 사항 해결
    - 추가 질문 응답

  다음 단계 (5분):
    - POC 또는 체험 제안
    - 타임라인 논의
    - 담당자 정의

데모 Dos & Don'ts:
  ✅ Do:
    - 고객 중심 시나리오
    - 간결 (핵심만)
    - 가치 강조
    - 대화형 (일방적 X)

  ❌ Don't:
    - 모든 기능 나열
    - 기술 전문 용어 과다
    - 고객 말 안 듣기
    - 경쟁사 비방

제안서 구성:
  1. 요약 (Executive Summary)
  2. 고객 니즈 & 우리 솔루션
  3. 범위 (Scope of Work)
  4. 가격 & 패키지
  5. 구현 일정
  6. 성공 사례
  7. 계약 조건

가격 협상:
  전략:
    - 앵커링 (높은 가격 먼저)
    - 번들링 (여러 제품/서비스)
    - 가치 강조 (ROI 계산)

  양보 순서:
    1. 할인 (최대 20%)
    2. 추가 서비스 (온보딩, 트레이닝)
    3. 장기 계약 (2-3년)

  절대 양보 안 함:
    - 핵심 기능 제거
    - 과도한 할인 (>30%)
    - 비현실적 약속

클로징 기법:
  - 직접 요청: "계약 진행하시겠습니까?"
  - 가정 질문: "다음 주 월요일 시작 가능하신가요?"
  - 선택 질문: "연간 계약과 월간 중 어느 것을 선호하시나요?"
  - 긴급성: "이번 분기 내 계약 시 10% 할인"

반대 처리:
  "가격이 비싸요"
  → ROI 강조, 비용 분석, 경쟁사 비교

  "지금은 아니에요"
  → 타임라인 재확인, 파일럿 제안

  "다른 도구 사용 중"
  → 차별화 강조, 전환 지원

성장 해킹​

바이럴 루프 (Viral Loop)​

공식:
  K (Viral Coefficient) = 초대 수 × 전환율

목표: K > 1 (바이럴 성장)

예시:
  - 사용자 1명이 평균 3명 초대
  - 초대 수락률: 40%
  - K = 3 × 0.4 = 1.2

  결과: 바이럴 성장!
  - 100명 → 120명 → 144명 → ...

바이럴 루프 설계:

1. 초대 동기 부여:
   - 추천인 보상: 무료 1개월
   - 피추천인 보상: 20% 할인
   - 양측 혜택 (Win-Win)

2. 마찰 제거:
   - 원클릭 초대 (이메일 자동)
   - 여러 초대 방법 (이메일, 링크, SNS)
   - 진행 상황 표시 (초대 3명/5명)

3. 제품 내 통합:
   - 팀 협업 기능 (초대 필수)
   - 공유 기능 (프로젝트, 템플릿)
   - 브랜드 노출 (무료 플랜 워터마크)

성공 사례:
  - Dropbox: 추천 프로그램
    초대: 500MB, 피초천: 500MB
    결과: 60% 가입이 추천
    성장: 100K → 4M (15개월)

  - Slack: 팀 초대 필수
    자연스러운 확산 (팀 단위)
    K = 1.3 (추정)

그로스 해킹 전술​

1. 프리미엄 최적화​

목표: 무료 → 유료 전환율 증가

전술:

1. 가치 우선 제공:
   - 무료로 충분한 가치 (만족)
   - 유료로 더 큰 가치 (욕구)

2. 사용량 제한:
   - 저장소: 1GB → 100GB
   - 프로젝트: 3개 → 무제한
   - 팀원: 5명 → 50명

3. 기능 제한:
   - 고급 기능 잠금 (타임라인, 리포팅)
   - "업그레이드" 배지 표시
   - 잠긴 기능 클릭 → 업그레이드 제안

4. 타이밍:
   - 가치 경험 후 (1-2주)
   - 제한 도달 시 (80% 사용)
   - 성공 순간 (Aha Moment 후)

측정:
  - 무료 활성 사용자: 1,000
  - 유료 전환율: 10% → 15% (목표)
  - 추가 MRR: 50 × $20 = $1,000/월

2. 온보딩 최적화​

목표: 신규 사용자 활성화

핵심 지표:
  - Time to Value (TTV): 가치 경험까지 시간
  - Activation Rate: 핵심 기능 사용률
  - Day 7 Retention: 7일 유지율

전술:

1. 간소화:
   - 가입: 3필드 (이름, 이메일, 비밀번호)
   - 소셜 로그인 (Google, GitHub)
   - 이메일 인증 지연 (나중에)

2. 인터랙티브 튜토리얼:
   - 첫 프로젝트 생성 가이드
   - 진행 표시 (1/5 완료)
   - 축하 메시지 (마일스톤)

3. 개인화:
   - 역할 질문: "PM, 개발자, 디자이너?"
   - 맞춤 템플릿 제공
   - 관련 기능 우선 소개

4. 빠른 성공:
   - 프리세팅 데이터 (샘플 프로젝트)
   - 즉시 사용 가능한 템플릿
   - 빠른 결과 (3분 내)

측정:
  - TTV: 30분 → 5분
  - Activation: 40% → 70%
  - Day 7 Retention: 20% → 40%

3. 리텐션 해킹​

목표: 이탈 방지, 활성 유지

전술:

1. 이메일 리인게이지먼트:
   - 비활성 감지 (7일 미사용)
   - 자동 이메일:
     Day 7: "놓친 업데이트"
     Day 14: "팀원들이 기다려요"
     Day 21: "새 기능 출시"

2. 인앱 알림:
   - 개인화 추천: "이 기능 써보세요"
   - 팀 활동: "@김철수가 작업 할당"
   - 마일스톤: "프로젝트 50% 완료!"

3. 제품 개선:
   - 이탈 인터뷰 (왜 떠나나?)
   - 코호트 분석 (언제 이탈?)
   - 기능 개선 (불만 해결)

4. 커뮤니티:
   - 사용자 그룹 (Slack, 포럼)
   - 웨비나 (사용 팁)
   - 뉴스레터 (베스트 프랙티스)

측정:
  - Monthly Churn: 10% → 5%
  - NRR: 95% → 105%
  - DAU/MAU: 20% → 30%

한국 시장 GTM​

현지화 전략​

언어:
  - UI/UX: 100% 한글화
  - 문서: 한국어 우선
  - 지원: 한국어 필수 (이메일, 채팅)

결제:
  - 한국 PG사 연동 (KG이니시스, NHN KCP)
  - 간편결제 (카카오페이, 네이버페이)
  - 세금계산서 자동 발행

문화:
  - 존댓말 (고객 소통)
  - 공휴일 인식 (설, 추석)
  - 업무 시간 (9-6시, 점심 12-1시)

법규:
  - 개인정보보호법 준수
  - 이용약관 (한국어)
  - 환불 정책 (7일 무조건 환불)

한국 채널 전략​

온라인:
  - 네이버 블로그, 카페 (SEO)
  - 카카오 채널 (고객 소통)
  - 유튜브 (튜토리얼)
  - 페이스북 그룹

오프라인:
  - 스타트업 행사 (컨퍼런스, 밋업)
  - 공동 작업 공간 (위워크, 패스트파이브)
  - 대학 특강 (학생 타겟)

파트너:
  - 액셀러레이터 (프라이머, 스파크랩)
  - 정부 지원 (K-Startup, TIPS)
  - 대기업 협업 (오픈 이노베이션)
