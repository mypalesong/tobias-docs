Prometheus 모니터링 가이드

Prometheus로 애플리케이션을 모니터링하고 메트릭을 수집하는 방법을 배워보세요.

Prometheus란?​

Prometheus는 오픈소스 모니터링 및 알림 시스템입니다. 시계열 데이터베이스를 기반으로 메트릭을 수집하고 저장합니다.

✅ 시계열 데이터베이스
✅ 강력한 쿼리 언어 (PromQL)
✅ Pull 기반 메트릭 수집
✅ 서비스 디스커버리 지원
✅ Grafana와 완벽한 통합

주요 특징​

1. 다차원 데이터 모델​

# 메트릭 이름 + 레이블로 데이터 식별
http_requests_total{method="GET", endpoint="/api/users", status="200"}

2. PromQL 쿼리​

# 지난 5분간 초당 요청 수
rate(http_requests_total[5m])

# 99th percentile 응답 시간
histogram_quantile(0.99, http_request_duration_seconds_bucket)

3. Pull 모델​

Prometheus가 타겟에서 메트릭을 주기적으로 가져옴
(Push 방식의 StatsD, Graphite와 차이점)

왜 Prometheus인가?​

vs 전통적인 모니터링​

**전통적 방식 (예: Nagios)**
- 호스트/서비스 상태 체크
- 임계값 기반 알람
- 제한적인 메트릭

**Prometheus**
- 풍부한 메트릭 수집
- 다차원 데이터 분석
- 동적 서비스 디스커버리
- 강력한 쿼리 언어

사용 사례​

✅ **웹 애플리케이션 모니터링**
   - 요청 수, 응답 시간, 에러율
   - 사용자 경험 메트릭

✅ **인프라 모니터링**
   - CPU, 메모리, 디스크
   - 네트워크 트래픽

✅ **비즈니스 메트릭**
   - 주문 수, 매출
   - 사용자 활동

✅ **마이크로서비스**
   - 서비스 간 통신
   - 분산 추적

기본 개념​

메트릭 타입​

from prometheus_client import Counter, Gauge, Histogram, Summary

# Counter: 증가만 하는 값 (요청 수)
requests = Counter('requests_total', 'Total requests')

# Gauge: 증가/감소하는 값 (온도, 메모리)
memory = Gauge('memory_usage_bytes', 'Memory usage')

# Histogram: 분포 측정 (응답 시간)
latency = Histogram('request_duration_seconds', 'Request latency')

# Summary: 분위수 계산
size = Summary('response_size_bytes', 'Response size')

레이블​

# 레이블로 메트릭 구분
http_requests = Counter(
    'http_requests_total',
    'Total HTTP requests',
    ['method', 'endpoint', 'status']
)

# 사용
http_requests.labels(method='GET', endpoint='/api', status='200').inc()

시작하기​

1. Prometheus 설치​

# Docker로 빠르게 시작
docker run -p 9090:9090 prom/prometheus

2. Python 앱에 메트릭 추가​

from prometheus_client import Counter, start_http_server
import time

# 메트릭 정의
requests = Counter('my_requests_total', 'Total requests')

# 메트릭 엔드포인트 시작 (포트 8000)
start_http_server(8000)

# 애플리케이션 로직
while True:
    requests.inc()  # 카운터 증가
    time.sleep(1)

3. Prometheus 설정​

# prometheus.yml
scrape_configs:
  - job_name: 'my_app'
    static_configs:
      - targets: ['localhost:8000']

4. 메트릭 확인​

http://localhost:9090 에서 Prometheus UI 접속
