1. 설치 및 설정

Prometheus는 오픈소스 모니터링 시스템으로, 시계열 데이터베이스를 기반으로 메트릭을 수집하고 저장합니다. 이 문서에서는 Prometheus를 설치하고 설정하는 다양한 방법을 다룹니다.

1.1 Prometheus 설치 방법​

Prometheus를 설치하는 방법은 여러 가지가 있습니다. 환경과 요구사항에 따라 적절한 방법을 선택하세요.

1.1.1 Docker를 이용한 설치 (권장)​

Docker는 가장 빠르고 간편하게 Prometheus를 시작할 수 있는 방법입니다.

# Prometheus 공식 이미지 다운로드 및 실행
docker run -d \
  --name prometheus \
  -p 9090:9090 \
  -v /path/to/prometheus.yml:/etc/prometheus/prometheus.yml \
  prom/prometheus:latest
팁

Docker를 사용하면 환경 격리와 버전 관리가 쉬워지며, 팀 간 동일한 환경을 구성할 수 있습니다.

1.1.2 바이너리 다운로드 설치​

공식 웹사이트에서 바이너리를 직접 다운로드하여 설치할 수 있습니다.

Linux/macOS:

# Prometheus 다운로드
wget https://github.com/prometheus/prometheus/releases/download/v2.45.0/prometheus-2.45.0.linux-amd64.tar.gz

# 압축 해제
tar xvfz prometheus-2.45.0.linux-amd64.tar.gz

# 디렉토리 이동
cd prometheus-2.45.0.linux-amd64

# Prometheus 실행
./prometheus --config.file=prometheus.yml

Windows:

  1. Prometheus 릴리스 페이지에서 Windows용 바이너리 다운로드
  2. 압축 해제
  3. 명령 프롬프트에서 실행:
prometheus.exe --config.file=prometheus.yml

1.1.3 패키지 매니저를 이용한 설치​

Ubuntu/Debian:

# APT 패키지 업데이트
sudo apt-get update

# Prometheus 설치
sudo apt-get install prometheus

# 서비스 시작
sudo systemctl start prometheus
sudo systemctl enable prometheus

macOS (Homebrew):

# Homebrew로 Prometheus 설치
brew install prometheus

# 서비스 시작
brew services start prometheus

CentOS/RHEL:

# Prometheus 사용자 생성
sudo useradd --no-create-home --shell /bin/false prometheus

# 다운로드 및 설치
sudo wget https://github.com/prometheus/prometheus/releases/download/v2.45.0/prometheus-2.45.0.linux-amd64.tar.gz
sudo tar -xvf prometheus-2.45.0.linux-amd64.tar.gz
sudo mv prometheus-2.45.0.linux-amd64 /etc/prometheus

# systemd 서비스 파일 생성
sudo vi /etc/systemd/system/prometheus.service
경고

바이너리 설치 시 버전 업데이트와 보안 패치를 수동으로 관리해야 합니다.

1.2 prometheus.yml 설정 파일 구조​

Prometheus의 모든 설정은 prometheus.yml YAML 파일에서 관리됩니다.

1.2.1 기본 구조​

# 전역 설정
global:
  scrape_interval: 15s      # 메트릭 수집 주기 (기본값: 15초)
  evaluation_interval: 15s  # 규칙 평가 주기
  external_labels:          # 외부 시스템에 전송될 때 추가되는 레이블
    cluster: 'production'
    region: 'us-west-1'

# Alertmanager 설정 (선택사항)
alerting:
  alertmanagers:
    - static_configs:
        - targets:
            - 'localhost:9093'

# 규칙 파일 로드 (선택사항)
rule_files:
  - 'alerts/*.yml'
  - 'rules/*.yml'

# 스크랩 설정
scrape_configs:
  # Prometheus 자체 모니터링
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

1.2.2 설정 파일 상세 설명​

섹션설명필수 여부
global전역 설정 (수집 간격, 외부 레이블 등)선택
alertingAlert Manager 연동 설정선택
rule_files알림 규칙 및 기록 규칙 파일 경로선택
scrape_configs메트릭 수집 대상 및 방법 설정필수

1.3 Scrape 설정​

Scrape는 Prometheus가 타겟으로부터 메트릭을 수집하는 과정입니다.

1.3.1 기본 Scrape 설정​

scrape_configs:
  - job_name: 'my-application'
    scrape_interval: 10s      # 이 job만의 수집 주기
    scrape_timeout: 5s        # 타임아웃 설정
    metrics_path: '/metrics'  # 메트릭 엔드포인트 경로
    static_configs:
      - targets:
          - 'app1.example.com:8080'
          - 'app2.example.com:8080'
        labels:
          environment: 'production'
          team: 'backend'

1.3.2 다중 타겟 설정​

scrape_configs:
  # 프로덕션 애플리케이션
  - job_name: 'production-apps'
    static_configs:
      - targets: ['app1:8080', 'app2:8080', 'app3:8080']
        labels:
          environment: 'production'

  # 스테이징 애플리케이션
  - job_name: 'staging-apps'
    static_configs:
      - targets: ['staging-app:8080']
        labels:
          environment: 'staging'

  # 데이터베이스 익스포터
  - job_name: 'postgres'
    static_configs:
      - targets: ['postgres-exporter:9187']
        labels:
          db_type: 'postgresql'

1.3.3 동적 서비스 디스커버리​

실제 운영 환경에서는 동적 서비스 디스커버리를 사용하는 것이 좋습니다.

Kubernetes 서비스 디스커버리:

scrape_configs:
  - job_name: 'kubernetes-pods'
    kubernetes_sd_configs:
      - role: pod
    relabel_configs:
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
        action: keep
        regex: true
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_path]
        action: replace
        target_label: __metrics_path__
        regex: (.+)

Docker 서비스 디스커버리:

scrape_configs:
  - job_name: 'docker'
    docker_sd_configs:
      - host: unix:///var/run/docker.sock
    relabel_configs:
      - source_labels: [__meta_docker_container_name]
        target_label: container_name
팁

동적 서비스 디스커버리를 사용하면 새로운 인스턴스가 추가되거나 제거될 때 자동으로 모니터링 대상이 업데이트됩니다.

1.4 Storage 설정​

Prometheus는 시계열 데이터를 로컬 디스크에 저장합니다.

1.4.1 Storage 기본 설정​

# prometheus.yml에는 직접 설정하지 않고, 실행 시 플래그로 지정

명령줄 옵션:

./prometheus \
  --config.file=prometheus.yml \
  --storage.tsdb.path=/data/prometheus \
  --storage.tsdb.retention.time=15d \
  --storage.tsdb.retention.size=50GB

1.4.2 Storage 옵션 설명​

옵션설명기본값
--storage.tsdb.path데이터 저장 경로data/
--storage.tsdb.retention.time데이터 보존 기간15d
--storage.tsdb.retention.size최대 저장 크기없음
--storage.tsdb.min-block-duration최소 블록 기간2h
--storage.tsdb.max-block-duration최대 블록 기간36h

1.4.3 Docker에서 Storage 설정​

docker run -d \
  --name prometheus \
  -p 9090:9090 \
  -v /my/prometheus/data:/prometheus \
  prom/prometheus:latest \
  --config.file=/etc/prometheus/prometheus.yml \
  --storage.tsdb.path=/prometheus \
  --storage.tsdb.retention.time=30d
경고

데이터 보존 기간과 크기는 디스크 용량을 고려하여 설정하세요. 기본값은 15일입니다.

1.5 기본 보안 설정​

프로덕션 환경에서는 반드시 보안 설정을 적용해야 합니다.

1.5.1 Basic Authentication 설정​

먼저 비밀번호 해시를 생성합니다:

# htpasswd 유틸리티 설치 (Apache utils)
sudo apt-get install apache2-utils

# 비밀번호 파일 생성
htpasswd -nBC 10 "" | tr -d ':\n'

web-config.yml 파일 생성:

basic_auth_users:
  admin: $2y$10$X.KjHpgf5J5Eq8F8Q8N9XOqP3Nz7J0rN3Q3X5XqE8N9X.KjHpgf5J

Prometheus 실행 시 적용:

./prometheus \
  --config.file=prometheus.yml \
  --web.config.file=web-config.yml

1.5.2 TLS/HTTPS 설정​

web-config.yml에 TLS 추가:

tls_server_config:
  cert_file: /path/to/cert.pem
  key_file: /path/to/key.pem

basic_auth_users:
  admin: $2y$10$X.KjHpgf5J5Eq8F8Q8N9XOqP3Nz7J0rN3Q3X5XqE8N9X.KjHpgf5J

1.5.3 네트워크 접근 제어​

방화벽 설정 (iptables):

# Prometheus 포트(9090)를 특정 IP에서만 접근 허용
sudo iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 9090 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 9090 -j DROP

Docker 네트워크 격리:

# 전용 네트워크 생성
docker network create monitoring

# Prometheus를 격리된 네트워크에서 실행
docker run -d \
  --name prometheus \
  --network monitoring \
  -p 127.0.0.1:9090:9090 \
  prom/prometheus:latest
팁

프로덕션 환경에서는 Prometheus 앞에 Nginx나 Traefik 같은 리버스 프록시를 두어 인증과 암호화를 처리하는 것이 좋습니다.

1.6 Node Exporter 설치​

Node Exporter는 시스템 레벨 메트릭(CPU, 메모리, 디스크, 네트워크 등)을 수집하는 Prometheus 공식 익스포터입니다.

1.6.1 Docker로 Node Exporter 설치​

docker run -d \
  --name node-exporter \
  --net="host" \
  --pid="host" \
  -v "/:/host:ro,rslave" \
  prom/node-exporter:latest \
  --path.rootfs=/host

1.6.2 바이너리로 Node Exporter 설치​

# Node Exporter 다운로드
wget https://github.com/prometheus/node_exporter/releases/download/v1.6.1/node_exporter-1.6.1.linux-amd64.tar.gz

# 압축 해제
tar xvfz node_exporter-1.6.1.linux-amd64.tar.gz

# 실행
cd node_exporter-1.6.1.linux-amd64
./node_exporter

1.6.3 systemd 서비스로 등록​

sudo vi /etc/systemd/system/node-exporter.service
[Unit]
Description=Node Exporter
After=network.target

[Service]
Type=simple
User=prometheus
ExecStart=/usr/local/bin/node_exporter

[Install]
WantedBy=multi-user.target
# 서비스 시작
sudo systemctl daemon-reload
sudo systemctl start node-exporter
sudo systemctl enable node-exporter

1.6.4 Prometheus에 Node Exporter 추가​

prometheus.yml 업데이트:

scrape_configs:
  - job_name: 'node-exporter'
    static_configs:
      - targets: ['localhost:9100']
        labels:
          instance: 'server-1'
          datacenter: 'dc1'
팁

Node Exporter는 기본적으로 9100 포트에서 메트릭을 제공합니다. 브라우저에서 http://localhost:9100/metrics로 접속하여 메트릭을 확인할 수 있습니다.

1.7 Python Exporter 설정​

Python 애플리케이션에서 커스텀 메트릭을 수집하는 방법을 알아봅니다.

1.7.1 Prometheus 클라이언트 라이브러리 설치​

pip install prometheus-client

1.7.2 기본 Python 애플리케이션 계측​

from prometheus_client import start_http_server, Counter, Gauge, Histogram
import random
import time

# 메트릭 정의
REQUEST_COUNT = Counter(
    'app_requests_total',
    'Total number of requests',
    ['method', 'endpoint', 'status']
)

ACTIVE_USERS = Gauge(
    'app_active_users',
    'Number of active users'
)

REQUEST_DURATION = Histogram(
    'app_request_duration_seconds',
    'Request duration in seconds',
    ['endpoint']
)

def process_request(method, endpoint):
    """요청 처리 시뮬레이션"""
    # 요청 처리 시간 측정
    with REQUEST_DURATION.labels(endpoint=endpoint).time():
        # 비즈니스 로직 시뮬레이션
        time.sleep(random.uniform(0.1, 0.5))

        # 요청 카운트 증가
        status = random.choice([200, 200, 200, 400, 500])
        REQUEST_COUNT.labels(method=method, endpoint=endpoint, status=status).inc()

    return status

if __name__ == '__main__':
    # 메트릭 서버 시작 (기본 포트: 8000)
    start_http_server(8000)
    print("Metrics server started on :8000")

    # 애플리케이션 로직 시뮬레이션
    while True:
        # 액티브 유저 업데이트
        ACTIVE_USERS.set(random.randint(50, 150))

        # 랜덤 요청 처리
        endpoints = ['/api/users', '/api/products', '/api/orders']
        methods = ['GET', 'POST', 'PUT', 'DELETE']

        for _ in range(random.randint(1, 5)):
            process_request(
                random.choice(methods),
                random.choice(endpoints)
            )

        time.sleep(1)

1.7.3 Flask 애플리케이션 통합​

from flask import Flask, request
from prometheus_client import Counter, Histogram, generate_latest, REGISTRY
import time

app = Flask(__name__)

# 메트릭 정의
HTTP_REQUESTS = Counter(
    'flask_http_requests_total',
    'Total HTTP requests',
    ['method', 'endpoint', 'status']
)

HTTP_REQUEST_DURATION = Histogram(
    'flask_http_request_duration_seconds',
    'HTTP request duration',
    ['method', 'endpoint']
)

@app.before_request
def before_request():
    """요청 시작 시간 기록"""
    request.start_time = time.time()

@app.after_request
def after_request(response):
    """요청 완료 후 메트릭 기록"""
    if hasattr(request, 'start_time'):
        duration = time.time() - request.start_time
        HTTP_REQUEST_DURATION.labels(
            method=request.method,
            endpoint=request.path
        ).observe(duration)

        HTTP_REQUESTS.labels(
            method=request.method,
            endpoint=request.path,
            status=response.status_code
        ).inc()

    return response

@app.route('/api/hello')
def hello():
    """샘플 API 엔드포인트"""
    time.sleep(0.1)  # 처리 시간 시뮬레이션
    return {'message': 'Hello, World!'}

@app.route('/api/users/<int:user_id>')
def get_user(user_id):
    """사용자 조회 엔드포인트"""
    time.sleep(0.2)
    return {'user_id': user_id, 'name': f'User {user_id}'}

@app.route('/metrics')
def metrics():
    """Prometheus 메트릭 엔드포인트"""
    return generate_latest(REGISTRY)

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8000)

1.7.4 FastAPI 애플리케이션 통합​

from fastapi import FastAPI, Request
from prometheus_client import Counter, Histogram, generate_latest, REGISTRY
import time

app = FastAPI()

# 메트릭 정의
HTTP_REQUESTS = Counter(
    'fastapi_http_requests_total',
    'Total HTTP requests',
    ['method', 'endpoint', 'status']
)

HTTP_REQUEST_DURATION = Histogram(
    'fastapi_http_request_duration_seconds',
    'HTTP request duration',
    ['method', 'endpoint']
)

@app.middleware("http")
async def prometheus_middleware(request: Request, call_next):
    """Prometheus 메트릭 미들웨어"""
    start_time = time.time()

    # 요청 처리
    response = await call_next(request)

    # 메트릭 기록
    duration = time.time() - start_time
    HTTP_REQUEST_DURATION.labels(
        method=request.method,
        endpoint=request.url.path
    ).observe(duration)

    HTTP_REQUESTS.labels(
        method=request.method,
        endpoint=request.url.path,
        status=response.status_code
    ).inc()

    return response

@app.get("/api/hello")
async def hello():
    """샘플 API 엔드포인트"""
    return {"message": "Hello, World!"}

@app.get("/api/users/{user_id}")
async def get_user(user_id: int):
    """사용자 조회 엔드포인트"""
    return {"user_id": user_id, "name": f"User {user_id}"}

@app.get("/metrics")
async def metrics():
    """Prometheus 메트릭 엔드포인트"""
    return generate_latest(REGISTRY)

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)

1.8 설정 검증 및 문제 해결​

1.8.1 설정 파일 검증​

Prometheus는 설정 파일 검증 도구를 제공합니다.

# 설정 파일 검증
./promtool check config prometheus.yml

# 규칙 파일 검증
./promtool check rules alerts.yml

성공 출력:

Checking prometheus.yml
  SUCCESS: 1 rule files found
  SUCCESS: prometheus.yml is valid prometheus config file syntax

1.8.2 설정 리로드​

Prometheus를 재시작하지 않고 설정을 다시 로드할 수 있습니다.

# HTTP POST로 리로드 (--web.enable-lifecycle 플래그 필요)
curl -X POST http://localhost:9090/-/reload

# 또는 SIGHUP 시그널 전송
kill -HUP $(pidof prometheus)

1.8.3 일반적인 문제 해결​

문제 1: 타겟이 수집되지 않음 (Target Down)

# 타겟 상태 확인
curl http://localhost:9090/api/v1/targets

# 타겟이 실행 중인지 확인
curl http://target-host:port/metrics

# 네트워크 연결 확인
telnet target-host port

문제 2: 메트릭이 표시되지 않음

Prometheus UI에서 Status > Targets로 이동하여 타겟 상태를 확인하세요.

  • UP: 정상 수집 중
  • DOWN: 연결 실패
  • UNKNOWN: 아직 수집 시도 안 함

문제 3: 디스크 공간 부족

# 현재 디스크 사용량 확인
du -sh /data/prometheus

# 데이터 보존 기간 줄이기
./prometheus --storage.tsdb.retention.time=7d

app/app.py:

from flask import Flask, jsonify
from prometheus_client import Counter, Histogram, Gauge, generate_latest, REGISTRY
import time
import random
import os

app = Flask(__name__)

# 메트릭 정의
REQUEST_COUNT = Counter(
    'app_http_requests_total',
    'Total HTTP requests',
    ['method', 'endpoint', 'status']
)

REQUEST_DURATION = Histogram(
    'app_http_request_duration_seconds',
    'HTTP request duration',
    ['method', 'endpoint'],
    buckets=[0.01, 0.05, 0.1, 0.5, 1.0, 5.0]
)

ACTIVE_CONNECTIONS = Gauge(
    'app_active_connections',
    'Number of active connections'
)

PROCESSING_TIME = Histogram(
    'app_processing_seconds',
    'Time spent processing request',
    ['endpoint']
)

@app.before_request
def before_request():
    """요청 전처리"""
    from flask import request
    request.start_time = time.time()
    ACTIVE_CONNECTIONS.inc()

@app.after_request
def after_request(response):
    """요청 후처리"""
    from flask import request

    if hasattr(request, 'start_time'):
        duration = time.time() - request.start_time
        REQUEST_DURATION.labels(
            method=request.method,
            endpoint=request.path
        ).observe(duration)

        REQUEST_COUNT.labels(
            method=request.method,
            endpoint=request.path,
            status=response.status_code
        ).inc()

    ACTIVE_CONNECTIONS.dec()
    return response

@app.route('/')
def index():
    """홈 페이지"""
    return jsonify({
        'message': 'Welcome to Prometheus Demo App',
        'endpoints': ['/api/users', '/api/products', '/api/slow', '/metrics']
    })

@app.route('/api/users')
def get_users():
    """사용자 목록 조회"""
    with PROCESSING_TIME.labels(endpoint='/api/users').time():
        time.sleep(random.uniform(0.05, 0.15))
        return jsonify([
            {'id': 1, 'name': 'Alice'},
            {'id': 2, 'name': 'Bob'},
            {'id': 3, 'name': 'Charlie'}
        ])

@app.route('/api/products')
def get_products():
    """상품 목록 조회"""
    with PROCESSING_TIME.labels(endpoint='/api/products').time():
        time.sleep(random.uniform(0.1, 0.3))
        return jsonify([
            {'id': 1, 'name': 'Laptop', 'price': 1200},
            {'id': 2, 'name': 'Mouse', 'price': 25},
            {'id': 3, 'name': 'Keyboard', 'price': 75}
        ])

@app.route('/api/slow')
def slow_endpoint():
    """느린 엔드포인트 (테스트용)"""
    with PROCESSING_TIME.labels(endpoint='/api/slow').time():
        time.sleep(random.uniform(1.0, 3.0))
        return jsonify({'message': 'This was slow!'})

@app.route('/api/error')
def error_endpoint():
    """에러 발생 엔드포인트 (테스트용)"""
    if random.random() > 0.5:
        return jsonify({'error': 'Something went wrong'}), 500
    return jsonify({'message': 'Success'})

@app.route('/metrics')
def metrics():
    """Prometheus 메트릭 엔드포인트"""
    return generate_latest(REGISTRY)

@app.route('/health')
def health():
    """헬스체크 엔드포인트"""
    return jsonify({'status': 'healthy'})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8000)
