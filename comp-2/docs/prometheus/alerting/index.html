5. Alerting

Prometheus의 알림 시스템을 사용하여 시스템 문제를 실시간으로 감지하고 대응하는 방법을 배워보세요.

5.1 Alerting 개념​

Alert Rule과 Alertmanager​

Prometheus의 알림 시스템은 두 가지 주요 컴포넌트로 구성됩니다:

┌──────────────┐
│  Prometheus  │
│   Server     │
│              │
│ ┌──────────┐ │
│ │  Alert   │ │  Alert Rule 평가
│ │  Rules   │ │  (PromQL 기반)
│ └────┬─────┘ │
└──────┼───────┘
       │ Firing Alerts
       ▼
┌──────────────┐
│ Alertmanager │  그룹화, 라우팅, Silence
│              │  알림 채널로 발송
└──────┬───────┘
       │
   ┌───┴───┬─────────┬──────────┐
   ▼       ▼         ▼          ▼
┌─────┐ ┌─────┐ ┌────────┐ ┌────────┐
│Email│ │Slack│ │PagerDuty│ │Webhook│
└─────┘ └─────┘ └────────┘ └────────┘

Prometheus Server:

  • Alert Rule을 주기적으로 평가
  • 조건이 충족되면 Alert를 Firing 상태로 전환
  • Alertmanager로 Alert 전송

Alertmanager:

  • Alert 수신 및 그룹화
  • Silence 및 Inhibition 처리
  • 라우팅 규칙에 따라 알림 채널로 발송
  • 알림 중복 제거 (Deduplication)

Alert 상태​

┌──────────┐
│ Inactive │  조건 미충족
└────┬─────┘
     │ 조건 충족
     ▼
┌──────────┐
│ Pending  │  for 기간 대기 중
└────┬─────┘
     │ for 기간 경과
     ▼
┌──────────┐
│ Firing   │  Alert 발송
└────┬─────┘
     │ 조건 미충족
     ▼
┌──────────┐
│ Resolved │  복구됨
└──────────┘
for 절의 중요성

for 절을 사용하여 일시적인 스파이크로 인한 불필요한 알림을 방지하세요. 예: for: 5m은 조건이 5분 이상 지속될 때만 알림을 발송합니다.

5.2 Alert Rule 작성​

기본 구조​

# /etc/prometheus/alerts.yml
groups:
  - name: example_alerts
    interval: 30s  # 평가 주기 (기본값: 1m)
    rules:
      - alert: HighErrorRate
        expr: |
          rate(http_requests_total{status=~"5.."}[5m])
          / rate(http_requests_total[5m]) > 0.05
        for: 5m
        labels:
          severity: critical
          team: backend
        annotations:
          summary: "높은 에러율 감지"
          description: "{{ $labels.instance }}에서 에러율이 {{ $value | humanizePercentage }}입니다."

Alert Rule 구성 요소​

- alert: AlertName          # Alert 이름 (PascalCase 권장)
  expr: <PromQL>            # PromQL 표현식 (boolean 결과)
  for: <duration>           # Pending 지속 시간 (선택)
  labels:                   # 라우팅용 레이블
    <label_name>: <value>
  annotations:              # 알림 메시지용 정보
    <annotation_name>: <template>

주요 필드:

  • expr: Alert 조건을 정의하는 PromQL 쿼리
  • for: Alert가 Firing되기 전 대기 시간
  • labels: 라우팅 및 그룹화에 사용
  • annotations: 알림 메시지에 포함될 상세 정보

템플릿 변수​

annotations:
  summary: "{{ $labels.instance }}에서 문제 발생"
  description: |
    현재 값: {{ $value }}
    인스턴스: {{ $labels.instance }}
    작업: {{ $labels.job }}

사용 가능한 변수:

  • {{ $labels.<label_name> }}: Alert의 레이블 값
  • {{ $value }}: PromQL 표현식 결과값
  • {{ $externalLabels }}: Prometheus 외부 레이블
레이블 vs 어노테이션
  • Labels: 라우팅, 그룹화, Silence에 사용 (간결하게)
  • Annotations: 사람이 읽을 정보 (상세하게)

5.4 알림 라우팅​

Route Tree 구조​

route:
  receiver: 'default-receiver'
  group_by: ['alertname']

  routes:
    # Database 팀
    - match:
        team: database
      receiver: 'database-team'
      group_by: ['alertname', 'instance']

      routes:
        # Critical alerts는 PagerDuty로
        - match:
            severity: critical
          receiver: 'database-pager'

        # Warning alerts는 Slack으로
        - match:
            severity: warning
          receiver: 'database-slack'

    # Backend 팀
    - match:
        team: backend
      receiver: 'backend-team'

      routes:
        - match_re:
            service: ^(api|web)$
          receiver: 'backend-web-team'

그룹화 옵션​

route:
  # 그룹화 기준
  group_by: ['alertname', 'cluster', 'service']

  # 첫 알림 전 대기 시간 (같은 그룹의 다른 알림 수집)
  group_wait: 10s

  # 그룹에 새 알림 추가 시 대기 시간
  group_interval: 5m

  # 동일 알림 재전송 간격
  repeat_interval: 4h

그룹화 전략:

# 전략 1: 서비스별 그룹화
group_by: ['service']
# → 서비스 단위로 모든 알림을 하나로 묶음

# 전략 2: 알림 이름별 그룹화
group_by: ['alertname']
# → 같은 종류의 알림끼리 묶음

# 전략 3: 다차원 그룹화
group_by: ['alertname', 'cluster', 'namespace']
# → 알림 종류 + 클러스터 + 네임스페이스로 세분화

5.6 Silence와 Inhibition​

Silence (일시 정지)​

UI에서 Silence 생성:

1. http://alertmanager:9093 접속
2. "Silences" 탭 클릭
3. "New Silence" 버튼 클릭
4. Matcher 설정 (예: alertname="HighCPU", instance="web-1")
5. 기간 설정 (예: 2시간)
6. 생성자 및 코멘트 입력

API로 Silence 생성:

# Silence 생성
curl -X POST http://alertmanager:9093/api/v2/silences \
  -H 'Content-Type: application/json' \
  -d '{
    "matchers": [
      {
        "name": "alertname",
        "value": "HighCPU",
        "isRegex": false
      },
      {
        "name": "instance",
        "value": "web-1",
        "isRegex": false
      }
    ],
    "startsAt": "2024-01-15T10:00:00Z",
    "endsAt": "2024-01-15T12:00:00Z",
    "createdBy": "admin",
    "comment": "서버 유지보수 작업 중"
  }'

Python으로 Silence 관리:

import requests
from datetime import datetime, timedelta

class AlertmanagerSilence:
    def __init__(self, alertmanager_url: str):
        self.url = alertmanager_url

    def create_silence(
        self,
        matchers: dict,
        duration_hours: int,
        created_by: str,
        comment: str
    ) -> str:
        """Silence 생성"""
        start = datetime.utcnow()
        end = start + timedelta(hours=duration_hours)

        silence = {
            "matchers": [
                {
                    "name": k,
                    "value": v,
                    "isRegex": False
                }
                for k, v in matchers.items()
            ],
            "startsAt": start.isoformat() + "Z",
            "endsAt": end.isoformat() + "Z",
            "createdBy": created_by,
            "comment": comment
        }

        response = requests.post(
            f"{self.url}/api/v2/silences",
            json=silence
        )
        response.raise_for_status()

        silence_id = response.json()["silenceID"]
        print(f"Silence created: {silence_id}")
        return silence_id

    def delete_silence(self, silence_id: str):
        """Silence 삭제"""
        response = requests.delete(
            f"{self.url}/api/v2/silence/{silence_id}"
        )
        response.raise_for_status()
        print(f"Silence deleted: {silence_id}")

    def list_silences(self) -> list:
        """활성 Silence 목록"""
        response = requests.get(f"{self.url}/api/v2/silences")
        response.raise_for_status()
        return response.json()

# 사용 예제
am = AlertmanagerSilence("http://localhost:9093")

# 유지보수를 위한 Silence 생성
silence_id = am.create_silence(
    matchers={"instance": "web-1"},
    duration_hours=2,
    created_by="ops-team",
    comment="서버 패치 작업"
)

# 작업 완료 후 Silence 해제
am.delete_silence(silence_id)

Inhibition (억제)​

Critical alert가 발생하면 관련된 warning alert 억제:

# alertmanager.yml
inhibit_rules:
  # Critical이면 Warning 억제
  - source_match:
      severity: critical
    target_match:
      severity: warning
    equal: ['alertname', 'instance']

  # 서비스 Down이면 모든 알림 억제
  - source_match:
      alertname: ServiceDown
    target_match_re:
      alertname: (HighLatency|HighErrorRate)
    equal: ['service']

  # 노드 Down이면 해당 노드의 모든 알림 억제
  - source_match:
      alertname: NodeDown
    target_match_re:
      alertname: .*
    equal: ['instance']

Inhibition 동작 방식:

┌──────────────────────┐
│  Critical Alert      │
│  (source)            │
│  - alertname: NodeDown
│  - instance: web-1   │
└──────────┬───────────┘
           │ inhibits
           ▼
┌──────────────────────┐
│  Warning Alerts      │
│  (target)            │
│  - alertname: HighCPU
│  - instance: web-1   │ ← 억제됨
└──────────────────────┘

5.8 Python으로 Webhook 수신기 구현​

FastAPI Webhook 서버​

from fastapi import FastAPI, Request
from pydantic import BaseModel
from typing import List, Dict, Any
from datetime import datetime
import httpx

app = FastAPI(title="Prometheus Alertmanager Webhook Receiver")

class Alert(BaseModel):
    status: str
    labels: Dict[str, str]
    annotations: Dict[str, str]
    startsAt: str
    endsAt: str
    generatorURL: str

class AlertWebhook(BaseModel):
    receiver: str
    status: str
    alerts: List[Alert]
    groupLabels: Dict[str, str]
    commonLabels: Dict[str, str]
    commonAnnotations: Dict[str, str]
    externalURL: str

# 알림 처리 핸들러
class AlertHandler:
    """Alert 처리 로직"""

    def __init__(self):
        self.slack_webhook_url = "YOUR_SLACK_WEBHOOK_URL"

    async def handle_alert(self, webhook: AlertWebhook):
        """Alert 수신 및 처리"""
        if webhook.status == "firing":
            await self.handle_firing(webhook)
        else:
            await self.handle_resolved(webhook)

    async def handle_firing(self, webhook: AlertWebhook):
        """Firing 알림 처리"""
        print(f"🔥 Alert Firing: {webhook.groupLabels.get('alertname')}")

        for alert in webhook.alerts:
            severity = alert.labels.get('severity', 'unknown')

            # Critical alert는 즉시 Slack 알림
            if severity == 'critical':
                await self.send_slack_notification(alert)

            # 데이터베이스에 저장
            await self.save_to_database(alert)

            # 자동 대응 실행
            await self.auto_remediation(alert)

    async def handle_resolved(self, webhook: AlertWebhook):
        """Resolved 알림 처리"""
        print(f"✅ Alert Resolved: {webhook.groupLabels.get('alertname')}")

        for alert in webhook.alerts:
            # 복구 알림
            await self.send_resolution_notification(alert)

    async def send_slack_notification(self, alert: Alert):
        """Slack 알림 전송"""
        message = {
            "text": f"🚨 {alert.annotations.get('summary')}",
            "attachments": [
                {
                    "color": "danger",
                    "fields": [
                        {
                            "title": "Severity",
                            "value": alert.labels.get('severity'),
                            "short": True
                        },
                        {
                            "title": "Instance",
                            "value": alert.labels.get('instance'),
                            "short": True

from sqlalchemy import create_engine, Column, Integer, String, DateTime
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker
from datetime import datetime

Base = declarative_base()

class AlertHistory(Base):
    """Alert 이력 테이블"""
    __tablename__ = 'alert_history'

    id = Column(Integer, primary_key=True)
    alertname = Column(String(100), index=True)
    instance = Column(String(100), index=True)
    severity = Column(String(20), index=True)
    status = Column(String(20))  # firing, resolved
    summary = Column(String(500))
    description = Column(String(2000))
    starts_at = Column(DateTime)
    ends_at = Column(DateTime, nullable=True)
    received_at = Column(DateTime, default=datetime.utcnow)

# 데이터베이스 초기화
engine = create_engine('sqlite:///alerts.db')
Base.metadata.create_all(engine)
Session = sessionmaker(bind=engine)

class AlertRepository:
    """Alert 저장소"""

    def __init__(self):
        self.session = Session()

    def save_alert(self, alert: Alert, status: str):
        """Alert 저장"""
        history = AlertHistory(
            alertname=alert.labels.get('alertname'),
            instance=alert.labels.get('instance'),
            severity=alert.labels.get('severity'),
            status=status,
            summary=alert.annotations.get('summary'),
            description=alert.annotations.get('description'),
            starts_at=datetime.fromisoformat(
                alert.startsAt.replace('Z', '+00:00')
            ),
            ends_at=datetime.fromisoformat(
                alert.endsAt.replace('Z', '+00:00')
            ) if alert.endsAt != '0001-01-01T00:00:00Z' else None
        )

        self.session.add(history)
        self.session.commit()

    def get_active_alerts(self) -> List[AlertHistory]:
        """현재 발생 중인 Alert 조회"""
        return self.session.query(AlertHistory).filter(
            AlertHistory.status == 'firing',
            AlertHistory.ends_at.is_(None)
        ).all()

    def get_alert_stats(self, hours: int = 24) -> dict:
        """Alert 통계"""
        from datetime import timedelta
        cutoff = datetime.utcnow() - timedelta(hours=hours)

        alerts = self.session.query(AlertHistory).filter(
            AlertHistory.received_at >= cutoff
        ).all()

        return {
            "total": len(alerts),
            "by_severity": {
                "critical": len([a for a in alerts if a.severity == 'critical']),
                "warning": len([a for a in alerts if a.severity == 'warning']),
                "info": len([a for a in alerts if a.severity == 'info'])
            },
            "by_alertname": {}
        }

import requests
from typing import Dict, List
from datetime import datetime, timedelta

class AlertmanagerClient:
    """Alertmanager API 클라이언트"""

    def __init__(self, base_url: str = "http://localhost:9093"):
        self.base_url = base_url

    def send_alert(
        self,
        alertname: str,
        labels: Dict[str, str],
        annotations: Dict[str, str],
        starts_at: datetime = None,
        ends_at: datetime = None
    ):
        """프로그래매틱하게 Alert 발송"""
        if starts_at is None:
            starts_at = datetime.utcnow()

        # ends_at이 없으면 5분 후 자동 해제
        if ends_at is None:
            ends_at = starts_at + timedelta(minutes=5)

        alert = {
            "labels": {
                "alertname": alertname,
                **labels
            },
            "annotations": annotations,
            "startsAt": starts_at.isoformat() + "Z",
            "endsAt": ends_at.isoformat() + "Z",
            "generatorURL": "http://custom-app/alerts"
        }

        response = requests.post(
            f"{self.base_url}/api/v2/alerts",
            json=[alert]
        )
        response.raise_for_status()
        return response.json()

    def get_alerts(self, filters: Dict[str, str] = None) -> List[dict]:
        """현재 Alert 목록 조회"""
        params = {}
        if filters:
            params['filter'] = ','.join(
                f'{k}="{v}"' for k, v in filters.items()
            )

        response = requests.get(
            f"{self.base_url}/api/v2/alerts",
            params=params
        )
        response.raise_for_status()
        return response.json()

from prometheus_client import Gauge, Counter, generate_latest
from datetime import datetime, timedelta
import asyncio

class BusinessAlertManager:
    """비즈니스 메트릭 모니터링 및 알림"""

    def __init__(self, am_client: AlertmanagerClient):
        self.am_client = am_client

        # 메트릭 정의
        self.payment_failure_rate = Gauge(
            'payment_failure_rate',
            'Payment failure rate'
        )
        self.order_processing_time = Gauge(
            'order_processing_time_seconds',
            'Order processing time'
        )

    async def monitor_payment_failures(self):
        """결제 실패율 모니터링"""
        while True:
            # 실제로는 데이터베이스나 API에서 가져옴
            failure_rate = await self.get_payment_failure_rate()
            self.payment_failure_rate.set(failure_rate)

            # 임계값 초과 시 알림
            if failure_rate > 0.10:  # 10%
                self.am_client.send_alert(
                    alertname="HighPaymentFailureRate",
                    labels={
                        "severity": "critical",
                        "team": "payment",
                        "service": "payment-processor"
                    },
                    annotations={
                        "summary": f"결제 실패율 {failure_rate*100:.1f}%",
                        "description": (
                            f"현재 결제 실패율이 {failure_rate*100:.1f}%입니다. "
                            "즉시 확인이 필요합니다."
                        ),
                        "runbook_url": "https://wiki.example.com/runbooks/payment-failures"
                    }
                )

            await asyncio.sleep(60)  # 1분마다 체크

    async def get_payment_failure_rate(self) -> float:
        """결제 실패율 계산"""
        # 실제 구현
        return 0.05  # 5%

    async def monitor_order_processing(self):
        """주문 처리 시간 모니터링"""
        while True:
            avg_time = await self.get_avg_order_processing_time()
            self.order_processing_time.set(avg_time)

            if avg_time > 30:  # 30초
                self.am_client.send_alert(
                    alertname="SlowOrderProcessing",
                    labels={
                        "severity": "warning",
                        "team": "backend",
                        "service": "order-service"
                    },
                    annotations={
                        "summary": f"주문 처리 지연 ({avg_time:.1f}초)",
                        "description": (
                            f"평균 주문 처리 시간이 {avg_time:.1f}초입니다."
                        )
                    }
                )

            await asyncio.sleep(60)

    async def get_avg_order_processing_time(self) -> float:
        """평균 주문 처리 시간"""
        return 15.0

# 실행
async def main():
    am_client = AlertmanagerClient()
    manager = BusinessAlertManager(am_client)

    # 모니터링 시작
    await asyncio.gather(
        manager.monitor_payment_failures(),

    )

if __name__ == "__main__":

      receiver: 'slack'
      active_time_intervals:
        - business_hours

    # 업무 외 시간에는 무시
    - match:
        severity: warning
      receiver: 'null'
      mute_time_intervals:
        - business_hours

    # Critical은 항상 PagerDuty
    - match:
        severity: critical
      receiver: 'pagerduty'

5.12 베스트 프랙티스​

1. Alert Rule 작성​

Alert 작성 원칙

증상 기반 알림 (Symptom-based)

# Good: 사용자가 경험하는 증상
- alert: HighLatency
  expr: http_request_duration_seconds{quantile="0.99"} > 1

# Bad: 원인 추정
- alert: HighCPU
  expr: cpu_usage > 80

실행 가능한 알림 (Actionable)

  • 알림을 받은 사람이 무엇을 해야 하는지 명확해야 함
  • Runbook 링크 필수

의미 있는 알림만 (Meaningful)

  • False positive가 많으면 무시됨
  • 임계값과 for 절을 적절히 조정

2. 알림 구조화​

# 팀별 레이블 체계
labels:
  severity: critical|warning|info
  team: backend|frontend|database|ops
  service: api|web|worker
  environment: production|staging

# 일관된 어노테이션
annotations:
  summary: "간단한 한 줄 요약"
  description: "상세한 설명 (현재 값 포함)"
  runbook_url: "https://wiki.example.com/runbooks/..."
  dashboard_url: "https://grafana.example.com/d/..."
  graph_url: "{{ .GeneratorURL }}"

3. 라우팅 전략​

route:
  # 기본 설정
  receiver: 'default'
  group_by: ['alertname', 'service']
  group_wait: 10s
  group_interval: 5m
  repeat_interval: 4h

  routes:
    # Critical → PagerDuty (24/7)
    - match:
        severity: critical
      receiver: 'pagerduty'
      continue: true  # 다른 수신자에게도 전송

    # Critical → Slack (즉시 알림)
    - match:
        severity: critical
      receiver: 'slack-critical'

    # Warning → Slack (묶어서 전송)
    - match:
        severity: warning
      receiver: 'slack-warning'
      group_wait: 30s
      group_interval: 10m

    # Info → 로그만
    - match:
        severity: info
      receiver: 'null'

4. Inhibition 활용​

inhibit_rules:
  # 노드 다운 시 해당 노드의 다른 알림 억제
  - source_match:
      alertname: NodeDown
    target_match_re:
      alertname: (HighCPU|HighMemory|DiskSpaceLow)
    equal: ['instance']

  # 서비스 다운 시 성능 알림 억제
  - source_match:
      alertname: ServiceDown
    target_match_re:
      alertname: (HighLatency|HighErrorRate)
    equal: ['service']

문제 1: Basic Alert Rule​

FastAPI 애플리케이션의 에러율이 3% 이상일 때 알림을 보내는 Alert Rule을 작성하세요.

해답 보기
groups:
  - name: api_alerts
    interval: 30s
    rules:
      - alert: APIHighErrorRate
        expr: |
          sum(rate(http_requests_total{status=~"5.."}[5m])) by (service)
          /
          sum(rate(http_requests_total[5m])) by (service)
          > 0.03
        for: 5m
        labels:
          severity: warning
          team: backend
          service: api
        annotations:
          summary: "API 에러율 상승"
          description: |
            서비스 {{ $labels.service }}의 에러율이 {{ $value | humanizePercentage }}입니다.
            5분 평균 기준 3%를 초과했습니다.
          runbook_url: "https://wiki.example.com/runbooks/api-errors"

  • for: 5m으로 일시적 스파이크 무시
  • 비율 계산으로 정확한 에러율 측정
  • Runbook URL 제공

문제 2: Webhook 수신기​

Alertmanager로부터 Webhook을 받아서 Slack으로 포맷팅하여 전송하는 FastAPI 앱을 작성하세요.

해답 보기
from fastapi import FastAPI, Request
from pydantic import BaseModel
from typing import List, Dict
import httpx

app = FastAPI()

class Alert(BaseModel):
    status: str
    labels: Dict[str, str]
    annotations: Dict[str, str]
    startsAt: str

class AlertWebhook(BaseModel):
    status: str
    alerts: List[Alert]
    groupLabels: Dict[str, str]

SLACK_WEBHOOK_URL = "YOUR_SLACK_WEBHOOK_URL"

def format_slack_message(webhook: AlertWebhook) -> dict:
    """Slack 메시지 포맷"""
    color = "danger" if webhook.status == "firing" else "good"
    emoji = "🔥" if webhook.status == "firing" else "✅"

    # Alert 목록
    alert_text = ""
    for alert in webhook.alerts:
        alert_text += f"\n• *{alert.labels.get('instance')}*: {alert.annotations.get('summary')}"

    return {
        "text": f"{emoji} *{webhook.groupLabels.get('alertname')}*",
        "attachments": [
            {
                "color": color,
                "fields": [
                    {
                        "title": "Status",
                        "value": webhook.status.upper(),
                        "short": True
                    },
                    {
                        "title": "Severity",
                        "value": webhook.alerts[0].labels.get('severity', 'unknown'),
                        "short": True
                    },
                    {
                        "title": "Alerts",
                        "value": alert_text,
                        "short": False
                    }
                ]
            }
        ]
    }

@app.post("/webhook")
async def receive_alert(webhook: AlertWebhook):
    """Webhook 수신 및 Slack 전송"""
    # Slack 메시지 생성
    slack_msg = format_slack_message(webhook)

    # Slack으로 전송
    async with httpx.AsyncClient() as client:
        response = await client.post(SLACK_WEBHOOK_URL, json=slack_msg)
        response.raise_for_status()

    return {"status": "ok"}

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)

문제 3: Silence 자동화​

배포 시 자동으로 30분 Silence를 생성하고, 배포 완료 시 해제하는 스크립트를 작성하세요.

해답 보기
import requests
from datetime import datetime, timedelta
from contextlib import contextmanager

class DeploymentSilence:
    """배포 중 Silence 관리"""

    def __init__(self, alertmanager_url: str):
        self.url = alertmanager_url

    def create_deployment_silence(
        self,
        service: str,
        environment: str,
        duration_minutes: int = 30,
        created_by: str = "deploy-bot"
    ) -> str:
        """배포 Silence 생성"""
        start = datetime.utcnow()
        end = start + timedelta(minutes=duration_minutes)

        silence = {
            "matchers": [
                {"name": "service", "value": service, "isRegex": False},
                {"name": "environment", "value": environment, "isRegex": False}
            ],
            "startsAt": start.isoformat() + "Z",
            "endsAt": end.isoformat() + "Z",
            "createdBy": created_by,
            "comment": f"Automated silence during {service} deployment"
        }

        response = requests.post(
            f"{self.url}/api/v2/silences",
            json=silence
        )
        response.raise_for_status()

        silence_id = response.json()["silenceID"]
        print(f"✅ Silence created: {silence_id} (expires in {duration_minutes}m)")
        return silence_id

    def delete_silence(self, silence_id: str):
        """Silence 삭제"""
        response = requests.delete(
            f"{self.url}/api/v2/silence/{silence_id}"
        )
        response.raise_for_status()
        print(f"✅ Silence deleted: {silence_id}")

    @contextmanager
    def deployment_silence(self, service: str, environment: str):
        """Context manager로 자동 Silence 관리"""
        silence_id = self.create_deployment_silence(service, environment)
        try:
            yield silence_id
        finally:
            # 배포 완료 후 자동 해제
            self.delete_silence(silence_id)
            print(f"✅ Deployment completed, silence removed")

# 사용 예제
ds = DeploymentSilence("http://localhost:9093")

# Context manager 사용
with ds.deployment_silence(service="api", environment="production"):
    print("🚀 Deploying...")
    # 실제 배포 작업
    import time
    time.sleep(5)
    print("✅ Deployment done")

# Silence가 자동으로 해제됨
