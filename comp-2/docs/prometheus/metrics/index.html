2. 메트릭과 Python 클라이언트

Prometheus의 핵심은 메트릭입니다. 이 문서에서는 Prometheus의 4가지 메트릭 타입과 Python에서 prometheus_client를 사용하는 방법을 상세히 다룹니다.

2.1 Prometheus 메트릭 타입​

Prometheus는 4가지 핵심 메트릭 타입을 제공합니다.

2.1.1 Counter (카운터)​

Counter는 증가만 가능한 누적 메트릭입니다. 재시작 시에만 0으로 리셋됩니다.

사용 사례:

  • HTTP 요청 수
  • 처리된 작업 수
  • 발생한 에러 수
  • 데이터베이스 쿼리 수
from prometheus_client import Counter

# Counter 정의
http_requests_total = Counter(
    'http_requests_total',
    'Total HTTP requests',
    ['method', 'endpoint', 'status']
)

# Counter 증가
http_requests_total.labels(method='GET', endpoint='/api/users', status='200').inc()
http_requests_total.labels(method='POST', endpoint='/api/users', status='201').inc(1)

# 특정 값만큼 증가
bytes_sent = Counter('bytes_sent_total', 'Total bytes sent')
bytes_sent.inc(1024)  # 1024 바이트 증가
Counter 사용 주의사항

Counter는 절대 감소하지 않습니다. 감소할 수 있는 값(예: 현재 활성 연결 수)은 Gauge를 사용하세요.

2.1.2 Gauge (게이지)​

Gauge는 증가와 감소가 모두 가능한 메트릭입니다. 현재 상태를 나타냅니다.

사용 사례:

  • 현재 메모리 사용량
  • 활성 연결 수
  • 큐의 현재 크기
  • 온도, CPU 사용률
from prometheus_client import Gauge

# Gauge 정의
active_users = Gauge('active_users', 'Number of active users')
memory_usage = Gauge('memory_usage_bytes', 'Memory usage in bytes')

# Gauge 조작
active_users.set(42)      # 특정 값으로 설정
active_users.inc()        # 1 증가
active_users.inc(5)       # 5 증가
active_users.dec()        # 1 감소
active_users.dec(3)       # 3 감소

# 현재 시간 추적
last_success = Gauge('job_last_success_timestamp', 'Last successful job completion')
last_success.set_to_current_time()

# 함수 실행 시간 추적
request_duration = Gauge('request_duration_seconds', 'Request duration')

@request_duration.time()
def process_request():
    # 처리 로직
    pass

# 컨텍스트 매니저 사용
with request_duration.time():
    # 처리 로직
    pass
Gauge의 유용한 메서드
  • set(value): 값 설정
  • inc(amount=1): 증가
  • dec(amount=1): 감소
  • set_to_current_time(): 현재 Unix 타임스탬프로 설정
  • track_inprogress(): 진행 중인 작업 수 추적

2.1.3 Histogram (히스토그램)​

Histogram은 관측값을 버킷에 분류하고, 합계와 개수를 제공합니다. 주로 요청 지속 시간이나 응답 크기를 측정합니다.

사용 사례:

  • API 응답 시간
  • 요청 크기
  • 쿼리 실행 시간
from prometheus_client import Histogram

# Histogram 정의 (기본 버킷 사용)
request_duration = Histogram(
    'http_request_duration_seconds',
    'HTTP request duration in seconds'
)

# 커스텀 버킷 사용
request_size = Histogram(
    'http_request_size_bytes',
    'HTTP request size in bytes',
    buckets=[100, 500, 1000, 5000, 10000, 50000, 100000]
)

# 레이블과 함께 사용
api_latency = Histogram(
    'api_latency_seconds',
    'API endpoint latency',
    ['method', 'endpoint'],
    buckets=[0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1.0, 5.0]
)

# 값 관측
request_duration.observe(0.25)  # 0.25초 기록
request_size.observe(2048)      # 2048 바이트 기록

# 데코레이터 사용
@request_duration.time()
def handle_request():
    # 요청 처리
    pass

# 컨텍스트 매니저 사용
with api_latency.labels(method='GET', endpoint='/users').time():
    # API 처리
    pass

# 수동 타이밍
import time
start = time.time()
# 작업 수행
duration = time.time() - start
request_duration.observe(duration)

Histogram vs Summary
  • Histogram: 서버 측에서 백분위수 계산 가능, 집계 가능
  • Summary: 클라이언트 측에서 백분위수 계산, 집계 불가능
  • 대부분의 경우 Histogram을 권장합니다.

2.1.4 Summary (요약)​

Summary는 Histogram과 유사하지만 클라이언트 측에서 백분위수를 계산합니다.

사용 사례:

  • 정확한 백분위수가 필요한 경우 (하지만 Histogram 권장)
from prometheus_client import Summary

# Summary 정의
request_latency = Summary(
    'request_latency_seconds',
    'Request latency in seconds'
)

# 백분위수 지정
response_time = Summary(
    'response_time_seconds',
    'Response time in seconds',
    ['endpoint']
)

# 값 관측
request_latency.observe(0.25)
response_time.labels(endpoint='/api').observe(0.15)

# 데코레이터 사용
@request_latency.time()
def process():
    pass

# 컨텍스트 매니저
with request_latency.time():
    pass

from prometheus_client import start_http_server, Counter
import time
import random

# 메트릭 정의
requests_total = Counter('requests_total', 'Total requests')

2.3 레이블 (Labels) 사용법​

레이블을 사용하면 같은 메트릭을 여러 차원으로 분류할 수 있습니다.

from prometheus_client import Counter, Histogram

# 레이블이 있는 Counter
http_requests = Counter(
    'http_requests_total',
    'Total HTTP requests',
    ['method', 'endpoint', 'status_code']
)

# 레이블 값 지정하여 증가
http_requests.labels(method='GET', endpoint='/api/users', status_code='200').inc()
http_requests.labels(method='POST', endpoint='/api/users', status_code='201').inc()
http_requests.labels(method='GET', endpoint='/api/orders', status_code='200').inc()

# 레이블이 있는 Histogram
request_duration = Histogram(
    'request_duration_seconds',
    'Request duration',
    ['method', 'endpoint']
)

request_duration.labels(method='GET', endpoint='/api/users').observe(0.25)
request_duration.labels(method='POST', endpoint='/api/users').observe(0.35)

# 레이블을 변수로 저장하여 재사용
get_users_counter = http_requests.labels(method='GET', endpoint='/api/users', status_code='200')
get_users_counter.inc()
get_users_counter.inc()
레이블 사용 주의사항
  • 레이블 값의 카디널리티(고유 값의 개수)가 너무 높으면 성능 문제가 발생합니다
  • 사용자 ID, 이메일, IP 주소 등을 레이블로 사용하지 마세요
  • 레이블 값은 제한된 집합이어야 합니다 (예: HTTP 메서드, 상태 코드)

2.3.1 레이블 기본값 설정​

from prometheus_client import Counter

# 특정 레이블에 기본값 제공
errors = Counter(
    'errors_total',
    'Total errors',
    ['service', 'severity']
)

# 기본값 사용
errors.labels(service='api', severity='warning').inc()

# 부분적으로만 지정
class ErrorTracker:
    def __init__(self, service_name):
        self.service_name = service_name

    def track_error(self, severity):
        errors.labels(service=self.service_name, severity=severity).inc()

tracker = ErrorTracker('payment-service')
tracker.track_error('error')
tracker.track_error('critical')

2.4 메트릭 명명 규칙​

Prometheus 메트릭은 일관된 명명 규칙을 따라야 합니다.

2.4.1 기본 규칙​

  1. 소문자와 언더스코어 사용: http_requests_total
  2. 단위 접미사 사용: _seconds, _bytes, _total
  3. 메트릭 타입 접미사:
    • Counter: _total (예: requests_total)
    • Gauge: 접미사 없음 (예: memory_usage_bytes)
    • Histogram/Summary: _seconds, _bytes 등

2.4.2 명명 예시​

from prometheus_client import Counter, Gauge, Histogram

# 좋은 예
http_requests_total = Counter('http_requests_total', '...')
database_queries_total = Counter('database_queries_total', '...')
memory_usage_bytes = Gauge('memory_usage_bytes', '...')
cpu_usage_ratio = Gauge('cpu_usage_ratio', '...')  # 0-1 범위
request_duration_seconds = Histogram('request_duration_seconds', '...')
response_size_bytes = Histogram('response_size_bytes', '...')

# 나쁜 예
requests = Counter('requests', '...')  # _total 없음
RequestsTotal = Counter('RequestsTotal', '...')  # 대문자 사용
http_requests_count = Counter('http_requests_count', '...')  # _count 대신 _total

2.4.3 네임스페이스 사용​

# 애플리케이션별 네임스페이스
myapp_http_requests_total = Counter('myapp_http_requests_total', '...')
myapp_database_errors_total = Counter('myapp_database_errors_total', '...')

# 서비스별 네임스페이스

2.5 Flask 통합​

Flask 애플리케이션에 Prometheus 메트릭을 통합하는 방법입니다.

2.5.1 기본 통합​

from flask import Flask, request
from prometheus_client import Counter, Histogram, generate_latest, CONTENT_TYPE_LATEST
import time

app = Flask(__name__)

# 메트릭 정의
REQUEST_COUNT = Counter(
    'flask_http_requests_total',
    'Total Flask HTTP requests',
    ['method', 'endpoint', 'status']
)

REQUEST_DURATION = Histogram(
    'flask_http_request_duration_seconds',
    'Flask HTTP request duration',
    ['method', 'endpoint']
)

# Before/After request 핸들러
@app.before_request
def before_request():
    request._prometheus_start_time = time.time()

@app.after_request
def after_request(response):
    if hasattr(request, '_prometheus_start_time'):
        duration = time.time() - request._prometheus_start_time
        REQUEST_DURATION.labels(
            method=request.method,
            endpoint=request.endpoint or 'unknown'
        ).observe(duration)

        REQUEST_COUNT.labels(
            method=request.method,
            endpoint=request.endpoint or 'unknown',
            status=response.status_code
        ).inc()

    return response

# 메트릭 엔드포인트
@app.route('/metrics')
def metrics():
    return generate_latest(), 200, {'Content-Type': CONTENT_TYPE_LATEST}

# 비즈니스 로직
@app.route('/')
def home():
    return 'Hello, World!'

@app.route('/api/users')
def get_users():
    # 사용자 목록 반환
    return {'users': []}

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

2.5.2 미들웨어를 사용한 고급 통합​

from flask import Flask, request, g
from prometheus_client import Counter, Histogram, Gauge, make_wsgi_app
from werkzeug.middleware.dispatcher import DispatcherMiddleware
import time

app = Flask(__name__)

# 메트릭 정의
REQUEST_COUNT = Counter(
    'flask_requests_total',
    'Total requests',
    ['method', 'endpoint', 'status']
)

REQUEST_DURATION = Histogram(
    'flask_request_duration_seconds',
    'Request duration',
    ['method', 'endpoint'],
    buckets=[0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1.0, 5.0]
)

REQUEST_IN_PROGRESS = Gauge(
    'flask_requests_in_progress',
    'Requests in progress',
    ['method', 'endpoint']
)

EXCEPTION_COUNT = Counter(
    'flask_exceptions_total',
    'Total exceptions',
    ['method', 'endpoint', 'exception_type']
)

@app.before_request
def before_request():
    g.start_time = time.time()
    endpoint = request.endpoint or 'unknown'
    REQUEST_IN_PROGRESS.labels(method=request.method, endpoint=endpoint).inc()

@app.after_request
def after_request(response):
    endpoint = request.endpoint or 'unknown'
    REQUEST_IN_PROGRESS.labels(method=request.method, endpoint=endpoint).dec()

    if hasattr(g, 'start_time'):
        duration = time.time() - g.start_time
        REQUEST_DURATION.labels(method=request.method, endpoint=endpoint).observe(duration)

    REQUEST_COUNT.labels(
        method=request.method,
        endpoint=endpoint,
        status=response.status_code
    ).inc()

    return response

@app.errorhandler(Exception)
def handle_exception(e):
    endpoint = request.endpoint or 'unknown'
    REQUEST_IN_PROGRESS.labels(method=request.method, endpoint=endpoint).dec()

    EXCEPTION_COUNT.labels(
        method=request.method,
        endpoint=endpoint,
        exception_type=type(e).__name__
    ).inc()

    raise e

# Prometheus 메트릭 엔드포인트 추가
app.wsgi_app = DispatcherMiddleware(app.wsgi_app, {
    '/metrics': make_wsgi_app()
})

@app.route('/')
def home():
    return 'Hello!'

@app.route('/api/data')
def get_data():
    time.sleep(0.1)  # 처리 시간 시뮬레이션
    return {'data': [1, 2, 3]}

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

2.5.3 비즈니스 메트릭 추가​

from flask import Flask, request, jsonify
from prometheus_client import Counter, Histogram, Gauge, make_wsgi_app
from werkzeug.middleware.dispatcher import DispatcherMiddleware

app = Flask(__name__)

# 비즈니스 메트릭
USER_REGISTRATIONS = Counter('user_registrations_total', 'Total user registrations')
USER_LOGINS = Counter('user_logins_total', 'Total user logins', ['status'])
ACTIVE_USERS = Gauge('active_users', 'Currently active users')
ORDER_VALUE = Histogram(
    'order_value_dollars',
    'Order value in dollars',
    buckets=[10, 50, 100, 500, 1000, 5000]
)
SHOPPING_CART_SIZE = Histogram(
    'shopping_cart_items',
    'Number of items in shopping cart',
    buckets=[1, 2, 5, 10, 20, 50]
)

# 엔드포인트
@app.route('/api/register', methods=['POST'])
def register():
    # 회원가입 로직
    USER_REGISTRATIONS.inc()
    ACTIVE_USERS.inc()
    return jsonify({'status': 'success'}), 201

@app.route('/api/login', methods=['POST'])
def login():
    # 로그인 로직
    success = True  # 실제로는 인증 확인

    if success:
        USER_LOGINS.labels(status='success').inc()
        ACTIVE_USERS.inc()
        return jsonify({'status': 'success'}), 200
    else:
        USER_LOGINS.labels(status='failure').inc()
        return jsonify({'status': 'failure'}), 401

@app.route('/api/logout', methods=['POST'])
def logout():
    ACTIVE_USERS.dec()
    return jsonify({'status': 'success'}), 200

@app.route('/api/order', methods=['POST'])
def create_order():
    data = request.get_json()
    order_value = data.get('value', 0)
    item_count = data.get('item_count', 0)

    ORDER_VALUE.observe(order_value)
    SHOPPING_CART_SIZE.observe(item_count)

    return jsonify({'status': 'success'}), 201

# 메트릭 엔드포인트
app.wsgi_app = DispatcherMiddleware(app.wsgi_app, {
    '/metrics': make_wsgi_app()
})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

2.6 FastAPI 통합​

FastAPI에서 Prometheus 메트릭을 사용하는 방법입니다.

2.6.1 기본 통합​

from fastapi import FastAPI, Request
from prometheus_client import Counter, Histogram, make_asgi_app
import time

app = FastAPI()

# 메트릭 정의
REQUEST_COUNT = Counter(
    'fastapi_requests_total',
    'Total requests',
    ['method', 'endpoint', 'status']
)

REQUEST_DURATION = Histogram(
    'fastapi_request_duration_seconds',
    'Request duration',
    ['method', 'endpoint']
)

# 미들웨어
@app.middleware("http")
async def prometheus_middleware(request: Request, call_next):
    start_time = time.time()

    response = await call_next(request)

    duration = time.time() - start_time
    endpoint = request.url.path

    REQUEST_DURATION.labels(method=request.method, endpoint=endpoint).observe(duration)
    REQUEST_COUNT.labels(
        method=request.method,
        endpoint=endpoint,
        status=response.status_code
    ).inc()

    return response

# Prometheus 메트릭 엔드포인트
metrics_app = make_asgi_app()
app.mount("/metrics", metrics_app)

# API 엔드포인트
@app.get("/")
async def root():
    return {"message": "Hello World"}

@app.get("/api/users")
async def get_users():
    return {"users": []}

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)

2.6.3 커스텀 메트릭 추가​

from fastapi import FastAPI, Request, Response
from prometheus_client import Counter, Histogram, Gauge, make_asgi_app
from prometheus_fastapi_instrumentator import Instrumentator, metrics
import time

app = FastAPI()

# 커스텀 메트릭
DB_QUERIES = Counter('db_queries_total', 'Total database queries', ['query_type'])
CACHE_HITS = Counter('cache_hits_total', 'Cache hits', ['cache_name'])
CACHE_MISSES = Counter('cache_misses_total', 'Cache misses', ['cache_name'])
ACTIVE_CONNECTIONS = Gauge('active_db_connections', 'Active database connections')

# 기본 계측
instrumentator = Instrumentator()

from prometheus_client import Counter, Histogram
from functools import wraps
import time

# Request: 요청 수
requests_total = Counter(
    'service_requests_total',
    'Total requests',
    ['method', 'endpoint']
)

# Error: 에러 수
errors_total = Counter(
    'service_errors_total',
    'Total errors',
    ['method', 'endpoint', 'error_type']
)

# Duration: 처리 시간
request_duration_seconds = Histogram(
    'service_request_duration_seconds',
    'Request duration',
    ['method', 'endpoint'],
    buckets=[0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1.0, 5.0]
)

def track_red_metrics(method, endpoint):
    """RED 메트릭을 추적하는 데코레이터"""
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            start_time = time.time()

            try:
                result = func(*args, **kwargs)
                requests_total.labels(method=method, endpoint=endpoint).inc()
                return result
            except Exception as e:
                errors_total.labels(
                    method=method,
                    endpoint=endpoint,
                    error_type=type(e).__name__
                ).inc()
                raise
            finally:
                duration = time.time() - start_time
                request_duration_seconds.labels(method=method, endpoint=endpoint).observe(duration)

        return wrapper
    return decorator

class BatchJob:
    def __init__(self, job_name, pushgateway_url='localhost:9091'):
        self.job_name = job_name
        self.pushgateway_url = pushgateway_url

        # 각 배치 작업마다 별도의 레지스트리 사용
        self.registry = CollectorRegistry()

        # 메트릭 정의
        self.records_processed = Counter(
            'batch_records_processed_total',
            'Total records processed',
            ['status'],
            registry=self.registry
        )

        self.processing_duration = Gauge(
            'batch_processing_duration_seconds',
            'Batch processing duration',
            registry=self.registry
        )

        self.last_success = Gauge(
            'batch_last_success_timestamp',
            'Timestamp of last successful run',
            registry=self.registry
        )

        self.errors = Counter(
            'batch_errors_total',
            'Total batch errors',
            ['error_type'],
            registry=self.registry
        )

        self.batch_size = Gauge(
            'batch_size',
            'Number of records in current batch',
            registry=self.registry
        )

    def run(self):
        """배치 작업 실행"""
        print(f"Starting batch job: {self.job_name}")
        start_time = time.time()

        try:
            # 배치 데이터 가져오기
            records = self.fetch_records()
            self.batch_size.set(len(records))

            # 레코드 처리
            for record in records:
                try:
                    self.process_record(record)
                    self.records_processed.labels(status='success').inc()
                except Exception as e:
                    self.records_processed.labels(status='failure').inc()
                    self.errors.labels(error_type=type(e).__name__).inc()

            # 성공 타임스탬프 기록
            self.last_success.set(time.time())

        except Exception as e:
            print(f"Batch job failed: {e}")
            self.errors.labels(error_type=type(e).__name__).inc()
            raise
        finally:
            # 처리 시간 기록
            duration = time.time() - start_time
            self.processing_duration.set(duration)

            # Pushgateway로 메트릭 전송
            self.push_metrics()

            print(f"Batch job completed in {duration:.2f} seconds")

    def fetch_records(self):
        """처리할 레코드 가져오기 (시뮬레이션)"""
        import random
        return [{'id': i} for i in range(random.randint(100, 1000))]

        """레코드 처리 (시뮬레이션)"""
        time.sleep(0.01)

        # 10% 확률로 에러 발생
        if random.random() < 0.1:
            raise Exception("Processing error")

    def push_metrics(self):
        """Pushgateway로 메트릭 전송"""
        try:

                self.pushgateway_url,
                job=self.job_name,

    def cleanup_metrics(self):
        """Pushgateway에서 메트릭 삭제"""
        try:
            delete_from_gateway(self.pushgateway_url, job=self.job_name)
            print(f"Metrics deleted from {self.pushgateway_url}")
        except Exception as e:
            print(f"Failed to delete metrics: {e}")
