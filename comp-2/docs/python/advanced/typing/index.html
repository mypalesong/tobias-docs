5. 타입 힌팅

타입 힌팅(Type Hinting)은 Python 3.5부터 도입된 기능으로, 변수, 함수 매개변수, 반환값에 대한 타입 정보를 명시할 수 있게 해줍니다. Python은 여전히 동적 타입 언어이지만, 타입 힌트를 통해 코드의 가독성과 유지보수성을 크게 향상시킬 수 있습니다.

5.1 타입 힌트를 사용하는 이유​

타입 힌트는 런타임에 강제되지 않습니다. 즉, 잘못된 타입을 전달해도 Python은 실행됩니다. 그럼에도 불구하고 타입 힌트를 사용하는 이유는 다음과 같습니다:

주요 이점​

  1. 코드 문서화: 함수의 입력과 출력 타입을 명확히 표현
  2. IDE 지원 향상: 자동완성, 타입 체크, 리팩토링 도구 지원
  3. 버그 조기 발견: mypy 같은 정적 분석 도구로 실행 전 타입 오류 검출
  4. 리팩토링 안전성: 타입 변경 시 영향받는 코드를 쉽게 파악
  5. 팀 협업 개선: 코드 의도를 명확히 전달
  6. 대규모 프로젝트 유지보수: 코드베이스가 커질수록 타입 정보의 가치 증가
# 타입 힌트 없이
def greet(name, times):
    return f"Hello, {name}! " * times

# 타입 힌트 사용
def greet(name: str, times: int) -> str:
    return f"Hello, {name}! " * times

# IDE가 타입 오류를 경고할 수 있음
result = greet("Alice", "3")  # 'times'는 int여야 함
팁

타입 힌트는 점진적으로 도입할 수 있습니다. 전체 코드베이스를 한 번에 변경할 필요 없이, 중요한 부분부터 시작하세요.

5.2 기본 타입 힌트​

Python의 기본 내장 타입들을 사용한 타입 힌팅입니다.

기본 타입들​

# 기본 타입
def calculate(x: int, y: int) -> int:
    return x + y

def get_message() -> str:
    return "Hello, World!"

def calculate_ratio(a: float, b: float) -> float:
    return a / b

def is_valid(data: str) -> bool:
    return len(data) > 0

# 여러 변수 타입 힌트
name: str = "Alice"
age: int = 30
height: float = 165.5
is_student: bool = False

None 타입​

from typing import Optional

# 반환값이 없는 함수
def print_message(message: str) -> None:
    print(message)

# None을 반환할 수 있는 함수
def find_user(user_id: int) -> Optional[str]:
    """사용자를 찾으면 이름을 반환, 못 찾으면 None 반환"""
    users = {1: "Alice", 2: "Bob"}
    return users.get(user_id)

# Python 3.10+에서는 | 연산자 사용 가능
def find_user_new(user_id: int) -> str | None:
    users = {1: "Alice", 2: "Bob"}
    return users.get(user_id)

5.3 typing 모듈​

복잡한 타입을 표현하기 위한 typing 모듈의 주요 타입들입니다.

컨테이너 타입​

from typing import List, Dict, Tuple, Set

# List - 같은 타입의 요소들
def process_numbers(numbers: List[int]) -> List[int]:
    return [n * 2 for n in numbers]

# Dict - 키와 값의 타입 지정
def get_user_scores() -> Dict[str, int]:
    return {"Alice": 95, "Bob": 87, "Charlie": 92}

# Tuple - 고정된 길이와 각 위치의 타입
def get_coordinate() -> Tuple[float, float]:
    return (37.5665, 126.9780)

# 가변 길이 튜플
def get_numbers() -> Tuple[int, ...]:
    return (1, 2, 3, 4, 5)

# Set - 중복 없는 요소들
def get_unique_tags() -> Set[str]:
    return {"python", "programming", "typing"}

# 중첩된 타입
def get_student_grades() -> Dict[str, List[int]]:
    return {
        "Alice": [95, 87, 92],
        "Bob": [78, 85, 88]
    }

Python 3.9+ 내장 제네릭​

Python 3.9부터는 typing 모듈 없이 내장 타입을 직접 사용할 수 있습니다:

# Python 3.9+
def process_items(items: list[str]) -> dict[str, int]:
    return {item: len(item) for item in items}

def get_point() -> tuple[float, float]:
    return (1.0, 2.0)

# Python 3.8 이하에서는 typing 모듈 필요
from typing import List, Dict, Tuple

def process_items_old(items: List[str]) -> Dict[str, int]:
    return {item: len(item) for item in items}

Union과 Optional​

from typing import Union, Optional

# Union - 여러 타입 중 하나
def process_id(user_id: Union[int, str]) -> str:
    return str(user_id)

# Optional - None이 될 수 있는 타입 (Union[T, None]의 단축)
def get_config(key: str) -> Optional[int]:
    config = {"timeout": 30, "retries": 3}
    return config.get(key)

# Python 3.10+ | 연산자
def process_data(data: int | str | float) -> str:
    return str(data)

def find_item(item_id: int) -> dict | None:
    items = {1: {"name": "Apple"}}
    return items.get(item_id)

5.4 타입 별칭 (Type Aliases)​

복잡한 타입을 재사용 가능한 이름으로 정의할 수 있습니다.

from typing import List, Dict, Tuple, Union

# 간단한 별칭
UserId = int
UserName = str

def create_user(user_id: UserId, name: UserName) -> None:
    print(f"Creating user {name} with ID {user_id}")

# 복잡한 타입 별칭
Coordinate = Tuple[float, float]
Path = List[Coordinate]

def calculate_distance(path: Path) -> float:
    total = 0.0
    for i in range(len(path) - 1):
        x1, y1 = path[i]
        x2, y2 = path[i + 1]
        total += ((x2 - x1) ** 2 + (y2 - y1) ** 2) ** 0.5
    return total

# JSON 타입 별칭
JsonValue = Union[str, int, float, bool, None, Dict[str, 'JsonValue'], List['JsonValue']]
JsonObject = Dict[str, JsonValue]

def parse_config(config: JsonObject) -> None:
    print(config)

# Python 3.10+ TypeAlias를 사용한 명시적 선언
from typing import TypeAlias

Vector: TypeAlias = List[float]
Matrix: TypeAlias = List[Vector]

def multiply_matrix(a: Matrix, b: Matrix) -> Matrix:
    # 행렬 곱셈 구현
    return [[0.0]]

5.5 Callable 타입​

함수를 매개변수나 반환값으로 사용할 때의 타입 힌트입니다.

from typing import Callable, List

# Callable[[인자타입들], 반환타입]
def apply_operation(x: int, y: int, operation: Callable[[int, int], int]) -> int:
    return operation(x, y)

def add(a: int, b: int) -> int:
    return a + b

def multiply(a: int, b: int) -> int:
    return a * b

result1 = apply_operation(5, 3, add)        # 8
result2 = apply_operation(5, 3, multiply)   # 15

# 인자가 없는 함수
def run_task(task: Callable[[], None]) -> None:
    print("Starting task...")
    task()
    print("Task completed!")

def hello() -> None:
    print("Hello!")

run_task(hello)

# 고차 함수 (함수를 반환하는 함수)
def make_multiplier(n: int) -> Callable[[int], int]:
    def multiplier(x: int) -> int:
        return x * n
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))  # 10
print(triple(5))  # 15

# 리스트 정렬에 사용되는 키 함수
def sort_by_length(items: List[str],
                   key_func: Callable[[str], int]) -> List[str]:
    return sorted(items, key=key_func)

words = ["apple", "pie", "zoo", "elephant"]
sorted_words = sort_by_length(words, len)

5.6 제네릭 타입 (Generic Types)​

타입 매개변수를 사용하여 재사용 가능한 제네릭 코드를 작성할 수 있습니다.

TypeVar 사용​

from typing import TypeVar, List, Sequence

# TypeVar 정의
T = TypeVar('T')

def first_element(items: Sequence[T]) -> T:
    """시퀀스의 첫 번째 요소를 반환"""
    return items[0]

# 타입이 보존됨
numbers: List[int] = [1, 2, 3]
first_num: int = first_element(numbers)  # int 반환

words: List[str] = ["hello", "world"]
first_word: str = first_element(words)   # str 반환

# 여러 TypeVar 사용
K = TypeVar('K')
V = TypeVar('V')

def get_first_item(dictionary: Dict[K, V]) -> Tuple[K, V]:
    """딕셔너리의 첫 번째 항목을 반환"""
    key = next(iter(dictionary))
    return key, dictionary[key]

# 제약이 있는 TypeVar
from typing import Union

NumberType = TypeVar('NumberType', int, float)

def add_numbers(a: NumberType, b: NumberType) -> NumberType:
    return a + b

print(add_numbers(1, 2))      # OK: int
print(add_numbers(1.5, 2.5))  # OK: float
# print(add_numbers("a", "b")) # mypy 오류: str은 허용되지 않음

제네릭 클래스​

from typing import TypeVar, Generic, List, Optional

T = TypeVar('T')

class Stack(Generic[T]):
    """제네릭 스택 자료구조"""

    def __init__(self) -> None:
        self._items: List[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

    def peek(self) -> Optional[T]:
        return self._items[-1] if self._items else None

    def is_empty(self) -> bool:
        return len(self._items) == 0

# 타입별로 사용
int_stack: Stack[int] = Stack()
int_stack.push(1)
int_stack.push(2)
print(int_stack.pop())  # 2

str_stack: Stack[str] = Stack()
str_stack.push("hello")
str_stack.push("world")
print(str_stack.pop())  # "world"

# 복잡한 제네릭 클래스
K = TypeVar('K')
V = TypeVar('V')

class Cache(Generic[K, V]):
    """제네릭 캐시"""

    def __init__(self, max_size: int = 100) -> None:
        self._cache: Dict[K, V] = {}
        self._max_size = max_size

    def get(self, key: K) -> Optional[V]:
        return self._cache.get(key)

    def set(self, key: K, value: V) -> None:
        if len(self._cache) >= self._max_size:
            # 가장 오래된 항목 제거
            oldest_key = next(iter(self._cache))
            del self._cache[oldest_key]
        self._cache[key] = value

# 사용 예시
user_cache: Cache[int, str] = Cache()
user_cache.set(1, "Alice")
user_cache.set(2, "Bob")

config_cache: Cache[str, dict] = Cache()
config_cache.set("database", {"host": "localhost", "port": 5432})

5.7 Protocol (구조적 서브타이핑)​

Protocol은 덕 타이핑(duck typing)을 타입 시스템에서 표현하는 방법입니다. 명시적 상속 없이 특정 메서드나 속성을 가진 객체를 정의할 수 있습니다.

from typing import Protocol, List

class Drawable(Protocol):
    """그릴 수 있는 객체의 프로토콜"""

    def draw(self) -> str:
        ...

class Circle:
    def __init__(self, radius: float) -> None:
        self.radius = radius

    def draw(self) -> str:
        return f"Drawing circle with radius {self.radius}"

class Square:
    def __init__(self, side: float) -> None:
        self.side = side

    def draw(self) -> str:
        return f"Drawing square with side {self.side}"

# Drawable을 상속하지 않았지만 draw() 메서드가 있어서 호환됨
def render_shapes(shapes: List[Drawable]) -> None:
    for shape in shapes:
        print(shape.draw())

circle = Circle(5.0)
square = Square(10.0)
render_shapes([circle, square])  # OK!

# 더 복잡한 프로토콜 예시
from typing import Iterator

class SupportsClose(Protocol):
    """close() 메서드를 가진 객체"""

    def close(self) -> None:
        ...

class DatabaseConnection:
    def close(self) -> None:
        print("Closing database connection")

class FileHandler:
    def close(self) -> None:
        print("Closing file")

def cleanup_resource(resource: SupportsClose) -> None:
    resource.close()

# 두 클래스 모두 사용 가능
db = DatabaseConnection()
file = FileHandler()

cleanup_resource(db)
cleanup_resource(file)

# 런타임 체크 가능한 프로토콜
from typing import runtime_checkable

@runtime_checkable
class Sizeable(Protocol):
    def __len__(self) -> int:
        ...

def print_size(obj: Sizeable) -> None:
    print(f"Size: {len(obj)}")

# 런타임에 프로토콜 준수 여부 확인
items = [1, 2, 3]
if isinstance(items, Sizeable):
    print_size(items)  # OK

5.8 Literal 타입​

특정 리터럴 값만 허용하는 타입입니다.

from typing import Literal

# 특정 문자열 값만 허용
Mode = Literal["read", "write", "append"]

def open_file(filename: str, mode: Mode) -> None:
    print(f"Opening {filename} in {mode} mode")

open_file("data.txt", "read")    # OK
open_file("data.txt", "write")   # OK
# open_file("data.txt", "delete") # mypy 오류

# 숫자 리터럴
HttpStatus = Literal[200, 404, 500]

def handle_response(status: HttpStatus) -> str:
    if status == 200:
        return "Success"
    elif status == 404:
        return "Not Found"
    else:
        return "Server Error"

# 불리언 리터럴
def set_debug(enabled: Literal[True]) -> None:
    """디버그는 항상 True만 가능"""
    print("Debug enabled")

# 여러 타입의 리터럴 조합
def process_data(data: str, format: Literal["json", "xml", "csv"] = "json") -> dict:
    if format == "json":
        import json
        return json.loads(data)
    elif format == "xml":
        # XML 파싱
        return {}
    else:
        # CSV 파싱
        return {}

# Literal을 사용한 타입 좁히기
from typing import Union

def get_config(key: Literal["timeout"]) -> int: ...
def get_config(key: Literal["url"]) -> str: ...
def get_config(key: str) -> Union[int, str]:
    config = {"timeout": 30, "url": "http://example.com"}
    return config.get(key, "")

5.9 Final​

변경할 수 없는 값이나 오버라이드할 수 없는 메서드를 표시합니다.

from typing import Final

# 상수 정의
MAX_CONNECTIONS: Final = 100
API_KEY: Final[str] = "secret-key-123"

# 재할당 시도 시 mypy가 오류 발생
# MAX_CONNECTIONS = 200  # mypy 오류

# 클래스에서 Final 사용
class Configuration:
    # 클래스 변수를 상수로
    MAX_RETRIES: Final = 3

    def __init__(self) -> None:
        # 인스턴스 변수를 상수로
        self.timeout: Final = 30

config = Configuration()
# config.timeout = 60  # mypy 오류

# 메서드 오버라이딩 방지
from typing import final

class BaseAPI:
    @final
    def authenticate(self) -> bool:
        """이 메서드는 오버라이드할 수 없음"""
        return True

    def fetch_data(self) -> dict:
        """이 메서드는 오버라이드 가능"""
        return {}

class ExtendedAPI(BaseAPI):
    # def authenticate(self) -> bool:  # mypy 오류
    #     return False

    def fetch_data(self) -> dict:  # OK
        return {"extended": True}

# 클래스 상속 방지
@final
class DatabaseConnection:
    """이 클래스는 상속할 수 없음"""

    def connect(self) -> None:
        print("Connected")

# class ExtendedConnection(DatabaseConnection):  # mypy 오류
#     pass

5.10 mypy를 사용한 정적 타입 체크​

mypy는 Python 코드의 타입을 정적으로 분석하는 도구입니다.

# example.py

5.11 타입 스텁 파일 (.pyi)​

5.12 점진적 타이핑 (Gradual Typing)​

기존 코드베이스에 타입 힌트를 점진적으로 추가하는 전략입니다.

단계별 접근법​

# 1단계: 타입 힌트가 전혀 없는 코드

    return {
        "name": user["name"].upper(),
        "email": user["email"].lower(),
        "age": user["age"] + 1
    }

# 3단계: 더 구체적인 타입 사용
from typing import Dict, Any

def process_user(user: Dict[str, Any]) -> Dict[str, Any]:
    return {
        "name": user["name"].upper(),
        "email": user["email"].lower(),
        "age": user["age"] + 1
    }

# 4단계: TypedDict로 구조 명확히
from typing import TypedDict

class User(TypedDict):
    name: str
    email: str
    age: int

    name: str
    email: str

        "name": user["name"].upper(),

예제 1: 타입이 지정된 REST API 클라이언트​

from typing import TypedDict, List, Optional, Literal
from dataclasses import dataclass
import requests

# API 응답 타입 정의
class ApiResponse(TypedDict):
    status: int
    message: str
    data: dict

class UserData(TypedDict):
    id: int
    username: str
    email: str
    is_active: bool

@dataclass
class ApiError(Exception):
    status_code: int
    message: str

HttpMethod = Literal["GET", "POST", "PUT", "DELETE"]

class ApiClient:
    """타입이 지정된 API 클라이언트"""

    def __init__(self, base_url: str, api_key: str) -> None:
        self.base_url = base_url
        self.api_key = api_key
        self._session = requests.Session()
        self._session.headers.update({"Authorization": f"Bearer {api_key}"})

    def _request(
        self,
        method: HttpMethod,
        endpoint: str,
        data: Optional[dict] = None
    ) -> ApiResponse:
        """HTTP 요청 수행"""
        url = f"{self.base_url}{endpoint}"
        response = self._session.request(method, url, json=data)

        if response.status_code >= 400:
            raise ApiError(response.status_code, response.text)

        return response.json()

    def get_user(self, user_id: int) -> UserData:
        """사용자 정보 조회"""
        response = self._request("GET", f"/users/{user_id}")
        return response["data"]

    def create_user(
        self,
        username: str,
        email: str,
        is_active: bool = True
    ) -> UserData:
        """사용자 생성"""
        data = {
            "username": username,
            "email": email,
            "is_active": is_active
        }
        response = self._request("POST", "/users", data)
        return response["data"]

    def list_users(
        self,
        page: int = 1,
        per_page: int = 10,
        active_only: bool = False
    ) -> List[UserData]:
        """사용자 목록 조회"""
        params = {
            "page": page,
            "per_page": per_page,
            "active_only": active_only
        }
        response = self._request("GET", f"/users?{self._build_query(params)}")
        return response["data"]

    def _build_query(self, params: dict) -> str:
        """쿼리 문자열 생성"""
        return "&".join(f"{k}={v}" for k, v in params.items())

# 사용 예시
client = ApiClient("https://api.example.com", "secret-key")

try:
    user = client.get_user(1)
    print(f"User: {user['username']}")

    new_user = client.create_user("alice", "alice@example.com")
    print(f"Created user with ID: {new_user['id']}")

    users = client.list_users(page=1, per_page=5, active_only=True)
    for user in users:
        print(f"- {user['username']}")
except ApiError as e:
    print(f"API Error: {e.message}")

예제 2: 제네릭 데이터 저장소​

from typing import TypeVar, Generic, Dict, Optional, List, Callable
from dataclasses import dataclass
from datetime import datetime
from abc import ABC, abstractmethod

T = TypeVar('T')
K = TypeVar('K')

@dataclass
class Entity:
    """모든 엔티티의 기본 클래스"""
    id: int
    created_at: datetime
    updated_at: datetime

class Repository(Generic[T], ABC):
    """추상 저장소 클래스"""

    @abstractmethod
    def save(self, entity: T) -> T:
        """엔티티 저장"""
        pass

    @abstractmethod
    def find_by_id(self, entity_id: int) -> Optional[T]:
        """ID로 엔티티 조회"""
        pass

    @abstractmethod
    def find_all(self) -> List[T]:
        """모든 엔티티 조회"""
        pass

    @abstractmethod
    def delete(self, entity_id: int) -> bool:
        """엔티티 삭제"""
        pass

class InMemoryRepository(Repository[T]):
    """메모리 기반 저장소 구현"""

    def __init__(self) -> None:
        self._storage: Dict[int, T] = {}
        self._next_id = 1

    def save(self, entity: T) -> T:
        if isinstance(entity, Entity):
            if entity.id == 0:
                entity.id = self._next_id
                self._next_id += 1
                entity.created_at = datetime.now()
            entity.updated_at = datetime.now()
            self._storage[entity.id] = entity
        return entity

    def find_by_id(self, entity_id: int) -> Optional[T]:
        return self._storage.get(entity_id)

    def find_all(self) -> List[T]:
        return list(self._storage.values())

    def delete(self, entity_id: int) -> bool:
        if entity_id in self._storage:
            del self._storage[entity_id]
            return True
        return False

    def find_by(self, predicate: Callable[[T], bool]) -> List[T]:
        """조건에 맞는 엔티티 찾기"""
        return [entity for entity in self._storage.values() if predicate(entity)]

# 구체적인 엔티티 정의
@dataclass
class User(Entity):
    username: str
    email: str
    is_active: bool = True

@dataclass
class Product(Entity):
    name: str
    price: float
    stock: int

# 사용 예시
user_repo: Repository[User] = InMemoryRepository()
product_repo: Repository[Product] = InMemoryRepository()

# 사용자 저장
user1 = User(id=0, username="alice", email="alice@example.com",
             created_at=datetime.now(), updated_at=datetime.now())
saved_user = user_repo.save(user1)
print(f"Saved user: {saved_user.username} (ID: {saved_user.id})")

# 제품 저장
product1 = Product(id=0, name="Laptop", price=999.99, stock=10,
                   created_at=datetime.now(), updated_at=datetime.now())
saved_product = product_repo.save(product1)
print(f"Saved product: {saved_product.name} (ID: {saved_product.id})")

# 조회
found_user = user_repo.find_by_id(1)
if found_user:
    print(f"Found user: {found_user.username}")

# 조건부 검색 (InMemoryRepository만)
if isinstance(user_repo, InMemoryRepository):
    active_users = user_repo.find_by(lambda u: u.is_active)
    print(f"Active users: {len(active_users)}")

예제 3: 이벤트 시스템​

from typing import Protocol, TypeVar, Generic, Callable, List, Dict, Type
from dataclasses import dataclass
from datetime import datetime
from abc import ABC

# 이벤트 프로토콜
class Event(Protocol):
    """모든 이벤트가 따라야 할 프로토콜"""
    timestamp: datetime

@dataclass
class UserRegisteredEvent:
    user_id: int
    username: str
    email: str
    timestamp: datetime

@dataclass
class OrderPlacedEvent:
    order_id: int
    user_id: int
    total_amount: float
    timestamp: datetime

# 이벤트 핸들러 타입
E = TypeVar('E', bound=Event)
EventHandler = Callable[[E], None]

class EventBus:
    """타입 안전한 이벤트 버스"""

    def __init__(self) -> None:
        self._handlers: Dict[Type[Event], List[EventHandler]] = {}

    def subscribe(self, event_type: Type[E], handler: EventHandler[E]) -> None:
        """이벤트 핸들러 등록"""
        if event_type not in self._handlers:
            self._handlers[event_type] = []
        self._handlers[event_type].append(handler)

    def publish(self, event: E) -> None:
        """이벤트 발행"""
        event_type = type(event)
        if event_type in self._handlers:
            for handler in self._handlers[event_type]:
                handler(event)

    def unsubscribe(self, event_type: Type[E], handler: EventHandler[E]) -> None:
        """이벤트 핸들러 제거"""
        if event_type in self._handlers:
            self._handlers[event_type].remove(handler)

# 이벤트 핸들러 정의
def send_welcome_email(event: UserRegisteredEvent) -> None:
    print(f"Sending welcome email to {event.email}")

def update_user_stats(event: UserRegisteredEvent) -> None:
    print(f"Updating stats for user {event.username}")

def process_payment(event: OrderPlacedEvent) -> None:
    print(f"Processing payment of ${event.total_amount}")

def update_inventory(event: OrderPlacedEvent) -> None:
    print(f"Updating inventory for order {event.order_id}")

# 사용 예시
event_bus = EventBus()

# 핸들러 등록
event_bus.subscribe(UserRegisteredEvent, send_welcome_email)
event_bus.subscribe(UserRegisteredEvent, update_user_stats)
event_bus.subscribe(OrderPlacedEvent, process_payment)
event_bus.subscribe(OrderPlacedEvent, update_inventory)

# 이벤트 발행
user_event = UserRegisteredEvent(
    user_id=1,
    username="alice",
    email="alice@example.com",
    timestamp=datetime.now()
)
event_bus.publish(user_event)

order_event = OrderPlacedEvent(
    order_id=100,
    user_id=1,
    total_amount=99.99,
    timestamp=datetime.now()
)
event_bus.publish(order_event)

5. 타입 가드 (Type Guards)​

from typing import Union, TypeGuard

def is_string_list(val: list) -> TypeGuard[List[str]]:
    """타입 가드 함수"""
    return all(isinstance(x, str) for x in val)

def process(items: Union[List[str], List[int]]) -> None:
    if is_string_list(items):
        # 이 블록에서 items는 List[str]로 좁혀짐
        print(items[0].upper())
    else:
        # 이 블록에서 items는 List[int]로 좁혀짐
        print(items[0] + 1)

문제 1: 타입이 지정된 계산기​

간단한 계산기 클래스를 작성하세요. 사칙연산을 지원하고, 모든 메서드에 적절한 타입 힌트를 추가하세요.

해답 보기
from typing import Union, Literal

Number = Union[int, float]
Operation = Literal["+", "-", "*", "/"]

class Calculator:
    """타입이 지정된 계산기"""

    def __init__(self) -> None:
        self.result: Number = 0

    def add(self, a: Number, b: Number) -> Number:
        """덧셈"""
        self.result = a + b
        return self.result

    def subtract(self, a: Number, b: Number) -> Number:
        """뺄셈"""
        self.result = a - b
        return self.result

    def multiply(self, a: Number, b: Number) -> Number:
        """곱셈"""
        self.result = a * b
        return self.result

    def divide(self, a: Number, b: Number) -> Number:
        """나눗셈"""
        if b == 0:
            raise ValueError("Cannot divide by zero")
        self.result = a / b
        return self.result

    def calculate(self, a: Number, b: Number, operation: Operation) -> Number:
        """주어진 연산 수행"""
        operations = {
            "+": self.add,
            "-": self.subtract,
            "*": self.multiply,
            "/": self.divide
        }
        return operations[operation](a, b)

    def get_result(self) -> Number:
        """마지막 결과 반환"""
        return self.result

# 사용 예시
calc = Calculator()
print(calc.add(10, 5))        # 15
print(calc.multiply(3, 4))    # 12
print(calc.calculate(20, 4, "/"))  # 5.0

문제 2: 제네릭 캐시 구현​

LRU(Least Recently Used) 캐시를 제네릭 클래스로 구현하세요. 키와 값의 타입을 매개변수화하고, 타입 힌트를 완전히 적용하세요.

해답 보기
from typing import TypeVar, Generic, Dict, Optional, OrderedDict as OrderedDictType
from collections import OrderedDict

K = TypeVar('K')
V = TypeVar('V')

class LRUCache(Generic[K, V]):
    """LRU 캐시 구현"""

    def __init__(self, capacity: int) -> None:
        if capacity <= 0:
            raise ValueError("Capacity must be positive")
        self.capacity = capacity
        self.cache: OrderedDictType[K, V] = OrderedDict()

    def get(self, key: K) -> Optional[V]:
        """키로 값 조회. 없으면 None 반환"""
        if key not in self.cache:
            return None

        # 최근 사용으로 이동
        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key: K, value: V) -> None:
        """키-값 쌍 저장"""
        if key in self.cache:
            # 기존 항목 업데이트
            self.cache.move_to_end(key)
        else:
            # 새 항목 추가
            if len(self.cache) >= self.capacity:
                # 가장 오래된 항목 제거
                self.cache.popitem(last=False)

        self.cache[key] = value

    def remove(self, key: K) -> bool:
        """키 제거. 성공 시 True, 키가 없으면 False"""
        if key in self.cache:
            del self.cache[key]
            return True
        return False

    def clear(self) -> None:
        """모든 항목 제거"""
        self.cache.clear()

    def size(self) -> int:
        """현재 캐시 크기"""
        return len(self.cache)

# 사용 예시
# 정수 키, 문자열 값
cache1: LRUCache[int, str] = LRUCache(3)
cache1.put(1, "one")
cache1.put(2, "two")
cache1.put(3, "three")
print(cache1.get(1))  # "one"

cache1.put(4, "four")  # 2가 제거됨
print(cache1.get(2))   # None

# 문자열 키, 딕셔너리 값
cache2: LRUCache[str, dict] = LRUCache(2)
cache2.put("user1", {"name": "Alice", "age": 30})
cache2.put("user2", {"name": "Bob", "age": 25})
print(cache2.get("user1"))  # {"name": "Alice", "age": 30}

문제 3: Protocol을 사용한 플러그인 시스템​

다양한 데이터 소스에서 데이터를 읽을 수 있는 플러그인 시스템을 구현하세요. Protocol을 사용하여 플러그인 인터페이스를 정의하고, 여러 구현체를 만드세요.

해답 보기
from typing import Protocol, List, Dict, Any, Optional
from abc import abstractmethod
import json
import csv
from io import StringIO

class DataSource(Protocol):
    """데이터 소스 프로토콜"""

    @abstractmethod
    def read(self) -> List[Dict[str, Any]]:
        """데이터 읽기"""
        ...

    @abstractmethod
    def validate(self) -> bool:
        """데이터 소스 유효성 검증"""
        ...

class JsonDataSource:
    """JSON 데이터 소스"""

    def __init__(self, json_string: str) -> None:
        self.json_string = json_string

    def read(self) -> List[Dict[str, Any]]:
        """JSON 데이터 파싱"""
        data = json.loads(self.json_string)
        if isinstance(data, list):
            return data
        return [data]

    def validate(self) -> bool:
        """JSON 유효성 검증"""
        try:
            json.loads(self.json_string)
            return True
        except json.JSONDecodeError:
            return False

class CsvDataSource:
    """CSV 데이터 소스"""

    def __init__(self, csv_string: str) -> None:
        self.csv_string = csv_string

    def read(self) -> List[Dict[str, Any]]:
        """CSV 데이터 파싱"""
        reader = csv.DictReader(StringIO(self.csv_string))
        return list(reader)

    def validate(self) -> bool:
        """CSV 유효성 검증"""
        try:
            reader = csv.DictReader(StringIO(self.csv_string))
            list(reader)
            return True
        except csv.Error:
            return False

class DatabaseDataSource:
    """데이터베이스 데이터 소스 (모의)"""

    def __init__(self, query: str) -> None:
        self.query = query

    def read(self) -> List[Dict[str, Any]]:
        """데이터베이스 쿼리 실행 (모의)"""
        # 실제로는 데이터베이스 연결 및 쿼리 실행
        return [
            {"id": 1, "name": "Alice"},
            {"id": 2, "name": "Bob"}
        ]

    def validate(self) -> bool:
        """쿼리 유효성 검증"""
        return len(self.query.strip()) > 0

class DataLoader:
    """데이터 로더"""

    def __init__(self, source: DataSource) -> None:
        self.source = source

    def load(self) -> Optional[List[Dict[str, Any]]]:
        """데이터 로드"""
        if not self.source.validate():
            print("Invalid data source")
            return None

        try:
            return self.source.read()
        except Exception as e:
            print(f"Error loading data: {e}")
            return None

# 사용 예시
# JSON 소스
json_data = '[{"id": 1, "name": "Alice"}, {"id": 2, "name": "Bob"}]'
json_source = JsonDataSource(json_data)
loader1 = DataLoader(json_source)
data1 = loader1.load()
print("JSON data:", data1)

# CSV 소스
csv_data = "id,name\n1,Alice\n2,Bob"
csv_source = CsvDataSource(csv_data)
loader2 = DataLoader(csv_source)
data2 = loader2.load()
print("CSV data:", data2)

# 데이터베이스 소스
db_source = DatabaseDataSource("SELECT * FROM users")
loader3 = DataLoader(db_source)
data3 = loader3.load()
print("Database data:", data3)

문제 4: TypedDict를 사용한 설정 관리​

애플리케이션 설정을 관리하는 시스템을 TypedDict를 사용하여 구현하세요. 중첩된 설정과 선택적 필드를 포함해야 합니다.

해답 보기
from typing import TypedDict, Optional, Literal, NotRequired
import json

class DatabaseConfig(TypedDict):
    """데이터베이스 설정"""
    host: str
    port: int
    username: str
    password: str
    database: str
    ssl: NotRequired[bool]  # Python 3.11+

class RedisConfig(TypedDict):
    """Redis 설정"""
    host: str
    port: int
    db: int
    password: NotRequired[str]

class LoggingConfig(TypedDict):
    """로깅 설정"""
    level: Literal["DEBUG", "INFO", "WARNING", "ERROR"]
    format: str
    output: Literal["console", "file", "both"]
    file_path: NotRequired[str]

class ApiConfig(TypedDict):
    """API 설정"""
    base_url: str
    timeout: int
    retries: int
    api_key: str

class AppConfig(TypedDict):
    """전체 애플리케이션 설정"""
    app_name: str
    version: str
    debug: bool
    database: DatabaseConfig
    redis: RedisConfig
    logging: LoggingConfig
    api: ApiConfig

class ConfigManager:
    """설정 관리자"""

    def __init__(self, config_file: Optional[str] = None) -> None:
        self.config: Optional[AppConfig] = None
        if config_file:
            self.load_from_file(config_file)

    def load_from_file(self, file_path: str) -> None:
        """파일에서 설정 로드"""
        with open(file_path, 'r') as f:
            self.config = json.load(f)

    def load_from_dict(self, config_dict: AppConfig) -> None:
        """딕셔너리에서 설정 로드"""
        self.config = config_dict

    def get_database_config(self) -> Optional[DatabaseConfig]:
        """데이터베이스 설정 반환"""
        if self.config:
            return self.config["database"]
        return None

    def get_redis_config(self) -> Optional[RedisConfig]:
        """Redis 설정 반환"""
        if self.config:
            return self.config["redis"]
        return None

    def get_logging_config(self) -> Optional[LoggingConfig]:
        """로깅 설정 반환"""
        if self.config:
            return self.config["logging"]
        return None

    def get_api_config(self) -> Optional[ApiConfig]:
        """API 설정 반환"""
        if self.config:
            return self.config["api"]
        return None

    def is_debug_mode(self) -> bool:
        """디버그 모드 여부"""
        return self.config["debug"] if self.config else False

    def validate(self) -> bool:
        """설정 유효성 검증"""
        if not self.config:
            return False

        # 필수 필드 확인
        required_keys = ["app_name", "version", "debug", "database", "redis", "logging", "api"]
        if not all(key in self.config for key in required_keys):
            return False

        # 데이터베이스 설정 검증
        db_config = self.config["database"]
        if db_config["port"] < 1 or db_config["port"] > 65535:
            return False

        # Redis 설정 검증
        redis_config = self.config["redis"]
        if redis_config["port"] < 1 or redis_config["port"] > 65535:
            return False

        return True

# 사용 예시
config_data: AppConfig = {
    "app_name": "MyApp",
    "version": "1.0.0",
    "debug": True,
    "database": {
        "host": "localhost",
        "port": 5432,
        "username": "admin",
        "password": "secret",
        "database": "myapp_db",
        "ssl": True
    },
    "redis": {
        "host": "localhost",
        "port": 6379,
        "db": 0
    },
    "logging": {
        "level": "INFO",
        "format": "%(asctime)s - %(name)s - %(levelname)s - %(message)s",
        "output": "both",
        "file_path": "/var/log/myapp.log"
    },
    "api": {
        "base_url": "https://api.example.com",
        "timeout": 30,
        "retries": 3,
        "api_key": "secret-key-123"
    }
}

manager = ConfigManager()
manager.load_from_dict(config_data)

if manager.validate():
    print("Configuration is valid")

    db_config = manager.get_database_config()
    if db_config:
        print(f"Database: {db_config['host']}:{db_config['port']}")

    if manager.is_debug_mode():
        print("Running in debug mode")
else:
    print("Invalid configuration")
