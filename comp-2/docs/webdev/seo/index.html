SEO 기초

검색 엔진 최적화를 통해 웹사이트의 검색 순위를 향상시키는 방법을 배워보세요.

1. SEO란?​

**SEO (Search Engine Optimization, 검색 엔진 최적화)**는 검색 엔진에서 웹사이트의 순위를 높이기 위해 사이트를 최적화하는 과정입니다.

1.1 왜 SEO가 중요한가?​

**통계:**
- Google 검색의 75%는 첫 페이지를 넘어가지 않음
- 첫 번째 결과는 전체 클릭의 28.5% 차지
- 유기적 검색이 전체 웹 트래픽의 53% 차지

**이점:**
✅ 무료 트래픽 증가
✅ 브랜드 인지도 향상
✅ 높은 전환율
✅ 장기적 투자 효과
✅ 경쟁 우위 확보

1.2 SEO 유형​

**1. On-Page SEO (온페이지)**
   - 웹사이트 내부 최적화
   - 메타 태그, 콘텐츠, HTML 구조
   - 제어 가능

**2. Off-Page SEO (오프페이지)**
   - 외부 요인
   - 백링크, 소셜 신호
   - 제어 어려움

**3. Technical SEO (기술적)**
   - 사이트 속도, 모바일 최적화
   - 구조화된 데이터
   - 크롤링/인덱싱

2. 검색 엔진 작동 방식​

2.1 크롤링 (Crawling)​

**검색 봇이 웹 페이지를 탐색하는 과정**

1. 시작점 (Seed URLs)
2. 링크 따라가기
3. 새로운 페이지 발견
4. 반복

**크롤러:**
- Googlebot (Google)
- Bingbot (Bing)
- DuckDuckBot (DuckDuckGo)

2.2 인덱싱 (Indexing)​

**발견한 페이지를 데이터베이스에 저장**

- 콘텐츠 분석
- 키워드 추출
- 관련성 판단
- 색인 저장

2.3 랭킹 (Ranking)​

**검색 결과 순위 결정**

**주요 요인 (200+ 요소):**
1. 콘텐츠 품질
2. 백링크 수와 품질
3. 사용자 경험
4. 페이지 속도
5. 모바일 친화성
6. 도메인 권위도

3. On-Page SEO​

3.1 메타 태그​

3.1.1 Title 태그​

<!-- 가장 중요한 SEO 요소 -->
<head>
  <!-- ✅ 좋은 예 -->
  <title>웹 개발 가이드 - HTML, CSS, JavaScript 기초부터 | MyBlog</title>

  <!-- ❌ 나쁜 예 -->
  <title>홈</title>
</head>

Best Practices:

  • 50-60자 권장
  • 주요 키워드를 앞쪽에 배치
  • 각 페이지마다 고유한 제목
  • 브랜드명 포함 (선택)
  • 클릭을 유도하는 문구

3.1.2 Description 태그​

<head>
  <!-- ✅ 좋은 예 -->
  <meta name="description" content="HTML, CSS, JavaScript 기초부터 실전 프로젝트까지. 웹 개발을 처음 시작하는 분들을 위한 완벽한 가이드입니다. 무료 강의와 예제 코드 제공.">

  <!-- ❌ 나쁜 예 -->
  <meta name="description" content="웹 개발">
</head>

Best Practices:

  • 150-160자 권장
  • 주요 키워드 포함
  • 행동 유도 문구 (CTA) 포함
  • 페이지 내용 정확히 요약

3.1.3 Keywords 태그 (더 이상 사용 안 함)​

<!-- ❌ Google은 더 이상 사용하지 않음 -->
<meta name="keywords" content="web, development, html, css">

3.2 헤딩 태그 (H1-H6)​

<!-- ✅ 좋은 예 - 계층 구조 -->
<h1>웹 개발 완벽 가이드</h1>
  <h2>1. HTML 기초</h2>
    <h3>1.1 HTML이란?</h3>
    <h3>1.2 HTML 태그</h3>
  <h2>2. CSS 기초</h2>
    <h3>2.1 CSS 선택자</h3>

<!-- ❌ 나쁜 예 - 계층 무시 -->
<h1>제목</h1>
<h3>소제목</h3>
<h5>내용</h5>

Best Practices:

  • H1은 페이지당 하나만
  • 계층 구조 유지
  • 키워드 포함 (자연스럽게)
  • 스타일링에 헤딩 사용 금지

3.3 URL 구조​

<!-- ✅ 좋은 예 -->
`https://example.com/blog/web-development-guide`
`https://example.com/products/category/item-name`

<!-- ❌ 나쁜 예 -->
`https://example.com/page?id=123&cat=5`
`https://example.com/product.php?p=456`

**Best Practices:**
- 짧고 설명적
- 소문자 사용
- 하이픈(-) 사용 (언더스코어 X)
- 키워드 포함
- 불필요한 파라미터 제거

3.4 이미지 최적화​

<!-- ✅ 좋은 예 -->
<img
  src="web-development-tutorial.jpg"
  alt="HTML, CSS, JavaScript를 배우는 초보자를 위한 웹 개발 튜토리얼"
  width="800"
  height="400"
  loading="lazy"
>

<!-- ❌ 나쁜 예 -->
<img src="img1.jpg">

Best Practices:

  • 설명적인 파일명
  • Alt 텍스트 필수 (키워드 포함)
  • Width/Height 명시 (CLS 방지)
  • 이미지 압축
  • WebP 포맷 사용
  • Lazy loading

3.5 내부 링크​

<!-- ✅ 좋은 예 -->
<p>
  웹 개발을 시작하려면 먼저
  <a href="/html-basics">HTML 기초</a>를
  배우는 것이 좋습니다.
</p>

<!-- ❌ 나쁜 예 -->
<a href="/page1">여기 클릭</a>

Best Practices:

  • 설명적인 앵커 텍스트
  • 관련 페이지 연결
  • 깨진 링크 수정
  • 너무 많은 링크 지양

4.1 시맨틱 태그 사용​

  <meta charset="UTF-8">
  <title>웹 개발 가이드</title>

  <!-- ✅ 시맨틱 구조 -->

      </ul>

  <main>
    <article>
      <h1>웹 개발 시작하기</h1>

        <h2>HTML 기초</h2>
        <p>HTML은 웹 페이지의 구조를 만듭니다...</p>

      <section>

        <li><a href="/js-guide">JavaScript 기초</a></li>

7. Sitemap과 Robots.txt​

7.1 Sitemap (sitemap.xml)​

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://example.com/</loc>
    <lastmod>2024-01-15</lastmod>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://example.com/blog/web-development-guide</loc>
    <lastmod>2024-01-20</lastmod>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>
  <url>
    <loc>https://example.com/blog/html-basics</loc>
    <lastmod>2024-01-18</lastmod>
    <changefreq>monthly</changefreq>
    <priority>0.6</priority>
  </url>
</urlset>

Priority 가이드:

  • 1.0: 홈페이지
  • 0.8: 중요한 페이지
  • 0.6: 일반 페이지
  • 0.4: 덜 중요한 페이지

7.2 Robots.txt​

# 모든 봇 허용
User-agent: *
Allow: /

# 특정 디렉토리 차단
Disallow: /admin/
Disallow: /private/
Disallow: /temp/

# 검색 결과 차단
Disallow: /search?

# Sitemap 위치
Sitemap: https://example.com/sitemap.xml

# 특정 봇 차단
User-agent: BadBot
Disallow: /

8. Core Web Vitals​

8.1 LCP (Largest Contentful Paint)​

가장 큰 콘텐츠가 로드되는 시간

**목표:**
- 좋음: `<` 2.5초
- 개선 필요: 2.5-4초
- 나쁨: `>` 4초

**개선 방법:**
1. 이미지 최적화
2. 서버 응답 시간 단축
3. CSS/JS 최적화
4. 리소스 사전 로드
<!-- 중요 리소스 사전 로드 -->
<head>
  <link rel="preload" href="hero-image.jpg" as="image">
  <link rel="preload" href="main.css" as="style">
  <link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>
</head>

8.2 FID (First Input Delay)​

사용자 첫 상호작용 응답 시간

**목표:**
- 좋음: `<` 100ms
- 개선 필요: 100-300ms
- 나쁨: `>` 300ms

**개선 방법:**
1. JavaScript 실행 시간 단축
2. 코드 분할 (Code Splitting)
3. Web Worker 사용
4. Third-party 스크립트 지연 로드
<!-- 스크립트 지연 로드 -->
<script src="analytics.js" defer></script>
<script src="widget.js" async></script>

8.3 CLS (Cumulative Layout Shift)​

레이아웃이 예상치 않게 이동하는 정도

**목표:**
- 좋음: `<` 0.1
- 개선 필요: 0.1-0.25
- 나쁨: `>` 0.25

**개선 방법:**
1. 이미지/비디오 크기 명시
2. 동적 콘텐츠에 공간 확보
3. Web Font 최적화
4. 광고 공간 미리 할당
<!-- ✅ 크기 명시로 CLS 방지 -->
<img src="image.jpg" width="800" height="600" alt="설명">

<!-- ✅ 종횡비 유지 -->
<div style="aspect-ratio: 16 / 9;">
  <img src="video-thumbnail.jpg" alt="비디오">
</div>

<!-- ✅ Font Loading 최적화 -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

10. 페이지 속도 최적화​

10.1 이미지 최적화​

<!-- 반응형 이미지 -->
<img
  src="image-800.jpg"
  srcset="image-400.jpg 400w,
          image-800.jpg 800w,
          image-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw,
         (max-width: 1200px) 50vw,
         33vw"
  alt="설명"
  loading="lazy"
>

<!-- WebP 포맷 -->
<picture>
  <source type="image/webp" srcset="image.webp">
  <source type="image/jpeg" srcset="image.jpg">
  <img src="image.jpg" alt="설명">
</picture>

10.2 CSS/JS 최소화​

<!-- CSS 압축 -->
<link rel="stylesheet" href="styles.min.css">

<!-- JS 압축 및 defer -->
<script src="script.min.js" defer></script>

<!-- Critical CSS 인라인 -->
<style>
  /* 최초 렌더링에 필요한 CSS만 */
  .hero { display: flex; }
</style>

10.3 리소스 힌트​

<head>
  <!-- DNS 사전 조회 -->
  <link rel="dns-prefetch" href="https://fonts.googleapis.com">

  <!-- 사전 연결 -->
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

  <!-- 사전 로드 -->
  <link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

  <!-- 사전 페치 (낮은 우선순위) -->
  <link rel="prefetch" href="/next-page.html">
</head>

SEO Best Practices
  1. 콘텐츠 품질이 최우선: 사용자에게 가치 있는 콘텐츠 제공
  2. 키워드는 자연스럽게: 키워드 stuffing 금지
  3. 모바일 최적화: 모바일 트래픽이 전체의 58%
  4. 페이지 속도: 3초 이내 로딩 목표
  5. 정기적 업데이트: 최신 정보 유지
  6. 내부 링크: 관련 페이지 연결
  7. 구조화된 데이터: Rich Results 활용
  8. HTTPS 필수: 보안 인증서 사용
  9. Alt 텍스트: 모든 이미지에 설명 추가
  10. Analytics 분석: 데이터 기반 개선
흔한 SEO 실수
  1. ❌ 중복 콘텐츠: 동일한 내용의 여러 페이지
  2. ❌ 키워드 stuffing: 과도한 키워드 반복
  3. ❌ 느린 페이지: 5초 이상 로딩
  4. ❌ 모바일 비친화적: 작은 텍스트, 터치 영역 부족
  5. ❌ 깨진 링크: 404 에러 페이지
  6. ❌ 얇은 콘텐츠: 300자 미만의 짧은 글
  7. ❌ hidden 텍스트: 숨겨진 키워드 (블랙햇)
  8. ❌ 자동 생성 콘텐츠: AI만으로 작성
  9. ❌ 제목 없음: H1 태그 누락
  10. ❌ Sitemap 미제출: Google에 알리지 않음
