CSS 프레임워크

CSS 프레임워크를 사용하여 빠르고 일관성 있는 웹 디자인을 구현하는 방법을 배워보세요.

1. CSS 프레임워크란?​

**CSS 프레임워크(CSS Framework)**는 미리 정의된 CSS 클래스와 컴포넌트를 제공하여 빠르게 웹사이트를 개발할 수 있도록 돕는 도구입니다.

1.1 왜 CSS 프레임워크를 사용하나?​

**장점:**
✅ 빠른 개발 속도 - 미리 만들어진 컴포넌트
✅ 일관된 디자인 - 통일된 스타일 가이드
✅ 반응형 기본 제공 - 모바일 대응 쉬움
✅ 크로스 브라우저 호환성 - 브라우저별 차이 해결
✅ 유지보수 용이 - 표준화된 코드

**단점:**
❌ 파일 크기 - 사용하지 않는 CSS도 포함
❌ 학습 곡선 - 프레임워크별 문법 학습 필요
❌ 커스터마이징 제한 - 독특한 디자인 구현 어려움
❌ 일반적인 디자인 - 다른 사이트와 비슷해 보일 수 있음

1.2 주요 CSS 프레임워크​

**인기 프레임워크:**

1. **Bootstrap** - 가장 인기 있는 프레임워크
   - 컴포넌트 중심
   - 풍부한 문서
   - jQuery 의존성 제거 (v5)

2. **Tailwind CSS** - 유틸리티 중심
   - 원자적 CSS 클래스
   - 커스터마이징 자유로움
   - 빌드 과정 필요

3. **Bulma** - 순수 CSS
   - JavaScript 없음
   - Flexbox 기반
   - 간단한 문법

4. **Foundation** - 엔터프라이즈급
   - 고급 기능
   - 접근성 중시
   - 복잡한 프로젝트에 적합

2. Bootstrap 5​

2.1 Bootstrap 소개​

Bootstrap은 Twitter에서 개발한 세계에서 가장 인기 있는 CSS 프레임워크입니다.

**특징:**
- 반응형 그리드 시스템 (12열)
- 미리 만들어진 컴포넌트
- Sass 변수로 커스터마이징
- jQuery 의존성 제거 (v5부터)
- 다크 모드 지원

2.2 Bootstrap 설치​

<!-- CDN 방식 (가장 쉬움) -->
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Bootstrap 예제</title>

  <!-- Bootstrap CSS -->
  <link
    href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css"
    rel="stylesheet"
  >
</head>
<body>
  <h1>Hello, Bootstrap!</h1>

  <!-- Bootstrap JS (선택사항, 인터랙티브 컴포넌트에 필요) -->
  <script
    src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"
  ></script>
</body>
</html>
# npm으로 설치
npm install bootstrap

# yarn으로 설치
yarn add bootstrap

3. Tailwind CSS​

3.1 Tailwind 소개​

Tailwind CSS는 유틸리티 우선(Utility-First) CSS 프레임워크로, 작은 단위의 클래스를 조합하여 디자인을 구성합니다.

**특징:**
- 유틸리티 클래스 중심
- 커스터마이징 매우 자유로움
- PurgeCSS로 사용하지 않는 CSS 제거
- 컴포넌트 없음 (직접 조합)
- 빌드 과정 필요

3.2 Tailwind 설치​

# npm으로 설치
npm install -D tailwindcss
npx tailwindcss init

# PostCSS와 함께 (권장)
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

tailwind.config.js:

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./src/**/*.{html,js}",
    "./public/**/*.html"
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}

CSS 파일 (styles.css):

@tailwind base;
@tailwind components;
@tailwind utilities;

4. Bootstrap vs Tailwind 비교​

4.1 비교표​

특징BootstrapTailwind CSS
접근 방식컴포넌트 중심유틸리티 중심
파일 크기큼 (~27KB min+gzip)작음 (PurgeCSS 후 ~3KB)
학습 곡선쉬움중간
커스터마이징제한적 (Sass 변수)매우 자유로움
디자인부트스트랩 스타일원하는 대로
빌드 과정불필요필요 (권장)
JavaScript포함없음
적합한 프로젝트빠른 프로토타입커스텀 디자인

4.2 코드 비교​

<!-- 같은 버튼을 만드는 방법 -->

<!-- Bootstrap -->

  클릭하세요
</button>

<!-- Tailwind -->

</button>

4.3 언제 무엇을 사용할까?​

**Bootstrap 추천:**
- 빠른 프로토타이핑 필요
- 표준 디자인 패턴 사용
- JavaScript 컴포넌트 필요
- 팀원이 Bootstrap에 익숙
- 빌드 과정 최소화

**Tailwind 추천:**
- 완전히 커스텀 디자인
- 파일 크기 최적화 중요
- 유틸리티 우선 접근 선호
- 디자인 시스템 구축
- 모던 빌드 도구 사용 중

CSS 프레임워크 선택 가이드

Bootstrap을 선택하세요:

  • 빠른 프로토타이핑이 필요할 때
  • 표준화된 UI 컴포넌트를 원할 때
  • 팀이 Bootstrap에 익숙할 때
  • 빌드 과정을 최소화하고 싶을 때

Tailwind를 선택하세요:

  • 완전히 커스텀 디자인이 필요할 때
  • 파일 크기 최적화가 중요할 때
  • 디자인 시스템을 직접 구축하고 싶을 때
  • 유틸리티 우선 접근을 선호할 때

둘 다 사용하지 마세요:

  • 충돌 가능성
  • 불필요하게 큰 번들 크기
  • 일관성 없는 코드
흔한 실수
  1. ❌ 너무 많은 커스텀 CSS: 프레임워크의 이점 상실
  2. ❌ !important 남용: 프레임워크 스타일 덮어쓰기 주의
  3. ❌ 모든 컴포넌트 포함: 사용하는 것만 import
  4. ❌ 반응형 무시: 프레임워크 제공 유틸리티 활용
  5. ❌ 버전 혼용: 일관된 버전 사용
