데이터베이스 기본

PostgreSQL의 기본 개념과 데이터베이스 관리 방법을 배웁니다.

데이터베이스란?​

데이터베이스는 체계적으로 조직된 데이터의 집합입니다. PostgreSQL은 관계형 데이터베이스 관리 시스템(RDBMS)으로, 데이터를 테이블 형태로 저장합니다.

기본 개념​

데이터베이스 (Database)​

  • 관련된 테이블들의 모음
  • 하나의 PostgreSQL 인스턴스에 여러 데이터베이스 존재 가능

스키마 (Schema)​

  • 데이터베이스 내의 논리적 그룹
  • 테이블, 뷰, 함수 등을 포함
  • 기본 스키마: public

테이블 (Table)​

  • 행(Row)과 열(Column)로 구성된 데이터 구조
  • 각 테이블은 특정 엔티티를 나타냄

행/레코드 (Row/Record)​

  • 테이블의 개별 항목
  • 하나의 데이터 단위

열/컬럼 (Column)​

  • 테이블의 속성
  • 특정 데이터 타입을 가짐

데이터베이스 관리​

데이터베이스 생성​

-- 기본 생성
CREATE DATABASE mydb;

-- 소유자 지정
CREATE DATABASE mydb OWNER myuser;

-- 인코딩 지정
CREATE DATABASE mydb
  ENCODING 'UTF8'
  LC_COLLATE 'ko_KR.UTF-8'
  LC_CTYPE 'ko_KR.UTF-8';

-- 템플릿 사용
CREATE DATABASE mydb TEMPLATE template0;

데이터베이스 목록 보기​

-- SQL 명령어
SELECT datname FROM pg_database;

-- psql 명령어
\l
\list

데이터베이스 선택​

# psql 접속 시 데이터베이스 지정
psql -U username -d mydb

# psql 내에서 전환
\c mydb
\connect mydb

현재 데이터베이스 확인​

SELECT current_database();

데이터베이스 삭제​

-- 주의: 삭제는 되돌릴 수 없습니다!
DROP DATABASE mydb;

-- 존재하는 경우에만 삭제
DROP DATABASE IF EXISTS mydb;

주의사항:

  • 연결된 데이터베이스는 삭제할 수 없음
  • 모든 연결을 종료한 후 삭제 가능

데이터베이스 이름 변경​

ALTER DATABASE oldname RENAME TO newname;

데이터베이스 소유자 변경​

ALTER DATABASE mydb OWNER TO newowner;

스키마 관리​

스키마 생성​

CREATE SCHEMA myschema;

-- 소유자 지정
CREATE SCHEMA myschema AUTHORIZATION myuser;

스키마 목록 보기​

-- SQL 명령어
SELECT schema_name FROM information_schema.schemata;

-- psql 명령어
\dn

스키마 설정​

-- 기본 스키마 설정
SET search_path TO myschema, public;

-- 영구적 변경
ALTER DATABASE mydb SET search_path TO myschema, public;

현재 스키마 확인​

SELECT current_schema();
SHOW search_path;

스키마 삭제​

-- 빈 스키마 삭제
DROP SCHEMA myschema;

-- 스키마와 모든 객체 삭제
DROP SCHEMA myschema CASCADE;

-- 존재하는 경우에만 삭제
DROP SCHEMA IF EXISTS myschema CASCADE;

사용자와 권한​

사용자(Role) 생성​

-- 기본 사용자 생성
CREATE USER username WITH PASSWORD 'password';

-- 로그인 가능한 롤 생성
CREATE ROLE username WITH LOGIN PASSWORD 'password';

-- 여러 속성 지정
CREATE USER username WITH
  PASSWORD 'password'
  CREATEDB
  CREATEROLE;

사용자 목록 보기​

-- SQL 명령어
SELECT usename FROM pg_user;

-- psql 명령어
\du

권한 부여​

-- 데이터베이스 권한
GRANT ALL PRIVILEGES ON DATABASE mydb TO username;

-- 스키마 권한
GRANT ALL ON SCHEMA myschema TO username;

-- 테이블 권한
GRANT SELECT, INSERT, UPDATE ON TABLE mytable TO username;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO username;

-- 시퀀스 권한
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO username;

권한 취소​

REVOKE ALL PRIVILEGES ON DATABASE mydb FROM username;
REVOKE INSERT ON TABLE mytable FROM username;

사용자 삭제​

-- 사용자가 소유한 객체가 없는 경우
DROP USER username;

-- 사용자가 소유한 모든 객체 재할당 후 삭제
REASSIGN OWNED BY username TO postgres;
DROP OWNED BY username;
DROP USER username;

psql 유용한 명령어​

메타 명령어​

\?              -- 도움말
\h              -- SQL 명령어 도움말
\h SELECT       -- 특정 SQL 명령어 도움말

\l              -- 데이터베이스 목록
\c dbname       -- 데이터베이스 연결
\dt             -- 테이블 목록
\d tablename    -- 테이블 구조 보기
\dn             -- 스키마 목록
\du             -- 사용자 목록
\df             -- 함수 목록

\i filename.sql -- SQL 파일 실행
\o filename.txt -- 출력을 파일로 저장
\o              -- 파일 출력 종료

\timing         -- 쿼리 실행 시간 표시
\x              -- 확장 표시 모드 토글
\q              -- 종료

설정 보기​

\conninfo       -- 현재 연결 정보
\encoding       -- 인코딩 확인
\set            -- 변수 목록

실전 예제​

새 프로젝트 설정​

-- 1. 데이터베이스 생성
CREATE DATABASE blogdb
  ENCODING 'UTF8'
  LC_COLLATE 'ko_KR.UTF-8'
  LC_CTYPE 'ko_KR.UTF-8';

-- 2. 사용자 생성
CREATE USER bloguser WITH PASSWORD 'secure_password';

-- 3. 권한 부여
GRANT ALL PRIVILEGES ON DATABASE blogdb TO bloguser;

-- 4. 데이터베이스 연결
\c blogdb

-- 5. 스키마 생성
CREATE SCHEMA blog;

-- 6. 스키마 권한 부여
GRANT ALL ON SCHEMA blog TO bloguser;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA blog TO bloguser;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA blog TO bloguser;

-- 7. 기본 스키마 설정
ALTER DATABASE blogdb SET search_path TO blog, public;

데이터베이스 정보 조회​

-- 데이터베이스 크기
SELECT
  pg_size_pretty(pg_database_size('mydb')) AS db_size;

-- 테이블 크기
SELECT
  schemaname,
  tablename,
  pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;

-- 활성 연결 수
SELECT count(*) FROM pg_stat_activity;

-- 데이터베이스별 연결 수
SELECT
  datname,
  count(*) as connections
FROM pg_stat_activity
GROUP BY datname;

백업과 복원​

백업 (pg_dump)​

# 전체 데이터베이스 백업
pg_dump -U username -d mydb > backup.sql

# 특정 테이블만 백업
pg_dump -U username -d mydb -t mytable > table_backup.sql

# 압축 백업
pg_dump -U username -d mydb | gzip > backup.sql.gz

# 커스텀 포맷 (복원 속도 빠름)
pg_dump -U username -Fc mydb > backup.dump

복원 (psql/pg_restore)​

# SQL 파일 복원
psql -U username -d newdb < backup.sql

# 압축 파일 복원
gunzip < backup.sql.gz | psql -U username -d newdb

# 커스텀 포맷 복원
pg_restore -U username -d newdb backup.dump
