Upgrade와 Downgrade

마이그레이션을 실행하고 롤백하는 방법을 배워봅시다!

🎯 학습 목표​

  • upgrade와 downgrade의 차이점 이해
  • 다양한 버전 이동 방법
  • 마이그레이션 상태 확인

⬆️ Upgrade: 데이터베이스 업그레이드​

기본 명령어​

# 최신 버전으로 업그레이드 (가장 많이 사용)
alembic upgrade head

# 특정 리비전으로 업그레이드
alembic upgrade 2f8b9c3d1a5e

# 상대적으로 N개 버전 올리기
alembic upgrade +2

# 다음 버전 하나만
alembic upgrade +1

실행 예시​

$ alembic upgrade head
INFO  [alembic.runtime.migration] Context impl SQLiteImpl.
INFO  [alembic.runtime.migration] Will assume non-transactional DDL.
INFO  [alembic.runtime.migration] Running upgrade  -> 2f8b9c3d1a5e, create users table
INFO  [alembic.runtime.migration] Running upgrade 2f8b9c3d1a5e -> 3a7c4e8f2b1d, create posts table

⬇️ Downgrade: 데이터베이스 다운그레이드​

기본 명령어​

# 이전 버전으로 롤백
alembic downgrade -1

# 특정 리비전으로 다운그레이드
alembic downgrade 2f8b9c3d1a5e

# 모든 마이그레이션 되돌리기
alembic downgrade base

# 상대적으로 N개 버전 내리기
alembic downgrade -2

실행 예시​

$ alembic downgrade -1
INFO  [alembic.runtime.migration] Context impl SQLiteImpl.
INFO  [alembic.runtime.migration] Will assume non-transactional DDL.
INFO  [alembic.runtime.migration] Running downgrade 3a7c4e8f2b1d -> 2f8b9c3d1a5e, create posts table

📊 현재 상태 확인​

current: 현재 리비전 확인​

alembic current

# 출력 예시:
# 2f8b9c3d1a5e (head)

show: 특정 리비전 정보 보기​

alembic show 2f8b9c3d1a5e

# 출력 예시:
# Rev: 2f8b9c3d1a5e
# Parent: <base>
# Path: /path/to/versions/2f8b9c3d1a5e_create_users_table.py
#
#     create users table
#
#     Revision ID: 2f8b9c3d1a5e
#     Revises:
#     Create Date: 2024-01-15 10:30:45.123456

🔄 실전 시나리오​

시나리오 1: 새 기능 배포​

# 1. 새 마이그레이션 생성
alembic revision -m "add user profile fields"

# 2. 마이그레이션 파일 작성
# ... (upgrade/downgrade 함수 구현)

# 3. 로컬 테스트
alembic upgrade head

# 4. 문제 없으면 Git 커밋
git add alembic/versions/*.py
git commit -m "Add user profile fields migration"

# 5. 프로덕션 배포
# (서버에서)
alembic upgrade head

시나리오 2: 긴급 롤백​

# 프로덕션에서 문제 발견!

# 1. 현재 상태 확인
alembic current
# 3a7c4e8f2b1d (head)

# 2. 이전 버전으로 롤백
alembic downgrade -1

# 3. 확인
alembic current
# 2f8b9c3d1a5e

# 4. 애플리케이션도 이전 버전으로 배포

시나리오 3: 특정 버전으로 점프​

# 현재: 5번째 마이그레이션
# 목표: 2번째 마이그레이션으로 이동

# 방법 1: 리비전 ID 사용
alembic downgrade 2f8b9c3d1a5e

# 방법 2: 상대적 이동
alembic downgrade -3

# 다시 최신으로
alembic upgrade head

🎓 버전 지정 방법​

절대 버전​

# 특정 리비전 ID
alembic upgrade 2f8b9c3d1a5e

# head (최신)
alembic upgrade head

# base (초기 상태)
alembic downgrade base

상대 버전​

# 현재에서 +N 버전 올리기
alembic upgrade +1
alembic upgrade +2

# 현재에서 -N 버전 내리기
alembic downgrade -1
alembic downgrade -2

범위 지정​

# from:to 형식으로 범위 지정
alembic upgrade 2f8b9c3d1a5e:head
alembic downgrade head:2f8b9c3d1a5e

🧪 테스트 패턴​

업그레이드/다운그레이드 사이클 테스트​

# 1. 현재 상태 저장
alembic current > current_revision.txt

# 2. 업그레이드 테스트
alembic upgrade head

# 3. 다운그레이드 테스트
alembic downgrade base

# 4. 다시 업그레이드
alembic upgrade head

# 5. 데이터 무결성 확인
# (애플리케이션 테스트 실행)

SQL 모드로 미리 확인​

# SQL만 출력 (실제 실행 안 함)
alembic upgrade head --sql

# 결과를 파일로 저장
alembic upgrade head --sql > migration.sql

# SQL 검토 후 직접 실행
psql -d mydb -f migration.sql

📝 실전 예제: 단계별 마이그레이션​

3개의 마이그레이션을 만들고 이동해봅시다.

1. 첫 번째 마이그레이션​

alembic revision -m "create users table"
# versions/aaa_create_users_table.py
def upgrade():
    op.create_table('users',
        sa.Column('id', sa.Integer(), primary_key=True),
        sa.Column('username', sa.String(50), nullable=False)
    )

def downgrade():
    op.drop_table('users')
alembic upgrade head
# Running upgrade  -> aaa, create users table

2. 두 번째 마이그레이션​

alembic revision -m "add email to users"
# versions/bbb_add_email_to_users.py
def upgrade():
    op.add_column('users', sa.Column('email', sa.String(100)))

def downgrade():
    op.drop_column('users', 'email')
alembic upgrade head
# Running upgrade aaa -> bbb, add email to users

3. 세 번째 마이그레이션​

alembic revision -m "create posts table"
# versions/ccc_create_posts_table.py
def upgrade():
    op.create_table('posts',
        sa.Column('id', sa.Integer(), primary_key=True),
        sa.Column('title', sa.String(200), nullable=False),
        sa.Column('author_id', sa.Integer(), nullable=False),
        sa.ForeignKeyConstraint(['author_id'], ['users.id'])
    )

def downgrade():
    op.drop_table('posts')
alembic upgrade head
# Running upgrade bbb -> ccc, create posts table

이동 테스트​

# 현재 위치 확인
alembic current
# ccc (head)

# 한 단계 뒤로
alembic downgrade -1
alembic current
# bbb

# 두 단계 뒤로
alembic downgrade -2
alembic current
# aaa (base)

# 다시 최신으로
alembic upgrade head
alembic current
# ccc (head)

# 모두 되돌리기
alembic downgrade base
alembic current
# (empty)

⚠️ 주의사항​

1. 데이터 손실 위험​

# ❌ 위험한 다운그레이드
def downgrade():
    op.drop_table('users')  # 모든 사용자 데이터 삭제!
# ✅ 안전한 다운그레이드 (백업 포함)
def upgrade():
    op.create_table('users_new', ...)
    op.execute('INSERT INTO users_new SELECT * FROM users')
    op.drop_table('users')
    op.rename_table('users_new', 'users')

def downgrade():
    # 백업 테이블이 있다면 복원 가능
    pass

2. 프로덕션 롤백​

# ❌ 절대 하지 말 것
alembic downgrade base  # 모든 데이터 삭제 위험!

# ✅ 안전한 롤백
alembic downgrade -1    # 한 단계만

3. 동시성 이슈​

여러 서버에서 동시에 마이그레이션 실행 방지:

# 락 메커니즘 사용
# alembic은 alembic_version 테이블로 상태 관리
# 동시 실행 시 자동으로 에러 발생

🔍 트러블슈팅​

문제 1: revision이 꼬인 경우​

$ alembic upgrade head
Error: Can't locate revision identified by 'xxxxx'

해결방법:

# 1. 현재 DB 버전 확인
SELECT * FROM alembic_version;

# 2. versions 디렉토리 파일 확인
ls alembic/versions/

# 3. 버전 테이블 수동 수정 (주의!)
UPDATE alembic_version SET version_num = 'correct_revision_id';

문제 2: downgrade 불가능​

$ alembic downgrade -1
Error: Target database is not up to date.

해결방법:

# 현재 상태 확인
alembic current

# 히스토리 확인
alembic history

# 강제 버전 설정 (매우 주의!)
alembic stamp head

📚 베스트 프랙티스​

1. 항상 테스트 먼저​

# 로컬/개발 환경에서 먼저 테스트
alembic upgrade head
alembic downgrade -1
alembic upgrade head
# 모든 테스트 통과 후 배포

2. 작은 단위로 마이그레이션​

# ❌ 나쁜 예: 한 번에 많은 변경
alembic revision -m "add 10 tables and modify 5 columns"

# ✅ 좋은 예: 논리적 단위로 분리
alembic revision -m "add users table"
alembic revision -m "add posts table"
alembic revision -m "add email column to users"

3. 프로덕션 배포 체크리스트​

# 1. 백업
pg_dump mydb > backup_$(date +%Y%m%d).sql

# 2. 마이그레이션 테스트 (SQL 모드)
alembic upgrade head --sql > migration.sql
# SQL 검토

# 3. 실행
alembic upgrade head

# 4. 검증
alembic current
# 애플리케이션 테스트

# 5. 문제 발생 시 롤백 준비
alembic downgrade -1

✅ 핵심 정리​

명령어설명사용 예시
upgrade head최신 버전으로배포 시
downgrade -1한 단계 뒤로롤백 시
current현재 버전 확인상태 확인
upgrade +NN단계 앞으로부분 업그레이드
downgrade base모두 되돌리기개발 환경 초기화
