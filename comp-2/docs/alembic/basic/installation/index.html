Alembic 설치 및 초기화

Alembic을 설치하고 프로젝트에 셋업하는 방법을 알아봅시다.

📦 Alembic 설치​

pip를 사용한 설치​

# Alembic 설치
pip install alembic

# SQLAlchemy와 함께 설치 (보통 함께 사용)
pip install alembic sqlalchemy

# 특정 데이터베이스 드라이버와 함께 설치
# PostgreSQL
pip install alembic sqlalchemy psycopg2-binary

# MySQL
pip install alembic sqlalchemy pymysql

# SQLite는 Python에 기본 포함됨

requirements.txt에 추가​

alembic==1.13.0
sqlalchemy==2.0.23
psycopg2-binary==2.9.9  # PostgreSQL 사용 시

설치 확인​

alembic --version
# 출력: alembic 1.13.0

🎬 프로젝트 초기화​

1단계: Alembic 초기화​

# 프로젝트 디렉토리로 이동
cd my_project

# Alembic 초기화
alembic init alembic

이 명령어는 다음과 같은 구조를 생성합니다:

my_project/
├── alembic/
│   ├── versions/          # 마이그레이션 파일 저장소
│   ├── env.py            # 환경 설정
│   ├── README            # 설명서
│   └── script.py.mako    # 마이그레이션 템플릿
├── alembic.ini           # Alembic 설정 파일
└── ...

2단계: 데이터베이스 URL 설정​

alembic.ini 파일을 열고 데이터베이스 URL을 설정합니다:

# alembic.ini

# SQLite (개발용으로 가장 간단)
sqlalchemy.url = sqlite:///./database.db

# PostgreSQL
sqlalchemy.url = postgresql://user:password@localhost/dbname

# MySQL
sqlalchemy.url = mysql+pymysql://user:password@localhost/dbname

# 환경변수 사용 (권장)
# sqlalchemy.url = driver://user:pass@localhost/dbname
팁

프로덕션에서는 설정 파일에 직접 비밀번호를 적지 말고, 환경변수를 사용하세요!

3단계: env.py 설정​

SQLAlchemy 모델과 연결하기 위해 alembic/env.py를 수정합니다:

# alembic/env.py
from logging.config import fileConfig
from sqlalchemy import engine_from_config
from sqlalchemy import pool
from alembic import context

# 여기에 여러분의 SQLAlchemy Base를 import
from myapp.models import Base  # 실제 모델 경로로 변경

# this is the Alembic Config object
config = context.config

# Interpret the config file for Python logging
if config.config_file_name is not None:
    fileConfig(config.config_file_name)

# SQLAlchemy MetaData 설정
target_metadata = Base.metadata

# 나머지 코드는 기본값 사용...

🔧 실전 예제: 블로그 프로젝트 초기화​

전체 과정을 하나의 예제로 따라해봅시다.

1. 프로젝트 구조 만들기​

mkdir blog_project
cd blog_project

# 가상환경 생성
python -m venv venv

# 가상환경 활성화
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

# 필요한 패키지 설치
pip install alembic sqlalchemy

2. SQLAlchemy 모델 작성​

# models.py
from sqlalchemy import Column, Integer, String, Text, DateTime
from sqlalchemy.ext.declarative import declarative_base
from datetime import datetime

Base = declarative_base()

class User(Base):
    __tablename__ = 'users'

    id = Column(Integer, primary_key=True)
    username = Column(String(50), unique=True, nullable=False)
    email = Column(String(100), unique=True, nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow)

class Post(Base):
    __tablename__ = 'posts'

    id = Column(Integer, primary_key=True)
    title = Column(String(200), nullable=False)
    content = Column(Text, nullable=False)
    author_id = Column(Integer, nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow)

3. Alembic 초기화​

alembic init alembic

4. alembic.ini 설정​

# alembic.ini
[alembic]
script_location = alembic
sqlalchemy.url = sqlite:///blog.db  # SQLite 사용

# 로깅 설정도 확인
[loggers]
keys = root,sqlalchemy,alembic

[handlers]
keys = console

[formatters]
keys = generic

[logger_root]
level = WARN
handlers = console

5. env.py 설정​

# alembic/env.py
from logging.config import fileConfig
from sqlalchemy import engine_from_config, pool
from alembic import context

# 우리의 모델 import
import sys
import os
sys.path.append(os.path.dirname(os.path.dirname(__file__)))
from models import Base

config = context.config
target_metadata = Base.metadata

# ... 나머지 코드는 기본값 유지

6. 설정 확인​

# 현재 리비전 확인 (아직 마이그레이션이 없으므로 에러 없이 실행되어야 함)
alembic current

💡 환경변수로 데이터베이스 URL 관리​

보안을 위해 환경변수를 사용하는 방법:

1. .env 파일 생성​

# .env
DATABASE_URL=postgresql://user:password@localhost/blog_db

2. python-dotenv 설치​

pip install python-dotenv

3. env.py 수정​

# alembic/env.py
from dotenv import load_dotenv
import os

load_dotenv()

# alembic.ini의 URL을 환경변수로 오버라이드
config.set_main_option('sqlalchemy.url', os.getenv('DATABASE_URL'))

4. .gitignore에 추가​

# .gitignore
.env
*.db
venv/
__pycache__/

🎯 초기화 체크리스트​

모든 설정이 완료되었는지 확인하세요:

# 1. Alembic이 설치되었는지 확인
alembic --version

# 2. 디렉토리 구조 확인
ls -la alembic/

# 3. 설정 파일 확인
cat alembic.ini

# 4. 현재 상태 확인 (에러가 없어야 함)
alembic current

🚨 자주 발생하는 문제​

문제 1: ModuleNotFoundError​

ModuleNotFoundError: No module named 'models'

해결방법: env.py에서 올바른 경로로 import하고 있는지 확인

import sys
import os
sys.path.append(os.path.dirname(os.path.dirname(__file__)))
from models import Base  # 정확한 경로로 수정

문제 2: Can't locate revision​

Can't locate revision identified by 'xxxxx'

해결방법: alembic/versions/ 디렉토리를 확인하고, 필요하면 초기화

# versions 디렉토리의 모든 파일 확인
ls alembic/versions/

# 필요시 모든 마이그레이션 파일 삭제 후 재시작

문제 3: Database connection failed​

해결방법: alembic.ini의 데이터베이스 URL이 올바른지 확인

# 연결 테스트
python -c "from sqlalchemy import create_engine; engine = create_engine('sqlite:///blog.db'); print('Connection successful!')"

📚 파일 구조 상세 설명​

alembic.ini​

메인 설정 파일로 다음을 포함:

  • 데이터베이스 연결 정보
  • 마이그레이션 파일 위치
  • 로깅 설정

alembic/env.py​

마이그레이션 실행 환경 설정:

  • SQLAlchemy 모델 연결
  • 데이터베이스 엔진 설정
  • 트랜잭션 관리

alembic/script.py.mako​

새 마이그레이션 파일 생성 시 사용되는 템플릿

alembic/versions/​

생성된 모든 마이그레이션 파일이 저장되는 디렉토리
