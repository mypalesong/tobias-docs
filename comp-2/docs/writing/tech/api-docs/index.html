4. API 문서 작성

API 문서는 개발자가 여러분의 API를 사용하는 데 필요한 모든 정보를 담고 있어야 합니다. 명확하고 완전한 문서는 API 채택률을 높입니다.

4.1 API 문서의 필수 요소​

1. 개요​

# User API

사용자 정보를 관리하는 RESTful API입니다.

**Base URL:** `https://api.example.com/v1`

**인증:** Bearer Token 필요

2. 인증​

## 인증

모든 요청에는 Bearer Token이 필요합니다.

**헤더:**
```http
Authorization: Bearer YOUR_ACCESS_TOKEN

토큰 발급: 인증 가이드 참고


### 3. 엔드포인트 문서

## 4.2 완벽한 엔드포인트 문서

```markdown
## GET /users/:id

사용자 정보를 조회합니다.

### URL 파라미터

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| `id` | string | ✅ | 사용자 고유 ID |

### 쿼리 파라미터

| 파라미터 | 타입 | 필수 | 기본값 | 설명 |
|----------|------|------|--------|------|
| `include` | string | ❌ | - | 포함할 관련 데이터 (posts,comments) |

### 요청 예시

```bash
curl -X GET \
  'https://api.example.com/v1/users/123?include=posts' \
  -H 'Authorization: Bearer YOUR_TOKEN'
// JavaScript (Fetch)
fetch('https://api.example.com/v1/users/123?include=posts', {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
})
.then(response => response.json())
.then(data => console.log(data));
# Python (requests)
import requests

headers = {'Authorization': 'Bearer YOUR_TOKEN'}
response = requests.get(
    'https://api.example.com/v1/users/123',
    params={'include': 'posts'},
    headers=headers
)
data = response.json()

응답​

성공 (200 OK)​

{
  "id": "123",
  "name": "김철수",
  "email": "user@example.com",
  "createdAt": "2024-01-01T00:00:00Z",
  "posts": [
    {
      "id": "1",
      "title": "첫 번째 글"
    }
  ]
}

응답 필드:

필드타입설명
idstring사용자 ID
namestring사용자 이름
emailstring이메일 주소
createdAtstring생성 일시 (ISO 8601)
postsarray사용자의 게시글 목록 (include=posts 시)

오류 응답​

404 Not Found

{
  "error": {
    "code": "USER_NOT_FOUND",
    "message": "해당 사용자를 찾을 수 없습니다."
  }
}

401 Unauthorized

{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "유효하지 않은 토큰입니다."
  }
}

Rate Limiting​

  • 시간당 1,000 요청
  • 초과 시 429 응답

참고사항​

팁

include 파라미터로 관련 데이터를 한 번에 가져오면 API 호출 횟수를 줄일 수 있습니다.

경고

이메일 주소는 민감 정보이므로 HTTPS 사용이 필수입니다.


## 4.3 POST 요청 문서

```markdown
## POST /users

새 사용자를 생성합니다.

### 요청 바디

```json
{
  "name": "김철수",
  "email": "user@example.com",
  "password": "securePassword123!",
  "role": "user"
}

필드:

필드타입필수제약 조건
namestring✅2-50자
emailstring✅유효한 이메일
passwordstring✅최소 8자, 특수문자 포함
rolestring❌user, admin (기본: user)

검증 규칙​

  • 이메일은 중복 불가
  • 비밀번호는 대소문자, 숫자, 특수문자 포함

요청 예시​

const response = await fetch('https://api.example.com/v1/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_TOKEN'
  },
  body: JSON.stringify({
    name: '김철수',
    email: 'user@example.com',
    password: 'securePassword123!'
  })
});

const data = await response.json();

응답​

성공 (201 Created)​

{
  "id": "124",
  "name": "김철수",
  "email": "user@example.com",
  "role": "user",
  "createdAt": "2024-01-15T10:30:00Z"
}

오류 (400 Bad Request)​

{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "입력 값이 올바르지 않습니다.",
    "details": [
      {
        "field": "email",
        "message": "이미 사용 중인 이메일입니다."
      },
      {
        "field": "password",
        "message": "특수문자를 포함해야 합니다."
      }
    ]
  }
}

## 4.4 오류 코드 문서

```markdown
## 오류 코드

### HTTP 상태 코드

| 코드 | 의미 | 설명 |
|------|------|------|
| 200 | OK | 요청 성공 |
| 201 | Created | 리소스 생성 성공 |
| 400 | Bad Request | 잘못된 요청 |
| 401 | Unauthorized | 인증 실패 |
| 403 | Forbidden | 권한 없음 |
| 404 | Not Found | 리소스 없음 |
| 429 | Too Many Requests | Rate limit 초과 |
| 500 | Internal Server Error | 서버 오류 |

### 커스텀 오류 코드

| 코드 | 설명 | 해결 방법 |
|------|------|-----------|
| `USER_NOT_FOUND` | 사용자 없음 | ID 확인 |
| `INVALID_TOKEN` | 토큰 만료/잘못됨 | 재로그인 |
| `VALIDATION_ERROR` | 입력 검증 실패 | details 확인 |
| `DUPLICATE_EMAIL` | 이메일 중복 | 다른 이메일 사용 |

4.5 SDK 문서 (선택사항)​

## JavaScript SDK

### 설치

```bash
npm install @example/api-client

초기화​

import { ApiClient } from '@example/api-client';

const client = new ApiClient({
  apiKey: 'YOUR_API_KEY',
  baseUrl: 'https://api.example.com/v1'
});

사용 예시​

// 사용자 조회
const user = await client.users.get('123');

// 사용자 생성
const newUser = await client.users.create({
  name: '김철수',
  email: 'user@example.com'
});

// 사용자 업데이트
await client.users.update('123', {
  name: '김영희'
});

// 사용자 삭제
await client.users.delete('123');

에러 처리​

try {
  const user = await client.users.get('123');
} catch (error) {
  if (error.code === 'USER_NOT_FOUND') {
    console.log('사용자를 찾을 수 없습니다');
  } else if (error.statusCode === 429) {
    console.log('너무 많은 요청. 잠시 후 다시 시도하세요');
  } else {
    console.error('알 수 없는 오류:', error);
  }
}

4.7 대화형 API 문서 도구​

Swagger/OpenAPI​

# openapi.yaml
openapi: 3.0.0
info:
  title: User API
  version: 1.0.0
paths:
  /users/{id}:
    get:
      summary: 사용자 조회
      parameters:
        - name: id
          in: path
          required: true
          schema:
            type: string
      responses:
        '200':
          description: 성공
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/User'
components:
  schemas:
    User:
      type: object
      properties:
        id:
          type: string
        name:
          type: string
        email:
          type: string

이를 통해 자동으로 대화형 문서를 생성할 수 있습니다.
