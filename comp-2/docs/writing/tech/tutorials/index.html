3. 튜토리얼 작성

좋은 튜토리얼은 사용자를 성공으로 이끕니다. 단계별로 따라하며 자연스럽게 학습할 수 있도록 구성하는 것이 핵심입니다.

3.1 튜토리얼 vs 가이드 vs 레퍼런스​

유형목적특징예시
튜토리얼학습단계별, 실습 중심"첫 React 앱 만들기"

레퍼런스정보 제공포괄적, 검색 중심"API 문서"

# React로 Todo 앱 만들기

이 튜토리얼에서는 React의 기본을 배우면서
실제로 작동하는 Todo 앱을 만들어봅니다.

**소요 시간:** 약 1시간

**배울 내용:**
- React 컴포넌트 만들기
- State 관리하기
- 이벤트 처리하기

**사전 요구사항:**
- JavaScript 기초 지식
- Node.js 설치 (v16 이상)
- 코드 에디터 (VS Code 추천)

**최종 결과물:**
![Final App](./final-result.png)

2. 준비 단계​

명확한 환경 설정:

## 프로젝트 설정

### 1. 새 React 앱 생성

터미널을 열고 다음 명령을 실행하세요:

```bash
npx create-react-app todo-app
cd todo-app

이 명령은 새 React 프로젝트를 생성합니다. 몇 분 정도 걸릴 수 있습니다.

2. 개발 서버 시작​

npm start

브라우저가 자동으로 열리며 http://localhost:3000에서 앱을 확인할 수 있습니다.

✅ 확인: 화면에 React 로고가 보이면 성공입니다!


### 3. 단계별 구현

**각 단계마다:**

```markdown
## Step 1: 기본 컴포넌트 만들기

### 목표
Todo 항목을 표시할 기본 컴포넌트를 만듭니다.

### 코드

`src/TodoItem.js` 파일을 생성하세요:

```jsx
// src/TodoItem.js
function TodoItem({ text }) {
  return (
    <div className="todo-item">
      <span>{text}</span>
    </div>
  );
}

export default TodoItem;

설명​

  • TodoItem: 함수형 컴포넌트를 정의합니다
  • { text }: props에서 text를 받습니다
  • <span>{text}</span>: 받은 텍스트를 표시합니다

테스트​

src/App.js를 수정하여 컴포넌트를 사용해봅시다:

import TodoItem from './TodoItem';

function App() {
  return (
    <div className="App">
      <h1>My Todos</h1>
      <TodoItem text="첫 번째 할 일" />
    </div>
  );
}

export default App;

브라우저에서 "첫 번째 할 일"이 보여야 합니다.

✅ 체크포인트: 할 일 항목이 화면에 나타나나요?

❌ 문제 해결:

  • 아무것도 안 보이면: 개발 서버가 실행 중인지 확인
  • 에러가 나면: 파일 이름과 import 경로 확인

### 4. 점진적인 난이도

```markdown
## Step 2: 할 일 목록 만들기 (쉬움)

여러 Todo 항목을 표시해봅시다...

## Step 3: 할 일 추가 기능 (보통)

이제 새 할 일을 추가하는 기능을 만들어봅시다...

## Step 4: 완료 표시 기능 (보통)

할 일을 완료 처리하는 기능을 추가합니다...

## Step 5: 삭제 기능 (어려움)

완료된 할 일을 삭제하는 기능을 구현합니다...

5. 마무리​

## 완성! 🎉

축하합니다! Todo 앱을 완성했습니다.

3.4 코드 예제 작성법​

✅ 좋은 코드 예제​

// ✅ 완전하고 실행 가능한 예제
function greetUser(name) {
  // 사용자에게 인사합니다
  const message = `안녕하세요, ${name}님!`;
  console.log(message);
  return message;
}

// 사용 예시
greetUser("철수");  // 출력: "안녕하세요, 철수님!"

❌ 피해야 할 코드 예제​

// ❌ 불완전하고 실행 불가능
function greet(n) {
  return `Hello, ${n}`;
}
// ... 어딘가에서 사용

코드 설명 기법​

Before/After 패턴​

// ❌ Before: 비효율적인 방법
for (let i = 0; i < items.length; i++) {
  console.log(items[i]);
}

// ✅ After: 더 나은 방법
items.forEach(item => console.log(item));

인라인 주석​

const user = {
  name: "김철수",          // 사용자 이름
  age: 25,                 // 나이
  email: "user@example.com"  // 이메일 주소
};

// 1. 사용자 입력 받기
const input = document.getElementById('input').value;

// 2. 입력 유효성 검사
if (input.length === 0) {

}

// 3. 데이터 처리

3.5 대화형 요소​

퀴즈​

독자가 직접 생각해보고 답을 확인할 수 있는 퀴즈를 추가하세요.

예시:

💭 생각해보기​

다음 코드의 출력은 무엇일까요?

const count = 0;
setCount(count + 1);
console.log(count);
정답 보기

0이 출력됩니다.

이유: setState는 비동기이므로 즉시 반영되지 않습니다. 다음 렌더링에서 count가 1이 됩니다.

도전 과제​

독자가 직접 해결할 수 있는 과제를 제공하세요.

예시:

🎯 도전 과제​

지금까지 배운 내용으로 다음을 만들어보세요:

목표: 카운터가 5를 넘으면 경고를 표시

힌트:

  • if 문 사용
  • count 값 확인
해답

Counter 컴포넌트를 수정하여 조건을 추가하세요:

function Counter() {
  const [count, setCount] = useState(0);

  const increment = () => {
    const newCount = count + 1;
    setCount(newCount);

    if (newCount > 5) {
      alert('5를 넘었습니다!');
    }
  };

  return (
    <button onClick={increment}>{count}</button>
  );
}
