Join과 로딩 전략

N+1 쿼리 문제를 해결하고 데이터 로딩을 최적화하는 방법을 배웁니다.

🎯 목표​

  • ✅ Lazy Loading vs Eager Loading 이해
  • ✅ N+1 쿼리 문제 인식 및 해결
  • ✅ selectinload(), joinedload() 활용
  • ✅ 로딩 전략 선택 기준
  • ✅ 성능 최적화 패턴

🐌 Lazy Loading (지연 로딩)​

기본 동작​

Lazy Loading은 SQLAlchemy의 기본 동작입니다.

from sqlalchemy import select

with Session(engine) as session:
    # User만 조회
    user = session.get(User, 1)

    # 이 시점에 posts는 로드되지 않음
    print(user.username)

    # posts 접근 시 추가 쿼리 발생!
    for post in user.posts:  # ← 여기서 SELECT 쿼리 실행
        print(post.title)

실행되는 SQL:

-- 1. User 조회
SELECT * FROM users WHERE id = 1;

-- 2. posts 접근 시 (Lazy Loading)
SELECT * FROM posts WHERE user_id = 1;

Lazy Loading의 장점​

# ✅ 필요한 데이터만 로드
user = session.get(User, 1)
print(user.username)  # posts는 로드 안 됨 (메모리 절약)

Lazy Loading의 문제: N+1 쿼리​

# ❌ N+1 쿼리 문제
with Session(engine) as session:
    users = session.execute(select(User)).scalars().all()  # 1개 쿼리

    for user in users:  # N개의 추가 쿼리!
        print(f"{user.username}:")
        for post in user.posts:  # 각 user마다 쿼리 실행
            print(f"  - {post.title}")

실행되는 SQL (users가 100명이면 101개 쿼리!):

-- 1번 쿼리
SELECT * FROM users;

-- 100번 쿼리 (각 user마다)
SELECT * FROM posts WHERE user_id = 1;
SELECT * FROM posts WHERE user_id = 2;
SELECT * FROM posts WHERE user_id = 3;
...
SELECT * FROM posts WHERE user_id = 100;

⚡ Eager Loading (즉시 로딩)​

selectinload() - SELECT IN 전략​

가장 권장되는 방법: 별도 쿼리로 데이터를 미리 로드

from sqlalchemy.orm import selectinload

with Session(engine) as session:
    # posts를 미리 로드
    stmt = select(User).options(selectinload(User.posts))
    users = session.execute(stmt).scalars().all()

    # 추가 쿼리 없이 접근
    for user in users:
        for post in user.posts:  # 쿼리 없음!
            print(f"{user.username}: {post.title}")

실행되는 SQL (단 2개 쿼리!):

-- 1. User 조회
SELECT * FROM users;

-- 2. 모든 posts를 한번에 (IN 사용)
SELECT * FROM posts
WHERE posts.user_id IN (1, 2, 3, ..., 100);

joinedload() - JOIN 전략​

하나의 쿼리로 User와 Post를 한번에 조회

from sqlalchemy.orm import joinedload

with Session(engine) as session:
    # JOIN으로 한번에 로드
    stmt = select(User).options(joinedload(User.posts))
    users = session.execute(stmt).scalars().unique().all()

    for user in users:
        for post in user.posts:  # 쿼리 없음!
            print(f"{user.username}: {post.title}")

실행되는 SQL (1개 쿼리!):

SELECT users.*, posts.*
FROM users
LEFT OUTER JOIN posts ON users.id = posts.user_id;

selectinload() vs joinedload()​

특성selectinload()joinedload()
쿼리 수2개 (User + Posts)1개 (JOIN)
중복 행없음있음 (unique() 필요)
성능One-to-Many에 좋음One-to-One에 좋음
복잡한 JOIN문제 없음느려질 수 있음
권장대부분의 경우간단한 관계

사용 가이드:

# ✅ One-to-Many → selectinload
stmt = select(User).options(selectinload(User.posts))

# ✅ One-to-One → joinedload
stmt = select(User).options(joinedload(User.profile))

# ✅ Many-to-Many → selectinload
stmt = select(Post).options(selectinload(Post.tags))

🔗 중첩 로딩​

여러 단계 로딩​

from sqlalchemy.orm import selectinload

with Session(engine) as session:
    # User → Post → Comment (3단계)
    stmt = select(User).options(
        selectinload(User.posts).selectinload(Post.comments)
    )
    users = session.execute(stmt).scalars().all()

    for user in users:
        for post in user.posts:
            print(f"{post.title} - {len(post.comments)}개 댓글")

실행되는 SQL (3개 쿼리):

-- 1. User
SELECT * FROM users;

-- 2. Posts
SELECT * FROM posts WHERE posts.user_id IN (...);

-- 3. Comments
SELECT * FROM comments WHERE comments.post_id IN (...);

여러 관계 동시 로딩​

from sqlalchemy.orm import selectinload, joinedload

with Session(engine) as session:
    # User의 여러 관계를 동시에 로드
    stmt = select(User).options(
        selectinload(User.posts),
        selectinload(User.comments),
        joinedload(User.profile)
    )
    users = session.execute(stmt).scalars().unique().all()

🎨 로딩 전략 상세​

subqueryload() - 서브쿼리 전략​

from sqlalchemy.orm import subqueryload

with Session(engine) as session:
    stmt = select(User).options(subqueryload(User.posts))
    users = session.execute(stmt).scalars().all()

실행되는 SQL:

-- 1. User
SELECT * FROM users;

-- 2. Posts (서브쿼리 사용)
SELECT * FROM posts
WHERE posts.user_id IN (
    SELECT users.id FROM users
);

selectinload() vs subqueryload():

  • selectinload(): 더 간단, 대부분 더 빠름 (권장)
  • subqueryload(): 복잡한 쿼리에서 유용할 수 있음

contains_eager() - 이미 JOIN된 경우​

from sqlalchemy.orm import contains_eager

with Session(engine) as session:
    # 이미 JOIN을 했을 때
    stmt = select(User)\
        .join(User.posts)\
        .where(Post.is_published == True)\
        .options(contains_eager(User.posts))

    users = session.execute(stmt).scalars().unique().all()

noload() - 로딩 방지​

from sqlalchemy.orm import noload

with Session(engine) as session:
    # posts를 아예 로드하지 않음
    stmt = select(User).options(noload(User.posts))
    users = session.execute(stmt).scalars().all()

    # 접근 시 빈 리스트 반환 (쿼리 없음)
    print(users[0].posts)  # []

raiseload() - 로딩 시 에러​

from sqlalchemy.orm import raiseload

with Session(engine) as session:
    # posts 접근 시 에러 발생 (개발 중 N+1 탐지용)
    stmt = select(User).options(raiseload(User.posts))
    users = session.execute(stmt).scalars().all()

    # 에러 발생!
    print(users[0].posts)  # InvalidRequestError

💡 실전 최적화 패턴​

패턴 1: 조건부 로딩​

from sqlalchemy.orm import selectinload

def get_users(
    session: Session,
    load_posts: bool = False,
    load_comments: bool = False
):
    """필요한 관계만 로드"""
    stmt = select(User)

    if load_posts:
        stmt = stmt.options(selectinload(User.posts))

    if load_comments:
        stmt = stmt.options(selectinload(User.comments))

    return session.execute(stmt).scalars().all()

# 사용
users = get_users(session, load_posts=True)  # posts만 로드
users = get_users(session, load_posts=True, load_comments=True)  # 둘 다 로드

패턴 2: 기본 로딩 전략 설정​

from sqlalchemy.orm import relationship

class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)

    # lazy='select' (기본값, Lazy Loading)
    posts: Mapped[list["Post"]] = relationship(lazy='select')

    # lazy='selectin' (항상 Eager Loading)
    profile: Mapped["Profile"] = relationship(lazy='selectin')

    # lazy='joined' (항상 JOIN)
    settings: Mapped["Settings"] = relationship(lazy='joined')

권장사항:

# ✅ 대부분: lazy='select' (기본값) + 필요시 options()
posts: Mapped[list["Post"]] = relationship()

# ⚠️ lazy='selectin' 신중히 사용
# 항상 로드하므로 불필요한 경우 메모리 낭비

패턴 3: API별 최적화​

from sqlalchemy.orm import selectinload, joinedload

# API 1: 사용자 목록 (포스트 수만 필요)
def get_users_list(session: Session):
    stmt = select(
        User.id,
        User.username,
        func.count(Post.id).label('post_count')
    ).outerjoin(Post)\
     .group_by(User.id, User.username)

    return session.execute(stmt).all()

# API 2: 사용자 상세 (모든 관계 필요)
def get_user_detail(session: Session, user_id: int):
    stmt = select(User)\
        .where(User.id == user_id)\
        .options(
            selectinload(User.posts).selectinload(Post.comments),
            selectinload(User.comments),
            joinedload(User.profile)
        )

    return session.execute(stmt).scalar_one()

# API 3: 포스트 목록 (작성자만 필요)
def get_posts_list(session: Session):
    stmt = select(Post).options(joinedload(Post.author))

    return session.execute(stmt).scalars().unique().all()

패턴 4: 페이지네이션 + Eager Loading​

from sqlalchemy import select
from sqlalchemy.orm import selectinload

def get_posts_page(
    session: Session,
    page: int = 1,
    per_page: int = 10
):
    """페이징 + Eager Loading"""
    stmt = select(Post)\
        .options(
            joinedload(Post.author),
            selectinload(Post.tags),
            selectinload(Post.comments).joinedload(Comment.author)
        )\
        .order_by(Post.created_at.desc())\
        .limit(per_page)\
        .offset((page - 1) * per_page)

    posts = session.execute(stmt).scalars().unique().all()

    # 총 개수 (별도 쿼리)
    count_stmt = select(func.count(Post.id))
    total = session.execute(count_stmt).scalar()

    return {
        'posts': posts,
        'total': total,
        'page': page,
        'per_page': per_page,
        'total_pages': (total + per_page - 1) // per_page
    }

📊 성능 측정​

쿼리 수 확인​

import logging

# SQL 로깅 활성화
logging.basicConfig()
logging.getLogger('sqlalchemy.engine').setLevel(logging.INFO)

with Session(engine) as session:
    # 모든 SQL 쿼리가 출력됨
    users = session.execute(select(User)).scalars().all()
    for user in users:
        print(user.posts)  # 쿼리 확인 가능

성능 비교​

import time
from sqlalchemy.orm import selectinload

# ❌ Lazy Loading
start = time.time()
with Session(engine) as session:
    users = session.execute(select(User)).scalars().all()
    for user in users:
        for post in user.posts:
            pass
lazy_time = time.time() - start
print(f"Lazy Loading: {lazy_time:.3f}s")

# ✅ Eager Loading
start = time.time()
with Session(engine) as session:
    stmt = select(User).options(selectinload(User.posts))
    users = session.execute(stmt).scalars().all()
    for user in users:
        for post in user.posts:
            pass
eager_time = time.time() - start
print(f"Eager Loading: {eager_time:.3f}s")

print(f"속도 향상: {lazy_time / eager_time:.1f}배")

🎯 실전 예제: 블로그 포스트 조회​

나쁜 예 (N+1 문제)​

# ❌ N+1 쿼리 발생
def get_posts_bad(session: Session):
    posts = session.execute(select(Post)).scalars().all()

    results = []
    for post in posts:
        results.append({
            'title': post.title,
            'author': post.author.username,  # 쿼리!
            'comment_count': len(post.comments),  # 쿼리!
            'tags': [tag.name for tag in post.tags]  # 쿼리!
        })

    return results

# 포스트 100개 → 301개 쿼리! (1 + 100 + 100 + 100)

좋은 예 (Eager Loading)​

# ✅ 4개 쿼리로 최적화
def get_posts_good(session: Session):
    stmt = select(Post).options(
        joinedload(Post.author),  # JOIN (같은 쿼리)
        selectinload(Post.comments),  # +1 쿼리
        selectinload(Post.tags)  # +1 쿼리
    )

    posts = session.execute(stmt).scalars().unique().all()

    results = []
    for post in posts:
        results.append({
            'title': post.title,
            'author': post.author.username,  # 쿼리 없음!
            'comment_count': len(post.comments),  # 쿼리 없음!
            'tags': [tag.name for tag in post.tags]  # 쿼리 없음!
        })

    return results

# 포스트 100개 → 3개 쿼리! (Post+Author JOIN + Comments + Tags)

💡 프로 팁​

1. 기본은 Lazy, 필요시 Eager​

# ✅ 기본 패턴
class User(Base):
    posts: Mapped[list["Post"]] = relationship()  # lazy='select'

# 필요한 곳에서만
stmt = select(User).options(selectinload(User.posts))

2. One-to-Many는 selectinload​

# ✅ One-to-Many
stmt = select(User).options(selectinload(User.posts))

# ❌ joinedload는 중복 행 발생
stmt = select(User).options(joinedload(User.posts))  # 비효율적

3. One-to-One은 joinedload​

# ✅ One-to-One
stmt = select(User).options(joinedload(User.profile))

# ⚠️ selectinload도 가능하지만 쿼리 2개
stmt = select(User).options(selectinload(User.profile))

4. 개발 중 raiseload 활용​

from sqlalchemy.orm import raiseload

# 개발 환경에서 N+1 탐지
if DEBUG:
    stmt = select(User).options(raiseload('*'))
    users = session.execute(stmt).scalars().all()
    for user in users:
        print(user.posts)  # InvalidRequestError 발생!

5. 복잡한 경우 명시적 JOIN​

# ✅ 복잡한 조건 + Eager Loading
stmt = select(User)\
    .join(Post)\
    .where(Post.is_published == True)\
    .options(contains_eager(User.posts))\
    .distinct()
