중급편 소개

SQLAlchemy의 핵심 기능을 마스터하여 실무 수준의 애플리케이션을 개발하세요.

🎯 학습 목표​

중급편에서는 실무에서 반드시 필요한 SQLAlchemy의 핵심 기능들을 배웁니다:

  • 관계(Relationship) 정의: 테이블 간 관계를 ORM으로 표현
  • 쿼리 작성: 복잡한 조건의 데이터 조회
  • Join 최적화: Eager/Lazy Loading으로 성능 최적화
  • 세션 관리: 트랜잭션과 세션 생명주기 이해

📚 학습 순서​

1. 관계(Relationship) 정의​

테이블 간의 관계를 Python 객체로 다루는 방법을 배웁니다.

class User(Base):
    __tablename__ = 'users'
    id: Mapped[int] = mapped_column(primary_key=True)
    posts: Mapped[list["Post"]] = relationship(back_populates="author")

class Post(Base):
    __tablename__ = 'posts'
    id: Mapped[int] = mapped_column(primary_key=True)
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))
    author: Mapped["User"] = relationship(back_populates="posts")

# 사용
user = session.get(User, 1)
for post in user.posts:  # 관계를 통한 접근
    print(post.title)

2. 쿼리 작성​

SQLAlchemy의 강력한 쿼리 API를 활용합니다.

from sqlalchemy import select

# 복잡한 조건
stmt = select(User).where(
    User.age > 25,
    User.email.like('%@gmail.com')
).order_by(User.created_at.desc())

users = session.execute(stmt).scalars().all()

3. Join과 로딩 전략​

N+1 쿼리 문제를 해결하고 성능을 최적화합니다.

from sqlalchemy.orm import selectinload

# Eager Loading으로 N+1 문제 해결
stmt = select(User).options(selectinload(User.posts))
users = session.execute(stmt).scalars().all()

# 단 2개의 쿼리로 모든 데이터 로드

4. 세션 관리​

세션의 생명주기와 트랜잭션을 올바르게 관리합니다.

# 세션 패턴
with Session(engine) as session:
    with session.begin():
        # 트랜잭션 내에서 작업
        user = User(username='alice')
        session.add(user)
        # 자동 커밋/롤백

💡 중급편의 핵심​

관계 vs Foreign Key​

# ❌ Foreign Key만 사용
class Post(Base):
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))
    # 사용 시: post.user_id로만 접근 가능

# ✅ Relationship 추가
class Post(Base):
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))
    author: Mapped["User"] = relationship()
    # 사용 시: post.author로 객체 접근!

N+1 쿼리 문제​

# ❌ N+1 쿼리 발생
users = session.execute(select(User)).scalars().all()
for user in users:
    print(user.posts)  # 각 사용자마다 쿼리!

# ✅ Eager Loading
stmt = select(User).options(selectinload(User.posts))
users = session.execute(stmt).scalars().all()
for user in users:
    print(user.posts)  # 추가 쿼리 없음!

🎯 실전 예제​

중급편에서는 실제 블로그 시스템을 구축하면서 배웁니다:

# User ↔ Post (1:N)
# Post ↔ Comment (1:N)
# Post ↔ Tag (N:M)
# User ↔ User (팔로우, N:M)

class User(Base):
    posts: Mapped[list["Post"]] = relationship()
    followers: Mapped[list["User"]] = relationship()

class Post(Base):
    author: Mapped["User"] = relationship()
    comments: Mapped[list["Comment"]] = relationship()
    tags: Mapped[list["Tag"]] = relationship(secondary=post_tags)

⚡ 성능 최적화 미리보기​

중급편에서 배우는 성능 최적화 기법:

  1. Eager Loading: N+1 쿼리 제거
  2. Select Only What You Need: 필요한 컬럼만 조회
  3. Bulk Operations: 대량 작업 최적화
  4. Connection Pooling: 연결 재사용
