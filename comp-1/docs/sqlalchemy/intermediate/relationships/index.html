관계(Relationship) 정의

SQLAlchemy ORM으로 테이블 간의 관계를 Python 객체로 표현하는 방법을 배웁니다.

🎯 목표​

  • ✅ Foreign Key와 Relationship 이해
  • ✅ One-to-Many (1:N) 관계
  • ✅ Many-to-One (N:1) 관계
  • ✅ One-to-One (1:1) 관계
  • ✅ Many-to-Many (N:M) 관계
  • ✅ Cascade 옵션 활용

🔑 Foreign Key vs Relationship​

Foreign Key만 사용​

from sqlalchemy import ForeignKey
from sqlalchemy.orm import Mapped, mapped_column

class Post(Base):
    __tablename__ = 'posts'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str]
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))

사용:

post = session.get(Post, 1)
print(post.user_id)  # 123 (ID만 알 수 있음)

# User 정보가 필요하면 별도 조회
user = session.get(User, post.user_id)

Relationship 추가​

from sqlalchemy.orm import relationship

class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)
    username: Mapped[str]

    # Relationship
    posts: Mapped[list["Post"]] = relationship(back_populates="author")

class Post(Base):
    __tablename__ = 'posts'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str]
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))

    # Relationship
    author: Mapped["User"] = relationship(back_populates="posts")

사용:

post = session.get(Post, 1)
print(post.author.username)  # 객체로 직접 접근!

user = session.get(User, 1)
for post in user.posts:  # 리스트로 접근
    print(post.title)

1️⃣ One-to-Many (1:N) 관계​

기본 설정​

User (1) ↔ Post (N): 한 사용자가 여러 게시글을 작성

from sqlalchemy import String, Text, ForeignKey
from sqlalchemy.orm import Mapped, mapped_column, relationship

class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)
    username: Mapped[str] = mapped_column(String(50))

    # One-to-Many: 한 유저가 여러 포스트
    posts: Mapped[list["Post"]] = relationship(
        back_populates="author",
        cascade="all, delete-orphan"
    )

class Post(Base):
    __tablename__ = 'posts'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(200))
    content: Mapped[str] = mapped_column(Text)

    # Foreign Key
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))

    # Many-to-One: 여러 포스트가 한 유저에 속함
    author: Mapped["User"] = relationship(back_populates="posts")

사용 예제​

with Session(engine) as session:
    # 사용자 생성
    user = User(username='alice')

    # 포스트 생성 및 연결
    post1 = Post(title='First Post', content='Hello')
    post2 = Post(title='Second Post', content='World')

    # 방법 1: append
    user.posts.append(post1)
    user.posts.append(post2)

    # 방법 2: 리스트 할당
    # user.posts = [post1, post2]

    # 방법 3: author 설정
    # post1.author = user
    # post2.author = user

    session.add(user)
    session.commit()

    # 조회
    user = session.get(User, 1)
    print(f"{user.username}의 포스트:")
    for post in user.posts:
        print(f"  - {post.title}")

관계를 통한 추가/삭제​

with Session(engine) as session:
    user = session.get(User, 1)

    # 새 포스트 추가
    new_post = Post(title='New Post', content='Content')
    user.posts.append(new_post)
    session.commit()

    # 포스트 제거 (cascade 설정 시 DB에서도 삭제)
    user.posts.remove(user.posts[0])
    session.commit()

2️⃣ Many-to-One (N:1) 관계​

Many-to-One은 One-to-Many의 반대편입니다.

class Comment(Base):
    __tablename__ = 'comments'

    id: Mapped[int] = mapped_column(primary_key=True)
    content: Mapped[str] = mapped_column(Text)

    # Foreign Key
    post_id: Mapped[int] = mapped_column(ForeignKey('posts.id'))

    # Many-to-One: 여러 댓글이 한 포스트에 속함
    post: Mapped["Post"] = relationship(back_populates="comments")

class Post(Base):
    __tablename__ = 'posts'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str]

    # One-to-Many: 한 포스트가 여러 댓글을 가짐
    comments: Mapped[list["Comment"]] = relationship(
        back_populates="post",
        cascade="all, delete-orphan"
    )

사용:

with Session(engine) as session:
    post = session.get(Post, 1)

    # 댓글 추가
    comment1 = Comment(content='Great post!')
    comment2 = Comment(content='Thanks for sharing')

    post.comments.extend([comment1, comment2])
    session.commit()

    # 댓글의 포스트 접근
    comment = session.get(Comment, 1)
    print(f"댓글이 달린 포스트: {comment.post.title}")

3️⃣ One-to-One (1:1) 관계​

기본 설정​

User (1) ↔ Profile (1): 한 사용자가 하나의 프로필을 가짐

from sqlalchemy import String, Text, Date, ForeignKey
from sqlalchemy.orm import Mapped, mapped_column, relationship

class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)
    username: Mapped[str] = mapped_column(String(50))

    # One-to-One: uselist=False
    profile: Mapped["Profile"] = relationship(
        back_populates="user",
        uselist=False,  # 리스트가 아닌 단일 객체
        cascade="all, delete-orphan"
    )

class Profile(Base):
    __tablename__ = 'profiles'

    id: Mapped[int] = mapped_column(primary_key=True)
    bio: Mapped[str | None] = mapped_column(Text)
    birth_date: Mapped[date | None] = mapped_column(Date)
    avatar_url: Mapped[str | None] = mapped_column(String(500))

    # Foreign Key (unique=True로 1:1 보장)
    user_id: Mapped[int] = mapped_column(
        ForeignKey('users.id'),
        unique=True
    )

    # One-to-One
    user: Mapped["User"] = relationship(back_populates="profile")

사용 예제​

from datetime import date

with Session(engine) as session:
    # 사용자와 프로필 생성
    user = User(username='alice')
    profile = Profile(
        bio='Python developer',
        birth_date=date(1990, 5, 15),
        avatar_url='https://example.com/avatar.jpg'
    )

    # 연결
    user.profile = profile

    session.add(user)
    session.commit()

    # 조회
    user = session.get(User, 1)
    print(f"{user.username}의 프로필:")
    print(f"  Bio: {user.profile.bio}")
    print(f"  생년월일: {user.profile.birth_date}")

4️⃣ Many-to-Many (N:M) 관계​

Association Table 사용​

Post (N) ↔ Tag (M): 한 포스트가 여러 태그, 한 태그가 여러 포스트

from sqlalchemy import Table, Column, Integer, ForeignKey, String
from sqlalchemy.orm import Mapped, mapped_column, relationship

# Association Table (중간 테이블)
post_tags = Table(
    'post_tags',
    Base.metadata,
    Column('post_id', Integer, ForeignKey('posts.id'), primary_key=True),
    Column('tag_id', Integer, ForeignKey('tags.id'), primary_key=True)
)

class Post(Base):
    __tablename__ = 'posts'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(200))

    # Many-to-Many
    tags: Mapped[list["Tag"]] = relationship(
        secondary=post_tags,
        back_populates="posts"
    )

class Tag(Base):
    __tablename__ = 'tags'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(50), unique=True)

    # Many-to-Many
    posts: Mapped[list["Post"]] = relationship(
        secondary=post_tags,
        back_populates="tags"
    )

생성되는 SQL:

CREATE TABLE post_tags (
    post_id INTEGER REFERENCES posts(id),
    tag_id INTEGER REFERENCES tags(id),
    PRIMARY KEY (post_id, tag_id)
);

사용 예제​

with Session(engine) as session:
    # 태그 생성
    tag_python = Tag(name='python')
    tag_sql = Tag(name='sqlalchemy')
    tag_db = Tag(name='database')

    # 포스트 생성
    post = Post(title='SQLAlchemy Guide')

    # 태그 추가
    post.tags.extend([tag_python, tag_sql, tag_db])

    session.add(post)
    session.commit()

    # 조회
    post = session.get(Post, 1)
    print(f"태그: {[tag.name for tag in post.tags]}")

    # 반대 방향
    tag = session.query(Tag).filter_by(name='python').first()
    print(f"Python 태그의 포스트: {[p.title for p in tag.posts]}")

Association Object Pattern​

중간 테이블에 추가 데이터가 필요한 경우:

from datetime import datetime
from sqlalchemy import DateTime, func

class PostTag(Base):
    """Association Object (추가 데이터 포함)"""
    __tablename__ = 'post_tags'

    post_id: Mapped[int] = mapped_column(
        ForeignKey('posts.id'),
        primary_key=True
    )
    tag_id: Mapped[int] = mapped_column(
        ForeignKey('tags.id'),
        primary_key=True
    )

    # 추가 데이터
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now()
    )
    created_by: Mapped[int | None] = mapped_column(ForeignKey('users.id'))

    # Relationships
    post: Mapped["Post"] = relationship(back_populates="post_tags")
    tag: Mapped["Tag"] = relationship(back_populates="post_tags")
    creator: Mapped["User"] = relationship()

class Post(Base):
    __tablename__ = 'posts'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str]

    # Association Object 사용
    post_tags: Mapped[list["PostTag"]] = relationship(
        back_populates="post",
        cascade="all, delete-orphan"
    )

    # 편의 속성 (선택사항)
    @property
    def tags(self):
        return [pt.tag for pt in self.post_tags]

class Tag(Base):
    __tablename__ = 'tags'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str]

    post_tags: Mapped[list["PostTag"]] = relationship(
        back_populates="tag"
    )

사용:

with Session(engine) as session:
    post = session.get(Post, 1)
    tag = session.query(Tag).filter_by(name='python').first()
    user = session.get(User, 1)

    # Association Object 생성
    post_tag = PostTag(
        post=post,
        tag=tag,
        creator=user
    )

    session.add(post_tag)
    session.commit()

    # 조회
    for pt in post.post_tags:
        print(f"Tag: {pt.tag.name}, Added by: {pt.creator.username}, At: {pt.created_at}")

🔄 Self-Referential 관계​

팔로우 시스템​

User ↔ User: 사용자가 다른 사용자를 팔로우

# Association Table
followers_table = Table(
    'followers',
    Base.metadata,
    Column('follower_id', Integer, ForeignKey('users.id'), primary_key=True),
    Column('following_id', Integer, ForeignKey('users.id'), primary_key=True)
)

class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)
    username: Mapped[str] = mapped_column(String(50))

    # Self-referential Many-to-Many
    following: Mapped[list["User"]] = relationship(
        secondary=followers_table,
        primaryjoin=id == followers_table.c.follower_id,
        secondaryjoin=id == followers_table.c.following_id,
        back_populates="followers"
    )

    followers: Mapped[list["User"]] = relationship(
        secondary=followers_table,
        primaryjoin=id == followers_table.c.following_id,
        secondaryjoin=id == followers_table.c.follower_id,
        back_populates="following"
    )

사용:

with Session(engine) as session:
    alice = User(username='alice')
    bob = User(username='bob')
    charlie = User(username='charlie')

    # Alice가 Bob과 Charlie를 팔로우
    alice.following.extend([bob, charlie])

    session.add_all([alice, bob, charlie])
    session.commit()

    # 조회
    user = session.query(User).filter_by(username='alice').first()
    print(f"{user.username}가 팔로우하는 사람:")
    for following in user.following:
        print(f"  - {following.username}")

    print(f"{user.username}의 팔로워:")
    for follower in user.followers:
        print(f"  - {follower.username}")

⚙️ Cascade 옵션​

주요 Cascade 옵션​

class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)
    username: Mapped[str]

    # Cascade 옵션
    posts: Mapped[list["Post"]] = relationship(
        back_populates="author",
        cascade="all, delete-orphan"
    )

Cascade 옵션:

  • all: save-update, merge, refresh, expunge, delete
  • save-update: 부모 추가 시 자식도 자동 추가
  • delete: 부모 삭제 시 자식도 삭제
  • delete-orphan: 부모에서 제거된 자식 삭제
  • merge: 병합 시 자식도 병합
  • refresh: 새로고침 시 자식도 새로고침

Cascade 예제​

# cascade="all, delete-orphan"
with Session(engine) as session:
    user = session.get(User, 1)

    # 부모 삭제 → 자식도 삭제
    session.delete(user)
    session.commit()  # user의 모든 posts도 삭제됨

# delete-orphan
with Session(engine) as session:
    user = session.get(User, 1)

    # 리스트에서 제거 → DB에서도 삭제
    removed_post = user.posts.pop()  # 리스트에서 제거
    session.commit()  # DB에서도 삭제됨

Cascade 비교​

# ❌ cascade 없음
class User(Base):
    posts: Mapped[list["Post"]] = relationship(back_populates="author")

# 사용 시:
user.posts.append(post)
session.add(user)
session.add(post)  # 수동으로 추가 필요

# ✅ cascade="save-update"
class User(Base):
    posts: Mapped[list["Post"]] = relationship(
        back_populates="author",
        cascade="save-update"
    )

# 사용 시:
user.posts.append(post)
session.add(user)  # post도 자동 추가

🔧 back_populates vs backref​

back_populates (권장)​

class User(Base):
    __tablename__ = 'users'
    id: Mapped[int] = mapped_column(primary_key=True)

    posts: Mapped[list["Post"]] = relationship(back_populates="author")

class Post(Base):
    __tablename__ = 'posts'
    id: Mapped[int] = mapped_column(primary_key=True)
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))

    author: Mapped["User"] = relationship(back_populates="posts")

backref (레거시)​

class User(Base):
    __tablename__ = 'users'
    id: Mapped[int] = mapped_column(primary_key=True)

    # backref: 한쪽에만 정의
    posts: Mapped[list["Post"]] = relationship(
        backref="author"  # Post.author가 자동 생성됨
    )

class Post(Base):
    __tablename__ = 'posts'
    id: Mapped[int] = mapped_column(primary_key=True)
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))
    # relationship 정의 불필요

권장사항: back_populates를 사용하면 양방향 관계가 명시적으로 보여 코드가 더 명확합니다.

💡 실전 예제: 블로그 시스템​

from datetime import datetime
from sqlalchemy import String, Text, DateTime, ForeignKey, Table, Column, func
from sqlalchemy.orm import Mapped, mapped_column, relationship

# Many-to-Many: Post ↔ Tag
post_tags = Table(
    'post_tags',
    Base.metadata,
    Column('post_id', Integer, ForeignKey('posts.id'), primary_key=True),
    Column('tag_id', Integer, ForeignKey('tags.id'), primary_key=True)
)

class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)
    username: Mapped[str] = mapped_column(String(50), unique=True)

    # One-to-Many: User → Post
    posts: Mapped[list["Post"]] = relationship(
        back_populates="author",
        cascade="all, delete-orphan"
    )

    # One-to-Many: User → Comment
    comments: Mapped[list["Comment"]] = relationship(
        back_populates="author",
        cascade="all, delete-orphan"
    )

class Post(Base):
    __tablename__ = 'posts'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(200))
    content: Mapped[str] = mapped_column(Text)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now()
    )

    # Many-to-One: Post → User
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))
    author: Mapped["User"] = relationship(back_populates="posts")

    # One-to-Many: Post → Comment
    comments: Mapped[list["Comment"]] = relationship(
        back_populates="post",
        cascade="all, delete-orphan"
    )

    # Many-to-Many: Post ↔ Tag
    tags: Mapped[list["Tag"]] = relationship(
        secondary=post_tags,
        back_populates="posts"
    )

class Comment(Base):
    __tablename__ = 'comments'

    id: Mapped[int] = mapped_column(primary_key=True)
    content: Mapped[str] = mapped_column(Text)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now()
    )

    # Many-to-One: Comment → Post
    post_id: Mapped[int] = mapped_column(ForeignKey('posts.id'))
    post: Mapped["Post"] = relationship(back_populates="comments")

    # Many-to-One: Comment → User
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))
    author: Mapped["User"] = relationship(back_populates="comments")

class Tag(Base):
    __tablename__ = 'tags'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(50), unique=True)

    # Many-to-Many: Tag ↔ Post
    posts: Mapped[list["Post"]] = relationship(
        secondary=post_tags,
        back_populates="tags"
    )

# 사용 예제
with Session(engine) as session:
    # 사용자 생성
    user = User(username='alice')

    # 태그 생성
    tag_python = Tag(name='python')
    tag_sql = Tag(name='sqlalchemy')

    # 포스트 생성
    post = Post(
        title='SQLAlchemy 완벽 가이드',
        content='SQLAlchemy를 배워봅시다!',
        author=user,
        tags=[tag_python, tag_sql]
    )

    # 댓글 생성
    comment1 = Comment(content='좋은 글이네요!', author=user, post=post)
    comment2 = Comment(content='감사합니다!', author=user, post=post)

    session.add(user)
    session.commit()

    # 조회
    user = session.query(User).filter_by(username='alice').first()
    for post in user.posts:
        print(f"포스트: {post.title}")
        print(f"태그: {[tag.name for tag in post.tags]}")
        print(f"댓글 수: {len(post.comments)}")

💡 프로 팁​

1. Lazy Loading vs Eager Loading​

# ❌ Lazy Loading (기본값): N+1 문제
users = session.query(User).all()
for user in users:
    print(user.posts)  # 각 사용자마다 쿼리 발생!

# ✅ Eager Loading: Join 최적화 (다음 장에서 자세히)
from sqlalchemy.orm import selectinload

stmt = select(User).options(selectinload(User.posts))
users = session.execute(stmt).scalars().all()

2. Relationship 방향 선택​

# ✅ 양방향 (권장)
class User(Base):
    posts: Mapped[list["Post"]] = relationship(back_populates="author")

class Post(Base):
    author: Mapped["User"] = relationship(back_populates="posts")

# ⚠️ 단방향 (필요한 경우만)
class Post(Base):
    author: Mapped["User"] = relationship()  # Post → User만

3. Cascade 신중히 사용​

# ✅ 종속적인 데이터는 cascade
class User(Base):
    posts: Mapped[list["Post"]] = relationship(
        cascade="all, delete-orphan"  # User 삭제 시 Post도 삭제
    )

# ❌ 독립적인 데이터는 cascade 금지
class Post(Base):
    tags: Mapped[list["Tag"]] = relationship()  # Post 삭제해도 Tag는 유지

🔗 관련 문서​

  • ORM 모델 정의
