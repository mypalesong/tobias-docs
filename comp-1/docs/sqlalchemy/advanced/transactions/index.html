트랜잭션 관리

🔐 ACID 속성​

Atomicity (원자성)​

트랜잭션의 모든 작업이 성공하거나 모두 실패합니다.

with Session(engine) as session:
    with session.begin():
        # 모든 작업이 하나의 단위
        account_from = session.get(Account, 1)
        account_to = session.get(Account, 2)

        account_from.balance -= 100
        account_to.balance += 100

        # 둘 다 성공 또는 둘 다 실패

Consistency (일관성)​

트랜잭션 전후 데이터베이스가 일관된 상태를 유지합니다.

class Account(Base):
    __tablename__ = 'accounts'

    id: Mapped[int] = mapped_column(primary_key=True)
    balance: Mapped[Decimal] = mapped_column(Numeric(10, 2))

    __table_args__ = (
        # 잔액은 0 이상
        CheckConstraint('balance >= 0', name='check_balance_positive'),
    )

with Session(engine) as session:
    with session.begin():
        account = session.get(Account, 1)
        account.balance -= 1000  # balance < 0이면 에러!

Isolation (격리성)​

동시 실행 중인 트랜잭션이 서로 영향을 주지 않습니다.

# Transaction 1
with Session(engine) as session1:
    with session1.begin():
        user = session1.get(User, 1)
        user.balance -= 100

        # Transaction 2는 이 변경을 볼 수 없음 (격리 수준에 따라)

Durability (지속성)​

커밋된 트랜잭션은 시스템 장애에도 보존됩니다.

with Session(engine) as session:
    with session.begin():
        order = Order(amount=100)
        session.add(order)
        # commit 후 DB 저장, 시스템 재시작해도 유지

📊 격리 수준 (Isolation Levels)​

PostgreSQL 격리 수준​

격리 수준Dirty ReadNon-Repeatable ReadPhantom Read
READ UNCOMMITTED❌❌❌
READ COMMITTED✅❌❌
REPEATABLE READ✅✅✅
SERIALIZABLE✅✅✅

PostgreSQL 특이사항: READ UNCOMMITTED는 READ COMMITTED와 동일하게 동작

격리 수준 설정​

from sqlalchemy import create_engine

# Engine 레벨 설정
engine = create_engine(
    'postgresql://...',
    isolation_level="REPEATABLE READ"
)

# Session 레벨 설정
with Session(engine) as session:
    session.connection(execution_options={"isolation_level": "SERIALIZABLE"})

    with session.begin():
        # SERIALIZABLE 격리 수준으로 실행
        pass

READ COMMITTED (기본값)​

# Transaction 1
with Session(engine) as session1:
    user = session1.get(User, 1)
    print(user.balance)  # 100

    # Transaction 2가 balance를 150으로 변경하고 커밋

    # 다시 조회하면 변경된 값 보임
    session1.refresh(user)
    print(user.balance)  # 150 (Non-Repeatable Read)

REPEATABLE READ​

# Transaction 1 (REPEATABLE READ)
with Session(engine) as session1:
    session1.connection(execution_options={"isolation_level": "REPEATABLE READ"})

    with session1.begin():
        user = session1.get(User, 1)
        print(user.balance)  # 100

        # Transaction 2가 balance를 150으로 변경하고 커밋

        # 다시 조회해도 처음 값 유지
        session1.refresh(user)
        print(user.balance)  # 100 (Repeatable Read)

SERIALIZABLE​

# 완전한 격리, 동시 실행 불가
engine = create_engine('postgresql://...', isolation_level="SERIALIZABLE")

# Transaction 1
with Session(engine) as session1:
    with session1.begin():
        stmt = select(func.sum(Account.balance))
        total = session1.execute(stmt).scalar()

        # Transaction 2가 새 계좌를 추가하면 Serialization Error

🔒 잠금 (Locking)​

비관적 잠금 (Pessimistic Locking)​

FOR UPDATE: 행을 독점 잠금

from sqlalchemy import select

# 재고 차감 (동시성 제어)
with Session(engine) as session:
    with session.begin():
        # 행 잠금 (다른 트랜잭션은 대기)
        stmt = select(Product).where(Product.id == 1).with_for_update()
        product = session.execute(stmt).scalar_one()

        if product.stock >= quantity:
            product.stock -= quantity
            order = Order(product=product, quantity=quantity)
            session.add(order)
        else:
            raise OutOfStockError()

FOR UPDATE SKIP LOCKED: 잠긴 행은 건너뜀

# 작업 큐 패턴
with Session(engine) as session:
    with session.begin():
        # 잠기지 않은 작업만 가져옴
        stmt = select(Task)\
            .where(Task.status == 'pending')\
            .limit(10)\
            .with_for_update(skip_locked=True)

        tasks = session.execute(stmt).scalars().all()

        for task in tasks:
            task.status = 'processing'
            process_task(task)
            task.status = 'completed'

FOR UPDATE NOWAIT: 즉시 에러 (대기 안 함)

from sqlalchemy.exc import OperationalError

with Session(engine) as session:
    try:
        with session.begin():
            stmt = select(Resource)\
                .where(Resource.id == 1)\
                .with_for_update(nowait=True)

            resource = session.execute(stmt).scalar_one()
            # 작업...

    except OperationalError as e:
        if 'could not obtain lock' in str(e):
            print("리소스가 사용 중입니다")

낙관적 잠금 (Optimistic Locking)​

Version Column 사용

from sqlalchemy import Integer
from sqlalchemy.orm import Mapped, mapped_column

class Product(Base):
    __tablename__ = 'products'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str]
    stock: Mapped[int]

    # Version 컬럼
    version: Mapped[int] = mapped_column(Integer, default=0)

# 사용
with Session(engine) as session:
    with session.begin():
        product = session.get(Product, 1)
        original_version = product.version

        # 작업 수행
        product.stock -= 10

        # 커밋 시 version 체크
        # UPDATE products SET stock=?, version=version+1
        # WHERE id=? AND version=?

        # 다른 트랜잭션이 먼저 수정했다면 StaleDataError

SQLAlchemy의 자동 낙관적 잠금

from sqlalchemy.orm import Mapped, mapped_column, version_id_col

class Product(Base):
    __tablename__ = 'products'

    id: Mapped[int] = mapped_column(primary_key=True)
    stock: Mapped[int]

    # version_id_col로 자동 버전 관리
    version_id: Mapped[int] = mapped_column(default=0)

    __mapper_args__ = {
        'version_id_col': version_id
    }

💾 Savepoint (중첩 트랜잭션)​

기본 Savepoint​

with Session(engine) as session:
    session.begin()

    try:
        # 사용자 생성
        user = User(username='alice')
        session.add(user)
        session.flush()

        # Savepoint 생성
        with session.begin_nested():
            # 위험한 작업
            risky_post = Post(title='Risky', user_id=user.id)
            session.add(risky_post)

            if check_spam(risky_post):
                raise ValueError("Spam detected!")

        session.commit()

    except ValueError:
        # risky_post만 롤백, user는 유지
        session.rollback()

생성되는 SQL:

BEGIN;

INSERT INTO users (username) VALUES ('alice');

SAVEPOINT sa_savepoint_1;

INSERT INTO posts (title, user_id) VALUES ('Risky', 1);

-- 에러 발생 시
ROLLBACK TO SAVEPOINT sa_savepoint_1;

COMMIT;

실전 예제: 배치 처리​

def batch_import_users(user_data_list):
    """사용자 일괄 임포트 (부분 실패 허용)"""
    success_count = 0
    error_count = 0

    with Session(engine) as session:
        session.begin()

        for user_data in user_data_list:
            try:
                with session.begin_nested():  # Savepoint
                    user = User(**user_data)
                    session.add(user)
                    session.flush()

                    success_count += 1

            except Exception as e:
                # 이 사용자만 롤백, 다음 사용자 계속
                print(f"Failed to import {user_data['username']}: {e}")
                error_count += 1

        session.commit()

    return {
        'success': success_count,
        'errors': error_count
    }

⚠️ Deadlock 처리​

Deadlock이란?​

# Transaction 1
with Session(engine) as session1:
    session1.begin()
    account_a = session1.get(Account, 1, with_for_update=True)
    time.sleep(1)
    account_b = session1.get(Account, 2, with_for_update=True)  # 대기
    session1.commit()

# Transaction 2 (동시 실행)
with Session(engine) as session2:
    session2.begin()
    account_b = session2.get(Account, 2, with_for_update=True)
    time.sleep(1)
    account_a = session2.get(Account, 1, with_for_update=True)  # 대기
    session2.commit()

# 결과: Deadlock! (서로를 기다림)

Deadlock 방지​

1. 일관된 순서로 잠금

# ✅ 항상 ID 순서대로 잠금
def transfer_money(session, from_id, to_id, amount):
    # ID 순서대로 정렬
    ids = sorted([from_id, to_id])

    # 순서대로 잠금 획득
    account1 = session.get(Account, ids[0], with_for_update=True)
    account2 = session.get(Account, ids[1], with_for_update=True)

    if from_id == ids[0]:
        account1.balance -= amount
        account2.balance += amount
    else:
        account2.balance -= amount
        account1.balance += amount

2. Deadlock 재시도

from sqlalchemy.exc import OperationalError
import time

def execute_with_retry(func, max_retries=3):
    """Deadlock 발생 시 재시도"""
    for attempt in range(max_retries):
        try:
            with Session(engine) as session:
                with session.begin():
                    return func(session)

        except OperationalError as e:
            if 'deadlock detected' in str(e) and attempt < max_retries - 1:
                print(f"Deadlock detected, retrying ({attempt + 1}/{max_retries})...")
                time.sleep(0.1 * (2 ** attempt))  # Exponential backoff
            else:
                raise

# 사용
def transfer(session):
    # 이체 로직
    pass

execute_with_retry(transfer)

🌐 분산 트랜잭션 (2-Phase Commit)​

PostgreSQL PREPARE TRANSACTION​

from sqlalchemy import text

# Phase 1: Prepare
with Session(engine) as session:
    session.begin()

    user = User(username='alice')
    session.add(user)
    session.flush()

    # 트랜잭션 준비
    session.execute(text("PREPARE TRANSACTION 'txn_001'"))

# Phase 2: Commit
with engine.connect() as conn:
    conn.execute(text("COMMIT PREPARED 'txn_001'"))

# 또는 Rollback
# conn.execute(text("ROLLBACK PREPARED 'txn_001'"))

여러 데이터베이스 간 트랜잭션​

# DB1: PostgreSQL
engine1 = create_engine('postgresql://...')

# DB2: PostgreSQL
engine2 = create_engine('postgresql://...')

# 2-Phase Commit
def distributed_transaction():
    session1 = Session(engine1)
    session2 = Session(engine2)

    try:
        # Phase 1: 작업 수행
        session1.begin()
        session2.begin()

        # DB1에 사용자 생성
        user = User(username='alice')
        session1.add(user)
        session1.flush()

        # DB2에 로그 기록
        log = AuditLog(action='user_created', user_id=user.id)
        session2.add(log)
        session2.flush()

        # Phase 2: Prepare
        session1.execute(text("PREPARE TRANSACTION 'txn1'"))
        session2.execute(text("PREPARE TRANSACTION 'txn2'"))

        # Phase 3: Commit
        with engine1.connect() as conn:
            conn.execute(text("COMMIT PREPARED 'txn1'"))
        with engine2.connect() as conn:
            conn.execute(text("COMMIT PREPARED 'txn2'"))

    except Exception as e:
        # Rollback
        try:
            with engine1.connect() as conn:
                conn.execute(text("ROLLBACK PREPARED 'txn1'"))
        except:
            pass

        try:
            with engine2.connect() as conn:
                conn.execute(text("ROLLBACK PREPARED 'txn2'"))
        except:
            pass

        raise

    finally:
        session1.close()
        session2.close()

💡 실전 패턴​

패턴 1: 재고 관리​

class InventoryService:
    def reserve_stock(self, product_id: int, quantity: int) -> bool:
        """재고 예약 (동시성 제어)"""
        with Session(engine) as session:
            with session.begin():
                # 비관적 잠금
                stmt = select(Product)\
                    .where(Product.id == product_id)\
                    .with_for_update()

                product = session.execute(stmt).scalar_one()

                if product.stock >= quantity:
                    product.stock -= quantity
                    reservation = StockReservation(
                        product_id=product_id,
                        quantity=quantity
                    )
                    session.add(reservation)
                    return True

                return False

패턴 2: 주문 처리​

def process_order(order_id: int):
    """주문 처리 (트랜잭션)"""
    with Session(engine) as session:
        with session.begin():
            # 주문 조회
            order = session.get(Order, order_id, with_for_update=True)

            if order.status != 'pending':
                raise ValueError("Order already processed")

            # Savepoint: 결제 시도
            try:
                with session.begin_nested():
                    payment = process_payment(order)
                    session.add(payment)

                    order.status = 'paid'

            except PaymentError as e:
                # 결제 실패, 주문은 pending 유지
                order.payment_error = str(e)
                raise

            # 재고 차감
            for item in order.items:
                product = session.get(Product, item.product_id, with_for_update=True)
                product.stock -= item.quantity

            order.status = 'confirmed'

패턴 3: 계좌 이체​

def transfer_money(from_account_id: int, to_account_id: int, amount: Decimal):
    """계좌 이체 (ACID 보장)"""
    with Session(engine) as session:
        with session.begin():
            # Deadlock 방지: ID 순서로 잠금
            ids = sorted([from_account_id, to_account_id])

            account1 = session.get(Account, ids[0], with_for_update=True)
            account2 = session.get(Account, ids[1], with_for_update=True)

            from_account = account1 if account1.id == from_account_id else account2
            to_account = account2 if account2.id == to_account_id else account1

            # 잔액 확인
            if from_account.balance < amount:
                raise InsufficientFundsError()

            # 이체
            from_account.balance -= amount
            to_account.balance += amount

            # 거래 기록
            transaction = Transaction(
                from_account_id=from_account_id,
                to_account_id=to_account_id,
                amount=amount
            )
            session.add(transaction)

💡 프로 팁​

1. 트랜잭션은 짧게​

# ❌ 나쁜 예: 긴 트랜잭션
with Session(engine) as session:
    with session.begin():
        data = fetch_data_from_api()  # 외부 API 호출 (느림)
        process_data(data)  # 복잡한 계산
        session.add(...)  # DB 작업

# ✅ 좋은 예: 짧은 트랜잭션
data = fetch_data_from_api()
processed = process_data(data)

with Session(engine) as session:
    with session.begin():
        session.add(processed)  # DB 작업만

2. 격리 수준 선택​

  • READ COMMITTED: 대부분의 경우 (기본값)
  • REPEATABLE READ: 일관된 읽기가 중요할 때
  • SERIALIZABLE: 완전한 격리가 필요할 때 (성능 저하)

3. 잠금 전략 선택​

  • 낙관적 잠금: 충돌이 드문 경우 (버전 컬럼)
  • 비관적 잠금: 충돌이 빈번한 경우 (FOR UPDATE)
