파일 조작

파일과 디렉토리를 생성, 복사, 이동, 삭제하는 방법을 배웁니다.

📝 파일 생성​

touch - 빈 파일 생성​

# 단일 파일 생성
touch file.txt

# 여러 파일 한 번에
touch file1.txt file2.txt file3.txt

# 공백이 있는 파일명 (따옴표 사용)
touch "my file.txt"

# 타임스탬프 업데이트 (파일이 이미 존재하면)
touch existing-file.txt

리다이렉션으로 파일 생성​

# 빈 파일
> newfile.txt

# 내용과 함께 생성
echo "Hello World" > file.txt

# 추가 (append)
echo "New line" >> file.txt

# 여러 줄
cat > multiline.txt << EOF
Line 1
Line 2
Line 3
EOF

📁 디렉토리 생성​

mkdir - 디렉토리 만들기​

# 단일 디렉토리
mkdir project

# 여러 디렉토리
mkdir dir1 dir2 dir3

# 중첩 디렉토리 (부모 디렉토리 포함)
mkdir -p parent/child/grandchild

# 권한 지정하며 생성
mkdir -m 755 public_dir

프로젝트 구조 한 번에 만들기​

# 프로젝트 디렉토리 구조
mkdir -p project/{src,tests,docs,config}
mkdir -p project/src/{components,utils,api}

# 확인
tree project
project/
├── config/
├── docs/
├── src/
│   ├── api/
│   ├── components/
│   └── utils/
└── tests/

📋 복사 (Copy)​

cp - 파일 복사​

# 파일 복사
cp source.txt destination.txt

# 디렉토리로 복사
cp file.txt ~/Documents/

# 여러 파일 복사
cp file1.txt file2.txt file3.txt ~/backup/

# 디렉토리 복사 (재귀적)
cp -r source_dir/ dest_dir/

# 권한 유지
cp -p file.txt backup.txt

# 확인 후 덮어쓰기
cp -i file.txt existing.txt

# 강제 덮어쓰기
cp -f file.txt existing.txt

# verbose (진행 상황 표시)
cp -v file.txt backup.txt

실전 예제​

# 백업 생성
cp important.txt important.txt.backup

# 날짜 포함 백업
cp config.json config.json.$(date +%Y%m%d)

# 전체 프로젝트 백업
cp -r ~/project ~/project.backup

# 특정 파일만 복사
cp ~/project/*.js ~/backup/

🚚 이동 및 이름 변경 (Move)​

mv - 파일 이동/이름 변경​

# 이름 변경
mv oldname.txt newname.txt

# 디렉토리로 이동
mv file.txt ~/Documents/

# 여러 파일 이동
mv *.txt ~/Documents/

# 디렉토리 이름 변경
mv old_dir new_dir

# 확인 후 이동
mv -i source.txt destination.txt

# verbose
mv -v file.txt new_location/

실전 예제​

# 파일 확장자 변경
mv document.md document.txt

# 공백 제거
mv "file with spaces.txt" file_without_spaces.txt

# 접두사 추가
for file in *.txt; do
  mv "$file" "backup_$file"
done

# 날짜 접두사 추가
for file in *.log; do
  mv "$file" "$(date +%Y%m%d)_$file"
done

🗑️ 삭제 (Remove)​

rm - 파일 삭제​

# 파일 삭제
rm file.txt

# 여러 파일
rm file1.txt file2.txt file3.txt

# 확인 후 삭제 (안전!)
rm -i file.txt

# 강제 삭제
rm -f file.txt

# 디렉토리 삭제 (재귀적)
rm -r directory/

# 디렉토리 강제 삭제 ⚠️ 주의!
rm -rf directory/

# verbose
rm -v file.txt

안전한 삭제​

# 항상 -i 사용 (확인)
rm -i *.txt

# 또는 alias 설정
alias rm='rm -i'

# 휴지통으로 이동 (삭제 대신)
# trash-cli 설치 필요
sudo apt install trash-cli
trash file.txt  # 복구 가능!

rmdir - 빈 디렉토리 삭제​

# 빈 디렉토리만 삭제
rmdir empty_dir/

# 부모 디렉토리까지 (모두 비어있으면)
rmdir -p parent/child/grandchild/

⚠️ 위험한 명령어​

절대 실행하지 마세요!

# ❌ 모든 것 삭제!
rm -rf /

# ❌ 홈 디렉토리 전체 삭제
rm -rf ~

# ❌ 현재 디렉토리의 모든 것
rm -rf *

# ❌ 숨김 파일 포함 모든 것
rm -rf .*

🔗 링크 생성​

ln - 하드 링크와 심볼릭 링크​

# 심볼릭 링크 (바로가기와 유사)
ln -s /path/to/original /path/to/link

# 예: 설정 파일 링크
ln -s ~/project/config.json ~/.config/myapp/config.json

# 상대 경로 심볼릭 링크
ln -s ../shared/utils.js ./utils.js

# 하드 링크 (같은 파일의 다른 이름)
ln original.txt hardlink.txt

# 링크 확인
ls -l link_name

실전 사용 예​

# Node.js 전역 링크
cd ~/my-package
npm link  # 심볼릭 링크 생성

# Python 가상환경 활성화 링크
ln -s ~/venv/bin/activate ~/activate

# 공유 리소스 링크
ln -s /mnt/c/Users/Shared ~/shared

🔍 파일 찾기 및 조작​

find + xargs 조합​

# .log 파일 모두 삭제
find . -name "*.log" -type f -delete

# 또는 확인 후 삭제
find . -name "*.log" -type f -exec rm -i {} \;

# 30일 이상 된 파일 삭제
find /tmp -type f -mtime +30 -delete

# 빈 디렉토리 삭제
find . -type d -empty -delete

# 특정 파일 복사
find . -name "*.conf" -exec cp {} ~/backup/ \;

🎯 실전 연습​

연습 1: 프로젝트 설정​

# 1. 프로젝트 구조 생성
mkdir -p ~/myproject/{src,tests,docs,build}
cd ~/myproject

# 2. 파일 생성
touch README.md
touch src/{app.js,utils.js,config.js}
touch tests/{app.test.js,utils.test.js}

# 3. 구조 확인
tree

연습 2: 백업 및 정리​

# 1. 테스트 파일 생성
mkdir ~/practice
cd ~/practice
touch file{1..10}.txt

# 2. 백업 디렉토리 생성 및 복사
mkdir backup
cp *.txt backup/

# 3. 일부 파일 이름 변경
for file in file{1..5}.txt; do
  mv "$file" "old_$file"
done

# 4. 특정 패턴 파일 삭제
rm -i old_file1.txt

# 5. 정리
rm -rf ~/practice

연습 3: 대량 파일 조작​

# 1. 테스트 파일 생성
mkdir ~/bulk-test
cd ~/bulk-test
touch photo_{1..100}.jpg

# 2. 디렉토리 분류
mkdir {2023,2024}

# 3. 파일 분배
mv photo_{1..50}.jpg 2023/
mv photo_{51..100}.jpg 2024/

# 4. 이름 일괄 변경
cd 2024
for file in photo_*.jpg; do
  mv "$file" "2024_$file"
done

# 5. 확인
ls

💡 프로 팁​

1. 와일드카드 마스터​

# * : 0개 이상의 문자
ls *.txt

# ? : 정확히 1개의 문자
ls file?.txt  # file1.txt, fileA.txt

# [] : 문자 범위
ls file[1-5].txt  # file1.txt ~ file5.txt
ls file[abc].txt  # filea.txt, fileb.txt, filec.txt

# {} : 확장
cp file.txt{,.backup}  # file.txt를 file.txt.backup으로
mkdir {2020..2024}  # 2020, 2021, 2022, 2023, 2024 생성

2. 안전한 조작​

# .bashrc에 추가
alias rm='rm -i'
alias cp='cp -i'
alias mv='mv -i'

# 또는 trash 사용
alias del='trash'  # trash-cli 필요

3. 파일 이름 일괄 변경​

# rename 설치
sudo apt install rename

# 사용
rename 's/old/new/' *.txt
rename 's/\.jpeg$/.jpg/' *

# 예: 공백을 언더스코어로
rename 's/ /_/g' *

4. 병렬 처리​

# parallel 설치
sudo apt install parallel

# 병렬로 압축
parallel gzip ::: *.log

# 병렬로 이미지 변환
parallel convert {} {.}.png ::: *.jpg
