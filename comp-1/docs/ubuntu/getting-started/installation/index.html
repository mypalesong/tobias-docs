WSL 2 설치하기

WSL 2와 Ubuntu를 설치하는 완벽한 가이드입니다. 단계별로 따라하면 10분 안에 설치 완료!

✅ 시스템 요구사항 확인​

설치하기 전에 시스템이 WSL 2를 지원하는지 확인하세요.

필수 요구사항​

  • Windows 10 버전 2004 (Build 19041) 이상
    • 또는 Windows 11
  • 64비트 시스템
  • 가상화 기능 활성화 (BIOS 설정)

Windows 버전 확인​

# PowerShell에서 실행
winver

또는:

# 상세 정보 확인
systeminfo | findstr /B /C:"OS Name" /C:"OS Version"

결과 예시:

OS Name:       Microsoft Windows 11 Pro
OS Version:    10.0.22621 Build 22621

가상화 지원 확인​

# 가상화가 활성화되어 있는지 확인
systeminfo | findstr /C:"Hyper-V"

만약 "Hyper-V Requirements: A hypervisor has been detected" 메시지가 보이면 준비 완료!

🚀 방법 1: 간편 설치 (권장)​

Windows 11 또는 Windows 10 (2004 이상)​

가장 쉬운 방법입니다!

1단계: PowerShell 관리자 권한으로 실행​

  1. 시작 메뉴에서 "PowerShell" 검색
  2. 우클릭 → "관리자 권한으로 실행"

2단계: WSL 설치 명령 실행​

wsl --install

이 명령은 자동으로:

  • ✅ WSL 기능 활성화
  • ✅ 가상 머신 플랫폼 활성화
  • ✅ Linux 커널 다운로드 및 설치
  • ✅ WSL 2를 기본 버전으로 설정
  • ✅ Ubuntu 배포판 설치

3단계: 재부팅​

# 설치 완료 후
Restart-Computer

4단계: Ubuntu 초기 설정​

재부팅 후 Ubuntu가 자동으로 실행됩니다.

Installing, this may take a few minutes...
Please create a default UNIX user account.
Username: your-username
Password:

중요 사항:

  • 사용자 이름은 소문자, 숫자, 하이픈만 사용
  • 비밀번호 입력 시 화면에 표시되지 않음 (정상)
  • 비밀번호는 잊지 마세요! (sudo 명령에 필요)

완료! 🎉

🔧 방법 2: 수동 설치​

자동 설치가 작동하지 않거나 더 세밀한 제어가 필요한 경우:

1단계: WSL 활성화​

# PowerShell 관리자 권한으로
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

2단계: 가상 머신 플랫폼 활성화​

dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

3단계: 재부팅​

Restart-Computer

4단계: Linux 커널 업데이트 패키지 다운로드​

WSL 2 Linux 커널 업데이트 패키지 다운로드

다운로드한 .msi 파일 실행 및 설치

5단계: WSL 2를 기본 버전으로 설정​

wsl --set-default-version 2

6단계: Linux 배포판 설치​

Microsoft Store에서 설치 (권장)​

  1. Microsoft Store 열기
  2. "Ubuntu" 검색
  3. Ubuntu 또는 Ubuntu 22.04 LTS 선택
  4. "설치" 클릭

명령줄로 설치​

# 사용 가능한 배포판 목록 확인
wsl --list --online

# Ubuntu 설치
wsl --install -d Ubuntu

7단계: Ubuntu 실행 및 초기 설정​

시작 메뉴에서 "Ubuntu" 실행

Username: your-username
Password: your-password

🎯 설치 확인​

WSL 버전 확인​

# 설치된 배포판 확인
wsl --list --verbose

출력 예시:

  NAME      STATE           VERSION
* Ubuntu    Running         2

VERSION 열이 2로 표시되어야 합니다!

Ubuntu에서 확인​

Ubuntu 터미널에서:

# Linux 버전 확인
lsb_release -a

# 커널 버전 확인
uname -r

출력 예시:

Description:    Ubuntu 22.04.3 LTS
Release:        22.04
Codename:       jammy

5.15.90.1-microsoft-standard-WSL2

간단한 테스트​

# 현재 디렉토리 확인
pwd

# 파일 목록 보기
ls -la

# 시스템 업데이트
sudo apt update

🔄 추가 배포판 설치​

여러 Linux 배포판을 동시에 사용할 수 있습니다!

사용 가능한 배포판 확인​

wsl --list --online

출력:

NAME               FRIENDLY NAME
Ubuntu             Ubuntu
Debian             Debian GNU/Linux
kali-linux         Kali Linux Rolling
Ubuntu-20.04       Ubuntu 20.04 LTS
Ubuntu-22.04       Ubuntu 22.04 LTS

배포판 설치​

# Debian 설치
wsl --install -d Debian

# Kali Linux 설치
wsl --install -d kali-linux

배포판 간 전환​

# 특정 배포판 실행
wsl -d Debian

# 기본 배포판 변경
wsl --set-default Debian

# 모든 배포판 확인
wsl --list --verbose

⚙️ 선택사항: Windows Terminal 설치​

Windows Terminal을 사용하면 훨씬 편리합니다!

설치 방법​

  1. Microsoft Store에서 "Windows Terminal" 검색
  2. 설치

장점​

  • 🎨 멋진 UI와 테마
  • 📑 탭 기능
  • 🎯 여러 셸 (PowerShell, CMD, WSL) 한 곳에서
  • ⚡ 빠르고 현대적인 터미널

사용법​

Windows Terminal 실행 후:

  • 새 탭: Ctrl + Shift + T
  • Ubuntu 열기: 드롭다운 메뉴에서 선택 또는 Ctrl + Shift + 숫자

🐛 문제 해결​

"WSL 2 requires an update to its kernel component"​

해결: WSL 2 커널 업데이트 다운로드 및 설치

"가상화를 사용할 수 없습니다"​

해결:

  1. BIOS 설정 진입 (재부팅 시 F2, F12, Del 등)
  2. "Virtualization Technology" 또는 "Intel VT-x" 찾기
  3. Enabled로 변경
  4. 저장 후 재부팅

"WslRegisterDistribution failed with error: 0x80370102"​

해결: Windows 기능에서 가상 머신 플랫폼 활성화:

  1. 제어판 → 프로그램 → Windows 기능 켜기/끄기
  2. "가상 머신 플랫폼" 체크
  3. 재부팅

Ubuntu가 시작되지 않음​

해결:

# WSL 재시작
wsl --shutdown

# 다시 시작
wsl

WSL 1로 설치됨 (WSL 2 원함)​

해결:

# WSL 2로 변환
wsl --set-version Ubuntu 2

# 기본 버전을 2로 설정
wsl --set-default-version 2
