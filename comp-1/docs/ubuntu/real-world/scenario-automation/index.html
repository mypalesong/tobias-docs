실전 시나리오: 자동화 및 Cron Job

매일 반복되는 작업을 자동화하여 시간을 절약하고 실수를 방지합니다.

📌 상황 설명​

DevOps 엔지니어 수진은 매일 아침 다음 작업을 수동으로 하고 있습니다:

  • 데이터베이스 백업
  • 로그 파일 정리
  • 시스템 상태 체크
  • 디스크 공간 확인
  • 리포트 이메일 발송

수진: "이거 자동화할 수 있지 않을까? 매일 같은 일을 반복하는 건 시간 낭비야!"

🎯 목표​

  • ✅ Shell 스크립트로 반복 작업 자동화
  • ✅ Cron으로 스케줄링
  • ✅ 에러 처리 및 로깅
  • ✅ 알림 및 모니터링

⚡ 실습​

1단계: 간단한 백업 스크립트​

backup.sh:

#!/bin/bash

# 백업 설정
BACKUP_DIR="/home/user/backups"
SOURCE_DIR="/home/user/important_data"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="backup_$DATE.tar.gz"

# 백업 디렉토리 생성
mkdir -p $BACKUP_DIR

# 백업 실행
echo "백업 시작: $(date)"
tar -czf "$BACKUP_DIR/$BACKUP_FILE" "$SOURCE_DIR"

# 결과 확인
if [ $? -eq 0 ]; then
    echo "백업 성공: $BACKUP_FILE"
else
    echo "백업 실패!"
    exit 1
fi

# 오래된 백업 삭제 (7일 이상)
find "$BACKUP_DIR" -name "backup_*.tar.gz" -mtime +7 -delete
echo "백업 완료: $(date)"
# 실행 권한 부여
chmod +x backup.sh

# 테스트 실행
./backup.sh

2단계: 데이터베이스 백업 스크립트​

db_backup.sh:

#!/bin/bash

# 설정
DB_USER="postgres"
DB_NAME="myapp"
BACKUP_DIR="/var/backups/postgresql"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/${DB_NAME}_$DATE.sql"
LOG_FILE="/var/log/db_backup.log"

# 로그 함수
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a $LOG_FILE
}

# 백업 디렉토리 생성
mkdir -p $BACKUP_DIR

log "데이터베이스 백업 시작"

# PostgreSQL 백업
pg_dump -U $DB_USER $DB_NAME > $BACKUP_FILE

if [ $? -eq 0 ]; then
    # 압축
    gzip $BACKUP_FILE
    log "백업 성공: ${BACKUP_FILE}.gz"

    # 파일 크기 확인
    SIZE=$(du -h "${BACKUP_FILE}.gz" | cut -f1)
    log "백업 파일 크기: $SIZE"

    # 30일 이상 된 백업 삭제
    OLD_COUNT=$(find $BACKUP_DIR -name "*.sql.gz" -mtime +30 | wc -l)
    find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete
    log "오래된 백업 $OLD_COUNT 개 삭제"
else
    log "❌ 백업 실패!"
    exit 1
fi

log "백업 완료"

3단계: 로그 정리 스크립트​

cleanup_logs.sh:

#!/bin/bash

LOG_DIR="/var/log/myapp"
RETENTION_DAYS=30

echo "로그 정리 시작: $(date)"

# 오래된 로그 파일 찾기
OLD_LOGS=$(find $LOG_DIR -name "*.log" -mtime +$RETENTION_DAYS)

if [ -z "$OLD_LOGS" ]; then
    echo "정리할 로그 없음"
else
    echo "정리할 로그:"
    echo "$OLD_LOGS"

    # 압축되지 않은 오래된 로그 압축
    find $LOG_DIR -name "*.log" -mtime +7 -mtime -$RETENTION_DAYS -exec gzip {} \;

    # 매우 오래된 로그 삭제
    find $LOG_DIR -name "*.log.gz" -mtime +$RETENTION_DAYS -delete

    echo "로그 정리 완료"
fi

# 디스크 사용량 확인
df -h $LOG_DIR

4단계: 시스템 상태 체크 스크립트​

health_check.sh:

#!/bin/bash

ALERT_EMAIL="admin@example.com"
LOG_FILE="/var/log/health_check.log"
ALERT=false

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a $LOG_FILE
}

alert() {
    ALERT=true
    log "⚠️  $1"
}

log "=== 시스템 상태 체크 시작 ==="

# 1. 디스크 사용량 체크 (80% 이상 경고)
DISK_USAGE=$(df / | tail -1 | awk '{print $5}' | sed 's/%//')
if [ $DISK_USAGE -gt 80 ]; then
    alert "디스크 사용량 높음: ${DISK_USAGE}%"
else
    log "✓ 디스크 사용량: ${DISK_USAGE}%"
fi

# 2. 메모리 사용량 체크
MEM_USAGE=$(free | grep Mem | awk '{print int($3/$2 * 100)}')
if [ $MEM_USAGE -gt 90 ]; then
    alert "메모리 사용량 높음: ${MEM_USAGE}%"
else
    log "✓ 메모리 사용량: ${MEM_USAGE}%"
fi

# 3. CPU 로드 체크
CPU_LOAD=$(uptime | awk -F'load average:' '{print $2}' | awk '{print $1}' | sed 's/,//')
CPU_CORES=$(nproc)
if (( $(echo "$CPU_LOAD > $CPU_CORES" | bc -l) )); then
    alert "CPU 로드 높음: $CPU_LOAD (코어: $CPU_CORES)"
else
    log "✓ CPU 로드: $CPU_LOAD"
fi

# 4. 주요 서비스 체크
SERVICES=("nginx" "postgresql" "redis-server")
for service in "${SERVICES[@]}"; do
    if systemctl is-active --quiet $service; then
        log "✓ $service 실행 중"
    else
        alert "$service 중지됨!"
    fi
done

# 5. 포트 체크
PORTS=(80 443 3000 5432)
for port in "${PORTS[@]}"; do
    if nc -z localhost $port 2>/dev/null; then
        log "✓ 포트 $port 열림"
    else
        alert "포트 $port 닫힘!"
    fi
done

# 알림 발송
if [ "$ALERT" = true ]; then
    log "⚠️  경고 발생! 이메일 발송"
    # 실제 환경에서는 mail 명령어 사용
    # cat $LOG_FILE | mail -s "시스템 경고" $ALERT_EMAIL
fi

log "=== 시스템 상태 체크 완료 ==="

5단계: Cron Job 설정​

# crontab 편집
crontab -e

Cron 스케줄 추가:

# 분 시 일 월 요일 명령어
# * * * * *

# 매일 새벽 2시에 데이터베이스 백업
0 2 * * * /home/user/scripts/db_backup.sh

# 매일 오전 3시에 백업
0 3 * * * /home/user/scripts/backup.sh

# 매일 오전 4시에 로그 정리
0 4 * * * /home/user/scripts/cleanup_logs.sh

# 매 시간마다 시스템 상태 체크
0 * * * * /home/user/scripts/health_check.sh

# 5분마다 디스크 사용량 체크
*/5 * * * * df -h / >> /var/log/disk_usage.log

# 매주 일요일 자정에 전체 시스템 백업
0 0 * * 0 /home/user/scripts/full_backup.sh

# 매월 1일에 월간 리포트 생성
0 0 1 * * /home/user/scripts/monthly_report.sh

Cron 시간 표현:

* * * * *
│ │ │ │ │
│ │ │ │ └─ 요일 (0-7, 0과 7은 일요일)
│ │ │ └─── 월 (1-12)
│ │ └───── 일 (1-31)
│ └─────── 시 (0-23)
└───────── 분 (0-59)

예시:
30 2 * * *        # 매일 02:30
0 */6 * * *       # 6시간마다
0 0 * * 1         # 매주 월요일 자정
0 0 1,15 * *      # 매월 1일과 15일
*/10 * * * *      # 10분마다
0 9-17 * * 1-5    # 평일 9시-17시 매 시간

7단계: 스크립트에 에러 처리 추가​

robust_backup.sh:

#!/bin/bash

set -euo pipefail  # 에러 발생 시 즉시 중단

# 설정
BACKUP_DIR="/home/user/backups"
SOURCE_DIR="/home/user/data"
LOG_FILE="/var/log/backup.log"
MAX_BACKUPS=7

# 에러 핸들러
error_exit() {
    echo "❌ 에러: $1" | tee -a $LOG_FILE
    # 알림 발송
    # echo "$1" | mail -s "백업 실패" admin@example.com
    exit 1
}

# 로그 함수
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a $LOG_FILE
}

# 디렉토리 확인
[ -d "$SOURCE_DIR" ] || error_exit "소스 디렉토리 없음: $SOURCE_DIR"
mkdir -p "$BACKUP_DIR" || error_exit "백업 디렉토리 생성 실패"

# 디스크 공간 확인
AVAILABLE_SPACE=$(df "$BACKUP_DIR" | tail -1 | awk '{print $4}')
REQUIRED_SPACE=$(du -s "$SOURCE_DIR" | awk '{print $1}')

if [ $AVAILABLE_SPACE -lt $REQUIRED_SPACE ]; then
    error_exit "디스크 공간 부족"
fi

# 백업 실행
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/backup_$DATE.tar.gz"

log "백업 시작"
tar -czf "$BACKUP_FILE" "$SOURCE_DIR" 2>&1 | tee -a $LOG_FILE

if [ ${PIPESTATUS[0]} -eq 0 ]; then
    SIZE=$(du -h "$BACKUP_FILE" | cut -f1)
    log "✓ 백업 성공: $BACKUP_FILE ($SIZE)"

    # 오래된 백업 삭제
    BACKUP_COUNT=$(ls -1 "$BACKUP_DIR"/backup_*.tar.gz 2>/dev/null | wc -l)
    if [ $BACKUP_COUNT -gt $MAX_BACKUPS ]; then
        ls -1t "$BACKUP_DIR"/backup_*.tar.gz | tail -n +$((MAX_BACKUPS+1)) | xargs rm -f
        log "오래된 백업 삭제 완료"
    fi
else
    error_exit "백업 실패"
fi

log "백업 프로세스 완료"

8단계: 병렬 실행 및 최적화​

parallel_backup.sh:

#!/bin/bash

# GNU parallel 설치
# sudo apt install parallel

# 여러 디렉토리 병렬 백업
DIRS=("dir1" "dir2" "dir3" "dir4")

backup_dir() {
    local dir=$1
    echo "백업 중: $dir"
    tar -czf "backup_${dir}_$(date +%Y%m%d).tar.gz" "$dir"
    echo "완료: $dir"
}

export -f backup_dir

# 병렬로 실행 (동시에 4개)
parallel -j 4 backup_dir ::: "${DIRS[@]}"

echo "모든 백업 완료"

9단계: 모니터링 대시보드​

dashboard.sh:

#!/bin/bash

while true; do
    clear
    echo "===================================="
    echo "   시스템 모니터링 대시보드"
    echo "===================================="
    echo "시간: $(date)"
    echo ""

    echo "📊 시스템 리소스:"
    echo "CPU: $(top -bn1 | grep "Cpu(s)" | awk '{print $2}')% 사용"
    echo "메모리: $(free -h | grep Mem | awk '{print $3 "/" $2}')"
    echo "디스크: $(df -h / | tail -1 | awk '{print $5 " 사용 (" $3 "/" $2 ")"}')"
    echo ""

    echo "🔄 주요 서비스:"
    for service in nginx postgresql redis-server; do
        if systemctl is-active --quiet $service; then
            echo "✓ $service"
        else
            echo "✗ $service (중지됨)"
        fi
    done
    echo ""

    echo "📝 최근 로그 (최근 5개):"
    tail -5 /var/log/syslog
    echo ""

    echo "다음 업데이트: 10초 후..."
    sleep 10
done

💡 프로 팁​

1. Cron 환경 변수​

# crontab 상단에 환경 변수 설정
SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin
MAILTO=admin@example.com

# 그 후 Cron Job
0 2 * * * /home/user/scripts/backup.sh

2. Cron 출력 로그​

# 표준 출력과 에러를 파일로
0 2 * * * /home/user/scripts/backup.sh >> /var/log/backup.log 2>&1

# 에러만 로그
0 2 * * * /home/user/scripts/backup.sh 2>> /var/log/backup_error.log

# 출력 무시
0 2 * * * /home/user/scripts/backup.sh > /dev/null 2>&1

3. 잠금 파일 (중복 실행 방지)​

#!/bin/bash

LOCKFILE=/var/run/backup.lock

# 잠금 파일 확인
if [ -f $LOCKFILE ]; then
    echo "이미 실행 중"
    exit 1
fi

# 잠금 파일 생성
touch $LOCKFILE

# 작업 실행
perform_backup

# 정리
rm -f $LOCKFILE

4. systemd 타이머 (Cron 대안)​

# /etc/systemd/system/backup.service
[Unit]
Description=Daily Backup

[Service]
Type=oneshot
ExecStart=/home/user/scripts/backup.sh
# /etc/systemd/system/backup.timer
[Unit]
Description=Daily Backup Timer

[Timer]
OnCalendar=daily
Persistent=true

[Install]
WantedBy=timers.target
# 활성화
sudo systemctl enable backup.timer
sudo systemctl start backup.timer

# 타이머 확인
systemctl list-timers
