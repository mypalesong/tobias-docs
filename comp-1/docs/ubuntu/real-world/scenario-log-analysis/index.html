실전 시나리오: 로그 분석 및 모니터링

서비스에서 에러가 발생했습니다! 로그를 분석하여 원인을 찾고 해결해야 합니다.

📌 상황 설명​

백엔드 개발자 지훈은 오전 9시에 출근하자마자 긴급 알림을 받았습니다.

"🚨 프로덕션 API 에러율 급증! 500 에러가 평소의 10배!"

지훈은 즉시 로그를 분석하여 문제를 찾아야 합니다.

🎯 목표​

  • ✅ 로그 파일 실시간 모니터링
  • ✅ 에러 패턴 및 원인 파악
  • ✅ 로그 필터링 및 통계 생성
  • ✅ 문제 해결 및 리포트 작성

⚡ 실습​

1단계: 로그 파일 확인​

# 로그 디렉토리 이동
cd /var/log

# 또는 애플리케이션 로그
cd ~/app/logs

# 로그 파일 목록
ls -lh

출력:

-rw-r--r-- 1 app app 2.1G Jan 15 09:00 app.log
-rw-r--r-- 1 app app 523M Jan 15 08:00 app.log.1
-rw-r--r-- 1 app app 487M Jan 14 23:59 app.log.2
-rw-r--r-- 1 app app  85M Jan 15 09:15 error.log

2단계: 실시간 로그 모니터링​

# 로그 실시간 감시 (tail -f)
tail -f app.log

# 마지막 100줄 보기
tail -n 100 app.log

# 여러 파일 동시 감시
tail -f app.log error.log

# 에러만 필터링하며 감시
tail -f app.log | grep ERROR

# 컬러 하이라이트 (grep --color)
tail -f app.log | grep --color=always -i error

3단계: 에러 로그 찾기​

로그 예시 (app.log):

2024-01-15 09:05:23 INFO  [main] Server started on port 3000
2024-01-15 09:05:45 ERROR [api] Database connection failed: timeout
2024-01-15 09:05:46 ERROR [api] Database connection failed: timeout
2024-01-15 09:05:47 WARN  [api] Retrying database connection (1/3)
2024-01-15 09:06:01 ERROR [api] Payment service returned 500
2024-01-15 09:06:15 ERROR [api] Database connection failed: timeout
# 모든 ERROR 찾기
grep ERROR app.log

# 케이스 무시 (error, Error, ERROR 모두)
grep -i error app.log

# 라인 번호 포함
grep -n ERROR app.log

# 파일명 표시 (여러 파일 검색)
grep -H ERROR *.log

# 매칭된 줄 개수
grep -c ERROR app.log

# 에러 주변 문맥 보기 (앞뒤 3줄)
grep -C 3 ERROR app.log
# 또는
grep -A 3 -B 3 ERROR app.log

4단계: 시간대별 에러 분석​

# 특정 시간 범위 에러 추출
grep "2024-01-15 09:0[5-9]" app.log | grep ERROR

# 오전 9시대 에러만
grep "2024-01-15 09:" app.log | grep ERROR

# 특정 분 (09:05)
grep "2024-01-15 09:05" app.log | grep ERROR

# 시간별 에러 카운트
grep ERROR app.log | cut -d' ' -f1,2 | cut -d':' -f1 | sort | uniq -c

출력:

    45 2024-01-15 09
    87 2024-01-15 10
    12 2024-01-15 11
     3 2024-01-15 12

5단계: 에러 패턴 분석​

# 에러 메시지별 그룹핑
grep ERROR app.log | awk '{print substr($0, index($0,$6))}' | sort | uniq -c | sort -rn

# 또는 더 간단히
grep ERROR app.log | cut -d']' -f3 | sort | uniq -c | sort -rn

출력:

    234 Database connection failed: timeout
     87 Payment service returned 500
     12 User not found
      5 Invalid token

지훈: "아하! 데이터베이스 연결 타임아웃이 가장 많네요!"

6단계: 특정 에러 상세 분석​

# "Database connection" 에러 전체 추출
grep "Database connection" app.log > db_errors.log

# 에러 발생 빈도 (분 단위)
grep "Database connection" app.log | cut -d' ' -f1,2 | cut -d':' -f1-2 | sort | uniq -c

# 패턴 매칭 (정규표현식)
grep -E "Database.*timeout|Connection.*refused" app.log

# 에러가 아닌 줄 (역 매칭)
grep -v ERROR app.log

7단계: 고급 로그 분석 (awk)​

# 에러 레벨별 통계
awk '{print $3}' app.log | sort | uniq -c

# API 엔드포인트별 에러
grep ERROR app.log | awk '{for(i=6;i<=NF;i++) printf "%s ", $i; print ""}' | grep -oP '/api/[^\s]+' | sort | uniq -c | sort -rn

# 시간대별 요청 수
awk '{print $2}' app.log | cut -d':' -f1 | sort | uniq -c

# 평균 응답 시간 계산 (로그에 응답 시간이 있다면)
grep "Response time" app.log | awk '{sum+=$NF; count++} END {print sum/count}'

8단계: 로그 압축 및 아카이빙​

# 에러 로그만 별도 저장
grep ERROR app.log > error_2024-01-15.log

# 날짜별로 분리
grep "2024-01-15" app.log > app_2024-01-15.log

# 압축
gzip app_2024-01-15.log

# 오래된 로그 찾기 및 압축
find /var/log/app -name "*.log" -mtime +7 -exec gzip {} \;

# 30일 이상 된 로그 삭제
find /var/log/app -name "*.log.gz" -mtime +30 -delete

9단계: 로그 리포트 생성​

analyze_logs.sh:

#!/bin/bash

LOG_FILE="app.log"
DATE=$(date +%Y-%m-%d)
REPORT="log_report_$DATE.txt"

echo "=== 로그 분석 리포트 ($DATE) ===" > $REPORT
echo "" >> $REPORT

echo "1. 총 로그 라인 수:" >> $REPORT
wc -l $LOG_FILE >> $REPORT
echo "" >> $REPORT

echo "2. 레벨별 통계:" >> $REPORT
awk '{print $3}' $LOG_FILE | sort | uniq -c | sort -rn >> $REPORT
echo "" >> $REPORT

echo "3. 에러 메시지 Top 10:" >> $REPORT
grep ERROR $LOG_FILE | cut -d']' -f3 | sort | uniq -c | sort -rn | head -10 >> $REPORT
echo "" >> $REPORT

echo "4. 시간대별 에러 수:" >> $REPORT
grep ERROR $LOG_FILE | cut -d' ' -f2 | cut -d':' -f1 | sort | uniq -c >> $REPORT
echo "" >> $REPORT

echo "5. 에러율:" >> $REPORT
TOTAL=$(wc -l < $LOG_FILE)
ERRORS=$(grep -c ERROR $LOG_FILE)
ERROR_RATE=$(echo "scale=2; $ERRORS * 100 / $TOTAL" | bc)
echo "총 $TOTAL 라인 중 $ERRORS 에러 ($ERROR_RATE%)" >> $REPORT

echo "리포트 생성 완료: $REPORT"
cat $REPORT
chmod +x analyze_logs.sh
./analyze_logs.sh

10단계: 실시간 대시보드​

monitor_logs.sh:

#!/bin/bash

LOG_FILE="app.log"

while true; do
  clear
  echo "=== 실시간 로그 모니터링 ==="
  echo "시간: $(date)"
  echo ""

  echo "최근 1분간 에러 수:"
  tail -n 1000 $LOG_FILE | grep ERROR | wc -l
  echo ""

  echo "최근 에러 Top 5:"
  tail -n 1000 $LOG_FILE | grep ERROR | cut -d']' -f3 | sort | uniq -c | sort -rn | head -5
  echo ""

  echo "최근 10개 에러:"
  tail -n 20 $LOG_FILE | grep ERROR | tail -10

  sleep 5
done
chmod +x monitor_logs.sh
./monitor_logs.sh

✅ 문제 해결​

지훈의 분석 결과:​

# 1. 에러 패턴 확인
grep "Database connection failed" app.log | wc -l
# 출력: 234

# 2. 발생 시간 분석
grep "Database connection failed" app.log | cut -d' ' -f2 | cut -d':' -f1-2 | sort | uniq -c

발견:

  • 09:05부터 급증
  • 데이터베이스 연결 풀 고갈 추정

조치:

# 데이터베이스 연결 설정 확인
cat config/database.yml

# 연결 풀 크기 확인
grep pool_size config/database.yml

# 설정 변경 (pool_size 증가)
sed -i 's/pool_size: 10/pool_size: 50/' config/database.yml

# 애플리케이션 재시작
sudo systemctl restart myapp

# 로그 확인
tail -f app.log | grep -i database

💡 프로 팁​

1. 로그 모니터링 도구​

# multitail - 여러 로그 동시 감시
sudo apt install multitail
multitail app.log error.log access.log

# lnav - 로그 네비게이터 (추천!)
sudo apt install lnav
lnav app.log

# ccze - 컬러 로그 뷰어
tail -f app.log | ccze -A

2. 로그 필터링 alias​

~/.bashrc:

alias errors='grep -i error'
alias warnings='grep -i warn'
alias today='grep $(date +%Y-%m-%d)'
alias logs='cd /var/log && ls -lht | head'

3. jq로 JSON 로그 파싱​

# JSON 로그 파싱
cat app.log | jq '.level, .message'

# 에러만 필터링
cat app.log | jq 'select(.level=="error")'

# 특정 필드 추출
cat app.log | jq -r '.timestamp + " " + .message'

4. 로그 로테이션​

/etc/logrotate.d/myapp:

/var/log/myapp/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    create 0640 app app
    sharedscripts
    postrotate
        systemctl reload myapp > /dev/null
    endscript
}
