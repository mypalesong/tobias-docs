systemd 완벽 가이드

현대 Linux의 핵심인 systemd로 서비스를 완벽하게 관리하세요.

🎯 목표​

  • ✅ systemd 이해
  • ✅ systemctl로 서비스 관리
  • ✅ Unit 파일 작성
  • ✅ 타이머로 작업 스케줄링
  • ✅ 로그 관리 (journald)

📖 systemd란?​

systemd는 Linux의 init 시스템이자 서비스 매니저입니다.

주요 기능​

  • 🚀 빠른 부팅: 병렬로 서비스 시작
  • 🔄 서비스 관리: 시작, 중지, 재시작
  • 📝 로깅: journald로 통합 로그 관리
  • ⏰ 스케줄링: 타이머로 cron 대체
  • 🔗 의존성 관리: 서비스 간 의존성 자동 처리

🔧 systemctl - 서비스 제어​

기본 명령어​

# 서비스 상태 확인
systemctl status nginx

# 서비스 시작
sudo systemctl start nginx

# 서비스 중지
sudo systemctl stop nginx

# 서비스 재시작
sudo systemctl restart nginx

# 설정 리로드 (재시작 없이)
sudo systemctl reload nginx

# 부팅 시 자동 시작 활성화
sudo systemctl enable nginx

# 부팅 시 자동 시작 비활성화
sudo systemctl disable nginx

# 활성화 + 시작 (한 번에)
sudo systemctl enable --now nginx

서비스 정보 확인​

# 모든 서비스 목록
systemctl list-units --type=service

# 실행 중인 서비스만
systemctl list-units --type=service --state=running

# 실패한 서비스
systemctl list-units --type=service --state=failed

# 부팅 시 활성화된 서비스
systemctl list-unit-files --type=service --state=enabled

# 특정 서비스 의존성 확인
systemctl list-dependencies nginx

# 서비스 설정 파일 위치 확인
systemctl cat nginx

시스템 제어​

# 시스템 재부팅
sudo systemctl reboot

# 시스템 종료
sudo systemctl poweroff

# 데몬 리로드 (unit 파일 변경 후)
sudo systemctl daemon-reload

# 시스템 상태 확인
systemctl status

# 부팅 시간 분석
systemd-analyze
systemd-analyze blame
systemd-analyze critical-chain

📝 Unit 파일 작성​

Unit 파일 위치​

# 시스템 unit 파일
/etc/systemd/system/       # 사용자 정의 (우선순위 높음)
/lib/systemd/system/       # 패키지가 설치한 파일
/usr/lib/systemd/system/   # 배포판 기본 파일

간단한 서비스 예제​

/etc/systemd/system/myapp.service:

[Unit]
Description=My Application
After=network.target

[Service]
Type=simple
User=myuser
WorkingDirectory=/home/myuser/myapp
ExecStart=/home/myuser/myapp/start.sh
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
# 서비스 활성화 및 시작
sudo systemctl daemon-reload
sudo systemctl enable myapp
sudo systemctl start myapp

# 상태 확인
systemctl status myapp

Node.js 앱 서비스 예제​

/etc/systemd/system/nodejs-app.service:

[Unit]
Description=Node.js Application
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/myapp
Environment="NODE_ENV=production"
Environment="PORT=3000"
ExecStart=/usr/bin/node /var/www/myapp/server.js
Restart=on-failure
RestartSec=10
StandardOutput=syslog
StandardError=syslog
SyslogIdentifier=nodejs-app

[Install]
WantedBy=multi-user.target

Python FastAPI 서비스​

/etc/systemd/system/fastapi-app.service:

[Unit]
Description=FastAPI Application
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/var/www/api
Environment="PATH=/var/www/api/venv/bin"
ExecStart=/var/www/api/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target

Unit 파일 섹션 설명​

[Unit] 섹션​

[Unit]
Description=서비스 설명
Documentation=https://example.com/docs
After=network.target postgresql.service  # 이 서비스들 다음에 시작
Before=nginx.service                     # 이 서비스 전에 시작
Requires=postgresql.service              # 필수 의존성
Wants=redis.service                      # 선택적 의존성

[Service] 섹션​

[Service]
Type=simple              # 가장 일반적
# Type=forking          # 데몬 프로세스
# Type=oneshot          # 한 번 실행 후 종료
# Type=notify           # 준비 완료 시 systemd에 알림

User=username
Group=groupname
WorkingDirectory=/path/to/app

ExecStart=/path/to/binary         # 시작 명령어
ExecReload=/bin/kill -HUP $MAINPID  # 리로드 명령어
ExecStop=/path/to/stop-script     # 중지 명령어

# 재시작 정책
Restart=always          # 항상 재시작
# Restart=on-failure   # 실패 시만
# Restart=on-abnormal  # 비정상 종료 시만
RestartSec=10          # 재시작 대기 시간

# 환경 변수
Environment="VAR1=value1"
Environment="VAR2=value2"
EnvironmentFile=/etc/myapp/env

# 리소스 제한
MemoryLimit=512M
CPUQuota=50%

[Install] 섹션​

[Install]
WantedBy=multi-user.target  # 일반적인 대상
# WantedBy=graphical.target  # GUI 환경

⏰ systemd 타이머 (Cron 대안)​

타이머 생성​

/etc/systemd/system/backup.service:

[Unit]
Description=Backup Service

[Service]
Type=oneshot
ExecStart=/usr/local/bin/backup.sh

/etc/systemd/system/backup.timer:

[Unit]
Description=Daily Backup Timer

[Timer]
OnCalendar=daily
OnCalendar=*-*-* 02:00:00  # 매일 02:00
Persistent=true  # 놓친 실행도 수행

[Install]
WantedBy=timers.target
# 타이머 활성화 및 시작
sudo systemctl enable backup.timer
sudo systemctl start backup.timer

# 타이머 목록 확인
systemctl list-timers

# 타이머 상태 확인
systemctl status backup.timer

타이머 스케줄 예제​

# 매 시간
OnCalendar=hourly

# 매일 자정
OnCalendar=daily

# 매주 월요일
OnCalendar=weekly
OnCalendar=Mon *-*-* 00:00:00

# 매월 1일
OnCalendar=monthly
OnCalendar=*-*-01 00:00:00

# 특정 시간
OnCalendar=*-*-* 14:30:00

# 여러 시간
OnCalendar=Mon,Tue,Wed 10:00:00

# 부팅 후 15분
OnBootSec=15min

# 마지막 실행 후 1시간
OnUnitActiveSec=1h

📋 journald - 로그 관리​

로그 확인​

# 모든 로그
journalctl

# 최근 로그 (페이지 단위)
journalctl -e

# 실시간 로그 (tail -f처럼)
journalctl -f

# 특정 서비스 로그
journalctl -u nginx

# 특정 서비스 실시간 로그
journalctl -u nginx -f

# 특정 기간
journalctl --since "2024-01-15"
journalctl --since "1 hour ago"
journalctl --since "2024-01-15 10:00" --until "2024-01-15 11:00"

# 우선순위별
journalctl -p err         # 에러 이상
journalctl -p warning     # 경고 이상

# 부팅별 로그
journalctl --list-boots   # 부팅 목록
journalctl -b             # 현재 부팅
journalctl -b -1          # 이전 부팅

# 커널 메시지만
journalctl -k

# JSON 형식
journalctl -o json

# 디스크 사용량
journalctl --disk-usage

# 로그 정리 (30일 이전)
sudo journalctl --vacuum-time=30d

# 크기 제한 (1GB)
sudo journalctl --vacuum-size=1G

journald 설정​

/etc/systemd/journald.conf:

[Journal]
Storage=persistent        # 디스크에 저장
SystemMaxUse=500M        # 최대 500MB 사용
SystemKeepFree=1G        # 1GB는 여유 공간으로
MaxRetentionSec=1month   # 1개월 보관
# 설정 적용
sudo systemctl restart systemd-journald

🎯 실전 예제​

1. Node.js 앱 자동 배포 서비스​

/etc/systemd/system/api-server.service:

[Unit]
Description=API Server
After=network.target

[Service]
Type=simple
User=api
Group=api
WorkingDirectory=/opt/api-server
Environment="NODE_ENV=production"
Environment="PORT=3000"
ExecStartPre=/usr/bin/npm install
ExecStart=/usr/bin/node server.js
Restart=always
RestartSec=10

# 건강 체크
ExecStartPost=/bin/sleep 5
ExecStartPost=/usr/bin/curl -f http://localhost:3000/health || exit 1

# 리소스 제한
MemoryLimit=512M
CPUQuota=80%

# 로깅
StandardOutput=journal
StandardError=journal
SyslogIdentifier=api-server

[Install]
WantedBy=multi-user.target

2. 데이터베이스 백업 타이머​

/etc/systemd/system/db-backup.service:

[Unit]
Description=PostgreSQL Backup

[Service]
Type=oneshot
User=postgres
ExecStart=/usr/local/bin/pg-backup.sh

/etc/systemd/system/db-backup.timer:

[Unit]
Description=Daily PostgreSQL Backup at 2 AM

[Timer]
OnCalendar=*-*-* 02:00:00
Persistent=true

[Install]
WantedBy=timers.target

/usr/local/bin/pg-backup.sh:

#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=/backups/postgresql
mkdir -p $BACKUP_DIR

pg_dump mydb > $BACKUP_DIR/mydb_$DATE.sql
gzip $BACKUP_DIR/mydb_$DATE.sql

# 30일 이상 된 백업 삭제
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete

echo "Backup completed: mydb_$DATE.sql.gz"
chmod +x /usr/local/bin/pg-backup.sh
sudo systemctl enable db-backup.timer
sudo systemctl start db-backup.timer

3. 웹 서버 자동 재시작 (감시)​

/etc/systemd/system/website-monitor.service:

[Unit]
Description=Website Monitor

[Service]
Type=oneshot
ExecStart=/usr/local/bin/check-website.sh

/etc/systemd/system/website-monitor.timer:

[Unit]
Description=Check Website Every 5 Minutes

[Timer]
OnBootSec=5min
OnUnitActiveSec=5min

[Install]
WantedBy=timers.target

/usr/local/bin/check-website.sh:

#!/bin/bash
if ! curl -f http://localhost 2>/dev/null; then
    systemctl restart nginx
    echo "Nginx restarted at $(date)" >> /var/log/nginx-monitor.log
fi

💡 프로 팁​

1. 서비스 디버깅​

# 상세 로그
journalctl -u myapp -xe

# 마지막 실행부터
journalctl -u myapp -f

# 설정 검증
systemd-analyze verify myapp.service

2. 빠른 재시작​

# 설정 변경 시 (daemon-reload + restart)
sudo systemctl restart myapp

# 또는
sudo systemctl daemon-reload && sudo systemctl restart myapp

3. 템플릿 서비스​

여러 인스턴스 실행:

/etc/systemd/system/worker@.service:

[Unit]
Description=Worker %i

[Service]
Type=simple
ExecStart=/usr/bin/worker --id %i

[Install]
WantedBy=multi-user.target
# 여러 인스턴스 실행
sudo systemctl start worker@1
sudo systemctl start worker@2
sudo systemctl start worker@3
