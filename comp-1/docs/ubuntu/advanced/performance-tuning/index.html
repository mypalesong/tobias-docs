성능 모니터링 및 튜닝

시스템 병목을 찾고, 리소스를 최적화하여 성능을 극대화하세요.

🎯 목표​

  • ✅ 시스템 리소스 모니터링 (CPU, 메모리, 디스크, 네트워크)
  • ✅ 병목 현상 찾기
  • ✅ 프로세스 우선순위 조정
  • ✅ 커널 파라미터 튜닝
  • ✅ 성능 벤치마킹

📊 CPU 모니터링​

top - 실시간 모니터링​

# 기본 실행
top

# 특정 사용자 프로세스만
top -u username

# CPU별로 표시
top (실행 후 '1' 키)

# 메모리 사용량 순 정렬
top (실행 후 'M' 키)

# CPU 사용량 순 정렬
top (실행 후 'P' 키)

# 특정 프로세스 모니터링
top -p 1234,5678

주요 정보:

top - 10:30:15 up 5 days, 2:15, 3 users, load average: 0.52, 0.58, 0.59
Tasks: 245 total,   1 running, 244 sleeping,   0 stopped,   0 zombie
%Cpu(s): 12.5 us,  3.2 sy,  0.0 ni, 84.0 id,  0.3 wa,  0.0 hi,  0.0 si
MiB Mem :  15872.2 total,   2048.5 free,   8234.1 used,   5589.6 buff/cache

htop - 향상된 top​

# 설치
sudo apt install htop

# 실행
htop

단축키:

  • F5: 트리 뷰
  • F6: 정렬 기준 변경
  • F9: 프로세스 종료
  • F10: 종료

프로세스 확인​

# 모든 프로세스
ps aux

# CPU 사용률 순
ps aux --sort=-%cpu | head -10

# 메모리 사용률 순
ps aux --sort=-%mem | head -10

# 특정 프로세스 찾기
ps aux | grep nginx

# 프로세스 트리
pstree

# 실시간 프로세스 통계
pidstat 1

CPU 사용률 측정​

# CPU 정보
lscpu

# CPU별 사용률
mpstat -P ALL 1

# 프로세스별 CPU 사용률
pidstat -u 1

💾 메모리 모니터링​

메모리 사용량 확인​

# 기본 메모리 정보
free -h

# 상세 정보
cat /proc/meminfo

# 메모리 통계 (1초 간격)
vmstat 1

# 프로세스별 메모리 사용
ps aux --sort=-%mem | head -10

free 출력 해석:

free -h
              total        used        free      shared  buff/cache   available
Mem:           15Gi       8.0Gi       1.0Gi       200Mi       6.0Gi       7.0Gi
Swap:         2.0Gi       100Mi       1.9Gi
  • total: 전체 메모리
  • used: 실제 사용 중
  • free: 완전히 비어있음
  • available: 사용 가능 (buff/cache 포함)
  • buff/cache: 캐시 (필요 시 해제 가능)

메모리 누수 찾기​

# 시간별 메모리 증가 추적
watch -n 1 'ps aux --sort=-%mem | head -10'

# 프로세스 메모리 상세
pmap 1234

# 메모리 매핑 확인
cat /proc/1234/maps

Swap 확인​

# Swap 사용량
swapon --show

# Swap 통계
vmstat -s

# Swappiness 확인 (0-100, 낮을수록 RAM 선호)
cat /proc/sys/vm/swappiness

💿 디스크 I/O 모니터링​

디스크 사용량​

# 파일 시스템 사용량
df -h

# 디렉토리 크기
du -sh /var/log
du -h --max-depth=1 /var

# 큰 파일 찾기
du -ah /home | sort -rh | head -20

# ncdu (대화형 디스크 사용량)
sudo apt install ncdu
ncdu /home

I/O 성능 측정​

# iostat 설치
sudo apt install sysstat

# 디스크 I/O 통계
iostat -x 1

# 프로세스별 I/O
sudo iotop

# 블록 디바이스 I/O
pidstat -d 1

iostat 해석:

Device    r/s    w/s    rkB/s    wkB/s  %util
sda     10.2   25.3    204.5    506.2   45.2
  • r/s, w/s: 초당 읽기/쓰기 횟수
  • rkB/s, wkB/s: 초당 읽기/쓰기 KB
  • %util: 디바이스 사용률 (100%면 병목)

디스크 벤치마크​

# 쓰기 속도 테스트
dd if=/dev/zero of=testfile bs=1G count=1 oflag=direct

# 읽기 속도 테스트
dd if=testfile of=/dev/null bs=1G count=1 iflag=direct

# 랜덤 I/O (fio)
sudo apt install fio
fio --name=random-read --ioengine=libaio --iodepth=16 --rw=randread --bs=4k --size=1G

🌐 네트워크 모니터링​

네트워크 사용량​

# 네트워크 통계
ifstat

# 실시간 대역폭
iftop

# 프로세스별 네트워크 사용
nethogs

# 연결 상태
netstat -tuln
ss -tuln

# 연결 수
ss -s

네트워크 성능 측정​

# 대역폭 테스트 (iperf3)
sudo apt install iperf3

# 서버 모드
iperf3 -s

# 클라이언트 모드
iperf3 -c server-ip

# Ping 테스트
ping -c 100 8.8.8.8

# 경로 추적
traceroute google.com
mtr google.com

⚙️ 프로세스 우선순위 조정​

nice - 시작 시 우선순위 설정​

# nice 값: -20 (최고) ~ 19 (최저), 기본 0

# 낮은 우선순위로 실행
nice -n 10 ./cpu-intensive-task.sh

# 높은 우선순위 (root 필요)
sudo nice -n -10 ./important-task.sh

renice - 실행 중인 프로세스​

# PID로 우선순위 변경
sudo renice -n -5 -p 1234

# 사용자의 모든 프로세스
sudo renice -n 10 -u username

# 프로세스 그룹
sudo renice -n 5 -g 5678

ionice - I/O 우선순위​

# I/O 우선순위 클래스
# 0: none (기본)
# 1: realtime (최우선)
# 2: best-effort (기본)
# 3: idle (유휴 시간에만)

# Best-effort 클래스, 우선순위 4
ionice -c 2 -n 4 ./backup.sh

# Idle 클래스 (백그라운드 작업에 적합)
ionice -c 3 ./low-priority-job.sh

🔧 커널 파라미터 튜닝​

sysctl - 런타임 커널 파라미터​

# 현재 설정 확인
sysctl -a

# 특정 파라미터 확인
sysctl vm.swappiness

# 임시 변경
sudo sysctl -w vm.swappiness=10

# 영구 변경
sudo nano /etc/sysctl.conf
# 파일에 추가: vm.swappiness=10

# 설정 적용
sudo sysctl -p

성능 관련 주요 파라미터​

메모리:

# Swappiness (0-100, 낮을수록 RAM 선호)
vm.swappiness=10

# Dirty page 임계값
vm.dirty_ratio=15
vm.dirty_background_ratio=5

# 캐시 압력
vm.vfs_cache_pressure=50

네트워크:

# TCP 버퍼 크기
net.core.rmem_max=16777216
net.core.wmem_max=16777216
net.ipv4.tcp_rmem=4096 87380 16777216
net.ipv4.tcp_wmem=4096 65536 16777216

# 연결 대기 큐 크기
net.core.somaxconn=65535
net.ipv4.tcp_max_syn_backlog=8192

# TIME_WAIT 소켓 재사용
net.ipv4.tcp_tw_reuse=1

# 파일 디스크립터
fs.file-max=2097152

파일 시스템:

# inotify 감시 한도 (파일 감시)
fs.inotify.max_user_watches=524288

# 파일 디스크립터 한도
fs.file-max=2097152

/etc/sysctl.conf 예제​

# 메모리 최적화
vm.swappiness=10
vm.dirty_ratio=15
vm.dirty_background_ratio=5

# 네트워크 최적화
net.core.somaxconn=65535
net.ipv4.tcp_max_syn_backlog=8192
net.ipv4.tcp_tw_reuse=1

# 파일 시스템
fs.file-max=2097152
fs.inotify.max_user_watches=524288

📈 벤치마킹 도구​

stress - 시스템 스트레스 테스트​

# 설치
sudo apt install stress

# CPU 스트레스 (4개 워커, 60초)
stress --cpu 4 --timeout 60

# 메모리 스트레스 (1GB)
stress --vm 2 --vm-bytes 512M --timeout 60

# I/O 스트레스
stress --io 4 --timeout 60

# 복합 테스트
stress --cpu 4 --vm 2 --vm-bytes 512M --io 2 --timeout 120

sysbench - 종합 벤치마크​

# 설치
sudo apt install sysbench

# CPU 벤치마크
sysbench cpu --threads=4 run

# 메모리 벤치마크
sysbench memory --memory-total-size=10G run

# 파일 I/O 벤치마크
sysbench fileio --file-total-size=2G prepare
sysbench fileio --file-total-size=2G --file-test-mode=rndrw run
sysbench fileio cleanup

🎯 실전 성능 최적화 시나리오​

시나리오: 웹 서버 느림​

1단계: 현재 상태 확인

# CPU 확인
top
htop

# 메모리 확인
free -h
vmstat 1

# 디스크 I/O
iostat -x 1

# 네트워크
iftop

2단계: 병목 찾기

# CPU 100%?
ps aux --sort=-%cpu | head -10

# 메모리 부족?
ps aux --sort=-%mem | head -10
free -h

# 디스크 I/O 100%?
iotop

# 네트워크 포화?
iftop
ss -s

3단계: 최적화

# CPU 병목: 프로세스 추가 또는 우선순위 조정
sudo renice -n 5 -p <high-cpu-pid>

# 메모리 부족: Swap 조정
sudo sysctl -w vm.swappiness=10

# 디스크 I/O: 캐시 조정
sudo sysctl -w vm.dirty_ratio=10

# 네트워크: 버퍼 증가
sudo sysctl -w net.core.somaxconn=65535

4단계: 모니터링

# 성능 개선 확인
watch -n 1 'top -bn1 | head -20'

💡 프로 팁​

1. 성능 모니터링 스크립트​

#!/bin/bash
# performance_monitor.sh

echo "=== CPU ==="
top -bn1 | grep "Cpu(s)" | awk '{print "CPU Usage: " 100 - $8 "%"}'

echo -e "\n=== Memory ==="
free -h | grep Mem | awk '{print "Used: " $3 "/" $2 " (" int($3/$2 * 100) "%)"}'

echo -e "\n=== Disk ==="
df -h / | tail -1 | awk '{print "Used: " $3 "/" $2 " (" $5 ")"}'

echo -e "\n=== Load Average ==="
uptime | awk -F'load average:' '{print $2}'

echo -e "\n=== Top Processes (CPU) ==="
ps aux --sort=-%cpu | head -6 | tail -5

echo -e "\n=== Top Processes (Memory) ==="
ps aux --sort=-%mem | head -6 | tail -5

2. 자동 알림​

#!/bin/bash
# alert_high_cpu.sh

THRESHOLD=80
CPU_USAGE=$(top -bn1 | grep "Cpu(s)" | awk '{print int(100 - $8)}')

if [ $CPU_USAGE -gt $THRESHOLD ]; then
    echo "⚠️  High CPU Usage: ${CPU_USAGE}%"
    # 이메일 또는 Slack 알림
    # mail -s "High CPU Alert" admin@example.com <<< "CPU: ${CPU_USAGE}%"
fi

3. 리소스 제한 (systemd)​

# /etc/systemd/system/myapp.service
[Service]
MemoryLimit=512M
CPUQuota=50%
IOWeight=500
