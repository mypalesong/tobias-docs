Linux CLI 고급

시스템 관리, 성능 튜닝, 보안 강화로 전문가가 되세요!

기초와 중급을 넘어, 이제 시스템의 깊은 곳까지 제어할 수 있습니다.

🎯 고급에서 배울 내용​

1. systemd 서비스 관리​

현대 Linux의 핵심! 서비스를 완벽하게 제어하세요.

  • systemctl - 서비스 시작/중지/재시작
  • Unit 파일 작성
  • 타이머 (Cron 대안)
  • 의존성 관리

2. 성능 모니터링 및 튜닝​

시스템 병목을 찾고 최적화하세요.

  • CPU, 메모리, I/O 모니터링
  • 프로세스 우선순위 (nice, renice)
  • 커널 파라미터 튜닝
  • 벤치마킹

3. 보안 강화​

시스템을 안전하게 보호하세요.

  • 파일 권한 고급 (ACL, SELinux)
  • 방화벽 설정 (ufw, iptables)
  • SSH 보안 강화
  • 보안 감사

4. 고급 네트워킹​

복잡한 네트워크 설정과 문제 해결

  • 네트워크 구성
  • VPN 설정
  • 패킷 분석 (tcpdump, wireshark)
  • 네트워크 트러블슈팅

5. 커널 및 시스템 깊이 파기​

Linux의 심장부를 이해하세요.

  • /proc, /sys 파일 시스템
  • 커널 모듈 관리
  • sysctl 설정
  • 부팅 프로세스 이해

💡 왜 고급이 중요한가?​

시스템 관리자로 성장​

기초/중급: 사용자로서 Linux 활용 고급: 관리자로서 Linux 제어

실무 시나리오​

문제: 프로덕션 서버의 성능이 저하됨

고급 기술로 해결:

# 1. 리소스 병목 찾기
htop
iotop
nethogs

# 2. 프로세스 우선순위 조정
renice -n -10 -p 1234

# 3. 커널 파라미터 튜닝
sysctl -w vm.swappiness=10
sysctl -w net.core.somaxconn=65535

# 4. 서비스 재시작 (무중단)
systemctl reload nginx

# 5. 모니터링 지속
watch -n 1 'ps aux | grep nginx'

자동화 및 신뢰성​

# systemd 서비스로 앱 관리
sudo systemctl enable myapp
sudo systemctl start myapp

# 자동 재시작 설정
# /etc/systemd/system/myapp.service
[Service]
Restart=always
RestartSec=10

🎓 학습 팁​

1. 실험 환경 구축​

# Docker로 안전한 실험 환경
docker run -it ubuntu:latest bash

# 또는 WSL에서 스냅샷 활용
wsl --export Ubuntu backup.tar
# 실험 후 문제 발생 시
wsl --import Ubuntu-test Ubuntu-test backup.tar

2. Man Page 깊이 읽기​

man systemd
man systemctl
man iptables
man sysctl

3. 로그 확인 습관​

# 시스템 로그
journalctl -xe

# 서비스별 로그
journalctl -u nginx

# 커널 로그
dmesg

4. 백업 먼저!​

# 설정 파일 백업
sudo cp /etc/sysctl.conf /etc/sysctl.conf.bak

# 전체 시스템 백업 (중요!)
sudo rsync -aAXv / --exclude={"/dev/*","/proc/*","/sys/*","/tmp/*","/run/*","/mnt/*","/media/*","/lost+found"} /backup/

⚡ Quick Reference​

systemd​

# 서비스 상태
systemctl status nginx

# 시작/중지/재시작
systemctl start nginx
systemctl stop nginx
systemctl restart nginx

# 부팅 시 자동 시작
systemctl enable nginx

# 로그 확인
journalctl -u nginx -f

성능 모니터링​

# CPU
top
htop

# 메모리
free -h
vmstat 1

# 디스크 I/O
iotop
iostat -x 1

# 네트워크
nethogs
iftop

보안​

# 방화벽
sudo ufw status
sudo ufw allow 22/tcp

# 파일 권한
ls -l
chmod 600 private.key

# 프로세스 확인
ps aux | grep suspicious

⚠️ 주의사항​

프로덕션 환경에서​

  1. 항상 백업: 설정 변경 전 백업
  2. 테스트 먼저: 개발 환경에서 먼저 테스트
  3. 문서화: 변경 사항을 기록
  4. 점진적 적용: 한 번에 하나씩 변경
  5. 모니터링: 변경 후 시스템 관찰

위험한 명령어​

# ❌ 절대 프로덕션에서 실행하지 마세요
rm -rf /
dd if=/dev/zero of=/dev/sda  # 디스크 포맷!
chmod -R 777 /  # 모든 권한 열기
iptables -F  # 방화벽 규칙 삭제
