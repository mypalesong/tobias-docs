시나리오 2: 민수의 Docker 컨테이너 개발 환경

민수가 마이크로서비스 아키텍처를 로컬에서 개발하기 위해 WSL 2와 Docker를 활용합니다.

📌 상황 설명​

백엔드 개발자 민수는 회사의 새로운 전자상거래 플랫폼을 개발하고 있습니다. 프로젝트는 마이크로서비스 아키텍처로 구성되어 있으며, API 게이트웨이, 사용자 서비스, 주문 서비스, 결제 서비스 등 여러 개의 독립적인 서비스가 있습니다. 민수는 로컬에서 모든 서비스를 Docker 컨테이너로 실행하여 프로덕션 환경과 동일한 환경을 구축하려고 합니다.

🎯 목표​

  • ✅ Docker Desktop for Windows 설치 및 WSL 2 통합
  • ✅ Docker Compose로 멀티 컨테이너 구성
  • ✅ 마이크로서비스 환경 구축
  • ✅ 네트워킹 및 볼륨 관리
  • ✅ 개발 워크플로우 최적화

💬 대화​

선배 재훈: "민수님, 로컬에서 모든 마이크로서비스를 실행하려면 Docker가 필수예요."

민수: "Docker Desktop을 설치하면 되는 건가요? WSL 2랑 어떻게 연동되나요?"

재훈: "Docker Desktop이 WSL 2를 자동으로 감지하고 통합됩니다. 성능도 훨씬 좋아요!"

민수: "좋아요! 바로 시작해봐요."

⚡ 실습​

1단계: Docker Desktop 설치​

재훈: "먼저 Docker Desktop을 설치합시다."

Windows에서:​

  1. Docker Desktop 다운로드

  2. 설치 파일 실행

  3. 설치 옵션:

    • ✅ "Use WSL 2 instead of Hyper-V" 체크
    • ✅ "Add shortcut to desktop" 체크 (선택)

  4. 설치 완료 후 재부팅

  5. Docker Desktop 실행

Docker Desktop 설정:​

  1. Settings → General

    • ✅ "Use the WSL 2 based engine" 활성화

  2. Settings → Resources → WSL Integration

    • ✅ "Enable integration with my default WSL distro" 활성화
    • ✅ "Ubuntu" 토글 활성화

  3. "Apply & Restart"

4단계: 간단한 마이크로서비스 작성​

API Gateway (Node.js + Express):

cd ~/projects/ecommerce-platform/services/api-gateway
npm init -y
npm install express axios cors

services/api-gateway/server.js:

const express = require('express');
const axios = require('axios');
const cors = require('cors');

const app = express();
const PORT = 3000;

app.use(cors());
app.use(express.json());

// Health check
app.get('/health', (req, res) => {
  res.json({ status: 'OK', service: 'API Gateway' });
});

// Proxy to user service
app.get('/api/users', async (req, res) => {
  try {
    const response = await axios.get('http://user-service:3001/users');
    res.json(response.data);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

// Proxy to order service
app.get('/api/orders', async (req, res) => {
  try {
    const response = await axios.get('http://order-service:3002/orders');
    res.json(response.data);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

app.listen(PORT, '0.0.0.0', () => {
  console.log(`API Gateway running on port ${PORT}`);
});

services/api-gateway/Dockerfile:

FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install --production

COPY . .

EXPOSE 3000

CMD ["node", "server.js"]

User Service:

cd ~/projects/ecommerce-platform/services/user-service
npm init -y
npm install express pg

services/user-service/server.js:

const express = require('express');
const { Pool } = require('pg');

const app = express();
const PORT = 3001;

// PostgreSQL 연결
const pool = new Pool({
  host: process.env.DB_HOST || 'database',
  port: 5432,
  database: process.env.DB_NAME || 'ecommerce',
  user: process.env.DB_USER || 'postgres',
  password: process.env.DB_PASSWORD || 'password'
});

app.use(express.json());

// Health check
app.get('/health', (req, res) => {
  res.json({ status: 'OK', service: 'User Service' });
});

// Get all users
app.get('/users', async (req, res) => {
  try {
    const result = await pool.query('SELECT id, name, email FROM users');
    res.json(result.rows);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

// Create user
app.post('/users', async (req, res) => {
  const { name, email } = req.body;
  try {
    const result = await pool.query(
      'INSERT INTO users (name, email) VALUES ($1, $2) RETURNING *',
      [name, email]
    );
    res.status(201).json(result.rows[0]);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

app.listen(PORT, '0.0.0.0', () => {
  console.log(`User Service running on port ${PORT}`);
});

services/user-service/Dockerfile:

FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install --production

COPY . .

EXPOSE 3001

CMD ["node", "server.js"]

5단계: Docker Compose 파일 작성​

재훈: "이제 모든 서비스를 Docker Compose로 연결합시다."

docker-compose.yml:

version: '3.8'

services:
  # PostgreSQL Database
  database:
    image: postgres:16-alpine
    container_name: ecommerce-db
    environment:
      POSTGRES_DB: ecommerce
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
    ports:
      - "5432:5432"
    volumes:
      - postgres-data:/var/lib/postgresql/data
      - ./services/database/init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - ecommerce-network

  # User Service
  user-service:
    build: ./services/user-service
    container_name: user-service
    environment:
      DB_HOST: database
      DB_NAME: ecommerce
      DB_USER: postgres
      DB_PASSWORD: password
    ports:
      - "3001:3001"
    depends_on:
      database:
        condition: service_healthy
    networks:
      - ecommerce-network
    volumes:
      - ./services/user-service:/app
      - /app/node_modules

  # Order Service
  order-service:
    build: ./services/order-service
    container_name: order-service
    environment:
      DB_HOST: database
      DB_NAME: ecommerce
    ports:
      - "3002:3002"
    depends_on:
      database:
        condition: service_healthy
    networks:
      - ecommerce-network
    volumes:
      - ./services/order-service:/app
      - /app/node_modules

  # API Gateway
  api-gateway:
    build: ./services/api-gateway
    container_name: api-gateway
    ports:
      - "3000:3000"
    depends_on:
      - user-service
      - order-service
    networks:
      - ecommerce-network
    volumes:
      - ./services/api-gateway:/app
      - /app/node_modules

  # Redis Cache (선택사항)
  redis:
    image: redis:7-alpine
    container_name: ecommerce-redis
    ports:
      - "6379:6379"
    networks:
      - ecommerce-network

networks:
  ecommerce-network:
    driver: bridge

volumes:
  postgres-data:

데이터베이스 초기화 스크립트:

services/database/init.sql:

-- Users table
CREATE TABLE IF NOT EXISTS users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Orders table
CREATE TABLE IF NOT EXISTS orders (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    product_name VARCHAR(255) NOT NULL,
    quantity INTEGER NOT NULL,
    total_price DECIMAL(10, 2) NOT NULL,
    status VARCHAR(50) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Sample data
INSERT INTO users (name, email) VALUES
    ('Alice Kim', 'alice@example.com'),
    ('Bob Park', 'bob@example.com'),
    ('Charlie Lee', 'charlie@example.com');

INSERT INTO orders (user_id, product_name, quantity, total_price) VALUES
    (1, 'Laptop', 1, 1299.99),

    (1, 'Keyboard', 1, 89.99);

9단계: 개발 워크플로우​

민수: "코드를 수정하면 어떻게 되나요?"

재훈: "볼륨 마운트 덕분에 실시간으로 반영돼요!"

services/user-service/server.js 수정:

// 새로운 엔드포인트 추가
app.get('/users/:id', async (req, res) => {
  const { id } = req.params;
  try {
    const result = await pool.query('SELECT * FROM users WHERE id = $1', [id]);
    if (result.rows.length === 0) {
      return res.status(404).json({ error: 'User not found' });
    }
    res.json(result.rows[0]);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});
# 서비스 재시작
docker-compose restart user-service

✅ 검증​

모든 것이 정상적으로 작동하는지 확인:

# 1. 모든 컨테이너 실행 중
docker-compose ps | grep Up

# 2. 네트워크 연결
docker exec api-gateway ping -c 3 user-service

# 3. 데이터베이스 연결
docker exec ecommerce-db psql -U postgres -d ecommerce -c "SELECT COUNT(*) FROM users;"

# 4. API 응답
curl -s http://localhost:3000/api/users | jq

# 5. Redis 연결
docker exec ecommerce-redis redis-cli PING

🎓 배운 내용​

Docker Compose 핵심 개념​

  1. 서비스 정의

    services:
      service-name:
        build: ./path
        image: image-name
        ports:
          - "host:container"

  2. 네트워크

    networks:
      app-network:
        driver: bridge

  3. 볼륨

    volumes:
      - ./source:/destination
      - named-volume:/data

  4. 환경 변수

    environment:
      KEY: value

  5. 의존성

    depends_on:
      database:
        condition: service_healthy

개발 워크플로우​

  1. 코드 수정 → 자동 반영 (볼륨 마운트)
  2. Dockerfile 수정 → 재빌드 필요
  3. docker-compose.yml 수정 → 재시작 필요
