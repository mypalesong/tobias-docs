시나리오 3: 지수의 Next.js 웹 개발 환경

프론트엔드 개발자 지수가 WSL 2에서 최신 웹 개발 환경을 구축하고 성능을 최적화합니다.

📌 상황 설명​

프론트엔드 개발자 지수는 회사의 새로운 SaaS 제품을 개발하고 있습니다. Next.js 14, TypeScript, TailwindCSS를 사용하며, 빠른 개발 경험을 위해 WSL 2 환경을 최적화하려고 합니다.

🎯 목표​

  • ✅ Next.js 14 + TypeScript 환경 구축
  • ✅ Fast Refresh 및 Hot Reload 최적화
  • ✅ TailwindCSS 및 UI 라이브러리 설정
  • ✅ API 모킹 및 개발 서버 설정
  • ✅ 빌드 및 배포 최적화

8단계: API Routes 작성​

src/app/api/users/route.ts:

import { NextRequest, NextResponse } from 'next/server';

// Mock data
const users = [
  { id: 1, name: 'Alice', email: 'alice@example.com' },
  { id: 2, name: 'Bob', email: 'bob@example.com' },
  { id: 3, name: 'Charlie', email: 'charlie@example.com' },
];

// GET /api/users
export async function GET(request: NextRequest) {
  return NextResponse.json(users);
}

// POST /api/users
export async function POST(request: NextRequest) {
  const body = await request.json();
  const newUser = {
    id: users.length + 1,
    ...body,
  };
  users.push(newUser);
  return NextResponse.json(newUser, { status: 201 });
}

9단계: 커스텀 Hook 작성​

src/hooks/useUsers.ts:

import { useState, useEffect } from 'react';

interface User {
  id: number;
  name: string;
  email: string;
}

export function useUsers() {
  const [users, setUsers] = useState<User[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    fetch('/api/users')
      .then((res) => res.json())
      .then((data) => {
        setUsers(data);
        setLoading(false);
      })
      .catch((err) => {
        setError(err);
        setLoading(false);
      });
  }, []);

  return { users, loading, error };
}

11단계: 성능 최적화​

.wslconfig 설정 (Windows 홈 디렉토리):

[wsl2]
memory=8GB
processors=4
localhostForwarding=true

파일 감시 한도 증가:

# inotify 한도 증가
echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

Next.js 설정 최적화 next.config.js:

/** @type {import('next').NextConfig} */
const nextConfig = {
