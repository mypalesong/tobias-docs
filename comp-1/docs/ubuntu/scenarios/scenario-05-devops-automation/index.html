시나리오 5: 현우의 DevOps 자동화 환경

DevOps 엔지니어 현우가 WSL 2에서 Kubernetes, Terraform, Ansible로 완벽한 CI/CD 및 자동화 환경을 구축합니다.

📌 상황 설명​

DevOps 엔지니어 현우는 회사의 인프라를 코드로 관리하고, CI/CD 파이프라인을 구축하며, 배포 자동화를 담당합니다. 로컬에서 모든 도구를 테스트하고 검증하기 위해 WSL 2 환경을 구축합니다.

🎯 목표​

  • ✅ kubectl 및 minikube 설치
  • ✅ Docker 및 Kubernetes 환경
  • ✅ Terraform 인프라 코드 작성
  • ✅ Ansible 자동화 플레이북
  • ✅ GitHub Actions 로컬 테스트
  • ✅ 모니터링 및 로깅 설정

⚡ 실습​

1단계: kubectl 설치​

# kubectl 최신 버전 다운로드
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# 실행 권한 부여
chmod +x kubectl

# PATH에 추가
sudo mv kubectl /usr/local/bin/

# 설치 확인
kubectl version --client

2단계: minikube 설치​

# minikube 바이너리 다운로드
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64

# 설치
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# 버전 확인
minikube version

3단계: minikube 클러스터 시작​

# Docker 드라이버로 시작
minikube start --driver=docker

# 클러스터 상태 확인
kubectl cluster-info
kubectl get nodes

출력:

NAME       STATUS   ROLES           AGE   VERSION
minikube   Ready    control-plane   1m    v1.28.3

4단계: 샘플 애플리케이션 배포​

k8s/deployment.yaml:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.25
        ports:
        - containerPort: 80
        resources:
          requests:
            memory: "64Mi"
            cpu: "250m"
          limits:
            memory: "128Mi"
            cpu: "500m"
---
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: NodePort
  selector:
    app: nginx
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
    nodePort: 30080
# 애플리케이션 배포
kubectl apply -f k8s/deployment.yaml

# 리소스 확인
kubectl get deployments
kubectl get pods
kubectl get services

# 서비스 접속
minikube service nginx-service --url

5단계: Terraform 설치​

# HashiCorp GPG 키 추가
wget -O- https://apt.releases.hashicorp.com/gpg | \
    gpg --dearmor | \
    sudo tee /usr/share/keyrings/hashicorp-archive-keyring.gpg

# 리포지토리 추가
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] \
    https://apt.releases.hashicorp.com $(lsb_release -cs) main" | \
    sudo tee /etc/apt/sources.list.d/hashicorp.list

# Terraform 설치
sudo apt update
sudo apt install terraform

# 버전 확인
terraform version

6단계: Terraform 인프라 코드​

terraform/main.tf:

terraform {
  required_providers {
    docker = {
      source  = "kreuzwerker/docker"
      version = "~> 3.0"
    }
  }
}

provider "docker" {
  host = "unix:///var/run/docker.sock"
}

# 네트워크 생성
resource "docker_network" "app_network" {
  name = "app_network"
}

# PostgreSQL 컨테이너
resource "docker_container" "postgres" {
  name  = "postgres-db"
  image = "postgres:16-alpine"

  env = [
    "POSTGRES_DB=myapp",
    "POSTGRES_USER=admin",
    "POSTGRES_PASSWORD=secret"
  ]

  ports {
    internal = 5432
    external = 5432
  }

  networks_advanced {
    name = docker_network.app_network.name
  }

  volumes {
    volume_name    = docker_volume.postgres_data.name
    container_path = "/var/lib/postgresql/data"
  }
}

# Redis 컨테이너
resource "docker_container" "redis" {
  name  = "redis-cache"
  image = "redis:7-alpine"

  ports {
    internal = 6379
    external = 6379
  }

  networks_advanced {
    name = docker_network.app_network.name
  }
}

# 볼륨
resource "docker_volume" "postgres_data" {
  name = "postgres_data"
}

# 출력
output "postgres_container_id" {
  value = docker_container.postgres.id
}

output "redis_container_id" {
  value = docker_container.redis.id
}
# Terraform 초기화
cd terraform
terraform init

# 실행 계획 확인
terraform plan

# 인프라 생성
terraform apply

# 리소스 확인
docker ps
terraform show

# 인프라 제거
terraform destroy

7단계: Ansible 설치​

# Ansible 설치
sudo apt update
sudo apt install -y software-properties-common
sudo add-apt-repository --yes --update ppa:ansible/ansible
sudo apt install -y ansible

# 버전 확인
ansible --version

8단계: Ansible 플레이북​

ansible/inventory.ini:

[local]
localhost ansible_connection=local

[webservers]
web1 ansible_host=localhost ansible_port=2221 ansible_user=vagrant
web2 ansible_host=localhost ansible_port=2222 ansible_user=vagrant

ansible/setup-webserver.yml:

---
- name: Setup Web Server
  hosts: local
  become: yes
  tasks:
    - name: Update apt cache
      apt:
        update_cache: yes

    - name: Install Nginx
      apt:
        name: nginx
        state: present

    - name: Start Nginx service
      service:
        name: nginx
        state: started
        enabled: yes

    - name: Copy custom index.html
      copy:
        content: |
          <!DOCTYPE html>
          <html>
          <head><title>Deployed by Ansible</title></head>
          <body>
            <h1>Hello from Ansible!</h1>
            <p>This server was configured automatically.</p>
          </body>
          </html>
        dest: /var/www/html/index.html
        owner: www-data
        group: www-data
        mode: '0644'

    - name: Install Docker
      shell: |
        curl -fsSL https://get.docker.com -o get-docker.sh
        sh get-docker.sh
      args:
        creates: /usr/bin/docker

    - name: Add user to docker group
      user:
        name: "{{ ansible_user }}"
        groups: docker
        append: yes
# 플레이북 실행
cd ansible
ansible-playbook -i inventory.ini setup-webserver.yml

# 특정 태스크만 실행
ansible-playbook -i inventory.ini setup-webserver.yml --tags "nginx"

# Dry run
ansible-playbook -i inventory.ini setup-webserver.yml --check

9단계: GitHub Actions 로컬 테스트 (act)​

# act 설치
curl https://raw.githubusercontent.com/nektos/act/master/install.sh | sudo bash

# act 버전 확인
act --version

.github/workflows/ci.yml:

name: CI Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm install

      - name: Run tests
        run: npm test

      - name: Run linting
        run: npm run lint

  build:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Build Docker image
        run: docker build -t myapp:${{ github.sha }} .

      - name: Push to registry
        run: |
          echo "Pushing to registry..."
          # docker push myapp:${{ github.sha }}
# GitHub Actions 로컬 실행
act -l  # 워크플로우 목록

act  # 전체 워크플로우 실행

act push  # push 이벤트 시뮬레이션

act -j test  # 특정 job만 실행

10단계: 모니터링 스택 (Prometheus + Grafana)​

docker-compose-monitoring.yml:

version: '3.8'

services:
  prometheus:
    image: prom/prometheus:latest
    container_name: prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
      - prometheus-data:/prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
    networks:
      - monitoring

  grafana:
    image: grafana/grafana:latest
    container_name: grafana
    ports:
      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin
    volumes:
      - grafana-data:/var/lib/grafana
    networks:
      - monitoring

  node-exporter:
    image: prom/node-exporter:latest
    container_name: node-exporter
    ports:
      - "9100:9100"
    networks:
      - monitoring

networks:
  monitoring:
    driver: bridge

volumes:
  prometheus-data:
  grafana-data:

prometheus/prometheus.yml:

global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  - job_name: 'node-exporter'
    static_configs:
      - targets: ['node-exporter:9100']
# 모니터링 스택 시작
docker-compose -f docker-compose-monitoring.yml up -d

# 접속
# Prometheus: http://localhost:9090
# Grafana: http://localhost:3000 (admin/admin)

🎓 배운 내용​

DevOps 핵심 도구​

  1. Kubernetes: 컨테이너 오케스트레이션
  2. Terraform: 인프라 as 코드
  3. Ansible: 설정 관리 및 자동화
  4. GitHub Actions: CI/CD 파이프라인
  5. Prometheus/Grafana: 모니터링

베스트 프랙티스​

  1. IaC (Infrastructure as Code): 모든 인프라를 코드로 관리
  2. GitOps: Git을 단일 진실 공급원으로
  3. 자동화: 반복 작업을 스크립트로
  4. 모니터링: 시스템 상태 실시간 추적

1. 유용한 kubectl alias​

# ~/.bashrc
alias k='kubectl'
alias kgp='kubectl get pods'
alias kgs='kubectl get services'
alias kgd='kubectl get deployments'
alias kdp='kubectl describe pod'
alias kl='kubectl logs'
alias kex='kubectl exec -it'

2. Terraform 작업 스크립트​

#!/bin/bash
# tf-deploy.sh

terraform init
terraform fmt
terraform validate
terraform plan -out=tfplan
terraform apply tfplan

# 암호화된 파일 생성
ansible-vault create secrets.yml

# 플레이북에서 사용
ansible-playbook playbook.yml --ask-vault-pass
