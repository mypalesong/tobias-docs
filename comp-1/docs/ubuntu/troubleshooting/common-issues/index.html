일반적인 문제 해결

WSL 2 사용 중 자주 발생하는 문제와 해결 방법입니다.

🔧 설치 및 실행 문제​

WSL이 설치되지 않음​

증상:

The Windows Subsystem for Linux optional component is not enabled.

해결:

# PowerShell 관리자 권한으로
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

# 재부팅
Restart-Computer

WSL 2 커널 업데이트 필요​

증상:

WSL 2 requires an update to its kernel component.

해결:

  1. WSL 2 Linux 커널 업데이트 다운로드
  2. 다운로드한 .msi 파일 실행
  3. WSL 재시작:
wsl --shutdown
wsl

가상화가 활성화되지 않음​

증상:

Please enable the Virtual Machine Platform Windows feature and ensure virtualization is enabled in the BIOS.

해결:

  1. BIOS 설정 진입 (재부팅 시 F2, F12, Del 등)
  2. "Virtualization Technology" 또는 "Intel VT-x" / "AMD-V" 찾기
  3. Enabled로 변경
  4. 저장 후 재부팅

Ubuntu가 시작되지 않음​

증상:

The system cannot find the path specified.

해결:

# WSL 완전 종료
wsl --shutdown

# 특정 배포판 재시작
wsl -d Ubuntu

# 배포판 목록 확인
wsl --list --verbose

# 배포판 재설정 (마지막 수단!)
wsl --unregister Ubuntu
wsl --install -d Ubuntu

🌐 네트워크 문제​

localhost 접속 안 됨​

증상: WSL에서 실행한 서버에 Windows에서 접속 불가

해결:

# WSL에서 서버를 0.0.0.0으로 바인딩
node server.js --host 0.0.0.0
# 또는
uvicorn app:app --host 0.0.0.0

포트가 이미 사용 중​

증상:

Error: listen EADDRINUSE: address already in use :::3000

해결:

WSL에서:

# 포트를 사용하는 프로세스 찾기
lsof -i :3000

# 또는
netstat -tuln | grep 3000

# 프로세스 종료
kill -9 <PID>

Windows에서:

# 포트를 사용하는 프로세스 찾기
netstat -ano | findstr :3000

# 프로세스 종료
taskkill /PID <PID> /F

DNS 문제​

증상:

Could not resolve host: github.com

해결:

/etc/resolv.conf 수정:

sudo rm /etc/resolv.conf
sudo bash -c 'echo "nameserver 8.8.8.8" > /etc/resolv.conf'
sudo bash -c 'echo "nameserver 8.8.4.4" >> /etc/resolv.conf'
sudo chattr +i /etc/resolv.conf  # 자동 재생성 방지

💾 파일 시스템 문제​

파일 권한 오류​

증상:

Permission denied

해결:

# 파일 소유권 변경
sudo chown -R $USER:$USER /path/to/directory

# 권한 변경
chmod -R 755 /path/to/directory

# 실행 권한 추가
chmod +x script.sh

Windows 파일 접근 느림​

증상: Windows 파일 시스템(/mnt/c)에서 작업 시 매우 느림

해결:

파일을 WSL 파일 시스템으로 이동:

# 나쁜 예 (느림)
cd /mnt/c/Users/username/projects
npm install

# 좋은 예 (빠름)
cd ~/projects
npm install

Windows에서 WSL 파일 접근:

\\wsl$\Ubuntu\home\username\projects

디스크 공간 부족​

증상:

No space left on device

해결:

# 디스크 사용량 확인
df -h

# 큰 파일/디렉토리 찾기
du -sh ~/* | sort -rh | head -10

# Docker 정리 (Docker 사용 시)
docker system prune -a

# APT 캐시 정리
sudo apt clean
sudo apt autoremove

# WSL 디스크 압축 (PowerShell에서)
wsl --shutdown
Optimize-VHD -Path "$env:LOCALAPPDATA\Packages\CanonicalGroupLimited.Ubuntu_*\LocalState\ext4.vhdx" -Mode Full

🐳 Docker 문제​

Docker 데몬 연결 실패​

증상:

Cannot connect to the Docker daemon

해결:

  1. Docker Desktop이 실행 중인지 확인
  2. Docker Desktop 설정:
    • Settings → Resources → WSL Integration
    • Ubuntu 토글 활성화
    • Apply & Restart
# Docker 그룹 권한
sudo usermod -aG docker $USER

# 로그아웃 후 재로그인 또는
newgrp docker

Docker 빌드 느림​

해결:

.dockerignore 추가:

node_modules
.git
.env
*.log

WSL 파일 시스템에서 작업:

# 나쁜 예
cd /mnt/c/projects
docker build .

# 좋은 예
cd ~/projects
docker build .

🔄 성능 문제​

시스템이 느림​

해결:

메모리 제한 설정 (.wslconfig):

Windows 홈 디렉토리에 .wslconfig 생성:

[wsl2]
memory=8GB
processors=4
swap=2GB
localhostForwarding=true

WSL 재시작:

wsl --shutdown
wsl

파일 감시 한도 초과​

증상:

System limit for number of file watchers reached

해결:

# inotify 한도 증가
echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

npm/node 느림​

해결:

  1. WSL 파일 시스템 사용
# 나쁜 예
cd /mnt/c/projects/myapp
npm install

# 좋은 예
cd ~/projects/myapp
npm install
  1. 캐시 정리
npm cache clean --force
  1. pnpm 사용 (더 빠름)
npm install -g pnpm
pnpm install

🔐 인증 문제​

Git 자격 증명 문제​

해결:

# Windows Credential Manager 사용
git config --global credential.helper "/mnt/c/Program\ Files/Git/mingw64/bin/git-credential-manager-core.exe"

# 또는 SSH 키 사용
ssh-keygen -t ed25519 -C "your.email@example.com"
cat ~/.ssh/id_ed25519.pub  # GitHub에 추가

SSH 권한 오류​

증상:

Permissions 0644 for '/home/user/.ssh/id_rsa' are too open

해결:

chmod 600 ~/.ssh/id_rsa
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
chmod 700 ~/.ssh

🖥️ WSL 명령어 문제​

wsl 명령어가 인식되지 않음​

해결:

  1. Windows 버전 확인:
winver

2004 이상이어야 합니다.

  1. Windows 업데이트 실행

특정 배포판 기본값 설정​

# 기본 배포판 변경
wsl --set-default Ubuntu

# 특정 배포판 실행
wsl -d Ubuntu

# 배포판 목록
wsl --list --verbose

🆘 긴급 복구​

WSL 완전 재설정​

주의: 모든 데이터가 삭제됩니다!

# 배포판 제거
wsl --unregister Ubuntu

# WSL 재설치
wsl --install -d Ubuntu

백업 및 복원​

백업:

# 배포판 내보내기
wsl --export Ubuntu D:\backup\ubuntu-backup.tar

복원:

# 배포판 가져오기
wsl --import Ubuntu D:\WSL\Ubuntu D:\backup\ubuntu-backup.tar

📞 추가 도움​

로그 확인​

# 시스템 로그
sudo dmesg | tail -50

# 서비스 로그
journalctl -xe

# WSL 로그 (PowerShell에서)
wsl --list --verbose

유용한 진단 명령어​

# 시스템 정보
uname -a
lsb_release -a

# 메모리 사용량
free -h

# 디스크 사용량
df -h

# 프로세스 확인
top
htop

# 네트워크 연결
netstat -tuln
ss -tuln
