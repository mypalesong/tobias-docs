고급 기법

최신 고급 프롬프팅 기법으로 AI 성능 극대화하기

🌳 Tree of Thoughts (ToT)​

개념​

Chain of Thought가 선형적 사고라면, Tree of Thoughts는 여러 경로를 탐색하는 기법입니다.

tot = {
    "정의": "여러 가지 추론 경로를 탐색하고 평가하는 기법",
    "구조": "트리 형태로 가능한 해결 경로 탐색",
    "장점": "복잡한 문제에서 최적 해법 발견",
    "적용": "창의적 문제, 다단계 의사결정"
}

CoT vs ToT 비교​

Chain of Thought (선형)​

문제 → 단계1 → 단계2 → 단계3 → 답

Tree of Thoughts (탐색)​

                문제
                 |
        +--------+--------+
        |        |        |
      경로A    경로B    경로C
        |        |        |
     +--+--+  +--+--+  +--+--+
     |  |  |  |  |  |  |  |  |
    ... ... ... ... ... ... ...

🔄 ReAct (Reasoning + Acting)​

개념​

**추론(Reasoning)**과 **행동(Acting)**을 번갈아 수행하는 기법입니다.

react = {
    "정의": "생각하고 행동하기를 반복하며 문제 해결",
    "구조": "Thought → Action → Observation → Thought → ...",
    "장점": "도구 사용, 정보 검색, 다단계 작업에 강력",
    "적용": "복잡한 조사, 다중 도구 사용, 탐색 작업"
}

ReAct 사이클​

1. Thought: 다음에 무엇을 해야 할까?
      ↓
2. Action: 도구 실행/정보 검색
      ↓
3. Observation: 결과 확인
      ↓
4. Thought: 이 결과를 어떻게 활용할까?
      ↓
   (반복)

기본 패턴​

"""
작업: [복잡한 작업]

사용 가능한 도구:
- search(query): 웹 검색
- calculate(expression): 계산
- read_file(path): 파일 읽기
- execute_code(code): 코드 실행

ReAct 프로세스:

Thought 1: [무엇을 해야 하는지 판단]
Action 1: [도구 사용]
Observation 1: [결과]

Thought 2: [결과를 바탕으로 다음 단계 판단]
Action 2: [도구 사용]
Observation 2: [결과]

[반복]

Final Answer: [최종 답변]
"""

실전 예제: 기술 조사​

"""
작업: FastAPI와 Flask의 성능을 비교하고,
     우리 프로젝트(REST API, 10만 req/day)에 맞는 것을 추천하세요.

사용 가능한 도구:
- search(query): 웹 검색
- benchmark(framework): 벤치마크 데이터 조회
- analyze_project(requirements): 프로젝트 요구사항 분석

---

Thought 1: 먼저 두 프레임워크의 기본 성능 차이를 알아야겠다.
Action 1: benchmark("FastAPI vs Flask")
Observation 1:
  - FastAPI: 초당 25,000 요청 처리
  - Flask: 초당 10,000 요청 처리
  - FastAPI가 약 2.5배 빠름

Thought 2: 성능 차이의 원인을 알아야 정확한 추천을 할 수 있다.
Action 2: search("FastAPI performance advantage reasons")
Observation 2:
  - 비동기 지원 (ASGI)
  - Pydantic 기반 빠른 검증
  - 자동 최적화

Thought 3: 우리 프로젝트 요구사항과 맞는지 확인해야 한다.
Action 3: analyze_project(requirements="REST API, 10만 req/day, 팀 3명")
Observation 3:
  - 하루 10만 요청 = 초당 약 1.16 요청
  - Flask도 충분히 처리 가능한 트래픽
  - 하지만 성장 대비 필요

Thought 4: 다른 요소들도 고려해야 한다.
Action 4: search("FastAPI vs Flask learning curve developer experience")
Observation 4:
  - FastAPI: 타입 힌트 필수, 자동 문서화
  - Flask: 유연하지만 수동 설정 많음
  - FastAPI 학습 곡선이 약간 높음

Thought 5: 이제 종합적인 추천을 할 수 있다.

Final Answer:
**추천: FastAPI**

이유:
1. 성능: 현재는 Flask로 충분하지만, 향후 트래픽 증가 대비
2. 개발 경험: 자동 문서화, 타입 안전성으로 팀 생산성 향상
3. 비동기: 외부 API 호출이 많다면 FastAPI가 유리
4. 최신 표준: Python 3.7+ 최신 기능 활용

주의사항:
- 팀원들의 타입 힌트 학습 시간 필요 (1-2주)
- 레거시 라이브러리 호환성 사전 확인 필요

결론: 단기간은 Flask가 쉽지만, 장기적으로 FastAPI가 유리
"""

실전 예제: 수학 문제​

"""
문제:
한 회사의 직원 수가 작년에 200명이었습니다.
올해는 작년보다 25% 증가했고,
내년에는 올해보다 20% 증가할 예정입니다.
내년 직원 수는 몇 명일까요?

3가지 방법으로 풀어주세요:

---

## 방법 1: 단계별 증가 계산

1단계: 올해 직원 수
- 작년: 200명
- 증가: 200 × 25% = 50명
- 올해: 200 + 50 = 250명

2단계: 내년 직원 수
- 올해: 250명
- 증가: 250 × 20% = 50명
- 내년: 250 + 50 = 300명

답: 300명

---

## 방법 2: 증가율 곱셈

올해 = 작년 × 1.25
내년 = 올해 × 1.20
따라서:
내년 = 작년 × 1.25 × 1.20
     = 200 × 1.25 × 1.20
     = 200 × 1.50
     = 300명

답: 300명

---

## 방법 3: 전체 증가율 계산

2년간 전체 증가율:
(1.25 × 1.20) - 1 = 1.50 - 1 = 0.50 = 50%

내년 = 작년 × (1 + 전체 증가율)
     = 200 × (1 + 0.50)
     = 200 × 1.50
     = 300명

답: 300명

---

## 최종 답변
**답: 300명**

신뢰도: 매우 높음 (3/3 일치)
검증: 세 가지 독립적인 방법이 모두 같은 답을 도출
"""

import bcrypt

class User:
    def __init__(self, id: int, username: str, password: str):
        self.id = id
        self.username = username
        self.password_hash = bcrypt.hashpw(
            password.encode(), bcrypt.gensalt()
        )

    def verify_password(self, password: str) -> bool:
        return bcrypt.checkpw(
            password.encode(), self.password_hash
        )

class TokenBlacklist:
    blacklist = set()

    @classmethod
    def add(cls, token: str):
        cls.blacklist.add(token)

    @classmethod
    def is_blacklisted(cls, token: str) -> bool:
        return token in cls.blacklist

class User:

def choose_technique(problem):
    if problem.has_multiple_valid_approaches:
        return "Tree of Thoughts"

    if problem.requires_external_tools:
        return "ReAct"

    if problem.accuracy_is_critical:
        return "Self-Consistency"

    if problem.is_complex_and_hierarchical:
        return "Least-to-Most"

    if problem.is_prompt_optimization:
        return "Meta-Prompting"

    return "Chain of Thought"  # 기본
