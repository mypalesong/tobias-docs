안티 패턴

피해야 할 일반적인 실수들

효과적인 프롬프트를 작성하는 것만큼, 흔한 실수를 피하는 것도 중요합니다. 이 가이드에서는 자주 발생하는 안티 패턴과 그 해결책을 다룹니다.

🎯 학습 목표​

  • 일반적인 프롬프팅 실수 인식
  • 각 안티 패턴의 문제점 이해
  • 올바른 대안 적용

❌ 모호함 (Vagueness)​

문제​

# ❌ 안티 패턴
"코드 좀 만들어줘"
"이거 고쳐줘"
"분석해줘"

왜 나쁜가?​

  • AI가 추측해야 함
  • 원하지 않는 결과
  • 반복 요청 필요

해결책​

# ✅ 올바른 방법
"""
Python으로 사용자 인증 API 작성:
- FastAPI 프레임워크
- JWT 토큰 방식
- 이메일/비밀번호 로그인
- 타입 힌트 포함
- Docstring 작성
"""

❌ 과도한 요구 (Overloading)​

문제​

# ❌ 안티 패턴
"""
전자상거래 플랫폼 전체를 만들어주세요.
- 프론트엔드 (React)
- 백엔드 (Node.js)
- 데이터베이스 설계
- 결제 시스템
- 배송 추적
- 관리자 대시보드
- 모바일 앱
- SEO 최적화
- 성능 최적화
- 보안
- CI/CD
- 문서화
한 번에 다 해주세요!
"""

왜 나쁜가?​

  • 품질 저하
  • 중요한 부분 누락
  • 깊이 없는 결과

해결책​

# ✅ 올바른 방법 (단계별 접근)
"""
1단계: 사용자 인증 API
- JWT 로그인
- 회원가입
- 비밀번호 재설정

이것만 먼저 구현해주세요.
"""

❌ 맥락 부족 (No Context)​

문제​

# ❌ 안티 패턴
"""
이 코드 최적화해줘:

def process(data):
    result = []
    for item in data:
        result.append(item * 2)
    return result
"""

왜 나쁜가?​

  • 최적화 방향 불명확
  • 데이터 규모 모름
  • 제약사항 파악 불가

해결책​

# ✅ 올바른 방법
"""
배경:
- 센서 데이터 처리 (초당 100만 개)
- 현재 성능: 10만 개/초
- 목표: 100만 개/초
- 메모리: 최대 1GB

환경:
- Python 3.11
- NumPy 사용 가능

코드:
def process(data):
    result = []
    for item in data:
        result.append(item * 2)
    return result

최적화해주세요.
"""

❌ 예제 없음 (No Examples for Complex Tasks)​

문제​

# ❌ 안티 패턴 (복잡한 패턴인데 예제 없음)
"""
다음 텍스트에서 개인정보를 식별하고 마스킹해주세요.
"""

왜 나쁜가?​

  • 원하는 형식 모호
  • 마스킹 방식 불명확
  • 일관성 없는 결과

해결책​

# ✅ Few-shot 사용
"""
다음 패턴으로 개인정보 마스킹:

예시 1:
입력: "제 이름은 김철수이고 전화번호는 010-1234-5678입니다."
출력: "제 이름은 [이름]이고 전화번호는 [전화번호]입니다."

예시 2:
입력: "이메일은 user@example.com이에요."
출력: "이메일은 [이메일]이에요."

이제 다음을 처리하세요:
입력: "저는 박영희이고 주소는 서울시 강남구입니다."
출력:
"""

❌ 일관성 없음 (Inconsistency)​

문제​

# ❌ 안티 패턴
"""
사용자를 생성하는 함수를 만들어주세요.

요구사항:
- user_name을 받음
- email 검증
- 비밀번호는 pwd로 받음
- Return: UserObj

그리고 camelCase 사용하고
snake_case로 변수 만들어주세요.
"""

왜 나쁜가?​

  • 혼란스러운 지시
  • 모순된 요구사항
  • 예측 불가능한 결과

해결책​

# ✅ 일관성 유지
"""
사용자 생성 함수 작성:

매개변수:
- username: str
- email: str
- password: str

반환: User 객체

코딩 스타일:
- 변수명: snake_case
- 클래스명: PascalCase
- 함수명: snake_case
"""

❌ 형식 미지정 (No Format Specification)​

문제​

# ❌ 안티 패턴
"""
사용자 데이터를 보기 좋게 정리해줘:

user1: Alice, 30, alice@example.com
user2: Bob, 25, bob@example.com
"""

왜 나쁜가?​

  • "보기 좋게"가 모호
  • 매번 다른 형식
  • 파싱 어려움

해결책​

# ✅ 명확한 형식 지정
"""
다음 사용자 데이터를 JSON 배열로 변환:

입력:
user1: Alice, 30, alice@example.com
user2: Bob, 25, bob@example.com

출력 형식:
[
  {"name": "...", "age": ..., "email": "..."},
  ...
]
"""

❌ 부정적 지시만 (Only Negatives)​

문제​

# ❌ 안티 패턴
"""
코드 작성 시:
- 클래스 사용하지 마세요
- 전역 변수 쓰지 마세요
- 긴 함수 만들지 마세요
- 주석 달지 마세요

함수를 만들어주세요.
"""

왜 나쁜가?​

  • 해야 할 것이 불명확
  • 부정적 제약만으로는 부족
  • 창의성 제한

해결책​

# ✅ 긍정적 + 부정적 균형
"""
함수 작성:

✅ 해야 할 것:
- 순수 함수 사용
- 타입 힌트 포함
- Docstring 작성
- 한 함수당 20줄 이하

❌ 하지 말아야 할 것:
- 전역 변수 사용
- 하드코딩된 값
"""

❌ 너무 많은 단계 (Too Many Steps)​

문제​

# ❌ 안티 패턴
"""
1단계: 변수 a 선언
2단계: 변수 b 선언
3단계: a에 1 할당
4단계: b에 2 할당
5단계: c 선언
6단계: a와 b를 더함
7단계: 결과를 c에 저장
8단계: c 출력
9단계: 함수로 감싸기
10단계: 문서화
...
20단계: 테스트 작성
"""

왜 나쁜가?​

  • 과도한 세분화
  • 중요한 것 가려짐
  • 불필요한 복잡성

해결책​

# ✅ 적절한 수준
"""
두 수를 더하는 함수 작성:

1단계: 함수 구현 (add 함수)
2단계: 타입 힌트 추가
3단계: Docstring 작성
4단계: 테스트 코드 작성
"""

❌ 과도한 격식 (Over-formality)​

문제​

# ❌ 안티 패턴
"""
귀하께서는 지금부터 최고의 소프트웨어 아키텍트로서의
역할을 수행하셔야 하며, 본인의 모든 지식과 경험을
총동원하여 다음과 같은 과업을 수행해 주시기를 간곡히
요청드리는 바입니다.

[100단어의 격식 차린 서론]

작업: Hello World 출력
"""

왜 나쁜가?​

  • 불필요한 토큰 낭비
  • 핵심 가려짐
  • 시간 낭비

해결책​

# ✅ 간결하고 직접적
"""
Python으로 Hello World 출력하세요.
"""

# 전문성이 필요할 때만 역할 부여
"""
당신은 시니어 보안 엔지니어입니다.
안전한 비밀번호 저장 방식을 설계해주세요.
"""

❌ 가정 강요 (Forcing Assumptions)​

문제​

# ❌ 안티 패턴
"""
당연히 React를 써야 하니까
당연히 TypeScript를 쓰겠지만
당연히 Tailwind를 쓰는 게 맞으니까

이걸로 대시보드 만들어줘
"""

왜 나쁜가?​

  • 최선의 선택 아닐 수 있음
  • 맥락 무시
  • 독단적 결정

해결책​

# ✅ 요구사항 명시 또는 조언 요청
"""
## 옵션 1: 기술 스택 지정
대시보드 제작:
- React + TypeScript
- Tailwind CSS
- Chart.js

## 옵션 2: 추천 요청
대시보드 제작 (기술 스택 추천 필요):

프로젝트 상황:
- 팀: 주니어 2명
- 일정: 2개월
- 복잡도: 중간

적합한 기술 스택을 추천하고 이유를 설명해주세요.
"""

❌ 테스트 없음 (No Testing Mentioned)​

문제​

# ❌ 안티 패턴
"""
결제 시스템 구현해줘
"""
# → 테스트 코드 없음, 검증 불가

왜 나쁜가?​

  • 버그 발견 어려움
  • 신뢰성 낮음
  • 유지보수 힘듦

해결책​

# ✅ 테스트 포함
"""
결제 시스템 구현:

기능:
- 결제 처리
- 환불 처리
- 결제 내역 조회

출력:
1. 구현 코드
2. 단위 테스트 (pytest)
3. 엣지 케이스 테스트
"""

✅ 안티 패턴 회피 체크리스트​

명확성​

  • 요청이 구체적인가?
  • 모호한 표현이 없는가?
  • 출력 형식이 명시되었는가?

범위​

  • 한 번에 요청하는 양이 적절한가?
  • 우선순위가 있는가?
  • 단계적 접근이 필요한가?

맥락​

  • 필요한 배경 정보를 제공했는가?
  • 환경이 명시되었는가?
  • 제약사항이 있는가?

일관성​

  • 용어가 통일되었는가?
  • 명명 규칙이 일관적인가?
  • 모순이 없는가?

실용성​

  • 테스트를 고려했는가?
  • 유지보수를 고려했는가?
  • 실제 사용 가능한가?
