코드 생성 시나리오

AI를 활용한 효율적인 코드 작성

코드 생성은 Prompt Engineering의 가장 인기있는 활용 사례입니다. 이 가이드에서는 다양한 코드 생성 상황에서 효과적인 프롬프트를 작성하는 방법을 다룹니다.

🎯 핵심 전략​

code_generation_strategy = {
    "명확한 명세": "입력, 출력, 동작 명확히",
    "환경 정의": "언어, 프레임워크, 버전",
    "품질 기준": "코딩 스타일, 테스트, 문서화",
    "단계적 접근": "복잡한 코드는 단계별로"
}

📝 시나리오 1: 함수/메서드 생성​

기본 패턴​

"""
{언어}로 {기능}을(를) 수행하는 함수 작성

함수명: {name}
매개변수:
- {param1}: {type} - {설명}
- {param2}: {type} - {설명}

반환값: {return_type} - {설명}

동작:
1. {step1}
2. {step2}
...

예외 처리:
- {condition} → {exception}

요구사항:
- 타입 힌트 포함
- Docstring 작성
- 사용 예제 포함
"""

실전 예제​

"""
Python으로 JSON 검증 및 파싱 함수 작성

함수명: parse_and_validate_json
매개변수:
- json_str: str - JSON 문자열
- schema: dict - 검증할 스키마 (선택)

반환값: dict - 파싱된 JSON 객체

동작:
1. JSON 문자열 파싱
2. 스키마 제공 시 검증
3. 검증 통과 시 dict 반환

예외 처리:
- 잘못된 JSON → JSONDecodeError
- 스키마 불일치 → ValidationError

요구사항:
- Python 3.10+
- 타입 힌트 필수
- Google 스타일 docstring
- pytest 테스트 2개 이상
- 사용 예제 포함

코드 스타일:
- snake_case 변수명
- Black 포맷터 준수
"""

📝 시나리오 2: 클래스 설계​

기본 패턴​

"""
{언어}로 {목적} 클래스 설계

클래스명: {ClassName}
책임: {주요 책임}

속성:
- {attr1}: {type} - {설명}
- {attr2}: {type} - {설명}

메서드:
- {method1}({params}): {설명}
- {method2}({params}): {설명}

디자인 패턴: {pattern if applicable}

요구사항:
- 캡슐화
- 불변성 (필요시)
- 타입 안전성
"""

실전 예제​

"""
Python으로 사용자 세션 관리 클래스 설계

클래스명: SessionManager
책임: 사용자 세션 생성, 검증, 만료 관리

속성:
- _sessions: dict[str, Session] - 활성 세션 저장
- _timeout: int - 세션 만료 시간 (초)

메서드:
- create_session(user_id: str) -> str:
  새 세션 생성, 세션 ID 반환

- validate_session(session_id: str) -> bool:
  세션 유효성 검증

- refresh_session(session_id: str) -> None:
  세션 만료 시간 갱신

- delete_session(session_id: str) -> None:
  세션 삭제

- cleanup_expired() -> int:
  만료된 세션 정리, 삭제 개수 반환

디자인 패턴: Singleton

요구사항:
- Python 3.11
- 타입 힌트 필수
- 스레드 안전 (threading.Lock 사용)
- Private 속성 (언더스코어)
- Docstring 모든 메서드
- 단위 테스트 5개 이상

추가:
- __repr__ 구현
- Context manager 지원 (선택)
"""

📝 시나리오 3: API 엔드포인트​

FastAPI 예제​

"""
FastAPI로 사용자 관리 CRUD API 작성

엔드포인트:
1. POST /users
   - 기능: 사용자 생성
   - 입력: UserCreate (name, email, password)
   - 출력: 201, User (id, name, email, created_at)
   - 검증: 이메일 형식, 비밀번호 8자 이상

2. GET /users/{user_id}
   - 기능: 사용자 조회
   - 출력: 200, User 또는 404

3. PUT /users/{user_id}
   - 기능: 사용자 수정
   - 입력: UserUpdate (name, email)
   - 출력: 200, User 또는 404

4. DELETE /users/{user_id}
   - 기능: 사용자 삭제
   - 출력: 204 또는 404

기술 스택:
- FastAPI 0.104+
- Pydantic v2
- SQLAlchemy 2.0 (ORM)
- PostgreSQL

구조:
- models.py: SQLAlchemy 모델
- schemas.py: Pydantic 스키마
- crud.py: 데이터베이스 작업
- router.py: API 라우터
- dependencies.py: 의존성 (DB 세션)

요구사항:
- 에러 처리 (HTTPException)
- 비밀번호 해싱 (bcrypt)
- 타입 힌트
- Docstring
- 각 엔드포인트별 curl 예제
"""

📝 시나리오 4: 알고리즘 구현​

기본 패턴​

"""
{알고리즘}을(를) {언어}로 구현

알고리즘: {name}
설명: {description}

입력: {input_spec}
출력: {output_spec}

시간 복잡도: O({complexity})
공간 복잡도: O({complexity})

요구사항:
- 효율적인 구현
- 엣지 케이스 처리
- 단계별 주석
- 테스트 케이스 3개 이상
"""

실전 예제​

"""
이진 탐색 트리 (BST)를 Python으로 구현

클래스: BinarySearchTree

노드 구조:
- value: int
- left: Node | None
- right: Node | None

메서드:
1. insert(value: int) -> None
   - 값 삽입 (중복 허용 안 함)

2. search(value: int) -> bool
   - 값 존재 여부 반환

3. delete(value: int) -> bool
   - 값 삭제, 성공 여부 반환

4. inorder_traversal() -> list[int]
   - 중위 순회 결과 반환 (정렬된 순서)

5. find_min() -> int | None
   - 최소값 반환

6. find_max() -> int | None
   - 최대값 반환

시간 복잡도:
- 평균: O(log n)
- 최악: O(n)

요구사항:
- Python 3.10+
- 타입 힌트
- 재귀 또는 반복 (선택)
- 각 메서드에 docstring
- 엣지 케이스 처리 (빈 트리, 단일 노드 등)
- pytest 테스트 10개 이상
- 시각화 메서드 (선택): print_tree()

테스트 케이스:
1. 빈 트리에 삽입
2. 중복 값 삽입
3. 삭제 (리프, 한 자식, 두 자식)
4. 존재하지 않는 값 검색/삭제
"""

📝 시나리오 5: 리팩토링​

기본 패턴​

"""
다음 코드를 리팩토링해주세요:

[현재 코드]

리팩토링 목표:
- {goal1}
- {goal2}

원칙:
- {principle1}
- {principle2}

출력:
1. 리팩토링된 코드
2. 변경 사항 설명
3. Before/After 비교
"""

실전 예제​

"""
다음 Python 코드를 리팩토링해주세요:

```python
def process_user_data(data):
    result = []
    for item in data:
        if item['age'] > 18:
            if item['status'] == 'active':
                if 'email' in item:
                    result.append({
                        'name': item['name'].upper(),
                        'email': item['email'].lower(),
                        'age': item['age']
                    })
    return result

리팩토링 목표:

  • 가독성 향상
  • 네스팅 줄이기
  • 타입 안전성 추가
  • 에러 처리 추가

원칙:

  • Early return 패턴 사용
  • 단일 책임 원칙
  • 타입 힌트 추가
  • Pydantic 모델 사용 (선택)

출력:

  1. 리팩토링된 코드
  2. 주요 변경 사항 3가지 설명
  3. 개선된 점 (가독성, 성능, 유지보수) """

## 📝 시나리오 6: 테스트 코드 생성

### pytest 예제

```python
"""
다음 함수에 대한 pytest 테스트 작성:

```python
def calculate_discount(price: float, discount_percent: int) -> float:
    '''가격에 할인율을 적용한 최종 가격 계산'''
    if discount_percent < 0 or discount_percent > 100:
        raise ValueError("Discount percent must be between 0 and 100")
    return price * (1 - discount_percent / 100)

테스트 케이스:

  1. 정상 케이스 (여러 조합)
  2. 경계값 (0%, 100%)
  3. 예외 케이스 (음수, 100 초과)
  4. 부동소수점 정밀도
  5. 큰 금액 처리

요구사항:

  • pytest fixture 사용
  • parametrize로 여러 케이스
  • 의미있는 테스트 이름
  • Assert 메시지 포함
  • 커버리지 100%

추가:

  • Given-When-Then 주석
  • 테스트 그룹화 (클래스) """

## 💡 코드 생성 Best Practices

### 1. 점진적 구축

```python
# 1단계: 기본 구조
"""
FastAPI 기본 앱 구조만 작성
- main.py
- 기본 라우터
"""

# 2단계: 모델 추가
"""
위 앱에 User 모델 추가
- SQLAlchemy 모델
- Pydantic 스키마
"""

# 3단계: CRUD 구현
"""
User CRUD 작업 구현
- Create, Read, Update, Delete
"""

# 4단계: 검증 추가
"""
입력 검증 강화
- 이메일 형식
- 비밀번호 강도
"""

2. 명확한 제약사항​

"""
제약사항:
✅ 사용 가능:
- Python 표준 라이브러리
- NumPy, Pandas

❌ 사용 불가:
- 외부 API 호출
- 파일 시스템 접근
- 전역 변수

성능:
- 최대 실행 시간: 1초
- 최대 메모리: 100MB
"""

"""
출력 형식:

## 1. 전체 코드
