AI를 활용한 효과적인 데이터 분석

데이터 분석에서 Prompt Engineering은 탐색적 분석, 시각화, 인사이트 도출을 돕습니다. 이 가이드에서는 다양한 데이터 분석 상황에서 효과적인 프롬프트 작성법을 다룹니다.

📊 시나리오 1: 탐색적 데이터 분석 (EDA)​

"""
다음 데이터셋의 탐색적 분석을 수행하세요:

데이터: sales_data.csv
크기: 10,000 행
컬럼:
- date: 날짜 (YYYY-MM-DD)
- product: 제품명
- category: 카테고리
- quantity: 판매 수량
- price: 단가
- region: 지역

분석 목표:
1. 기본 통계 (평균, 중앙값, 표준편차)
2. 결측치 확인
3. 이상치 탐지
4. 상관관계 분석

출력:
## 1. 데이터 개요
- 행/열 개수
- 데이터 타입
- 메모리 사용량

## 2. 기술 통계
[표 형식]

## 3. 결측치 분석
[컬럼별 결측치 비율]

## 4. 인사이트
- 주요 발견 3가지
- 추가 분석 제안

도구: pandas, matplotlib
시각화: 필요 시 포함
"""

월별 매출 데이터 시계열 분석:

데이터:
| 월 | 매출 (백만원) |
|----|---------------|
| 2024-01 | 150 |
| 2024-02 | 180 |
| 2024-03 | 165 |
| ... | ... |
| 2024-12 | 250 |

분석 작업:
1. 추세 분석

   - 이동 평균 (3개월)

2. 계절성 분석
   - 월별 패턴 확인
   - 분기별 비교

3. 예측
   - 다음 3개월 매출 예측
   - 신뢰 구간 제시

4. 이상 감지
   - 급격한 증감 월 식별
   - 원인 가설 제시

출력 형식:
## 추세
- 회귀식: y = ...
- 월평균 증가율: ...%
- 그래프 (matplotlib 코드)

## 계절성
- 성수기: [월]
- 비수기: [월]
- 분기별 평균

## 예측
| 월 | 예측값 | 하한 | 상한 |
|----|--------|------|------|

## 인사이트
[핵심 발견 + 비즈니스 시사점]

도구: pandas, statsmodels, matplotlib

📊 시나리오 3: 고객 세그멘테이션​

"""
고객 데이터 기반 세그멘테이션:

데이터: customers.csv (5,000명)
- customer_id
- age: 나이
- income: 연소득 (만원)
- purchase_frequency: 연간 구매 횟수
- avg_purchase: 평균 구매금액
- tenure: 가입 기간 (월)

분석 목표:
고객을 3-5개 그룹으로 세분화하여 마케팅 전략 수립

방법:
1. 특성 선택
   - RFM 분석 또는 K-means 클러스터링
   - 주요 변수 선정 (3-4개)

2. 전처리
   - 정규화 (StandardScaler)
   - 이상치 처리

3. 클러스터링
   - 최적 클러스터 수 결정 (엘보우 방법)
   - K-means 실행

4. 세그먼트 프로파일링
   - 각 그룹의 특성
   - 그룹명 제안
   - 크기 및 비율

출력:
## 세그먼트 요약
| 그룹 | 크기 | 특징 | 제안 전략 |
|------|------|------|-----------|
| VIP  | 500명 | 고소득, 고빈도 | ... |
| ... | ... | ... | ... |

## 시각화
- 클러스터 분포도 (2D PCA)
- 특성별 비교 (레이더 차트)

## 마케팅 제안
[그룹별 구체적 전략]

도구: pandas, scikit-learn, matplotlib, seaborn
"""

📊 시나리오 4: A/B 테스트 분석​

"""
웹사이트 A/B 테스트 결과 분석:

실험 설정:
- 대조군 (A): 기존 버튼 디자인
- 실험군 (B): 새 버튼 디자인
- 지표: 클릭률 (CTR)
- 기간: 2주
- 표본 크기: 각 10,000명

데이터:
Group A: 10,000명 중 1,200명 클릭 (12.0%)
Group B: 10,000명 중 1,380명 클릭 (13.8%)

분석 요청:
1. 기술 통계
   - 각 그룹의 CTR
   - 차이 (절대값, 상대값)

2. 통계적 유의성 검정
   - 카이제곱 검정 또는 t-test
   - p-value 계산
   - 신뢰구간 (95%)

3. 실무적 유의성
   - 최소 감지 가능 효과 (MDE)
   - 비즈니스 임팩트 추정

4. 결론 및 추천
   - 통계적 결론
   - 비즈니스 의사결정
   - 추가 실험 제안

출력 형식:
## 결과 요약
- CTR 차이: +1.8%p (+15%)
- p-value: ...
- 통계적 유의성: [유의/비유의]
- 신뢰구간: [하한, 상한]

## 비즈니스 임팩트
- 월간 예상 추가 클릭: ...
- 예상 매출 증가: ...

## 결론
[채택/기각 + 이유]

## 추가 제안
[후속 실험 아이디어]

도구: scipy.stats, pandas
"""

📊 시나리오 5: 대시보드 데이터 준비​

"""
경영진 대시보드용 데이터 가공:

원본 데이터: transactions.csv (50,000 건)
- transaction_id
- date
- customer_id
- product_id
- amount
- status: 'completed', 'pending', 'cancelled'

대시보드 요구사항:
1. KPI 카드
   - 총 매출 (완료 건만)
   - 거래 건수
   - 평균 거래금액
   - 전월 대비 증감률

2. 일별 매출 추이
   - 최근 30일
   - 7일 이동평균 포함

3. Top 10 제품
   - 매출액 기준
   - 거래 건수 포함

4. 상태별 분포
   - 완료/대기/취소 비율
   - 파이 차트 데이터

출력:
## KPI (JSON)
```json
{
  "total_revenue": 150000000,
  "total_transactions": 45000,
  "avg_transaction": 3333,
  "growth_rate": 12.5
}

일별 데이터 (CSV)​

date,revenue,transactions,moving_avg_7d
2024-10-01,5000000,150,4800000
...

Top 10 제품 (Table)​

순위제품매출건수

시각화 코드​

[matplotlib/plotly 코드]

도구: pandas, numpy 형식: JSON + CSV """

"""
❌ 불충분
'이 데이터 분석해줘'

✅ 충분
'전자상거래 월별 매출 데이터 (2년치)
목표: 계절성 패턴 파악 및 내년 Q1 예측
비즈니스 맥락: 재고 계획 수립용'
"""

2. 분석 목표 명확화​

"""
명확한 질문:
- '어떤 제품이 가장 많이 팔리는가?' (기술적)
- '어떤 고객군이 가장 수익성이 높은가?' (분석적)
- '다음 분기 매출은 얼마일까?' (예측적)
"""

3. 단계별 분석​

"""
1단계: 데이터 이해

2단계: 전처리
- 이상치 처리
- 정규화

3단계: 분석
- 모델 적용
- 결과 해석

4단계: 시각화
- 인사이트 표현
"""
