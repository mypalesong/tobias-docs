콘텐츠 작성 시나리오

AI를 활용한 효과적인 콘텐츠 제작

콘텐츠 작성은 Prompt Engineering의 창의적 활용 분야입니다. 이 가이드에서는 블로그, 문서, 마케팅 자료 등 다양한 콘텐츠를 효과적으로 작성하는 방법을 다룹니다.

content_writing_strategy = {

    "스타일 가이드": "어조, 문체, 용어",

✍️ 시나리오 1: 기술 블로그 포스트​

"""
기술 블로그 포스트 작성:

주제: FastAPI로 고성능 API 만들기
독자: 중급 Python 개발자
목적: FastAPI 장점 소개 및 실전 예제 제공
길이: 1,500-2,000 단어

구조:
## 도입부
- FastAPI 간단 소개
- 왜 FastAPI를 선택해야 하는가?

## 핵심 내용
1. FastAPI의 주요 특징 (3가지)
   - 빠른 성능
   - 자동 문서화
   - 타입 안전성

2. 실전 예제
   - 기본 API 구조
   - 데이터 검증 (Pydantic)
   - 비동기 처리

3. 성능 벤치마크
   - Flask와 비교
   - 실제 수치 제시

## 마무리
- 주요 포인트 요약
- 다음 학습 방향 제안

톤앤매너:
- 전문적이지만 친근하게
- 기술 용어 사용하되 설명 포함
- 코드 예제는 실행 가능하게

추가 요구사항:
- 코드 블록에 주석 포함
- 각 섹션에 소제목
- SEO 키워드 자연스럽게 배치
- CTA (Call to Action): 관련 강좌 링크
"""

# 빠른 시작 가이드

## 1. 계정 설정 (5분)
- [ ] 회원가입
- [ ] 이메일 인증
- [ ] 첫 봇 생성

## 2. 챗봇 설정 (10분)
### 기본 정보
- 봇 이름
- 환영 메시지
- 대응 언어

### 학습 데이터
- FAQ 업로드 (CSV)
- 웹사이트 크롤링
- 수동 입력

## 3. 배포 (5분)
- 웹사이트 임베드 코드
- 테스트하기

# 자주 묻는 질문 (FAQ)

Q1: 봇이 답변을 못 하는 경우는?
A: [해결 방법]

Q2: 요금제 변경 방법은?
A: [단계별 설명]

# 문제 해결

문제: 봇이 응답하지 않음
원인: [...]
해결: [...]

스타일:
- 단계별 번호 매기기
- 체크리스트 활용
- 스크린샷 위치 표시
- 주의사항 강조 (⚠️)

형식: Markdown
길이: 2,000 단어 이내
"""

✍️ 시나리오 3: 마케팅 카피​

"""
제품 랜딩 페이지 카피 작성:

제품: AI 기반 코드 리뷰 도구
타겟: 스타트업 개발팀 리더
USP: 리뷰 시간 70% 단축

구성:

## 히어로 섹션
- 헤드라인: 강렬한 가치 제안 (10단어 이내)
- 서브헤드라인: 구체적 설명 (20단어 이내)
- CTA 버튼: 액션 지향적 문구

예시 헤드라인 3가지 제안

## 문제 제기
현재 개발팀이 겪는 고충 3가지
- 느린 코드 리뷰
- 일관성 없는 품질
- 주니어 개발자 온보딩

## 솔루션
우리 제품이 해결하는 방법
- AI 자동 리뷰
- 커스텀 룰셋
- 학습 리소스 제공

## 기능 (3가지)
각 기능마다:
- 아이콘 제안
- 제목 (5단어)
- 설명 (30단어)
- 혜택 강조

## 사회적 증거
- 고객 후기 (가상의 3개)
- 사용 통계 (인상적인 숫자)

## CTA
- 주 CTA: "무료로 시작하기"
- 부 CTA: "데모 보기"

톤:
- 문제에 공감하되
- 솔루션에 자신감
- 기술 용어 최소화
- 혜택 중심 서술

제약:
- 전문 용어 설명 필요 시 각주
- 과장 금지
- 구체적 수치 사용
"""

✍️ 시나리오 4: 이메일 템플릿​

"""
고객 온보딩 이메일 시리즈 작성:

시나리오: SaaS 신규 가입 고객
목표: 활성 사용자로 전환
시리즈: 5통 (1주일)

## 이메일 1: 환영 (가입 직후)
제목: [제품명]에 오신 것을 환영합니다! 🎉

본문:
- 환영 인사
- 첫 단계 안내 (3가지)
- CTA: "시작하기"
- 도움이 필요하면 연락 (Support)

톤: 따뜻하고 친근하게
길이: 150 단어

## 이메일 2: 가이드 (1일 후)
제목: 5분 만에 첫 [기능] 만들기

본문:
- 간단한 튜토리얼
- 비디오/GIF 링크
- 팁 1-2개
- CTA: "직접 해보기"

톤: 교육적, 격려하는
길이: 200 단어

## 이메일 3: 성공 사례 (3일 후)
제목: 다른 팀은 이렇게 사용합니다

본문:
- 고객 성공 사례 (간략히)
- 구체적 성과 (수치)
- 적용 방법
- CTA: "더 알아보기"

톤: 영감을 주는
길이: 180 단어

## 이메일 4: 고급 기능 (5일 후)
제목: 생산성을 2배로: [고급 기능]

본문:
- 고급 기능 소개
- 사용 시점
- 간단한 예제
- CTA: "업그레이드"

톤: 가치 중심
길이: 160 단어

## 이메일 5: 피드백 요청 (7일 후)
제목: 여러분의 의견이 중요합니다

본문:
- 지금까지의 경험 질문
- 설문 링크 (2-3분)
- 감사 표현
- 인센티브 (할인 쿠폰)

톤: 감사하는, 경청하는
길이: 120 단어

공통 요구사항:
- 각 이메일에 명확한 1개 CTA
- 모바일 친화적 (짧은 문단)
- 개인화 포인트 표시 ([이름], [회사])
- 탈퇴 링크 포함
"""
