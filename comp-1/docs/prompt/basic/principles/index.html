핵심 원칙

좋은 프롬프트를 만드는 5가지 핵심 원칙

Prompt Engineering의 성공은 기본 원칙을 얼마나 잘 이해하고 적용하는가에 달려있습니다. 이 가이드에서는 효과적인 프롬프트를 작성하기 위한 5가지 핵심 원칙을 다룹니다.

🎯 학습 목표​

  • 명확성(Clarity)의 중요성과 실제 적용 방법
  • 구체성(Specificity)을 통한 정확한 결과 도출
  • 맥락 제공(Context)으로 AI 이해도 향상
  • 일관성(Consistency)을 유지하는 방법
  • 간결성(Conciseness)과 완전성의 균형

1️⃣ 명확성 (Clarity)​

정의​

명확성은 AI가 요청을 정확히 이해할 수 있도록 모호함 없이 표현하는 것입니다.

왜 중요한가?​

# 문제: 모호한 표현은 예상치 못한 결과를 초래합니다
모호한 프롬프트 → AI의 추측 → 원하지 않는 결과
명확한 프롬프트 → AI의 정확한 이해 → 원하는 결과

실전 예제​

❌ 나쁜 예: 모호함​

"웹사이트 만들어줘"

문제점:

  • 어떤 종류의 웹사이트인지 불명확
  • 기술 스택 미지정
  • 기능 범위 불명확
  • 디자인 요구사항 없음

✅ 좋은 예: 명확함​

## 프로젝트: 개인 블로그 웹사이트 개발

### 요구사항
- **목적**: 기술 블로그 포스팅 및 포트폴리오 전시
- **기술 스택**: Next.js 14, TypeScript, Tailwind CSS
- **핵심 기능**:
  1. 마크다운 기반 블로그 포스팅
  2. 태그 기반 분류 시스템
  3. 검색 기능
  4. 반응형 디자인 (모바일 우선)
- **우선순위**: 블로그 기능부터 구현

### 1단계 요청
먼저 프로젝트 구조와 메인 레이아웃 컴포넌트를 작성해주세요.

개선 포인트:

  • 웹사이트 유형 명시 (기술 블로그)
  • 기술 스택 지정
  • 구체적인 기능 나열
  • 우선순위 표시
  • 단계별 진행 방향 제시

명확성 체크리스트​

clarity_checklist = {
    "목적": "무엇을 만들거나 해결하려는가?",
    "범위": "어디까지 포함되는가?",
    "형식": "어떤 형식으로 출력되어야 하는가?",
    "제약사항": "지켜야 할 조건이 있는가?",
    "우선순위": "어떤 것이 가장 중요한가?"
}

2️⃣ 구체성 (Specificity)​

정의​

구체성은 일반적인 요청 대신 세부적이고 정확한 요구사항을 제공하는 것입니다.

구체성의 차원​

specificity_dimensions = {
    "양적 지표": ["개수", "크기", "시간", "범위"],
    "질적 요구": ["스타일", "톤", "수준", "형식"],
    "기술적 세부": ["버전", "라이브러리", "플랫폼", "환경"],
    "제약사항": ["해야 할 것", "하지 말아야 할 것"]
}

실전 예제​

예제 1: 함수 작성​

# ❌ 일반적
"데이터 정렬하는 함수 만들어줘"

# ✅ 구체적
"""
요구사항: 사용자 객체 리스트를 다중 조건으로 정렬하는 함수

함수 사양:
- 함수명: sort_users
- 매개변수:
  - users: List[User] (User는 name, age, created_at 속성 보유)
  - sort_by: List[Tuple[str, str]] (예: [("age", "desc"), ("name", "asc")])
- 반환값: List[User] (정렬된 새 리스트, 원본 변경 안 함)
- 타입 힌트 필수

- 유효하지 않은 필드명 입력 시 ValueError 발생
- docstring 작성 (Google 스타일)
- 단위 테스트 예제 3개 포함

예시:
users = [User("Alice", 30), User("Bob", 25), User("Alice", 25)]
sorted_users = sort_users(users, [("name", "asc"), ("age", "desc")])
# 결과: [User("Alice", 30), User("Alice", 25), User("Bob", 25)]
"""

3️⃣ 맥락 제공 (Context)​

정의​

맥락은 AI가 요청을 올바르게 이해하고 적절한 답변을 생성하는 데 필요한 배경 정보입니다.

맥락의 유형​

context_types = {
    "상황 맥락": "현재 상황, 문제 발생 배경",
    "기술 맥락": "사용 중인 기술, 환경, 버전",
    "목표 맥락": "최종 목표, 의도, 제약사항",
    "사용자 맥락": "사용자 수준, 선호도, 경험"
}

실전 예제​

예제 1: 버그 수정 요청​

# ❌ 맥락 없음
"""
이 코드 고쳐줘:

def process_data(data):
    return [x * 2 for x in data]
"""

# ✅ 맥락 제공
"""
상황:
- 10만 개의 숫자 데이터를 처리하는 배치 작업
- 현재 실행 시간: 약 30초
- 목표 실행 시간: 5초 이내

환경:
- Python 3.11
- 메모리: 8GB 사용 가능
- CPU: 8코어

현재 코드:
def process_data(data):
    return [x * 2 for x in data]

문제:
- 대용량 데이터 처리 시 성능 저하
- 메모리 사용량이 지나치게 높음 (6GB+)

요청:
메모리 사용량을 2GB 이하로 유지하면서 실행 시간을 5초 이내로
줄일 수 있는 최적화 방법을 제안하고 코드를 수정해주세요.

고려사항:
- 데이터 정확성은 반드시 유지
- NumPy 또는 multiprocessing 사용 가능
- 결과는 리스트 형태로 반환되어야 함
"""

4️⃣ 일관성 (Consistency)​

정의​

일관성은 프롬프트 내에서 용어, 형식, 스타일을 통일하여 혼란을 방지하는 것입니다.

실전 예제​

❌ 비일관적​

"""
사용자 데이터를 처리하는 함수를 만들어주세요.

입력: user object
출력: dict
기능:
- 유저 이름 대문자로 변환
- user_age를 계산
- Return processed data

스타일: camelCase 사용
"""

문제점:

  • 용어 혼재: "사용자" vs "user" vs "유저"
  • 형식 혼재: "user object" vs "dict"
  • 스타일 혼재: snake_case vs camelCase
  • 언어 혼재: 한국어 vs 영어

✅ 일관적​

"""
사용자 데이터를 처리하는 함수를 작성해주세요.

입력: User 객체 (name: str, birth_year: int)
출력: dict
기능:
- 사용자 이름을 대문자로 변환
- 사용자 나이를 계산 (현재 연도 - birth_year)
- 처리된 데이터를 딕셔너리로 반환

코딩 스타일:
- 변수명: snake_case
- 함수명: snake_case
- 타입 힌트 사용
"""

일관성 체크리스트​

consistency_checklist = {
    "용어": "같은 개념은 동일한 단어 사용",
    "형식": "데이터 타입 표현 방식 통일",
    "스타일": "코딩 컨벤션 일관성 유지",
    "언어": "한국어/영어 혼용 최소화",
    "구조": "비슷한 항목은 비슷한 형식으로"
}

5️⃣ 간결성 (Conciseness)​

정의​

간결성은 필요한 정보는 모두 포함하되, 불필요한 내용은 제거하는 것입니다.

균형 잡기​

# ⚠️ 너무 짧음 (정보 부족)
"API 만들어줘"

# ⚠️ 너무 김 (불필요한 정보)
"""
안녕하세요. 저는 FastAPI를 배우고 있는 초보 개발자입니다.
지난주에 파이썬을 시작했고, 오늘 FastAPI에 대해 알게 되었습니다.
제가 만들고 싶은 건 사용자 API인데요, 친구가 API를 만들어보라고
추천했거든요. 근데 어떻게 시작해야 할지 모르겠어요.
아, 그리고 데이터베이스는 PostgreSQL을 쓰려고 하는데,
MySQL도 괜찮을까요? 아니면 MongoDB가 나을까요?
사실 아직 데이터베이스에 대해서도 잘 모릅니다...
[계속 이어짐...]
"""

# ✅ 적절한 길이 (핵심 정보만)
"""
FastAPI로 사용자 CRUD API를 만들어주세요.

환경: Python 3.11, FastAPI, PostgreSQL
엔드포인트:
- POST /users (생성)
- GET /users/{id} (조회)
- PUT /users/{id} (수정)
- DELETE /users/{id} (삭제)

출력:
- 전체 코드
- API 테스트 예제
"""

간결성 향상 기법​

1. 핵심만 남기기​

# Before
"저는 파이썬으로 웹 스크래핑을 해보고 싶은데, 아직 초보라서
어떤 라이브러리를 써야 할지 모르겠고, 설치 방법도 잘 모르겠어요.
