기본 기법

실전에서 바로 사용할 수 있는 핵심 프롬프팅 기법

이 가이드에서는 Prompt Engineering의 기본이 되는 실용적인 기법들을 다룹니다. 각 기법은 즉시 적용 가능하며, 조합하여 사용하면 더욱 강력한 효과를 발휘합니다.

🎯 학습 목표​

  • 명령형 프롬프팅 (Instruction Following)
  • Zero-shot 프롬프팅
  • 구분자와 형식 지정
  • 출력 형식 제어
  • 반복 개선 (Iterative Refinement)
  • 제약 조건 활용

1️⃣ 명령형 프롬프팅 (Instruction Following)​

개념​

AI에게 명확한 지시를 제공하여 원하는 작업을 수행하게 하는 가장 기본적인 기법입니다.

핵심 원칙​

instruction_principles = {
    "명확성": "모호하지 않은 명령어 사용",
    "구체성": "정확한 행동 지시",
    "완전성": "필요한 모든 정보 포함",
    "순서": "단계가 있다면 명확히 표시"
}

"이 JSON 바꿔줘"

2️⃣ Zero-shot 프롬프팅​

개념​

예시 없이 AI의 기존 지식만으로 작업을 수행하게 하는 기법입니다.

언제 사용하는가?​

use_zero_shot_when = {
    "일반적인 작업": "번역, 요약, 기본 코드 작성",
    "간단한 질문": "사실 확인, 정의 요청",
    "표준 형식": "JSON 변환, SQL 쿼리 생성",
    "창의적 작업": "아이디어 생성, 브레인스토밍"
}

5️⃣ 반복 개선 (Iterative Refinement)​

개념​

첫 번째 시도에서 완벽한 결과를 기대하지 않고, 점진적으로 개선해 나가는 기법입니다.

개선 프로세스​

refinement_process = {
    "1단계": "초기 프롬프트 작성 및 실행",
    "2단계": "결과 분석 (무엇이 잘못되었는가?)",
    "3단계": "프롬프트 수정 (구체화, 제약 추가)",
    "4단계": "재실행 및 비교",
    "5단계": "만족할 때까지 반복"
}

6️⃣ 제약 조건 활용​

    "금지": "절대 하지 말아야 할 것",

    "우선순위": "중요도 순서",
    "형식": "특정 형식 준수"
}
