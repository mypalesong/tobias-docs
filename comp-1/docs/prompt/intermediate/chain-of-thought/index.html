Chain of Thought (CoT)

단계별 추론으로 복잡한 문제 해결하기

Chain of Thought (생각의 사슬)는 AI가 최종 답변에 도달하기 전에 중간 추론 단계를 명시적으로 보여주도록 하는 강력한 기법입니다. 특히 복잡한 문제 해결에 효과적입니다.

🎯 학습 목표​

  • Chain of Thought의 개념과 작동 원리
  • Zero-shot CoT vs Few-shot CoT
  • 다양한 문제 유형에 CoT 적용
  • CoT 성능 최적화 방법
  • 실무에서의 CoT 활용

📚 개념 이해​

정의​

chain_of_thought = {
    "정의": "AI가 답을 도출하는 사고 과정을 단계별로 표현하는 기법",
    "핵심": "중간 추론 단계를 명시적으로 보이기",
    "효과": "복잡한 문제의 정확도 향상",
    "적용": "수학, 논리, 추론, 분석 작업"
}

문제: 한 가게에서 사과 5개에 10,000원입니다.

답:
"""

# AI 응답: "24,000원입니다."
# → 답만 제시, 과정 없음

Chain of Thought 프롬프트​

"""
문제: 한 가게에서 사과 5개에 10,000원입니다.
     사과 12개를 사려면 얼마가 필요한가요?

단계별로 풀어주세요:
"""

# AI 응답:
# "1단계: 사과 1개의 가격을 계산합니다.
#  10,000원 ÷ 5개 = 2,000원/개
#
#  2단계: 사과 12개의 가격을 계산합니다.
#  2,000원/개 × 12개 = 24,000원
#
#  답: 24,000원"

# → 추론 과정 명시, 검증 가능

1. Zero-shot CoT​

"""

# 또는

"""
문제: [복잡한 문제]

이 문제를 단계별로 풀어주세요:
"""

실전 예제​

"""
문제:
레스토랑에서 점심 메뉴가 12,000원이고, 저녁 메뉴가 점심보다 30% 비쌉니다.
4명이 저녁을 먹는다면 총 얼마를 내야 하나요?

단계별로 생각해봅시다:
"""

# AI 응답:
"""
1단계: 저녁 메뉴의 가격 증가분 계산
12,000원 × 30% = 3,600원

2단계: 저녁 메뉴 가격 계산
12,000원 + 3,600원 = 15,600원

3단계: 4명의 총 비용 계산
15,600원 × 4명 = 62,400원

답: 62,400원
"""

### 1. 수학 문제

```python
"""
문제:
한 학교의 학생 수가 작년에는 800명이었습니다.
올해는 작년보다 15% 증가했고, 내년에는 올해보다 10% 증가할 예정입니다.
내년 예상 학생 수는?

단계별로 계산해주세요:
"""

# 응답:
"""
1단계: 올해 학생 수 계산
작년 학생 수: 800명
증가율: 15%
증가 인원: 800 × 0.15 = 120명
올해 학생 수: 800 + 120 = 920명

2단계: 내년 학생 수 계산
올해 학생 수: 920명
증가율: 10%
증가 인원: 920 × 0.10 = 92명
내년 학생 수: 920 + 92 = 1,012명

답: 1,012명
"""

2. 논리 추론​

"""
문제:
- 모든 개발자는 컴퓨터를 사용한다
- 철수는 컴퓨터를 사용하지 않는다
- 영희는 개발자다

다음 중 반드시 참인 것은?
A) 철수는 개발자가 아니다
B) 영희는 컴퓨터를 사용한다
C) 컴퓨터를 사용하지 않는 사람은 개발자가 아니다

논리적으로 단계별로 추론해주세요:
"""

# 응답:
"""
1단계: 주어진 명제 정리
명제1: 개발자 → 컴퓨터 사용
명제2: 철수 → 컴퓨터 미사용
명제3: 영희 → 개발자

2단계: 각 선택지 검증

A) 철수는 개발자가 아니다
- 명제1: 개발자면 컴퓨터를 사용해야 함
- 명제2: 철수는 컴퓨터를 사용하지 않음
- 대우: 컴퓨터 미사용 → 개발자 아님
- 결론: 참 ✓

B) 영희는 컴퓨터를 사용한다
- 명제3: 영희는 개발자
- 명제1: 개발자 → 컴퓨터 사용
- 결론: 참 ✓

C) 컴퓨터를 사용하지 않는 사람은 개발자가 아니다
- 이것은 명제1의 대우
- 대우는 원명제와 동치
- 결론: 참 ✓

3단계: 최종 답
모든 선택지가 참이지만, 가장 직접적으로 추론되는 것은 A와 B
"""

3. 코드 디버깅​

"""
다음 코드에 버그가 있습니다. 단계별로 분석하고 수정해주세요:

```python
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)

result = calculate_average([])
print(result)

분석 단계:

  1. 코드 실행 흐름 추적
  2. 잠재적 에러 식별
  3. 에러 발생 조건 설명
  4. 수정 방안 제시
  5. 수정된 코드 작성 """

응답:

""" 1단계: 코드 실행 흐름 추적

  • 빈 리스트 []가 입력으로 전달됨
  • total = 0으로 초기화
  • for 루프는 빈 리스트라 실행 안 됨
  • return total / len(numbers) 실행

2단계: 잠재적 에러 식별

  • len(numbers) = len([]) = 0
  • total / 0 → ZeroDivisionError 발생

3단계: 에러 발생 조건

  • 빈 리스트가 입력될 때
  • 0으로 나누기 시도

4단계: 수정 방안 방안1: 빈 리스트 체크 후 0 또는 None 반환 방안2: 빈 리스트 체크 후 예외 발생 방안3: 기본값 사용

5단계: 수정된 코드 (방안1 적용)

def calculate_average(numbers):
    if not numbers:  # 빈 리스트 체크
        return 0

    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)

result = calculate_average([])
print(result)  # 0

"""


### 4. 데이터 분석

```python
"""
다음 판매 데이터를 분석해주세요:

| 월 | 매출 |
|----|------|
| 1월 | 100 |
| 2월 | 120 |
| 3월 | 110 |
| 4월 | 150 |

다음 순서로 분석하세요:
1. 월별 증감률 계산

3. 추세 분석
4. 5월 예상 매출 추정
"""

# 응답:
"""
1단계: 월별 증감률 계산
- 1월 → 2월: (120-100)/100 = +20%
- 2월 → 3월: (110-120)/120 = -8.3%
- 3월 → 4월: (150-110)/110 = +36.4%

2단계: 평균 매출 계산
- 총 매출: 100 + 120 + 110 + 150 = 480
- 평균: 480 / 4 = 120

3단계: 추세 분석
- 전체적으로 상승 추세 (100 → 150, +50%)
- 3월에 일시적 하락 (계절성 또는 특수 요인 가능)
- 4월에 급등 (프로모션 또는 수요 증가)

4단계: 5월 예상 매출 추정
방법1: 평균 증가율 사용
- 평균 증가율: (20% - 8.3% + 36.4%) / 3 = 16.03%
- 예상: 150 × 1.16 = 174

방법2: 선형 회귀 추세
- 월평균 증가: (150-100) / 3 = 16.67
- 예상: 150 + 16.67 = 167

예상 매출: 167~174 사이
"""
