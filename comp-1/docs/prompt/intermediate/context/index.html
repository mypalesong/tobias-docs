Context Management

적절한 맥락 제공으로 정확한 답변 이끌어내기

Context Management(맥락 관리)는 AI가 작업을 올바르게 이해하고 수행하는 데 필요한 배경 정보를 효과적으로 제공하는 기법입니다. 너무 적거나 많지 않은, "딱 맞는" 맥락을 제공하는 것이 핵심입니다.

🎯 학습 목표​

  • Context의 중요성과 영향
  • Context 유형과 제공 시점
  • 적절한 Context 양 결정
  • Context 구조화 방법
  • 긴 Context 관리 기법

📚 개념 이해​

Context란?​

context = {
    "정의": "AI가 작업을 이해하는 데 필요한 배경 정보",
    "포함": ["상황", "환경", "제약", "목표", "역사"],
    "목적": "AI의 이해도 향상 → 답변 품질 향상",
    "원칙": "필요한 만큼, 명확하게"
}

Context의 영향​

Context 없음​

"""
이 코드를 최적화해주세요:

def process(data):
    result = []
    for item in data:
        if item > 0:
            result.append(item * 2)
    return result
"""

# AI 응답: 일반적인 최적화 (리스트 컴프리헨션 등)
# → 실제 필요한 최적화를 모름

Context 제공​

"""
배경:
- 이 함수는 센서 데이터(초당 100만 개)를 실시간 처리합니다
- 현재 처리 속도: 10만 개/초
- 목표: 100만 개/초 이상
- 메모리: 제한적 (최대 1GB)

환경:
- Python 3.11
- NumPy 사용 가능
- 멀티프로세싱 가능

코드:

    for item in data:
        if item > 0:
            result.append(item * 2)

# AI 응답: NumPy 벡터화, 메모리 효율적 처리 제안

Context의 종류​

context_types = {
