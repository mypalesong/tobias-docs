🤔 OAuth 2.0이란?​

정의​

OAuth 2.0은 사용자가 비밀번호를 공유하지 않고도 다른 애플리케이션에 리소스 접근 권한을 위임할 수 있게 하는 인가(Authorization) 프레임워크입니다.

문제: OAuth 이전​

# ❌ 나쁜 예: 비밀번호 직접 공유
class PhotoPrinterApp:
    def __init__(self):
        # 사용자의 Google 비밀번호를 직접 입력받음
        self.google_password = input("Enter your Google password: ")

    def print_photos(self):
        # Google에 비밀번호로 로그인
        google.login(self.email, self.google_password)
        photos = google.get_photos()
        self.print(photos)

# 문제점:
# 1. 보안 위험 (비밀번호 노출)
# 2. 과도한 권한 (모든 Google 서비스 접근 가능)
# 3. 권한 취소 불가 (비밀번호 변경만 가능)
# 4. 신뢰 문제 (앱을 믿어야 함)

해결: OAuth 2.0​

# ✅ 좋은 예: OAuth 2.0
class PhotoPrinterApp:
    def print_photos(self):
        # 1. Google에 권한 요청
        auth_url = "https://accounts.google.com/oauth/authorize"
        redirect_user_to(auth_url)

        # 2. 사용자가 Google에서 로그인 & 승인
        # 3. Access Token 받음 (비밀번호 X)
        token = receive_token()

        # 4. Token으로 사진만 접근 (제한된 권한)
        photos = google.get_photos(token)
        self.print(photos)

# 장점:
# 1. 비밀번호 공유 안 함
# 2. 필요한 권한만 (사진 접근만)
# 3. 언제든 권한 취소 가능
# 4. 앱을 신뢰할 필요 없음

🎭 주요 역할 (Roles)​

1. Resource Owner (리소스 소유자)​

누구: 사용자 (You)

역할: 리소스의 주인, 권한을 부여하는 사람

# 예: Alice (사용자)
resource_owner = {
    "name": "Alice",
    "email": "alice@example.com",
    "google_photos": [...],  # Alice의 사진들
    "github_repos": [...]    # Alice의 저장소들
}

2. Client (클라이언트)​

무엇: 애플리케이션 (앱, 웹사이트, 모바일 앱)

역할: 사용자를 대신하여 리소스에 접근하려는 애플리케이션

# 예: Photo Printer App
client = {
    "client_id": "photo-printer-12345",
    "client_secret": "secret-key-xyz",
    "redirect_uri": "https://photoprinter.com/callback",
    "requested_scopes": ["read:photos"]
}

3. Authorization Server (인증 서버)​

무엇: Google, GitHub, Facebook 등의 OAuth 서버

역할: 사용자 인증 & Access Token 발급

# 예: Google OAuth Server
authorization_server = {
    "authorize_endpoint": "https://accounts.google.com/o/oauth2/v2/auth",
    "token_endpoint": "https://oauth2.googleapis.com/token",
    "userinfo_endpoint": "https://www.googleapis.com/oauth2/v2/userinfo"
}

4. Resource Server (리소스 서버)​

무엇: 실제 데이터를 가진 API 서버

역할: Access Token을 검증하고 리소스 제공

# 예: Google Photos API
resource_server = {
    "base_url": "https://photoslibrary.googleapis.com",
    "endpoints": {
        "list_photos": "/v1/mediaItems",
        "upload_photo": "/v1/uploads"
    }
}

🔑 주요 구성 요소​

1. Client Credentials​

# 클라이언트 식별 정보
CLIENT_ID = "abc123xyz"           # 공개 정보 (Public)
CLIENT_SECRET = "super-secret"    # 비밀 정보 (Private)

# Client ID: 앱 식별
# Client Secret: 앱 인증 (서버 측 앱만)

2. Redirect URI​

# 인증 완료 후 돌아올 URL
REDIRECT_URI = "https://myapp.com/oauth/callback"

# 보안을 위해 미리 등록 필수
# 다른 URI로 redirect 시도 → 거부

3. Scope (권한 범위)​

# 요청할 권한 목록
scopes = [
    "read:user",       # 사용자 정보 읽기
    "write:repo",      # 저장소 쓰기
    "delete:repo"      # 저장소 삭제
]

# 최소 권한 원칙 (Principle of Least Privilege)
# ✅ 필요한 권한만 요청
scopes = ["read:user"]

# ❌ 과도한 권한 요청
scopes = ["read:user", "write:repo", "delete:repo", "admin"]

4. Tokens​

# Access Token (단기)
access_token = {
    "token": "ya29.a0AfH6SMBx...",
    "expires_in": 3600,  # 1시간
    "token_type": "Bearer",
    "scope": "read:user read:repo"
}

# Refresh Token (장기)
refresh_token = {
    "token": "1//0gkXy9nZ...",
    "expires_in": 2592000  # 30일
}

# ID Token (OIDC only)
id_token = {
    "token": "eyJhbGciOiJSUzI1NiIs...",  # JWT
    "payload": {
        "sub": "1234567890",
        "name": "Alice",
        "email": "alice@example.com"
    }
}

🔄 기본 플로우 (Authorization Code Flow)​

단계별 설명​

# Step 1: 사용자가 로그인 버튼 클릭
@app.get("/login")
def login():
    # Authorization URL 생성
    auth_url = (
        "https://accounts.google.com/o/oauth2/v2/auth"
        f"?client_id={CLIENT_ID}"
        f"&redirect_uri={REDIRECT_URI}"
        "&response_type=code"
        "&scope=openid email profile"
        f"&state={generate_state()}"  # CSRF 방지
    )
    return RedirectResponse(auth_url)

# Step 2: Google 로그인 페이지로 이동
# (사용자가 Google에서 로그인 & 권한 승인)

# Step 3: Callback - Authorization Code 받음
@app.get("/oauth/callback")
def callback(code: str, state: str):
    # State 검증 (CSRF 방지)
    verify_state(state)

    # Step 4: Authorization Code를 Access Token으로 교환
    token_response = requests.post(
        "https://oauth2.googleapis.com/token",
        data={
            "code": code,
            "client_id": CLIENT_ID,
            "client_secret": CLIENT_SECRET,
            "redirect_uri": REDIRECT_URI,
            "grant_type": "authorization_code"
        }
    )

    tokens = token_response.json()
    # {
    #   "access_token": "ya29...",
    #   "refresh_token": "1//0g...",
    #   "id_token": "eyJhbGc...",
    #   "expires_in": 3600
    # }

    # Step 5: Access Token으로 사용자 정보 가져오기
    user_info = requests.get(
        "https://www.googleapis.com/oauth2/v2/userinfo",
        headers={"Authorization": f"Bearer {tokens['access_token']}"}
    ).json()

    return {"user": user_info}

🌍 실생활 비유​

호텔 키카드 시스템​

# 1. Resource Owner: 당신 (호텔 투숙객)
resource_owner = "You"

# 2. Client: 발레 파킹 직원
client = "Valet Parking Service"

# 3. Authorization Server: 호텔 프론트 데스크
authorization_server = "Hotel Front Desk"

# 4. Resource Server: 주차장
resource_server = "Parking Garage"

# 플로우:
# 1. 당신이 발레 파킹 서비스 요청
# 2. 프론트 데스크가 "주차장 전용 키카드" 발급
# 3. 발레 직원이 주차장 키카드로 주차장만 접근
# 4. 객실 키카드 아님 (제한된 권한)
# 5. 체크아웃 시 키카드 자동 비활성화

# OAuth 매핑:
authorization_code = "임시 번호표"
access_token = "주차장 전용 키카드"
scope = "주차장 접근만 (객실 X)"
token_expiration = "체크아웃 시"

📊 OAuth vs 전통적 인증​

특성전통적 인증OAuth 2.0
비밀번호 공유필요불필요
권한 범위전체 접근제한된 접근
권한 취소비밀번호 변경만언제든 취소 가능
보안낮음높음
사용자 경험불편편리
신뢰앱을 완전 신뢰최소 신뢰

💡 핵심 개념 요약​

1. 인가 vs 인증​

# 인가 (Authorization): "무엇을 할 수 있는가?"
# OAuth 2.0의 주요 목적
authorization = {
    "question": "Can this app access my photos?",
    "answer": "Yes, read-only access for 1 hour"
}

# 인증 (Authentication): "누구인가?"
# OpenID Connect의 목적
authentication = {
    "question": "Who is this user?",
    "answer": "Alice (alice@example.com)"
}

2. 왜 Authorization Code를 사용하는가?​

# ❌ 나쁜 방법: Access Token을 URL로 직접 전달
# https://myapp.com/callback?access_token=secret-token
# 문제: 브라우저 히스토리, 프록시 로그에 노출

# ✅ 좋은 방법: Authorization Code 사용
# https://myapp.com/callback?code=temporary-code
# 장점:
# 1. Code는 일회용 (한 번만 사용 가능)
# 2. Code는 짧은 수명 (10분)
# 3. Client Secret로 보호 (서버에서만 Token 교환)

3. State Parameter의 중요성​

# CSRF 공격 방지
import secrets

# 요청 시
state = secrets.token_urlsafe(32)
save_to_session(state)

auth_url = f"...&state={state}"

# 콜백 시
if request.args['state'] != get_from_session():
    raise SecurityError("CSRF attack detected!")

🎯 실전 예제​

GitHub으로 로그인​

from fastapi import FastAPI, Request
from authlib.integrations.starlette_client import OAuth

app = FastAPI()
oauth = OAuth()

# GitHub OAuth 앱 등록
oauth.register(
    name='github',
    client_id='YOUR_CLIENT_ID',
    client_secret='YOUR_CLIENT_SECRET',
    authorize_url='https://github.com/login/oauth/authorize',
    access_token_url='https://github.com/login/oauth/access_token',
    api_base_url='https://api.github.com/',
    client_kwargs={'scope': 'user:email'}
)

@app.get('/login')
async def login(request: Request):
    """Step 1: GitHub 로그인 페이지로 리다이렉트"""
    redirect_uri = request.url_for('auth')
    return await oauth.github.authorize_redirect(request, redirect_uri)

@app.get('/auth')
async def auth(request: Request):
    """Step 2: Authorization Code 받고 Token 교환"""
    token = await oauth.github.authorize_access_token(request)

    # Step 3: Token으로 사용자 정보 가져오기
    resp = await oauth.github.get('user', token=token)
    user = resp.json()

    return {
        'username': user['login'],
        'email': user['email'],
        'avatar': user['avatar_url']
    }
