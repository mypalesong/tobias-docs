용어 정리

OAuth 2.0과 OpenID Connect에서 사용되는 주요 용어들을 정리합니다.

🎯 목표​

  • ✅ OAuth/OIDC 핵심 용어 이해
  • ✅ 역할과 구성 요소 구분
  • ✅ 토큰 유형 파악
  • ✅ 보안 개념 숙지

👥 역할 (Roles)​

Resource Owner​

리소스 소유자 - 보호된 리소스에 대한 접근 권한을 부여할 수 있는 주체

# 일반적으로 사용자 (You)
resource_owner = {
    "정의": "리소스의 주인",
    "권한": "리소스 접근 권한 부여/거부",
    "예시": "Google 계정을 가진 사용자",
    "역할": "클라이언트에게 권한 승인"
}

# 실제 예
# Alice가 Photo Printer 앱에게 자신의 Google Photos 접근 권한을 부여

Client​

클라이언트 - Resource Owner를 대신하여 보호된 리소스에 접근을 요청하는 애플리케이션

client = {
    "정의": "리소스 접근을 요청하는 앱",
    "예시": "Photo Printer 앱, Notion, Slack",
    "유형": {
        "Confidential Client": "서버 측 앱 (Client Secret 보관 가능)",
        "Public Client": "SPA, 모바일 앱 (Client Secret 보관 불가)"
    }
}

# Client Types
confidential_client = {
    "예시": "FastAPI 웹 애플리케이션",
    "특징": "서버에서 실행, Client Secret 안전하게 보관",
    "플로우": "Authorization Code Flow"
}

public_client = {
    "예시": "React SPA, React Native 앱",
    "특징": "브라우저/모바일에서 실행, Client Secret 노출 위험",
    "플로우": "Authorization Code + PKCE"
}

Authorization Server​

인증 서버 - Resource Owner 인증 및 Access Token 발급

authorization_server = {
    "정의": "사용자 인증 및 토큰 발급 서버",
    "예시": "Google OAuth Server, GitHub OAuth Server",
    "역할": [
        "사용자 인증 (로그인)",
        "권한 승인 확인",
        "Authorization Code 발급",
        "Access Token 발급",
        "Refresh Token 발급"
    ],
    "엔드포인트": {
        "Authorization Endpoint": "/authorize",
        "Token Endpoint": "/token",
        "Revocation Endpoint": "/revoke"
    }
}

# 예: Google
google_auth_server = {
    "authorization_endpoint": "https://accounts.google.com/o/oauth2/v2/auth",
    "token_endpoint": "https://oauth2.googleapis.com/token",
    "revocation_endpoint": "https://oauth2.googleapis.com/revoke"
}

Resource Server​

리소스 서버 - 보호된 리소스를 호스팅하는 서버

resource_server = {
    "정의": "실제 데이터를 가진 API 서버",
    "예시": "Google Photos API, GitHub API",
    "역할": [
        "Access Token 검증",
        "요청된 리소스 제공",
        "권한 확인 (Scope)"
    ]
}

# 예: Google Photos API
google_photos_api = {
    "base_url": "https://photoslibrary.googleapis.com",
    "endpoint": "/v1/mediaItems",
    "required_token": "Access Token",
    "required_scope": "https://www.googleapis.com/auth/photoslibrary.readonly"
}

# Authorization Server와 Resource Server는 같을 수도, 다를 수도 있음
# Google: 같은 조직, 다른 서버
# 자체 구현: 같은 서버에 다른 엔드포인트

🔐 인증 요소 (Credentials)​

Client ID​

클라이언트 식별자 - 클라이언트를 고유하게 식별하는 공개 문자열

client_id = {
    "정의": "클라이언트 앱의 고유 ID",
    "특징": "공개 정보 (Public)",
    "형식": "문자열 (UUID 형태가 일반적)",
    "예시": "abc123xyz456",
    "용도": "클라이언트 식별, Authorization Request에 포함"
}

# 예시
GOOGLE_CLIENT_ID = "123456789-abc.apps.googleusercontent.com"
GITHUB_CLIENT_ID = "Iv1.a1b2c3d4e5f6g7h8"

# Authorization URL에 포함
# https://accounts.google.com/o/oauth2/v2/auth?client_id=123456789-abc...

Client Secret​

클라이언트 비밀키 - 클라이언트를 인증하는 비밀 문자열

client_secret = {
    "정의": "클라이언트 앱의 비밀 키",
    "특징": "비밀 정보 (Private), 절대 노출 금지",
    "용도": "Token Endpoint에서 클라이언트 인증",
    "보관": "서버 환경변수, 비밀 관리 시스템"
}

# ⚠️ 주의사항
warnings = {
    "절대 노출 금지": "GitHub에 커밋 금지!",
    "서버만": "브라우저/모바일 앱에서 사용 금지",
    "환경변수": ".env 파일 사용, .gitignore에 추가"
}

# .env 파일
# GOOGLE_CLIENT_SECRET=your-secret-here
# GITHUB_CLIENT_SECRET=your-secret-here

# ❌ 나쁜 예
# const CLIENT_SECRET = "abc123"  // 코드에 하드코딩

# ✅ 좋은 예
# import os
# CLIENT_SECRET = os.getenv("CLIENT_SECRET")

🎫 토큰 (Tokens)​

Authorization Code​

인가 코드 - 임시 코드, Access Token으로 교환

authorization_code = {
    "정의": "Access Token 교환용 임시 코드",
    "수명": "짧음 (보통 10분)",
    "사용": "1회만 (재사용 불가)",
    "전달": "URL 쿼리 파라미터 (Redirect)",
    "교환": "Token Endpoint에서 Access Token으로 교환"
}

# 플로우
# 1. Authorization Server → Client (URL로 전달)
#    https://myapp.com/callback?code=AUTH_CODE_HERE

# 2. Client → Token Endpoint (Code + Secret)
#    POST /token
#    code=AUTH_CODE_HERE&client_id=...&client_secret=...

# 3. Token Endpoint → Client (Access Token 반환)
#    {"access_token": "...", "refresh_token": "..."}

# 보안: Client Secret과 함께 사용하여 보호

Access Token​

액세스 토큰 - 보호된 리소스에 접근하기 위한 자격증명

access_token = {
    "정의": "API 접근용 토큰",
    "수명": "짧음 (15분 ~ 1시간)",
    "형식": "Opaque String 또는 JWT",
    "용도": "Resource Server API 호출",
    "전달": "Authorization Header (Bearer)"
}

# 사용 예
import httpx

headers = {
    "Authorization": f"Bearer {access_token}"
}

response = httpx.get(
    "https://api.github.com/user",
    headers=headers
)

# Access Token 구조 (Opaque)
opaque_token = {
    "토큰": "ya29.a0AfH6SMBx...",
    "특징": "불투명, 내용 확인 불가",
    "검증": "Resource Server가 Authorization Server에 확인"
}

# Access Token 구조 (JWT)
jwt_token = {
    "토큰": "eyJhbGciOiJSUzI1NiIs...",
    "특징": "자체 검증 가능 (서명 확인)",
    "내용": "Payload에 권한 정보 포함"
}

# 보안: HTTPS 필수, 짧은 수명

Refresh Token​

리프레시 토큰 - 만료된 Access Token을 갱신하기 위한 토큰

refresh_token = {
    "정의": "Access Token 갱신용 토큰",
    "수명": "김 (수일 ~ 수개월)",
    "용도": "새로운 Access Token 발급",
    "보안": "매우 중요, 안전하게 보관",
    "제한": "일부 플로우에서만 제공"
}

# Refresh Token 사용
refresh_flow = {
    "1. Access Token 만료": "API 호출 시 401 Unauthorized",
    "2. Refresh Token 사용": "Token Endpoint에 Refresh Token 전송",
    "3. 새 Access Token 받음": "사용자 재로그인 없이"
}

# 코드 예시
async def refresh_access_token(refresh_token: str):
    response = await httpx.post(
        "https://oauth2.googleapis.com/token",
        data={
            "grant_type": "refresh_token",
            "refresh_token": refresh_token,
            "client_id": CLIENT_ID,
            "client_secret": CLIENT_SECRET
        }
    )
    return response.json()["access_token"]

# 보안: Rotation 권장 (매번 새 Refresh Token 발급)

ID Token (OIDC)​

ID 토큰 - 사용자 인증 정보를 담은 JWT

id_token = {
    "정의": "사용자 신원 정보를 담은 JWT",
    "형식": "JWT (JSON Web Token)",
    "용도": "사용자가 누구인지 확인 (인증)",
    "포함 정보": "sub, email, name, picture 등",
    "검증": "클라이언트가 서명 검증"
}

# ID Token vs Access Token
comparison = {
    "ID Token": {
        "목적": "사용자 인증 (Who)",
        "사용처": "클라이언트 앱",
        "형식": "JWT (항상)",
        "내용": "사용자 정보"
    },
    "Access Token": {
        "목적": "API 접근 권한 (What)",
        "사용처": "Resource Server",
        "형식": "Opaque 또는 JWT",
        "내용": "권한 정보"
    }
}

# ID Token 구조
id_token_example = {
    "header": {
        "alg": "RS256",
        "kid": "1234"
    },
    "payload": {
        "iss": "https://accounts.google.com",  # 발급자
        "sub": "10769150350006150715113082367",  # 사용자 ID
        "aud": "your-client-id",  # 수신자
        "exp": 1701234567,  # 만료 시간
        "iat": 1701230967,  # 발급 시간
        "email": "alice@example.com",
        "name": "Alice Smith"
    },
    "signature": "..."
}

# ⚠️ 주의: ID Token은 API 호출에 사용하지 않음!

🎚️ 권한 (Scopes)​

Scope​

스코프 - 클라이언트가 요청하는 권한의 범위

scope = {
    "정의": "권한의 범위, 접근 가능한 리소스",
    "형식": "공백으로 구분된 문자열",
    "예시": "read:user write:repo",
    "원칙": "최소 권한 (Least Privilege)"
}

# OAuth 2.0 Scopes (GitHub 예시)
github_scopes = {
    "user": "사용자 정보 읽기",
    "user:email": "이메일 주소 읽기",
    "repo": "저장소 전체 접근",
    "public_repo": "공개 저장소만 접근",
    "delete_repo": "저장소 삭제",
    "gist": "Gist 접근"
}

# OIDC Standard Scopes
oidc_scopes = {
    "openid": "OIDC 활성화 (필수)",
    "profile": "프로필 정보 (name, picture 등)",
    "email": "이메일 주소",
    "address": "주소",
    "phone": "전화번호"
}

# 사용 예
# ✅ 최소 권한
scopes = "openid email"

# ❌ 과도한 권한
scopes = "openid profile email address phone repo delete_repo"

# Authorization URL에 포함
# https://accounts.google.com/o/oauth2/v2/auth?scope=openid%20email

claims = {

# Standard Claims
standard_claims = {

    "website": "웹사이트",

    "email_verified": "이메일 인증 여부 (boolean)",
    "gender": "성별",
    "birthdate": "생년월일 (YYYY-MM-DD)",
    "zoneinfo": "시간대 (Europe/Paris)",

scope_to_claims = {

🔗 엔드포인트 (Endpoints)​

Authorization Endpoint​

인증 엔드포인트 - 사용자 인증 및 권한 승인을 처리

authorization_endpoint = {
    "정의": "사용자 로그인 & 권한 승인 페이지",
    "메서드": "GET (redirect)",
    "위치": "Authorization Server",
    "예시": "https://accounts.google.com/o/oauth2/v2/auth"
}

# 요청 파라미터
auth_params = {
    "client_id": "필수 - 클라이언트 ID",
    "redirect_uri": "필수 - 콜백 URL",
    "response_type": "필수 - 'code' (Authorization Code Flow)",
    "scope": "필수 - 요청 권한",
    "state": "권장 - CSRF 방지",
    "code_challenge": "PKCE - Code Challenge",
    "code_challenge_method": "PKCE - 'S256'"
}

# 예시 URL
example_url = (
    "https://accounts.google.com/o/oauth2/v2/auth"
    "?client_id=abc123"
    "&redirect_uri=https://myapp.com/callback"
    "&response_type=code"
    "&scope=openid email"
    "&state=random-state"
)

Token Endpoint​

토큰 엔드포인트 - Access Token 발급 및 갱신

token_endpoint = {
    "정의": "토큰 발급/갱신 엔드포인트",
    "메서드": "POST",
    "인증": "Client ID + Secret (Confidential Client)",
    "예시": "https://oauth2.googleapis.com/token"
}

# Authorization Code → Access Token
token_request = {
    "grant_type": "authorization_code",
    "code": "authorization-code-here",
    "client_id": "your-client-id",
    "client_secret": "your-client-secret",
    "redirect_uri": "https://myapp.com/callback",
    "code_verifier": "pkce-verifier (PKCE)"
}

# Refresh Token → New Access Token
refresh_request = {
    "grant_type": "refresh_token",
    "refresh_token": "refresh-token-here",
    "client_id": "your-client-id",
    "client_secret": "your-client-secret"
}

# 응답
token_response = {
    "access_token": "ya29.a0AfH6SMBx...",
    "refresh_token": "1//0gkXy9nZ...",
    "id_token": "eyJhbGciOiJSUzI1NiIs...",  # OIDC only
    "token_type": "Bearer",
    "expires_in": 3600  # seconds
}

UserInfo Endpoint (OIDC)​

사용자 정보 엔드포인트 - 사용자 프로필 정보 제공

userinfo_endpoint = {
    "정의": "사용자 추가 정보 조회",
    "메서드": "GET",
    "인증": "Access Token (Bearer)",
    "예시": "https://www.googleapis.com/oauth2/v2/userinfo"
}

# 요청
headers = {"Authorization": f"Bearer {access_token}"}
response = httpx.get(
    "https://www.googleapis.com/oauth2/v2/userinfo",
    headers=headers
)

# 응답
userinfo_response = {
    "sub": "10769150350006150715113082367",
    "name": "Alice Smith",
    "given_name": "Alice",
    "family_name": "Smith",
    "picture": "https://lh3.googleusercontent.com/...",
    "email": "alice@example.com",
    "email_verified": True,
    "locale": "en"
}

# ID Token vs UserInfo Endpoint
when_to_use = {
    "ID Token": "기본 정보만 필요 (빠름, 토큰에 포함)",
    "UserInfo Endpoint": "최신 정보 또는 추가 정보 필요 (API 호출)"
}

Revocation Endpoint​

토큰 폐기 엔드포인트 - 토큰 무효화

revocation_endpoint = {
    "정의": "토큰 무효화 (로그아웃)",
    "메서드": "POST",
    "예시": "https://oauth2.googleapis.com/revoke"
}

# Access Token 또는 Refresh Token 폐기
revoke_request = {
    "token": "token-to-revoke",
    "token_type_hint": "access_token"  # 또는 "refresh_token"
}

# FastAPI 예시
@app.post('/logout')
async def logout(access_token: str):
    async with httpx.AsyncClient() as client:
        await client.post(
            'https://oauth2.googleapis.com/revoke',
            data={'token': access_token}
        )
    return {"message": "Logged out"}

🔐 보안 개념​

State Parameter​

상태 파라미터 - CSRF 공격 방지

state = {
    "정의": "CSRF 공격 방지용 랜덤 문자열",
    "생성": "요청 시 생성, 세션에 저장",
    "검증": "콜백 시 세션 값과 비교",
    "필수": "강력 권장"
}

# 사용 예
import secrets

# 1. Authorization Request 전
state = secrets.token_urlsafe(32)
session['oauth_state'] = state

auth_url = f"...&state={state}"

# 2. Callback에서 검증
if request.args['state'] != session.get('oauth_state'):
    raise SecurityError("CSRF attack detected!")

# CSRF 공격 예
csrf_attack = {
    "공격자": "악의적인 Authorization URL 생성",
    "피해자": "클릭 시 공격자의 계정과 연결됨",
    "방어": "State parameter로 요청 출처 검증"
}

PKCE (Proof Key for Code Exchange)​

코드 교환 증명 키 - Authorization Code 탈취 방지

pkce = {
    "정의": "Authorization Code 보호 메커니즘",
    "구성": "Code Verifier + Code Challenge",
    "알고리즘": "SHA-256",
    "필수": "Public Clients (SPA, Mobile)"
}

# Code Verifier
code_verifier = {
    "정의": "랜덤 문자열 (43-128자)",
