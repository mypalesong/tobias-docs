🚀 OAuth 2.0 vs OpenID Connect​

OAuth 2.0​

목적: 권한 위임 (Authorization)

# OAuth 2.0: "이 앱이 내 사진에 접근하도록 허용"
# 사용자가 리소스 접근 권한을 앱에 위임

사용 사례:

  • Google Drive 파일 접근
  • GitHub 저장소 읽기/쓰기
  • Twitter 트윗 게시

OpenID Connect (OIDC)​

목적: 인증 (Authentication) + 권한 위임

# OIDC: "Google 계정으로 로그인"
# 사용자 신원 확인 + 기본 프로필 정보 제공

사용 사례:

  • 소셜 로그인 (Google, GitHub, Facebook)
  • Single Sign-On (SSO)
  • 사용자 프로필 정보 가져오기

🔄 OAuth 2.0 플로우 개요​

Authorization Code Flow (가장 일반적)​

User                    Client App              Auth Server         Resource Server
 |                          |                        |                      |
 |  1. 로그인 버튼 클릭       |                        |                      |
 |------------------------->|                        |                      |
 |                          |  2. 인증 요청           |                      |
 |                          |----------------------->|                      |
 |  3. 로그인 페이지         |                        |                      |
 |<-------------------------------------------------|                      |
 |  4. 로그인 & 권한 승인     |                        |                      |
 |------------------------------------------------->|                      |
 |                          |  5. Authorization Code |                      |
 |                          |<-----------------------|                      |
 |                          |  6. Code + Client Secret                     |
 |                          |----------------------->|                      |
 |                          |  7. Access Token       |                      |
 |                          |<-----------------------|                      |
 |                          |  8. API 요청 (Token)   |                      |
 |                          |----------------------------------------------->|
 |                          |  9. 리소스 (데이터)    |                      |
 |                          |<-----------------------------------------------|

🌟 빠른 시작​

5분 만에 Google 로그인 구현​

from fastapi import FastAPI, Depends
from authlib.integrations.starlette_client import OAuth
from starlette.config import Config

app = FastAPI()

# OAuth 설정
config = Config('.env')
oauth = OAuth(config)

oauth.register(
    name='google',
    client_id=config('GOOGLE_CLIENT_ID'),
    client_secret=config('GOOGLE_CLIENT_SECRET'),
    server_metadata_url='https://accounts.google.com/.well-known/openid-configuration',
    client_kwargs={'scope': 'openid email profile'}
)

@app.get('/login')
async def login(request: Request):
    """Google 로그인 시작"""
    redirect_uri = request.url_for('auth')
    return await oauth.google.authorize_redirect(request, redirect_uri)

@app.get('/auth')
async def auth(request: Request):
    """콜백: Google에서 돌아옴"""
    token = await oauth.google.authorize_access_token(request)
    user = token.get('userinfo')

    return {
        'email': user['email'],
        'name': user['name'],
        'picture': user['picture']
    }

완료! 🎉

📖 주요 개념 미리보기​

1. Roles (역할)​

  • Resource Owner: 사용자 (리소스 소유자)
  • Client: 앱 (리소스 접근을 요청하는 애플리케이션)
  • Authorization Server: 인증 서버 (Google, GitHub 등)
  • Resource Server: 리소스 서버 (API 서버)

2. Tokens (토큰)​

# Access Token (짧은 수명, 15분~1시간)
{
    "access_token": "eyJhbGc...",
    "token_type": "Bearer",
    "expires_in": 3600
}

# Refresh Token (긴 수명, 수일~수개월)
{
    "refresh_token": "def50200...",
    "expires_in": 2592000  # 30일
}

# ID Token (OIDC 전용, 사용자 정보 포함)
{
    "id_token": "eyJhbGc...",  # JWT
    "sub": "1234567890",
    "email": "user@example.com",
    "name": "John Doe"
}

3. Scopes (권한 범위)​

# OAuth 2.0 Scopes
scopes = [
    "read:user",      # 사용자 정보 읽기
    "repo",           # 저장소 접근
    "gist"            # Gist 접근
]

# OIDC Scopes
scopes = [
    "openid",         # 필수 (OIDC 활성화)
    "profile",        # 프로필 정보
    "email"           # 이메일 주소
]

🔐 보안 핵심 개념​

State Parameter (CSRF 방지)​

# 요청 시 랜덤 state 생성
state = secrets.token_urlsafe(32)
save_state(state)  # 세션에 저장

# 콜백 시 state 검증
if request_state != saved_state:
    raise HTTPException(status_code=400, detail="Invalid state")

PKCE (Proof Key for Code Exchange)​

# Code Verifier 생성
code_verifier = secrets.token_urlsafe(32)

# Code Challenge 생성
code_challenge = base64.urlsafe_b64encode(
    hashlib.sha256(code_verifier.encode()).digest()
).decode().rstrip('=')
