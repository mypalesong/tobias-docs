GitHub Issues

Issues는 버그 추적, 기능 요청, 작업 관리를 위한 GitHub의 강력한 도구입니다.

Issue란?​

Issue는 프로젝트의 작업, 버그, 기능 요청 등을 추적하는 티켓 시스템입니다. 팀원들과 소통하고 작업을 조직화하는 데 사용됩니다.

Issue 생성​

웹 UI에서​

  1. GitHub 저장소의 "Issues" 탭 클릭
  2. "New issue" 버튼 클릭
  3. 제목과 설명 작성
  4. "Submit new issue" 클릭

GitHub CLI로​

# 간단한 issue 생성
gh issue create --title "Bug: Login fails" --body "Users cannot login"

# 대화형으로 생성
gh issue create

# 웹 브라우저로 열기
gh issue create --web

# 라벨과 담당자 지정
gh issue create --title "Feature: Dark mode" --label "enhancement" --assignee username

좋은 Issue 작성법​

버그 리포트​

**버그 설명**
로그인 버튼 클릭 시 아무 반응이 없습니다.

**재현 방법**
1. 홈페이지 방문
2. 이메일과 비밀번호 입력
3. "로그인" 버튼 클릭
4. 아무 일도 일어나지 않음

**예상 동작**
로그인 후 대시보드로 이동해야 합니다.

**실제 동작**
버튼 클릭 시 아무 반응이 없습니다.

**환경**
- OS: Windows 11
- 브라우저: Chrome 120
- 버전: v1.2.3

**스크린샷**
[스크린샷 첨부]

**추가 정보**
콘솔에 다음 에러가 표시됩니다:
`TypeError: Cannot read property 'token' of undefined`

기능 요청​

**기능 설명**
다크 모드를 지원했으면 좋겠습니다.

**동기**
밤에 작업할 때 눈이 부시기 때문입니다.

**제안하는 해결책**
헤더에 테마 전환 버튼을 추가하고, 사용자 설정을 localStorage에 저장합니다.

**대안**
OS의 다크 모드 설정을 자동으로 감지합니다.

**추가 컨텍스트**
대부분의 최신 웹사이트가 다크 모드를 지원합니다.

Issue 관리​

라벨 활용​

# 라벨 추가
gh issue edit 123 --add-label "bug,high-priority"

# 라벨 제거
gh issue edit 123 --remove-label "bug"

# 라벨 목록 보기
gh label list

일반적인 라벨:

  • bug: 버그
  • enhancement: 기능 개선
  • documentation: 문서 관련
  • good first issue: 초보자에게 적합
  • help wanted: 도움 필요
  • wontfix: 수정하지 않음
  • duplicate: 중복
  • invalid: 유효하지 않음

담당자 지정​

# 담당자 추가
gh issue edit 123 --add-assignee username

# 본인에게 할당
gh issue edit 123 --add-assignee @me

# 담당자 제거
gh issue edit 123 --remove-assignee username

마일스톤 설정​

# 마일스톤 생성
gh api repos/:owner/:repo/milestones -f title="v2.0" -f description="Version 2.0 release"

# issue에 마일스톤 지정
gh issue edit 123 --milestone "v2.0"

Issue 상태​

# Issue 닫기
gh issue close 123

# 코멘트와 함께 닫기
gh issue close 123 --comment "Fixed in PR #124"

# Issue 다시 열기
gh issue reopen 123

Issue와 PR 연결​

자동 닫기 키워드​

커밋 메시지나 PR 설명에 다음 키워드 사용:

# 커밋 메시지에
git commit -m "fix: resolve login issue

Closes #123"

# PR 설명에
Fixes #123
Resolves #456
Closes #789

키워드:

  • Closes #123
  • Fixes #123
  • Resolves #123
  • Close #123, Fix #123, Resolve #123

수동 연결​

PR에서 "Development" 섹션을 통해 Issue 연결 가능

Issue 검색​

웹 UI 필터​

is:open label:bug
is:closed author:username
is:issue assignee:@me
label:"good first issue"

CLI 검색​

# 열린 issue 목록
gh issue list

# 내가 할당된 issue
gh issue list --assignee @me

# 라벨로 필터링
gh issue list --label bug

# 상태별
gh issue list --state closed

# 검색
gh issue list --search "login"

실습 예제​

예제 1: 버그 리포트 생성​

# 버그 issue 생성
gh issue create \
  --title "Bug: Login button not working" \
  --body "$(cat <<EOF
## 버그 설명
로그인 버튼 클릭 시 응답 없음

## 재현 방법
1. 로그인 페이지 방문
2. 자격 증명 입력
3. 로그인 버튼 클릭

## 환경
- Browser: Chrome 120
- OS: Windows 11
EOF
)" \
  --label "bug" \
  --assignee "@me"

예제 2: Issue에서 브랜치 생성​

# 1. Issue 번호 확인
gh issue list

# 2. Issue에서 브랜치 생성
gh issue develop 123 --checkout

# 또는 수동으로
git checkout -b fix/issue-123-login-bug

# 3. 작업 후 커밋
git commit -m "fix: resolve login button issue

Fixes #123"

# 4. PR 생성
gh pr create --title "Fix login button issue" --body "Closes #123"

예제 3: Issue 토론 참여​

# 코멘트 추가
gh issue comment 123 --body "이 문제를 재현했습니다. Chrome에서만 발생하네요."

# 코멘트 목록 보기
gh issue view 123 --comments

# 이모지 반응 추가
gh api repos/:owner/:repo/issues/comments/:comment_id/reactions \
  -f content="+1"

Issue 템플릿 선택​

여러 템플릿이 있을 때:

# .github/ISSUE_TEMPLATE/config.yml
blank_issues_enabled: false
contact_links:
  - name: 질문하기
    url: https://discussions.github.com/
    about: 질문은 Discussions에서 해주세요

Saved Replies​

자주 사용하는 답변을 저장:

  1. Settings → Saved replies
  2. "Add a saved reply" 클릭
  3. 제목과 내용 작성
# 예: "더 많은 정보 필요"
더 많은 정보를 제공해주시겠어요?

다음 정보가 필요합니다:
- OS와 버전
- 브라우저와 버전
- 재현 방법
- 에러 메시지

Discussions와의 차이​

FeatureIssuesDiscussions
용도버그, 작업 추적질문, 토론
구조선형스레드형
상태Open/ClosedOpen/Answered
담당자지정 가능없음

프로젝트 관리 팁​

1. 라벨 체계 정립​

타입:
- bug (🐛)
- enhancement (✨)
- documentation (📚)

우선순위:
- priority-high (🔴)
- priority-medium (🟡)
- priority-low (🟢)

상태:
- in-progress (🔄)
- blocked (⛔)
- needs-review (👀)

2. Issue 번호로 브랜치 명명​

git checkout -b feature/123-dark-mode
git checkout -b fix/456-login-bug

3. 주기적인 Issue 정리​

# 오래된 issue 찾기
gh issue list --state open --label "needs-info" --json number,createdAt

# 중복 issue 닫기
gh issue close 123 --comment "Duplicate of #124"
