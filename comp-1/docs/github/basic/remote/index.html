원격 저장소 작업

원격 저장소와 로컬 저장소를 동기화하는 방법을 배웁니다.

Pull vs Fetch​

git fetch - 데이터만 가져오기​

# 모든 원격 브랜치의 최신 정보 가져오기
git fetch

# 특정 원격 저장소에서만
git fetch origin

# 특정 브랜치만
git fetch origin main

fetch는 원격 저장소의 변경사항을 가져오기만 하고 병합하지 않습니다.

git pull - 가져오기 + 병합​

# 현재 브랜치의 원격 변경사항 가져오기 및 병합
git pull

# 특정 브랜치에서
git pull origin main

# rebase로 병합
git pull --rebase

pull은 fetch + merge와 동일합니다.

# 다음 두 명령은 동일함
git pull

git fetch
git merge origin/main

기본 워크플로우​

1. 원격 저장소의 최신 상태 확인​

# 원격 저장소 정보 가져오기
git fetch

# 로컬과 원격의 차이 확인
git log HEAD..origin/main --oneline

# 또는
git diff HEAD origin/main

2. 변경사항 가져오기​

# Pull (권장)
git pull

# Fetch + Merge (세밀한 제어 필요 시)
git fetch
git merge origin/main

3. 작업 후 푸시​

# 변경사항 푸시
git push

# 새 브랜치 푸시
git push -u origin feature-branch

Pull 전략​

Fast-forward (기본값)​

git pull

로컬에 새 커밋이 없으면 fast-forward로 병합됩니다.

Merge​

# 병합 커밋 생성
git pull --no-ff

# 항상 merge 사용하도록 설정
git config pull.rebase false

Rebase​

# Rebase로 병합
git pull --rebase

# 항상 rebase 사용하도록 설정
git config pull.rebase true
팁

rebase는 깔끔한 히스토리를 유지하지만, 충돌 해결이 더 복잡할 수 있습니다.

충돌 해결​

Pull 중 충돌 발생 시​

# 1. Pull 시도
git pull
# Auto-merging file.txt
# CONFLICT (content): Merge conflict in file.txt

# 2. 충돌 파일 확인
git status

# 3. 충돌 해결
# 파일을 열어 수정
# <<<<<<< HEAD
# 로컬 변경사항
# =======
# 원격 변경사항
# >>>>>>> origin/main

# 4. 해결 후 스테이징
git add file.txt

# 5. 병합 완료
git commit

Rebase 중 충돌 발생 시​

# 1. Rebase 시도
git pull --rebase
# CONFLICT (content): Merge conflict in file.txt

# 2. 충돌 해결
# 파일 수정

# 3. 스테이징
git add file.txt

# 4. Rebase 계속
git rebase --continue

# 또는 취소
git rebase --abort

고급 Pull 사용법​

Autostash​

작업 중인 변경사항을 자동으로 stash했다가 복원합니다.

git pull --autostash

# 또는 기본값으로 설정
git config --global pull.rebase true
git config --global rebase.autostash true

특정 깊이만 가져오기​

# 최근 10개 커밋만
git pull --depth 10

# 최근 1개 커밋만 (shallow)
git pull --depth 1

Push 고급 사용법​

Upstream 설정​

# 브랜치를 푸시하고 upstream 설정
git push -u origin feature-branch

# 이후부터는 간단하게
git push

모든 브랜치 푸시​

git push --all

태그 푸시​

# 특정 태그
git push origin v1.0.0

# 모든 태그
git push --tags

# 브랜치와 태그 모두
git push --follow-tags

강제 푸시 (주의!)​

# 위험한 강제 푸시
git push -f

# 더 안전한 강제 푸시
git push --force-with-lease
위험

-f는 원격의 커밋을 덮어쓰므로 협업 시 사용하지 마세요! --force-with-lease는 원격이 예상과 다르면 푸시를 거부합니다.

푸시 거부 처리​

# 에러: 원격에 새로운 커밋이 있음
git push
# ! [rejected]        main -> main (fetch first)

# 해결 방법 1: Pull 후 Push
git pull
git push

# 해결 방법 2: Rebase 후 Push
git pull --rebase
git push

원격 브랜치 관리​

원격 브랜치 확인​

# 원격 브랜치 목록
git branch -r

# 모든 브랜치 (로컬 + 원격)
git branch -a

# 원격 브랜치 상세 정보
git remote show origin

원격 브랜치 추적​

# 원격 브랜치를 로컬로 가져오기
git checkout -b local-branch origin/remote-branch

# 또는 (Git 2.23+)
git switch -c local-branch origin/remote-branch

# 동일한 이름으로
git checkout --track origin/feature-branch

삭제된 원격 브랜치 정리​

# 원격에서 삭제된 브랜치 참조 제거
git fetch --prune

# 또는
git remote prune origin

# 자동으로 prune 설정
git config --global fetch.prune true

실습 예제​

예제 1: 기본 동기화 워크플로우​

# 1. 최신 상태 확인
git fetch

# 2. 로컬과 원격의 차이 확인
git log HEAD..origin/main --oneline

# 3. 변경사항 가져오기
git pull

# 4. 작업 수행
echo "new feature" >> feature.txt
git add feature.txt
git commit -m "feat: add new feature"

# 5. 푸시
git push

예제 2: 충돌 해결​

# 1. Pull 시도
git pull
# CONFLICT in README.md

# 2. 충돌 파일 확인
git status

# 3. 파일 열어서 충돌 해결
code README.md

# 4. 해결 후 스테이징
git add README.md

# 5. 병합 완료
git commit -m "merge: resolve conflicts"

# 6. 푸시
git push

예제 3: Rebase 워크플로우​

# 1. 최신 코드 rebase로 가져오기
git pull --rebase

# 2. 작업
echo "changes" >> file.txt
git add file.txt
git commit -m "feat: update file"

# 3. 푸시
git push

예제 4: 여러 원격 저장소 관리​

# 1. 원본 저장소 추가
git remote add upstream https://github.com/original/repo.git

# 2. 원격 저장소 목록 확인
git remote -v

# 3. 원본 저장소의 최신 코드 가져오기
git fetch upstream

# 4. 원본의 main을 로컬 main에 병합
git checkout main
git merge upstream/main

# 5. 내 저장소에 푸시
git push origin main

일반적인 시나리오​

시나리오 1: 팀원이 푸시한 코드 가져오기​

# 방법 1: Pull (간단)
git pull

# 방법 2: Fetch + Merge (세밀한 제어)
git fetch
git log HEAD..origin/main  # 변경사항 확인
git merge origin/main

시나리오 2: 로컬 변경사항과 원격 변경사항이 충돌​

# 방법 1: 로컬 변경사항 stash
git stash
git pull
git stash pop
# 충돌 해결 후
git add .
git commit

# 방법 2: Rebase
git pull --rebase
# 충돌 해결 후
git add .
git rebase --continue

시나리오 3: 잘못 푸시한 커밋 되돌리기​

# 방법 1: Revert (권장 - 안전)
git revert HEAD
git push

# 방법 2: Reset (위험 - 팀과 협의 필요)
git reset --hard HEAD~1
git push --force-with-lease

유용한 설정​

# Pull 시 항상 rebase
git config --global pull.rebase true

# Fetch 시 자동으로 prune
git config --global fetch.prune true

# Push 시 현재 브랜치만
git config --global push.default current

# Pull 시 자동으로 stash
git config --global rebase.autostash true

정리 표​

명령어설명
git fetch원격 변경사항 가져오기 (병합 안 함)
git pull원격 변경사항 가져오기 + 병합
git pull --rebase원격 변경사항을 rebase로 적용
git push로컬 변경사항을 원격에 업로드
git push -u origin branch브랜치 푸시 + upstream 설정
git push --force-with-lease안전한 강제 푸시
git remote prune origin삭제된 원격 브랜치 참조 제거
