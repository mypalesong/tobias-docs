시나리오 8: 보안 사고 대응

수현이 API 키가 GitHub에 노출된 것을 발견하고 긴급 대응합니다!

📌 상황 설명​

금요일 저녁 7시, 집에 가려던 수현의 Slack에 GitHub Security Alert가 울립니다. "AWS Access Key detected in repository!" 😱

더 큰 문제는 이 키가 3주 전 커밋에 포함되어 있고, 이미 여러 브랜치에 퍼져있다는 것입니다!

🎯 목표​

  • Git 히스토리에서 민감 정보 완전 제거
  • 모든 브랜치에서 제거 확인
  • 보안 스캐닝 자동화 설정
  • 팀원 교육 및 재발 방지
  • 사고 대응 프로세스 확립

💬 대화​

GitHub Security Bot (Slack):

🚨 SECURITY ALERT 🚨
Repository: codecraft/shop-backend
Type: AWS Access Key
File: config/aws-credentials.js
Commit: abc1234 (3 weeks ago)
Severity: CRITICAL

This key has FULL AWS access!

수현 (긴급 소집): "@channel 긴급! AWS 키가 노출됐습니다. 즉시 대응합니다!"

준호: "어떻게 된 거죠?"

민지 (창백한 얼굴): "혹시... 제가 3주 전에 푸시한 거..."

수현: "괜찮아요. 지금부터 단계별로 처리합시다."

⚡ 실습​

Phase 1: 즉시 조치 (5분 내)​

1단계: 노출된 키 즉시 무효화​

# AWS CLI로 즉시 키 비활성화
aws iam delete-access-key \
  --access-key-id AKIA**************** \
  --user-name app-user

# 새 키 생성 (임시)
aws iam create-access-key --user-name app-user

수현: "일단 키를 무효화했습니다. 피해는 없는 것 같아요."

2단계: Private 설정 (임시 조치)​

# GitHub CLI로 저장소 임시 비공개 전환
gh repo edit --visibility private

# 또는 웹에서: Settings → Danger Zone → Change visibility

Phase 2: Git 히스토리 완전 제거 (30분)​

3단계: BFG Repo-Cleaner 사용​

# BFG 설치
brew install bfg

# 저장소 미러 클론 (모든 히스토리 포함)
git clone --mirror https://github.com/codecraft/shop-backend.git
cd shop-backend.git

# 민감한 정보가 포함된 파일 목록 작성
cat > secrets.txt << EOF
AKIA****************
AWS_SECRET_ACCESS_KEY=.*
aws_access_key_id.*
EOF

# BFG로 제거
bfg --replace-text secrets.txt

# 또는 파일 자체를 완전 삭제
bfg --delete-files aws-credentials.js

# 변경사항 확인
git log --all --full-history --source -- config/aws-credentials.js

# RefLog 정리
git reflog expire --expire=now --all

# Garbage collection
git gc --prune=now --aggressive

# 강제 푸시 (⚠️ 주의!)
git push --force --all
git push --force --tags

4단계: git-filter-repo로 더 정밀하게 제거​

# git-filter-repo 설치
pip install git-filter-repo

# 일반 클론 (미러 아님)
git clone https://github.com/codecraft/shop-backend.git
cd shop-backend

# 특정 파일 완전 제거
git filter-repo --path config/aws-credentials.js --invert-paths

# 또는 특정 문자열 치환
git filter-repo --replace-text <(echo '
AKIA****************==>REMOVED_AWS_KEY
aws_secret_access_key=.*==>REMOVED_SECRET
')

# 강제 푸시
git remote add origin https://github.com/codecraft/shop-backend.git
git push --force --all
git push --force --tags

Phase 3: 팀원 동기화​

5단계: 팀원들에게 공지 및 재클론 요청​

Slack 공지:

@channel 🚨 긴급 조치 완료

**상황**:
AWS 키가 Git 히스토리에 노출되어 완전 제거했습니다.

**팀원 필수 조치**:
1. ⛔ 현재 작업 중단
2. 💾 로컬 변경사항 백업 (stash 또는 patch)
3. 🗑️ 로컬 저장소 완전 삭제
4. 🔄 새로 클론

**명령어**:
\`\`\`bash
# 1. 변경사항 백업
git stash save "backup before repo clean"

# 2. 로컬 저장소 삭제
cd ..
rm -rf shop-backend

# 3. 새로 클론
git clone https://github.com/codecraft/shop-backend.git
cd shop-backend

# 4. 작업 복구 (필요시)
git stash pop
\`\`\`

**절대 하지 말 것**:
❌ Pull/Fetch (오래된 히스토리 다시 받음)
❌ 기존 로컬 저장소 계속 사용

**질문**: #dev-support 채널로

Phase 4: 보안 강화​

6단계: Pre-commit Hook 설정​

# .git/hooks/pre-commit
#!/bin/sh

echo "🔍 Scanning for secrets..."

# 민감한 패턴 정의
PATTERNS=(
  "AKIA[0-9A-Z]{16}"                    # AWS Access Key
  "aws_secret_access_key"
  "-----BEGIN RSA PRIVATE KEY-----"
  "password\s*=\s*['\"][^'\"]+['\"]"
  "api[_-]?key\s*=\s*['\"][^'\"]+['\"]"
  "secret[_-]?key\s*=\s*['\"][^'\"]+['\"]"
  "token\s*=\s*['\"][^'\"]+['\"]"
)

# 스테이징된 파일 검사
for pattern in "${PATTERNS[@]}"; do
  if git diff --cached | grep -iE "$pattern"; then
    echo "❌ ERROR: Potential secret detected!"
    echo "Pattern: $pattern"
    echo ""
    echo "Please remove sensitive data before committing."
    echo "Use environment variables instead."
    exit 1
  fi
done

echo "✅ No secrets detected"
exit 0
# Hook 설치 및 실행 권한
chmod +x .git/hooks/pre-commit

7단계: git-secrets 설치​

# git-secrets 설치
brew install git-secrets

# 현재 저장소에 설정
git secrets --install

# AWS 패턴 추가
git secrets --register-aws

# 커스텀 패턴 추가
git secrets --add 'password\s*=\s*.+'
git secrets --add 'api[_-]?key\s*=\s*.+'

# 전체 히스토리 스캔
git secrets --scan-history

# 전역 설정 (모든 새 저장소)
git secrets --install ~/.git-templates/git-secrets
git config --global init.templateDir ~/.git-templates/git-secrets

8단계: GitHub Secret Scanning 활성화​

# .github/workflows/secret-scan.yml
name: Secret Scanning

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  gitleaks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0

      - name: Run Gitleaks
        uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

  trufflehog:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0

      - name: TruffleHog OSS
        uses: trufflesecurity/trufflehog@main
        with:
          path: ./
          base: ${{ github.event.repository.default_branch }}
          head: HEAD

9단계: 환경 변수 관리 개선​

# .env.example (커밋 가능)
cat > .env.example << EOF
# AWS Configuration
AWS_ACCESS_KEY_ID=your_access_key_here
AWS_SECRET_ACCESS_KEY=your_secret_key_here
AWS_REGION=ap-northeast-2

# Database
DATABASE_URL=postgresql://localhost:5432/mydb

# API Keys
STRIPE_PUBLIC_KEY=pk_test_...
STRIPE_SECRET_KEY=sk_test_...
EOF

# .env (절대 커밋 금지)
cat > .env << EOF
AWS_ACCESS_KEY_ID=AKIA****************
AWS_SECRET_ACCESS_KEY=****************************************
AWS_REGION=ap-northeast-2
DATABASE_URL=postgresql://user:pass@host:5432/db
STRIPE_PUBLIC_KEY=pk_live_***
STRIPE_SECRET_KEY=sk_live_***
EOF

# .gitignore 업데이트
cat >> .gitignore << EOF

# Environment variables
.env
.env.local
.env.*.local

# Credentials
**/secrets/
**/*secret*
**/*credential*
*.key
*.pem
*.p12

# IDE
.vscode/settings.json
.idea/workspace.xml
EOF
// config/aws.js (안전한 방식)
require('dotenv').config();

module.exports = {
  accessKeyId: process.env.AWS_ACCESS_KEY_ID,
  secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY,
  region: process.env.AWS_REGION,
};

- **Date**: 2024-01-19 19:00 KST
