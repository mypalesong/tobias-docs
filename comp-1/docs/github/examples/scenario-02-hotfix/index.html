시나리오 2: 긴급 버그 발생!

민지의 두 번째 날, 프로덕션에서 치명적인 버그가 발견됩니다!

📌 상황 설명​

오전 9시 30분, Slack에 긴급 메시지가 쏟아집니다. 결제 시스템에서 금액이 잘못 계산되는 버그가 발견되었습니다! 고객들이 항의 전화를 하고 있고, CEO가 즉시 수정을 요구합니다.

민지는 새로운 로그인 폼을 개발하던 중이었지만, 긴급 상황에 투입됩니다.

🎯 목표​

  • 현재 작업을 안전하게 보관
  • Hotfix 브랜치에서 긴급 수정
  • 프로덕션에 즉시 배포
  • 개발 브랜치에도 수정 반영
  • 원래 작업으로 복귀

💬 대화​

준호 (Slack): "@channel 긴급! 결제 금액 계산 버그 발견. 할인율이 두 번 적용되고 있음. 즉시 수정 필요!"

현우 (PM): "모든 결제를 일시 중단했습니다. 최대한 빨리 수정 부탁드립니다!"

민지 (당황하며): "저... 지금 로그인 폼 작업 중인데 어떻게 하죠?"

준호: "괜찮아요. Stash로 임시 저장하고 hotfix 브랜치에서 작업하면 돼요. 같이 해봅시다!"

⚡ 실습​

1단계: 현재 작업 임시 저장​

민지의 현재 상황:

# 현재 작업 중인 브랜치
git status
# On branch feature/login-form
# Changes not staged for commit:
#   modified:   src/components/LoginForm.jsx
#   modified:   src/styles/LoginForm.css
# Untracked files:
#   src/components/LoginButton.jsx

준호: "먼저 현재 작업을 stash로 저장하세요."

# 현재 작업 임시 저장
git stash save "WIP: login form - 50% complete"

# Stash 목록 확인
git stash list
# stash@{0}: On feature/login-form: WIP: login form - 50% complete

# 깨끗한 상태 확인
git status
# On branch feature/login-form
# nothing to commit, working tree clean

민지: "작업이 사라진 것 같은데... 괜찮은 건가요?"

준호: "안전하게 저장되어 있어요. 나중에 stash pop으로 복구할 수 있습니다."

2단계: Main에서 Hotfix 브랜치 생성​

# Main 브랜치로 전환
git checkout main

# 최신 코드 받기
git pull origin main

# Hotfix 브랜치 생성
git checkout -b hotfix/payment-calculation

# 현재 위치 확인
git branch
#   feature/login-form
# * hotfix/payment-calculation
#   main

3단계: 버그 찾기 및 수정​

준호: "결제 계산 로직은 src/utils/payment.js에 있어요."

민지, 파일을 열어 버그를 발견합니다:

// src/utils/payment.js
// 🐛 버그가 있는 코드
export function calculateFinalPrice(originalPrice, discountRate) {
  const discountedPrice = originalPrice * (1 - discountRate);
  const finalPrice = discountedPrice * (1 - discountRate); // ❌ 할인이 두 번 적용됨!
  return finalPrice;
}

민지: "찾았어요! 할인이 두 번 적용되고 있어요!"

// 수정된 코드
export function calculateFinalPrice(originalPrice, discountRate) {
  const discountedPrice = originalPrice * (1 - discountRate);
  return discountedPrice; // ✅ 수정 완료
}

4단계: 테스트 및 커밋​

# 테스트 실행
npm test -- payment.test.js

# ✅ PASS  src/utils/payment.test.js
#   calculateFinalPrice
#     ✓ applies discount correctly (5ms)
#     ✓ handles zero discount (2ms)
#     ✓ handles 100% discount (1ms)

# 변경사항 확인
git diff

# 커밋
git add src/utils/payment.js
git commit -m "fix: correct double discount calculation in payment

- Remove duplicate discount application
- Final price now correctly calculated once
- All existing tests pass

CRITICAL: Production bug affecting all payments
Closes #156"

5단계: 긴급 배포​

# Hotfix 브랜치 푸시
git push -u origin hotfix/payment-calculation

# GitHub에서 PR 생성 (대상: main)
gh pr create \
  --title "🚨 HOTFIX: Fix payment calculation bug" \
  --body "Critical bug fix for production

## Problem
Discount was being applied twice, causing incorrect payment amounts.

## Solution
Removed duplicate discount calculation.

## Testing
- [x] Unit tests pass
- [x] Manual testing completed
- [x] Ready for immediate deployment

Closes #156" \
  --label "hotfix,critical"

준호: "제가 즉시 리뷰할게요."

30초 후...

준호 (GitHub): "LGTM! Approved. 바로 머지합니다."

# 준호가 PR 병합
gh pr merge --squash --delete-branch

수현 (DevOps): "배포 시작합니다. 2분 소요 예상."

6단계: Develop 브랜치에도 적용​

준호: "프로덕션은 수정됐지만, develop 브랜치에도 같은 버그가 있어요. Cherry-pick으로 가져올게요."

# Develop 브랜치로 전환
git checkout develop
git pull origin develop

# Main에서 hotfix 커밋 해시 찾기
git log origin/main --oneline -5
# abc1234 fix: correct double discount calculation in payment
# def5678 feat: add welcome message to home page

# Cherry-pick으로 커밋 가져오기
git cherry-pick abc1234

# 푸시
git push origin develop

민지: "와, 똑같은 수정을 다시 안 해도 되네요!"

7단계: 원래 작업으로 복귀​

# 로그인 폼 브랜치로 돌아가기
git checkout feature/login-form

# Develop 최신화 (hotfix 포함)
git fetch origin
git rebase origin/develop

# 저장했던 작업 복구
git stash pop

# 상태 확인
git status
# On branch feature/login-form
# Changes not staged for commit:
#   modified:   src/components/LoginForm.jsx
#   modified:   src/styles/LoginForm.css
# Untracked files:
#   src/components/LoginButton.jsx

민지: "모든 작업이 그대로 돌아왔어요!"

8단계: 배포 확인 및 사후 조치​

수현: "배포 완료. 프로덕션 정상 동작 확인!"

현우: "결제 시스템 재개합니다. 수고하셨습니다!"

준호: "민지님, 첫 hotfix 잘 해냈어요! 이제 포스트모템 문서를 작성해볼까요?"

# 결제 버그 포스트모템 (2024-01-15)

## 문제
- **발생 시간**: 09:30
- **발견 경로**: 고객 신고
- **영향 범위**: 모든 결제 (약 50건 영향)

## 원인
할인율이 두 번 적용되는 로직 오류

## 해결
- **수정 시간**: 10:15 (45분 소요)
- **배포 시간**: 10:17
- **수정 방법**: 중복 할인 계산 제거

## 재발 방지
1. ✅ 결제 관련 테스트 케이스 추가
2. ✅ Code review 체크리스트에 계산 로직 검증 추가
3. 🔄 프로덕션 모니터링 강화 (진행 중)

## 영향받은 고객
50명의 고객에게 환불 처리 예정

✅ 검증​

# Hotfix가 main과 develop 모두에 적용되었는지 확인
git log origin/main --oneline --grep="payment"
git log origin/develop --oneline --grep="payment"

# Stash가 비었는지 확인
git stash list

# 현재 브랜치 확인
git branch
#   develop
# * feature/login-form
#   main

🎓 배운 내용​

Git Stash​

# 작업 임시 저장
git stash save "설명"

# 저장 목록 확인
git stash list

# 복구
git stash pop

# 복구 (stash 유지)
git stash apply

Hotfix 워크플로우​

  1. 현재 작업 stash
  2. Main에서 hotfix 브랜치 생성
  3. 버그 수정 및 테스트
  4. PR → 리뷰 → 병합
  5. Develop에 cherry-pick
  6. 원래 작업 복귀

Cherry-pick​

# 특정 커밋만 가져오기
git cherry-pick <커밋해시>

# 여러 커밋
git cherry-pick abc123 def456

💡 실전 팁​

1. Hotfix 체크리스트​

- [ ] Main에서 브랜치 생성
- [ ] 테스트 작성/실행
- [ ] 최소한의 변경
- [ ] 빠른 리뷰 요청
- [ ] 배포 후 모니터링
- [ ] Develop에 반영
- [ ] 포스트모템 작성

2. Stash 활용 팁​

# 특정 파일만 stash
git stash push -m "message" path/to/file

# Stash 내용 미리보기
git stash show -p stash@{0}

# Stash를 브랜치로 만들기
git stash branch new-branch-name

3. 긴급 상황 대처​

# 1단계: 패닉하지 말기! 😌
# 2단계: 현재 작업 저장
git stash save "WIP: before emergency fix"

# 3단계: 안전한 브랜치로 이동
git checkout main
git pull

# 4단계: 수정 후 즉시 배포
# 5단계: 원래 작업 복귀
git stash pop

4. 실수 방지​

# Hotfix 브랜치에서 작업 중인지 확인
git branch --show-current

# Main이 최신인지 확인
git fetch origin
git status
