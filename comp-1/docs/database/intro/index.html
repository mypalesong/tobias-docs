PostgreSQL 데이터베이스 설계 완벽 가이드

웹 시스템을 위한 효과적인 데이터베이스 설계 전략

데이터베이스 설계는 웹 시스템의 성능, 확장성, 유지보수성을 결정하는 핵심 요소입니다. 이 가이드에서는 PostgreSQL 기반 웹 시스템 데이터베이스 설계의 모든 것을 다룹니다.

🎯 학습 목표​

  • 기초편: 데이터베이스 설계의 기본 개념과 정규화
  • 설계편: 관계 설계, 인덱싱, 제약조건 전략
  • 패턴편: 실무에서 자주 사용하는 설계 패턴
  • 실전편: 실제 시스템별 데이터베이스 설계 사례

📚 학습 순서​

1️⃣ 기초편 (Basic)​

데이터베이스 설계의 기본 개념을 배웁니다.

  • 기본 개념: 테이블, 컬럼, 키, 관계의 이해
  • 정규화: 데이터 중복 제거와 무결성 보장
  • 데이터 타입: PostgreSQL 데이터 타입 선택 전략

2️⃣ 설계편 (Design)​

효과적인 데이터베이스 설계 방법을 학습합니다.

  • 설계 원칙: SOLID, DRY를 데이터베이스에 적용
  • 관계 설계: 1:1, 1:N, N:M 관계 모델링
  • 인덱싱 전략: 쿼리 성능 최적화
  • 제약조건: 데이터 무결성 보장

3️⃣ 패턴편 (Patterns)​

실무에서 자주 사용하는 설계 패턴을 익힙니다.

  • 일반 패턴: 공통적으로 사용하는 테이블 설계
  • 논리 삭제: Soft Delete 패턴
  • 감사 로그: Audit Trail 구현
  • 버전 관리: 데이터 이력 관리

4️⃣ 실전편 (Scenarios)​

실제 시스템별 데이터베이스 설계 사례를 학습합니다.

  • 전자상거래: 상품, 주문, 결제 시스템
  • 소셜 미디어: 사용자, 게시물, 관계 시스템
  • SaaS: 멀티 테넌시, 구독 관리
  • CMS: 콘텐츠, 분류, 버전 관리

💡 PostgreSQL이란?​

정의​

-- PostgreSQL은 세계에서 가장 진보된 오픈소스 관계형 데이터베이스
CREATE DATABASE my_app;

-- 특징:
-- ✅ ACID 완벽 지원
-- ✅ 복잡한 쿼리 처리
-- ✅ 확장성 (Extensions)
-- ✅ JSON 지원
-- ✅ 풍부한 데이터 타입

🔑 핵심 개념 미리보기​

1. 테이블 설계​

-- ❌ 나쁜 설계
CREATE TABLE users (
    id SERIAL,
    data TEXT  -- 모든 데이터를 하나의 컬럼에
);

-- ✅ 좋은 설계
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(50) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

2. 관계 설계​

-- 1:N 관계 (한 사용자, 여러 게시물)
CREATE TABLE posts (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES users(id),
    title VARCHAR(200) NOT NULL,
    content TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- N:M 관계 (게시물 ↔ 태그)
CREATE TABLE post_tags (
    post_id BIGINT REFERENCES posts(id),
    tag_id BIGINT REFERENCES tags(id),
    PRIMARY KEY (post_id, tag_id)
);

3. 인덱스​

-- 단일 컬럼 인덱스
CREATE INDEX idx_users_email ON users(email);

-- 복합 인덱스
CREATE INDEX idx_posts_user_created
ON posts(user_id, created_at DESC);

-- 부분 인덱스
CREATE INDEX idx_active_users
ON users(email)
WHERE is_active = true;

🚀 빠른 시작​

5분 만에 데이터베이스 설계하기​

-- 1단계: 데이터베이스 생성
CREATE DATABASE blog_app;

-- 2단계: 사용자 테이블
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(50) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 3단계: 게시물 테이블 (관계 설정)
CREATE TABLE posts (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    title VARCHAR(200) NOT NULL,
    content TEXT,
    published_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 4단계: 인덱스 추가
CREATE INDEX idx_posts_user_id ON posts(user_id);
CREATE INDEX idx_posts_published ON posts(published_at) WHERE published_at IS NOT NULL;

-- 5단계: 샘플 데이터
INSERT INTO users (email, username) VALUES
    ('user1@example.com', 'user1'),
    ('user2@example.com', 'user2');

INSERT INTO posts (user_id, title, content, published_at) VALUES
    (1, 'First Post', 'Hello World!', CURRENT_TIMESTAMP),
    (1, 'Second Post', 'PostgreSQL is awesome!', CURRENT_TIMESTAMP);

-- 6단계: 쿼리 테스트
SELECT
    u.username,
    p.title,
    p.published_at
FROM posts p
JOIN users u ON p.user_id = u.id
WHERE p.published_at IS NOT NULL
ORDER BY p.published_at DESC;
