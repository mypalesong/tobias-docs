중급편 시작하기

기초편에서 Git과 GitHub의 기본을 배웠다면, 이제 협업과 고급 워크플로우를 마스터할 시간입니다!

중급편에서 배울 내용​

1. GitHub 협업 도구​

  • Pull Request: 코드 리뷰와 협업의 핵심
  • Issues: 작업 추적과 버그 관리
  • GitHub Projects: 프로젝트 관리

2. 고급 Git 명령어​

  • Rebase: 깔끔한 커밋 히스토리 유지
  • Stash: 임시로 변경사항 저장
  • Tags: 버전 관리

3. 충돌 해결​

  • 병합 충돌 이해하기
  • 효과적인 충돌 해결 전략
  • 충돌 방지 팁

4. 협업 워크플로우​

  • Git Flow
  • GitHub Flow
  • Trunk-based Development
  • Forking Workflow

5. GitHub Actions​

  • CI/CD 기초
  • 자동화된 테스트
  • 배포 자동화

학습 전 준비사항​

중급편을 시작하기 전에 다음 내용을 숙지하고 있어야 합니다:

  • ✅ Git 기본 명령어 (add, commit, push, pull)
  • ✅ 브랜치 생성과 전환
  • ✅ 기본 병합
  • ✅ 원격 저장소 작업

실습 환경 준비​

1. GitHub 계정 설정​

# GitHub CLI 설치 (선택사항)
# Windows
winget install --id GitHub.cli

# macOS
brew install gh

# Linux
sudo apt install gh

# 로그인
gh auth login

2. 협업 연습용 저장소 생성​

# 1. 새 저장소 생성
mkdir collab-practice
cd collab-practice
git init

# 2. README 추가
echo "# Collaboration Practice" > README.md
git add README.md
git commit -m "Initial commit"

# 3. GitHub에 저장소 생성 및 푸시
gh repo create collab-practice --public --source=. --remote=origin --push

# 또는 수동으로
# GitHub에서 저장소 생성 후
git remote add origin https://github.com/username/collab-practice.git
git push -u origin main
