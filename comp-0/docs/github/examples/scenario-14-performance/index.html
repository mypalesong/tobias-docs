시나리오 14: Git 성능 최적화

저장소가 커지면서 느려진 Git 성능을 개선합니다.

📌 상황​

코드크래프트의 저장소가 3년간 성장하면서 .git 폴더가 5GB를 넘어섰습니다. Clone에 10분, Push에 5분이 걸립니다. 팀원들이 불편해하고 있습니다.

🎯 목표​

  • 저장소 크기 분석 및 최적화
  • Git LFS로 대용량 파일 관리
  • Shallow Clone & Partial Clone
  • Git GC 및 유지보수
  • CI/CD 최적화

💬 대화​

민지: "아침마다 Clone하는데 10분씩 걸려요... ☕ 마시고 와야 해요."

수현: "저장소 크기가 5GB래요. 문제는 .git 폴더가 4GB라는 거..."

준호: "오래된 디자인 파일들이 히스토리에 그대로 있어서 그래요. 최적화가 필요해요."

⚡ 실습​

1단계: 저장소 분석​

# 저장소 크기 확인
du -sh .git
# 4.2G    .git

# 큰 파일들 찾기
git rev-list --objects --all | \
  git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | \
  awk '/^blob/ {print substr($0,6)}' | \
  sort --numeric-sort --key=2 | \
  tail -20

# 결과
# abc1234 52428800 designs/mockup-v1.psd
# def5678 104857600 designs/mockup-v2.psd
# ghi9012 209715200 video/demo-old.mp4

# 누가 추가했는지 확인
git log --all --find-object=abc1234

민지의 분석 결과:

총 저장소 크기: 5.2GB
  - 소스 코드: 500MB
  - 디자인 파일: 2GB (PSD, AI)
  - 비디오: 1.5GB
  - 기타 바이너리: 1.2GB

2단계: Git LFS 도입​

Git LFS란?​

Large File Storage - 대용량 파일을 별도 서버에 저장하고 Git에는 포인터만 저장

# Git LFS 설치
# macOS
brew install git-lfs

# Windows
# https://git-lfs.github.com/ 에서 설치

# Ubuntu
sudo apt install git-lfs

# 초기화
git lfs install
# Updated Git hooks.
# Git LFS initialized.

# 추적할 파일 타입 지정
git lfs track "*.psd"
git lfs track "*.ai"
git lfs track "*.mp4"
git lfs track "*.sketch"
git lfs track "designs/**"

# .gitattributes에 자동 추가됨
cat .gitattributes
# *.psd filter=lfs diff=lfs merge=lfs -text
# *.ai filter=lfs diff=lfs merge=lfs -text
# *.mp4 filter=lfs diff=lfs merge=lfs -text

기존 파일을 LFS로 마이그레이션​

# 히스토리에서 대용량 파일 제거 후 LFS로 다시 추가
git lfs migrate import \
  --include="*.psd,*.ai,*.mp4" \
  --everything

# 진행 상황 표시
# migrate: Sorting commits: ..., done.
# migrate: Rewriting commits: 100% (1234/1234), done.
# migrate: Updating refs: ..., done.
# migrate: checkout: ..., done.

# 결과 확인
git lfs ls-files
# abc1234 * designs/mockup-v1.psd
# def5678 * designs/mockup-v2.psd

# 저장소 크기 확인
du -sh .git
# 1.2G    .git  (4.2G → 1.2G 감소!)

Force Push (팀 협의 후)​

# 모든 브랜치에 적용
git lfs migrate import --everything --include="*.psd,*.ai,*.mp4"

# Force push (주의!)
git push --force --all
git push --force --tags

# 팀원들에게 공지
# "모두 re-clone 해주세요!"

3단계: Shallow Clone (CI용)​

# 최신 커밋만 (히스토리 없음)
git clone --depth 1 https://github.com/codecraft/shop.git

# .git 크기 비교
# 일반 clone: 4.2GB
# shallow clone: 200MB (21배 빠름!)

# 특정 브랜치만
git clone --depth 1 --branch main --single-branch \
  https://github.com/codecraft/shop.git

# 히스토리 확인
git log --oneline
# abc1234 (HEAD -> main) Latest commit
# (그 이전은 없음)

# 필요하면 나중에 히스토리 가져오기
git fetch --unshallow

CI/CD에 적용:

# .github/workflows/ci.yml
jobs:
  test:
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 1  # Shallow clone
          lfs: true        # LFS 파일 가져오기

4단계: Partial Clone (일부만)​

# Blob 없이 Clone (필요할 때 다운로드)
git clone --filter=blob:none \
  https://github.com/codecraft/shop.git

# 작동 방식
# 1. 트리와 커밋만 다운로드
# 2. 파일 내용은 checkout 시 다운로드

# 크기 비교
# 일반: 4.2GB
# blob:none: 800MB

# 특정 경로만 (Sparse + Partial)
git clone --filter=blob:none --sparse \
  https://github.com/codecraft/shop.git
cd shop
git sparse-checkout init --cone
git sparse-checkout set frontend packages/ui

5단계: Git Garbage Collection​

# 불필요한 객체 정리
git gc --aggressive --prune=now

# 진행 상황
# Enumerating objects: 12345, done.
# Counting objects: 100% (12345/12345), done.
# Compressing objects: 100% (12345/12345), done.

# 결과
# Before: 1.2GB
# After: 950MB (250MB 절약)

# 자동 GC 설정
git config gc.auto 256
git config gc.autopacklimit 4

# Reflog 정리 (오래된 기록 삭제)
git reflog expire --expire=30.days --all
git gc --prune=now

6단계: 중복 제거​

# 같은 파일이 여러 버전에 있을 때
git repack -a -d --depth=250 --window=250

# Packfile 정보 확인
git count-objects -v -H
# count: 0
# size: 0 bytes
# in-pack: 12345
# packs: 1
# size-pack: 950.00 MiB
# prune-packable: 0
# garbage: 0
# size-garbage: 0 bytes

7단계: GitHub에서 최적화​

# GitHub에 저장소 최적화 요청
# Settings → Manage access → Request repository maintenance

# GitHub LFS 대역폭 확인
gh api repos/:owner/:repo/lfs
# {
#   "bandwidth": 5368709120,  # 5GB
#   "storage": 1073741824     # 1GB
# }

# LFS 파일 목록
gh api repos/:owner/:repo/git/lfs/objects

8단계: .gitignore 최적화​

# 불필요한 파일 추적 방지
cat >> .gitignore << 'EOF'
# 빌드 산출물
dist/
build/
*.min.js
*.min.css

# 로그
*.log
logs/

# OS 파일
.DS_Store
Thumbs.db

# IDE
.idea/
.vscode/
*.swp

# 대용량 임시 파일
*.tmp
*.cache
node_modules/
EOF

# 이미 추적 중인 파일 제거
git rm -r --cached dist/
git rm -r --cached node_modules/
git commit -m "chore: remove build artifacts from tracking"

9단계: Commit 크기 제한​

# Pre-commit hook으로 대용량 파일 방지
cat > .git/hooks/pre-commit << 'EOF'
#!/bin/bash

# 5MB 이상 파일 감지
large_files=$(git diff --cached --name-only | \
  xargs -I{} find {} -type f -size +5M 2>/dev/null)

if [ -n "$large_files" ]; then
  echo "❌ Large files detected (>5MB):"
  echo "$large_files"
  echo ""
  echo "Please use Git LFS:"
  echo "  git lfs track \"*.ext\""
  exit 1
fi
EOF

chmod +x .git/hooks/pre-commit

10단계: Clone 최적화 가이드​

# 빠른 Clone 가이드

## 일반 개발자
\`\`\`bash
# Sparse checkout으로 필요한 부분만
git clone --filter=blob:none --sparse \\
  https://github.com/codecraft/shop.git
cd shop
git sparse-checkout set frontend packages/ui
\`\`\`

## CI/CD
\`\`\`bash
# Shallow clone
git clone --depth 1 --branch main \\
  https://github.com/codecraft/shop.git
\`\`\`

## 전체 기록 필요한 경우
\`\`\`bash
# LFS만 사용
git clone https://github.com/codecraft/shop.git
git lfs pull
\`\`\`

✅ 검증​

# Before vs After 비교
echo "=== BEFORE ==="
echo "Repository size: 5.2GB"
echo "Clone time: 10 minutes"
echo "Push time: 5 minutes"

echo ""
echo "=== AFTER ==="

# 저장소 크기
du -sh .git
# 950MB (82% 감소!)

# Clone 시간 측정
time git clone --depth 1 https://github.com/codecraft/shop.git
# real    0m45s (vs 10분)

# LFS 통계
git lfs ls-files | wc -l
# 156 files tracked by LFS

🎓 배운 내용​

Git LFS 활용​

# 설치 및 추적
git lfs install
git lfs track "*.psd"

# 마이그레이션
git lfs migrate import --include="*.ext" --everything

# 확인
git lfs ls-files

Clone 옵션 비교​

# 일반 clone (모든 히스토리)
git clone <url>

# Shallow (최신만)
git clone --depth 1 <url>

# Partial (blob 제외)
git clone --filter=blob:none <url>

# Sparse (특정 경로만)
git clone --sparse <url>
git sparse-checkout set path/

유지보수 명령어​

# 정리
git gc --aggressive

# 압축
git repack -a -d

# 오래된 객체 삭제
git prune

# Reflog 정리
git reflog expire --expire=30.days --all

💡 실전 팁​

1. 정기적인 저장소 검사​

# 주간 스크립트
cat > weekly-maintenance.sh << 'EOF'
#!/bin/bash
echo "🔍 Repository Health Check"

# 크기 확인
echo "Size: $(du -sh .git | cut -f1)"

# 큰 파일 찾기
echo "Top 10 largest files:"
git rev-list --objects --all | \
  git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | \
  awk '/^blob/ {print substr($0,6)}' | \
  sort -k2 -n -r | head -10

# GC
echo "Running GC..."
git gc --auto

echo "✅ Done"
EOF

chmod +x weekly-maintenance.sh

2. LFS 대역폭 절약​

# LFS 파일 다운로드 스킵
GIT_LFS_SKIP_SMUDGE=1 git clone <url>

# 나중에 필요한 파일만
git lfs pull --include="designs/current/*"

3. GitHub Actions 최적화​

# .github/workflows/optimized.yml
name: Fast CI

on: [push]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 1       # Shallow
          lfs: false           # LFS 스킵 (필요 없으면)

      - name: Cache dependencies
        uses: actions/cache@v3
        with:
          path: node_modules
          key: ${{ runner.os }}-${{ hashFiles('package-lock.json') }}

      - name: Test
        run: npm test

4. Monorepo에서 Sparse Checkout​

# Frontend 개발자는 frontend만
git clone --filter=blob:none --sparse <url>
git sparse-checkout set apps/frontend packages/ui

# Backend 개발자는 backend만
git sparse-checkout set apps/backend services
