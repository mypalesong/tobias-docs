시나리오 15: 고급 GitHub Actions 활용

복잡한 CI/CD 파이프라인을 GitHub Actions로 구축합니다.

📌 상황​

코드크래프트가 여러 플랫폼(Web, iOS, Android)을 동시에 개발하게 되었습니다. 빌드, 테스트, 배포를 자동화하고, 각 플랫폼별로 다른 전략이 필요합니다.

🎯 목표​

  • Matrix Build로 여러 환경 동시 테스트
  • Reusable Workflow 작성
  • Custom Action 개발
  • 조건부 실행 및 의존성 관리
  • Secrets와 환경 변수 관리
  • 배포 자동화 및 Rollback

💬 대화​

수현 (DevOps): "지금은 빌드만 20분 걸려요. 3개 플랫폼 순차 실행하면 1시간이에요."

준호: "병렬로 돌리면 되잖아요?"

수현: "맞아요! Matrix build 쓰면 동시에 돌릴 수 있어요. 보여드릴게요."

⚡ 실습​

1단계: Matrix Build로 병렬 테스트​

# .github/workflows/matrix-test.yml
name: Matrix Tests

on: [push, pull_request]

jobs:
  test:
    name: Test on ${{ matrix.os }} - Node ${{ matrix.node }}
    runs-on: ${{ matrix.os }}

    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        node: [16, 18, 20]
        # 3 OS × 3 Node versions = 9개 작업 동시 실행!

      fail-fast: false  # 하나 실패해도 나머지 계속
      max-parallel: 5   # 동시 실행 제한

    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js ${{ matrix.node }}
        uses: actions/setup-node@v3
        with:
          node-version: ${{ matrix.node }}

      - name: Install dependencies
        run: npm ci

      - name: Run tests
        run: npm test

      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          flags: ${{ matrix.os }}-node${{ matrix.node }}

2단계: Include/Exclude로 세밀한 제어​

strategy:
  matrix:
    os: [ubuntu-latest, windows-latest, macos-latest]
    node: [16, 18, 20]

    # 특정 조합 추가
    include:
      # Ubuntu에서만 Redis 테스트
      - os: ubuntu-latest
        node: 20
        redis: true

    # 특정 조합 제외
    exclude:
      # Windows + Node 16은 스킵
      - os: windows-latest
        node: 16

steps:
  - name: Start Redis
    if: matrix.redis == true
    run: |
      sudo systemctl start redis

3단계: Reusable Workflow 만들기​

# .github/workflows/reusable-deploy.yml
name: Reusable Deploy

on:
  workflow_call:
    inputs:
      environment:
        required: true
        type: string
      version:
        required: false
        type: string
        default: 'latest'

    secrets:
      DEPLOY_TOKEN:
        required: true
      API_KEY:
        required: true

    outputs:
      deployment-url:
        description: "Deployment URL"
        value: ${{ jobs.deploy.outputs.url }}

jobs:
  deploy:
    name: Deploy to ${{ inputs.environment }}
    runs-on: ubuntu-latest
    environment: ${{ inputs.environment }}

    outputs:
      url: ${{ steps.deploy.outputs.url }}

    steps:
      - uses: actions/checkout@v3

      - name: Deploy
        id: deploy
        run: |
          echo "Deploying version ${{ inputs.version }} to ${{ inputs.environment }}"
          # 배포 로직...
          echo "url=https://${{ inputs.environment }}.example.com" >> $GITHUB_OUTPUT
        env:
          TOKEN: ${{ secrets.DEPLOY_TOKEN }}
          API_KEY: ${{ secrets.API_KEY }}

사용하기:

# .github/workflows/main.yml
name: Main Pipeline

on:
  push:
    branches: [main]

jobs:
  deploy-staging:
    uses: ./.github/workflows/reusable-deploy.yml
    with:
      environment: staging
      version: ${{ github.sha }}
    secrets:
      DEPLOY_TOKEN: ${{ secrets.STAGING_TOKEN }}
      API_KEY: ${{ secrets.STAGING_API_KEY }}

  deploy-production:
    needs: deploy-staging
    uses: ./.github/workflows/reusable-deploy.yml
    with:
      environment: production
      version: ${{ github.sha }}
    secrets:
      DEPLOY_TOKEN: ${{ secrets.PROD_TOKEN }}
      API_KEY: ${{ secrets.PROD_API_KEY }}

4단계: Custom Action 개발​

# .github/actions/slack-notify/action.yml
name: 'Slack Notification'
description: 'Send notification to Slack'

inputs:
  webhook-url:
    description: 'Slack webhook URL'
    required: true
  message:
    description: 'Message to send'
    required: true
  status:
    description: 'Status (success/failure)'
    required: false
    default: 'success'

runs:
  using: 'composite'
  steps:
    - name: Send to Slack
      shell: bash
      run: |
        COLOR="${{ inputs.status == 'success' && 'good' || 'danger' }}"

        curl -X POST ${{ inputs.webhook-url }} \
          -H 'Content-Type: application/json' \
          -d '{
            "attachments": [{
              "color": "'$COLOR'",
              "title": "GitHub Actions",
              "text": "${{ inputs.message }}",
              "fields": [
                {
                  "title": "Repository",
                  "value": "${{ github.repository }}",
                  "short": true
                },
                {
                  "title": "Branch",
                  "value": "${{ github.ref_name }}",
                  "short": true
                }
              ]
            }]
          }'

사용하기:

jobs:

5단계: 복잡한 조건부 실행​

name: Complex Conditions

on:
  push:
  pull_request:

jobs:
  check-changes:
    runs-on: ubuntu-latest
    outputs:
      frontend: ${{ steps.filter.outputs.frontend }}
      backend: ${{ steps.filter.outputs.backend }}

    steps:
      - uses: actions/checkout@v3

      - uses: dorny/paths-filter@v2
        id: filter
        with:
          filters: |
            frontend:
              - 'frontend/**'
              - 'packages/ui/**'
            backend:
              - 'backend/**'
              - 'api/**'

  build-frontend:
    needs: check-changes
    if: |
      needs.check-changes.outputs.frontend == 'true' &&
      (github.event_name == 'push' ||
       github.event.pull_request.draft == false)

    runs-on: ubuntu-latest
    steps:
      - name: Build Frontend
        run: npm run build:frontend

  build-backend:
    needs: check-changes
    if: needs.check-changes.outputs.backend == 'true'

    runs-on: ubuntu-latest
    steps:
      - name: Build Backend
        run: npm run build:backend

6단계: Composite Action (JavaScript)​

// .github/actions/semantic-version/index.js
const core = require('@actions/core');
const github = require('@actions/github');
const semver = require('semver');

async function run() {
  try {
    const token = core.getInput('github-token');
    const octokit = github.getOctokit(token);

    // 최신 태그 가져오기
    const { data: tags } = await octokit.rest.repos.listTags({
      owner: github.context.repo.owner,
      repo: github.context.repo.repo,
      per_page: 1
    });

    const currentVersion = tags[0]?.name || 'v0.0.0';

    // PR 라벨 기반으로 버전 결정
    const labels = github.context.payload.pull_request?.labels || [];
    const isMajor = labels.some(l => l.name === 'major');
    const isMinor = labels.some(l => l.name === 'minor');

    let newVersion;
    if (isMajor) {
      newVersion = semver.inc(currentVersion, 'major');
    } else if (isMinor) {
      newVersion = semver.inc(currentVersion, 'minor');
    } else {
      newVersion = semver.inc(currentVersion, 'patch');
    }

    core.setOutput('version', newVersion);
    core.setOutput('previous-version', currentVersion);

  } catch (error) {
    core.setFailed(error.message);
  }
}

run();
# .github/actions/semantic-version/action.yml
name: 'Semantic Version'
description: 'Calculate next semantic version'

inputs:
  github-token:
    description: 'GitHub Token'
    required: true

outputs:
  version:
    description: 'Next version'
  previous-version:
    description: 'Previous version'

runs:
  using: 'node20'
  main: 'index.js'

7단계: 의존성 관리 및 캐싱​

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      # NPM 캐시
      - name: Cache NPM dependencies
        uses: actions/cache@v3
        with:
          path: ~/.npm
          key: ${{ runner.os }}-npm-${{ hashFiles('package-lock.json') }}
          restore-keys: |
            ${{ runner.os }}-npm-

      # 빌드 결과 캐시
      - name: Cache build
        uses: actions/cache@v3
        with:
          path: dist
          key: ${{ runner.os }}-build-${{ github.sha }}

      - name: Install
        run: npm ci

      - name: Build
        run: npm run build

      # 다음 job에서 사용할 artifact
      - name: Upload artifact
        uses: actions/upload-artifact@v3
        with:
          name: build-output
          path: dist/
          retention-days: 7

  deploy:
    needs: build
    runs-on: ubuntu-latest

    steps:
      - name: Download artifact
        uses: actions/download-artifact@v3
        with:
          name: build-output
          path: dist/

      - name: Deploy
        run: ./deploy.sh

8단계: 동적 Matrix 생성​

jobs:
  generate-matrix:
    runs-on: ubuntu-latest
    outputs:
      matrix: ${{ steps.set-matrix.outputs.matrix }}

    steps:
      - uses: actions/checkout@v3

      - name: Generate matrix
        id: set-matrix
        run: |
          # 변경된 서비스만 빌드
          SERVICES=$(git diff --name-only HEAD~1 | \
            grep '^services/' | \
            cut -d/ -f2 | \
            sort -u | \
            jq -R -s -c 'split("\n")[:-1]')

          echo "matrix={\"service\":$SERVICES}" >> $GITHUB_OUTPUT

  build:
    needs: generate-matrix
    if: needs.generate-matrix.outputs.matrix != '{"service":[]}'

    strategy:
      matrix: ${{ fromJson(needs.generate-matrix.outputs.matrix) }}

    runs-on: ubuntu-latest
    steps:
      - name: Build ${{ matrix.service }}
        run: |
          cd services/${{ matrix.service }}
          npm run build

9단계: 배포 승인 및 Rollback​

# .github/workflows/production-deploy.yml
name: Production Deploy

on:
  push:
    tags:
      - 'v*'

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment:
      name: production
      url: https://shop.example.com

    steps:
      - uses: actions/checkout@v3

      - name: Deploy to production
        id: deploy
        run: |
          VERSION=${GITHUB_REF#refs/tags/}
          echo "Deploying $VERSION"

          # 배포
          ./deploy.sh $VERSION

          echo "version=$VERSION" >> $GITHUB_OUTPUT

      - name: Health check
        run: |
          for i in {1..10}; do
            STATUS=$(curl -s -o /dev/null -w "%{http_code}" https://shop.example.com/health)
            if [ $STATUS -eq 200 ]; then
              echo "✅ Health check passed"
              exit 0
            fi
            echo "⏳ Waiting... ($i/10)"
            sleep 10
          done
          echo "❌ Health check failed"
          exit 1

      - name: Rollback on failure
        if: failure()
        run: |
          echo "🔄 Rolling back..."
          PREVIOUS_VERSION=$(git describe --tags --abbrev=0 HEAD^)
          ./deploy.sh $PREVIOUS_VERSION

      - name: Notify
        if: always()
        uses: ./.github/actions/slack-notify
        with:
          webhook-url: ${{ secrets.SLACK_WEBHOOK }}
          message: |
            Deployment ${{ job.status }}
            Version: ${{ steps.deploy.outputs.version }}
          status: ${{ job.status }}

10단계: Secrets 관리 Best Practices​

jobs:
  secure-deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      # ❌ 나쁜 예
      - name: Bad practice
        run: |
          echo "API Key: ${{ secrets.API_KEY }}"  # 로그에 노출!

      # ✅ 좋은 예
      - name: Good practice
        run: |
          # 환경 변수로만 사용
          ./deploy.sh
        env:
          API_KEY: ${{ secrets.API_KEY }}

      # Secrets 마스킹
      - name: Use secret
        run: |
          # Secret 값은 자동으로 ***로 표시됨
          echo "Deploying with key..."
          # API_KEY는 로그에서 마스킹됨
        env:
          API_KEY: ${{ secrets.API_KEY }}

      # Vault에서 가져오기 (더 안전)
      - name: Get secrets from Vault
        uses: hashicorp/vault-action@v2
        with:
          url: ${{ secrets.VAULT_URL }}
          token: ${{ secrets.VAULT_TOKEN }}
          secrets: |
            secret/data/production api_key | API_KEY ;
            secret/data/production db_password | DB_PASSWORD
