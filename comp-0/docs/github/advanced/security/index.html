Git & GitHub 보안

Git과 GitHub를 안전하게 사용하는 보안 Best Practices를 배웁니다.

주요 보안 위협​

1. 민감한 정보 노출​

  • API 키, 비밀번호, 토큰
  • 데이터베이스 자격 증명
  • 개인 키 파일

2. 악의적인 커밋​

  • 악성 코드 삽입
  • 백도어
  • 공급망 공격

3. 계정 탈취​

  • 약한 비밀번호
  • 2FA 미사용
  • SSH 키 유출

Pre-commit Hook로 검사​

#!/bin/sh

# 민감한 패턴 검색
PATTERNS=(
  "API_KEY.*=.*['\"][^'\"]+['\"]"
  "PASSWORD.*=.*['\"][^'\"]+['\"]"
  "SECRET.*=.*['\"][^'\"]+['\"]"
  "aws_secret_access_key"
  "private_key"
  "BEGIN RSA PRIVATE KEY"
)

for pattern in "${PATTERNS[@]}"; do
  if git diff --cached | grep -iE "$pattern"; then
    echo "❌ Error: Potential secret detected!"
    echo "Pattern: $pattern"
    echo "Please remove sensitive data before committing."
    exit 1
  fi
done

exit 0

git-secrets 도구​

# 설치
brew install git-secrets

# AWS 패턴 추가
git secrets --register-aws --global

# 커스텀 패턴 추가
git secrets --add 'password\s*=\s*.+'

# 저장소 스캔
git secrets --scan

# Pre-commit hook 설치
git secrets --install

이미 커밋된 Secrets 제거​

git filter-branch (구버전)​

# 특정 파일 완전히 삭제
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch path/to/secret-file' \
  --prune-empty --tag-name-filter cat -- --all

# 강제 푸시
git push --force --all
git push --force --tags

BFG Repo-Cleaner (권장)​

# 설치
brew install bfg

# 특정 파일 제거
bfg --delete-files secret-file.txt

# 특정 텍스트 치환
bfg --replace-text passwords.txt

# 정리
git reflog expire --expire=now --all
git gc --prune=now --aggressive

# 강제 푸시
git push --force

git-filter-repo (최신)​

# 설치
pip install git-filter-repo

# 특정 파일 제거
git filter-repo --path secret-file.txt --invert-paths

# 특정 문자열 치환
git filter-repo --replace-text expressions.txt

GPG 서명​

GPG 키 생성​

# GPG 키 생성
gpg --full-generate-key

# 키 목록 확인
gpg --list-secret-keys --keyid-format LONG

# 공개 키 내보내기
gpg --armor --export YOUR_KEY_ID

Git 설정​

# GPG 키 설정
git config --global user.signingkey YOUR_KEY_ID

# 모든 커밋에 서명
git config --global commit.gpgsign true

# 모든 태그에 서명
git config --global tag.gpgsign true

GitHub Security Features​

Dependabot​

자동으로 의존성 취약점을 검사하고 업데이트합니다.

version: 2
updates:
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
    open-pull-requests-limit: 10
    reviewers:
      - "security-team"
    labels:
      - "dependencies"
      - "security"

  - package-ecosystem: "pip"
    directory: "/"
    schedule:
      interval: "daily"

Code Scanning​

name: "CodeQL"

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]
  schedule:
    - cron: '0 0 * * 0'

jobs:
  analyze:
    name: Analyze
    runs-on: ubuntu-latest

    strategy:
      matrix:
        language: [ 'javascript', 'python' ]

    steps:
      - name: Checkout repository
        uses: actions/checkout@v3

      - name: Initialize CodeQL
        uses: github/codeql-action/init@v2
        with:
          languages: ${{ matrix.language }}

      - name: Autobuild
        uses: github/codeql-action/autobuild@v2

      - name: Perform CodeQL Analysis
        uses: github/codeql-action/analyze@v2

Secret Scanning​

GitHub이 자동으로 푸시된 secrets를 검사합니다:

  • AWS 키
  • Azure 키
  • Google Cloud 키
  • GitHub 토큰
  • Stripe 키
  • 기타 100+ 패턴

발견 시:

  1. Repository 관리자에게 알림
  2. Security 탭에 경고 표시
  3. 해당 secret을 즉시 무효화 권장

Secrets 관리​

GitHub Actions Secrets​

# GitHub CLI로 Secret 추가
gh secret set API_KEY

# 또는 웹 UI:
# Settings → Secrets and variables → Actions → New repository secret

사용:

# .github/workflows/deploy.yml
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Deploy
        env:
          API_KEY: ${{ secrets.API_KEY }}
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
        run: npm run deploy

외부 Secrets 관리​

# AWS Secrets Manager
steps:
  - name: Configure AWS credentials
    uses: aws-actions/configure-aws-credentials@v2
    with:
      aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
      aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}

  - name: Read secrets from AWS
    run: |
      SECRET=$(aws secretsmanager get-secret-value --secret-id my-secret)
      echo "::add-mask::$SECRET"

SSH 키 보안​

안전한 SSH 키 생성​

# Ed25519 알고리즘 사용 (권장)
ssh-keygen -t ed25519 -C "your_email@example.com"

# 또는 RSA 4096비트
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# 강력한 암호 설정 (필수!)

SSH 키 권한 설정​

# 개인 키 권한
chmod 600 ~/.ssh/id_ed25519

# 공개 키 권한
chmod 644 ~/.ssh/id_ed25519.pub

# .ssh 디렉토리 권한
chmod 700 ~/.ssh

SSH 키 관리​

# 여러 키 사용 (~/.ssh/config)
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_github

Host work.github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_work

# Fine-grained token 생성 (권장)
# Settings → Developer settings → Personal access tokens → Fine-grained tokens

# 최소 권한 원칙
# - 필요한 권한만 부여

git clone https://<TOKEN>@github.com/user/repo.git

Token 저장​

# ❌ 나쁜 예
export GITHUB_TOKEN=ghp_xxxxxxxxxxxx

# ✅ 좋은 예
# 1. Credential helper 사용
git config --global credential.helper cache

git config --global credential.helper osxkeychain

# 3. 환경 변수 파일 (.env)
GITHUB_TOKEN=ghp_xxxxxxxxxxxx
# .env는 .gitignore에 추가!
