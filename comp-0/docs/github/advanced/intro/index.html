고급편 시작하기

Git과 GitHub의 고급 기능을 마스터하여 전문가 수준의 개발자가 되어보세요!

고급편에서 배울 내용​

1. Git 내부 구조​

  • Git 객체 모델
  • Refs와 포인터
  • Packfiles
  • Git의 작동 원리

2. 고급 Git 명령어​

  • Git Hooks: 이벤트 기반 자동화
  • Cherry-pick: 선택적 커밋 가져오기
  • Bisect: 버그 발생 시점 찾기
  • Worktree: 여러 작업 공간 동시 관리
  • Submodules: 프로젝트 내 프로젝트 관리

3. GitHub 고급 기능​

  • Advanced GitHub Actions: 복잡한 CI/CD 파이프라인
  • GitHub API: 프로그래밍 방식으로 GitHub 제어
  • Security Features: 보안 스캐닝, Dependabot
  • GitHub Packages: 패키지 호스팅

4. 성능 최적화​

  • Large File Storage (LFS)
  • Shallow Clone
  • Sparse Checkout
  • Repository 최적화

5. 고급 워크플로우​

  • Monorepo 관리
  • Release 자동화
  • Multi-environment 배포

필수 선행 지식​

고급편을 시작하기 전에 다음을 숙지해야 합니다:

  • ✅ Git 기본 및 중급 명령어
  • ✅ 브랜치 전략 및 병합
  • ✅ Pull Request 워크플로우
  • ✅ 기본 GitHub Actions
  • ✅ Rebase 기초

실습 환경 준비​

Git 고급 설정​

# 성능 향상
git config --global core.preloadindex true
git config --global core.fscache true
git config --global gc.auto 256

# 더 나은 diff
git config --global diff.algorithm histogram
git config --global diff.colorMoved zebra

# GPG 서명
git config --global commit.gpgsign true
git config --global user.signingkey YOUR_GPG_KEY

# Rerere (재사용 기록된 해결책)
git config --global rerere.enabled true

유용한 도구 설치​

# Git LFS
git lfs install

# Hub (GitHub CLI 보완)
brew install hub  # macOS
# 또는
choco install hub  # Windows

# Git Flow
brew install git-flow
# 또는
apt install git-flow  # Linux

# BFG (대용량 파일 제거)
brew install bfg

고급 기능 미리보기​

Git Hooks 예시​

# .git/hooks/pre-commit
#!/bin/sh
npm test || exit 1
npm run lint || exit 1

Cherry-pick 예시​

# 다른 브랜치의 특정 커밋만 가져오기
git cherry-pick abc123

Bisect 예시​

# 버그를 도입한 커밋 찾기
git bisect start
git bisect bad
git bisect good v1.0.0
# Git이 자동으로 이진 탐색

Submodules 예시​

# 다른 프로젝트를 하위 프로젝트로
git submodule add https://github.com/user/lib.git lib

실전 시나리오​

시나리오 1: 대규모 Monorepo 관리​

  • Git LFS로 대용량 파일 관리
  • Sparse checkout으로 필요한 부분만 체크아웃
  • Worktree로 여러 브랜치 동시 작업
  • Hooks로 코드 품질 자동 검증

시나리오 2: 복잡한 릴리스 프로세스​

  • GitHub Actions로 자동화된 빌드
  • 여러 환경(dev, staging, prod)으로 배포
  • Semantic versioning 자동 관리
  • Release notes 자동 생성

시나리오 3: 오픈소스 프로젝트 유지보수​

  • Dependabot으로 의존성 자동 업데이트
  • Security scanning
  • GitHub API로 이슈/PR 자동 관리
  • Bot으로 반복 작업 자동화

성능 최적화 팁​

1. 대용량 저장소​

# Shallow clone
git clone --depth 1 https://github.com/user/repo.git

# Partial clone
git clone --filter=blob:none https://github.com/user/repo.git

# Sparse checkout
git sparse-checkout init --cone
git sparse-checkout set src/app

2. Git 최적화​

# 저장소 압축
git gc --aggressive --prune=now

# 불필요한 파일 제거
git prune

# Reflog 정리
git reflog expire --expire=30.days --all

3. 네트워크 최적화​

# 병렬 fetch
git config --global fetch.parallel 4

# 압축 레벨 조정
git config --global core.compression 9

보안 Best Practices​

1. GPG 서명​

# 모든 커밋에 서명
git config --global commit.gpgsign true

# 태그 서명
git tag -s v1.0.0 -m "Signed release"

2. Secrets 검증​

# Pre-commit hook로 secrets 검사
# .git/hooks/pre-commit
git diff --cached --name-only | xargs grep -l "API_KEY"
if [ $? -eq 0 ]; then
  echo "Warning: Potential secret detected!"
  exit 1
fi

3. 의존성 관리​

# .github/dependabot.yml
version: 2
updates:
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"

문제 해결 고급 기법​

복잡한 히스토리 수정​

# Filter-branch로 전체 히스토리 수정
git filter-branch --env-filter '
if [ "$GIT_AUTHOR_EMAIL" = "old@email.com" ]; then
  export GIT_AUTHOR_EMAIL="new@email.com"
fi
'

대용량 파일 제거​

# BFG로 대용량 파일 히스토리에서 제거
bfg --strip-blobs-bigger-than 50M
git reflog expire --expire=now --all
git gc --prune=now --aggressive

손상된 저장소 복구​

# Fsck로 문제 확인
git fsck --full

# Reflog로 복구
git reflog
git reset --hard HEAD@{5}
