Cherry-pick

특정 커밋만 선택적으로 다른 브랜치에 적용하는 Cherry-pick 사용법을 배웁니다.

Cherry-pick이란?​

Cherry-pick은 다른 브랜치의 특정 커밋만 현재 브랜치에 적용하는 명령어입니다.

기본 사용법​

# 특정 커밋을 현재 브랜치에 적용
git cherry-pick abc123

# 여러 커밋 적용
git cherry-pick abc123 def456 ghi789

# 범위로 적용
git cherry-pick abc123..def456

실습 예제​

예제 1: 버그 수정 적용하기​

# 상황: develop 브랜치에서 버그를 수정했는데,
# main 브랜치에도 같은 수정이 필요함

# 1. develop에서 버그 수정
git checkout develop
echo "fix bug" >> file.txt
git commit -am "fix: resolve critical bug"
# 커밋 해시: abc123

# 2. main으로 전환
git checkout main

# 3. 버그 수정 커밋만 적용
git cherry-pick abc123

# 4. 푸시
git push

예제 2: 특정 기능만 가져오기​

# feature 브랜치에서 특정 커밋만 가져오기
git log feature-branch --oneline
# abc123 feat: add login button
# def456 feat: add logout button
# ghi789 style: update colors

# logout 기능만 현재 브랜치에 적용
git cherry-pick def456

예제 3: 여러 커밋 적용​

# 연속된 커밋들 적용
git cherry-pick abc123^..def456

# 또는 개별적으로
git cherry-pick abc123 def456 ghi789

Cherry-pick 옵션​

-n (--no-commit)​

커밋하지 않고 스테이징만 합니다.

# 여러 커밋을 하나로 합치기
git cherry-pick -n abc123
git cherry-pick -n def456
git commit -m "Combined changes from abc123 and def456"

-e (--edit)​

커밋 메시지를 수정합니다.

git cherry-pick -e abc123
# 에디터가 열리고 메시지 수정 가능

-x​

원본 커밋 해시를 메시지에 추가합니다.

git cherry-pick -x abc123

# 커밋 메시지:
# fix: resolve critical bug
#
# (cherry picked from commit abc123)

--signoff​

커밋에 서명을 추가합니다.

git cherry-pick --signoff abc123

# 커밋 메시지:
# fix: resolve critical bug
#
# Signed-off-by: Your Name <your.email@example.com>

충돌 해결​

# 1. Cherry-pick 시도
git cherry-pick abc123
# CONFLICT (content): Merge conflict in file.txt

# 2. 충돌 파일 확인
git status

# 3. 충돌 해결
code file.txt
# ... 수정 ...

# 4. 스테이징
git add file.txt

# 5. Cherry-pick 계속
git cherry-pick --continue

# 또는 취소
git cherry-pick --abort

# 또는 건너뛰기
git cherry-pick --skip

고급 사용법​

범위로 Cherry-pick​

# abc123부터 def456까지 (abc123 포함 안 됨)
git cherry-pick abc123..def456

# abc123부터 def456까지 (abc123 포함)
git cherry-pick abc123^..def456

# HEAD부터 3개 커밋
git cherry-pick HEAD~3..HEAD

Mainline 옵션 (-m)​

병합 커밋을 cherry-pick할 때 사용합니다.

# 병합 커밋의 첫 번째 부모 선택
git cherry-pick -m 1 abc123

# 병합 커밋의 두 번째 부모 선택
git cherry-pick -m 2 abc123

전략 옵션​

# 특정 병합 전략 사용
git cherry-pick -X theirs abc123
git cherry-pick -X ours abc123

# Recursive 전략 옵션
git cherry-pick -X ignore-space-change abc123

실전 시나리오​

시나리오 1: Hotfix를 여러 브랜치에 적용​

# 1. main에서 hotfix 적용
git checkout main
git cherry-pick hotfix-commit

# 2. develop에도 적용
git checkout develop
git cherry-pick hotfix-commit

# 3. release 브랜치에도 적용
git checkout release/v2.0
git cherry-pick hotfix-commit

시나리오 2: 실험적 기능 중 일부만 가져오기​

# experimental 브랜치에서 안정적인 부분만 가져오기
git log experimental --oneline
# abc123 feat: stable feature A
# def456 feat: experimental feature B (buggy)
# ghi789 feat: stable feature C

# 안정적인 것만 cherry-pick
git cherry-pick abc123 ghi789

시나리오 3: 릴리스 브랜치에 선택적으로 기능 추가​

# develop에서 release 브랜치로 특정 기능만 이동
git checkout release/v1.5
git log develop --oneline --since="1 week ago"

# 완성된 기능만 선택
git cherry-pick feat-commit-1
git cherry-pick feat-commit-2
# 미완성 기능은 제외

Cherry-pick vs Merge​

Cherry-pick​

# 장점: 선택적으로 가져올 수 있음
# 단점: 같은 변경사항이 다른 해시로 존재 (중복)

git cherry-pick abc123

Merge​

# 장점: 전체 히스토리 유지, 중복 없음
# 단점: 모든 커밋을 가져와야 함

git merge feature-branch

Best Practices​

1. Cherry-pick 남용 피하기​

# ❌ 나쁜 예: 많은 커밋을 cherry-pick
git cherry-pick abc123 def456 ghi789 ... (20개)

# ✅ 좋은 예: Merge 사용
git merge feature-branch

2. 메시지에 원본 기록​

# -x 옵션 사용
git cherry-pick -x abc123

# 또는 수동으로
git cherry-pick abc123
git commit --amend -m "fix: bug fix

(cherry-picked from abc123 on develop branch)"

3. 충돌 최소화​

# 작은 단위로 cherry-pick
git cherry-pick abc123  # 하나씩
# 충돌 해결
git cherry-pick def456  # 다음 것

4. 테스트​

# Cherry-pick 후 반드시 테스트
git cherry-pick abc123
npm test
git push

일반적인 문제​

문제 1: 중복 커밋​

# 문제: 같은 변경사항이 다른 해시로 존재
# develop: abc123
# main: def456 (cherry-picked from abc123)

# 나중에 merge 시 충돌 가능
git checkout main
git merge develop
# CONFLICT!

# 해결: -X ours/theirs 사용
git merge -X ours develop

문제 2: 빈 커밋​

# 이미 적용된 변경사항을 cherry-pick하면
git cherry-pick abc123
# The previous cherry-pick is now empty

# --allow-empty로 강제 적용
git cherry-pick --allow-empty abc123

# 또는 건너뛰기
git cherry-pick --skip

문제 3: 큰 충돌​

# 충돌이 너무 크면 포기하고 수동으로
git cherry-pick --abort

# 수동으로 변경사항 적용
git show abc123
# 내용 확인하고 수동 적용

유용한 Alias​

# Cherry-pick alias
git config --global alias.cp 'cherry-pick'
git config --global alias.cpc 'cherry-pick --continue'
git config --global alias.cpa 'cherry-pick --abort'

# 사용
git cp abc123
git cpc

고급 패턴​

패턴 1: 선택적 백포트​

# 최신 기능 중 일부를 이전 버전에 백포트
git checkout v1.x-stable
git cherry-pick new-feature-commit
git push

패턴 2: 커밋 재정렬​

# Cherry-pick으로 커밋 순서 변경
git checkout -b reordered
git cherry-pick commit-3
git cherry-pick commit-1
git cherry-pick commit-2

패턴 3: 부분 기능 이동​

# 큰 feature에서 완성된 부분만 main으로
git checkout main
git cherry-pick feature-part-1
git cherry-pick feature-part-2
# feature-part-3는 아직 미완성이므로 제외

디버깅​

Cherry-pick 히스토리 확인​

# 커밋이 어디서 왔는지 확인
git log --all --oneline --graph | grep "cherry picked"

# 원본 커밋 찾기
git log --all --grep="abc123"

Dry-run​

# 실제로 적용하지 않고 미리 보기
git show abc123
git diff HEAD..abc123
