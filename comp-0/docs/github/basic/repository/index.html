저장소 생성 및 클론

Git 저장소(Repository)는 프로젝트의 모든 파일과 변경 이력을 저장하는 공간입니다.

새 저장소 생성​

로컬에서 새 저장소 만들기​

# 새 디렉토리 생성 및 이동
mkdir my-project
cd my-project

# Git 저장소 초기화
git init

# 확인
git status

git init 명령을 실행하면 .git 숨김 폴더가 생성됩니다. 이 폴더에 Git이 관리하는 모든 정보가 저장됩니다.

GitHub에서 새 저장소 만들기​

  1. GitHub에 로그인
  2. 우측 상단의 + → New repository 클릭
  3. 저장소 이름 입력
  4. Public/Private 선택
  5. (선택) README, .gitignore, License 추가
  6. Create repository 클릭

저장소 클론 (Clone)​

HTTPS로 클론​

git clone https://github.com/username/repository.git
팁

HTTPS는 설정이 간단하지만 매번 인증이 필요할 수 있습니다.

SSH로 클론​

git clone git@github.com:username/repository.git
팁

SSH는 초기 설정이 필요하지만, 설정 후에는 매번 인증할 필요가 없습니다.

특정 디렉토리 이름으로 클론​

git clone https://github.com/username/repository.git my-folder

특정 브랜치만 클론​

git clone -b develop https://github.com/username/repository.git

얕은 클론 (최근 커밋만)​

# 최근 1개의 커밋만 클론
git clone --depth 1 https://github.com/username/repository.git

대용량 저장소를 빠르게 클론할 때 유용합니다.

로컬 저장소와 원격 저장소 연결​

로컬에서 저장소를 만들고 나중에 GitHub 저장소와 연결할 수 있습니다.

# 로컬 저장소 생성
git init

# 원격 저장소 추가
git remote add origin https://github.com/username/repository.git

# 원격 저장소 확인
git remote -v

# 첫 번째 푸시
git branch -M main
git push -u origin main

원격 저장소 관리​

원격 저장소 목록 보기​

git remote -v

원격 저장소 추가​

git remote add [이름] [URL]

# 예시
git remote add upstream https://github.com/original/repository.git

원격 저장소 이름 변경​

git remote rename [기존이름] [새이름]

# 예시
git remote rename origin github

원격 저장소 URL 변경​

git remote set-url [이름] [새URL]

# 예시: HTTPS에서 SSH로 변경
git remote set-url origin git@github.com:username/repository.git

원격 저장소 제거​

git remote remove [이름]

# 예시
git remote remove origin

원격 저장소 정보 보기​

git remote show origin

저장소 구조​

my-project/
├── .git/           # Git 메타데이터 (숨김 폴더)
│   ├── config      # 로컬 설정
│   ├── objects/    # Git 객체 저장소
│   ├── refs/       # 참조 저장소
│   └── HEAD        # 현재 브랜치 포인터
├── .gitignore      # 무시할 파일 목록
├── README.md       # 프로젝트 설명
└── [프로젝트 파일들]
경고

.git 폴더를 삭제하면 모든 Git 이력이 사라집니다!

README.md 작성​

README.md는 프로젝트의 첫인상입니다. 다음 내용을 포함하는 것이 좋습니다:

# 프로젝트 제목

프로젝트에 대한 간단한 설명

## 기능

- 주요 기능 1
- 주요 기능 2
- 주요 기능 3

## 설치 방법

\`\`\`bash
npm install
\`\`\`

## 사용 방법

\`\`\`bash
npm start
\`\`\`

## 기여 방법

Pull Request를 환영합니다!

## 라이선스

MIT License

실습 예제​

1. 새 프로젝트 시작하기​

# 1. 디렉토리 생성
mkdir hello-git
cd hello-git

# 2. Git 저장소 초기화
git init

# 3. README 파일 생성
echo "# Hello Git" > README.md

# 4. 파일 추가 및 커밋
git add README.md
git commit -m "Initial commit"

# 5. GitHub 저장소 생성 후 연결
git remote add origin https://github.com/username/hello-git.git
git branch -M main
git push -u origin main

2. 기존 프로젝트 클론하기​

# 1. 저장소 클론
git clone https://github.com/username/project.git

# 2. 디렉토리 이동
cd project

# 3. 상태 확인
git status

# 4. 브랜치 확인
git branch -a
