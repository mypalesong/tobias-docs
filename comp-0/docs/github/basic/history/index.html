변경 이력 확인

Git의 커밋 히스토리를 확인하고 변경사항을 비교하는 방법을 배웁니다.

git log - 커밋 이력 보기​

기본 사용법​

# 전체 커밋 이력
git log

# 최근 n개의 커밋만
git log -n 5

# 한 줄로 간단하게
git log --oneline

# 그래프와 함께
git log --graph --oneline

상세 정보 포함​

# 변경된 파일 목록 포함
git log --stat

# 실제 변경 내용 포함
git log -p

# 특정 파일의 이력만
git log -- filename.txt

# 특정 디렉토리의 이력만
git log -- src/

시각화된 로그​

# 브랜치 그래프와 함께
git log --graph --oneline --all

# 더 보기 좋게
git log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --all

커밋 필터링​

날짜로 필터링​

# 특정 날짜 이후
git log --since="2024-01-01"

# 특정 날짜 이전
git log --until="2024-12-31"

# 상대적 날짜
git log --since="2 weeks ago"
git log --since="3 days ago"
git log --after="2024-01-01" --before="2024-12-31"

작성자로 필터링​

# 특정 작성자의 커밋만
git log --author="John"

# 여러 작성자
git log --author="John\|Jane"

커밋 메시지로 검색​

# 커밋 메시지에 특정 단어 포함
git log --grep="fix"

# 대소문자 구분 없이
git log --grep="bug" -i

# 여러 단어 (OR)
git log --grep="bug\|fix"

# 여러 단어 (AND)
git log --grep="bug" --grep="fix" --all-match

코드 변경 내용으로 검색​

# 특정 문자열이 추가되거나 제거된 커밋 찾기
git log -S "function login()"

# 정규표현식으로 검색
git log -G "function.*login"

git diff - 변경사항 비교​

기본 사용법​

# 작업 디렉토리 vs 스테이징 영역
git diff

# 스테이징 영역 vs 마지막 커밋
git diff --staged
# 또는
git diff --cached

# 작업 디렉토리 vs 마지막 커밋
git diff HEAD

커밋 간 비교​

# 두 커밋 비교
git diff commit1 commit2

# 브랜치 간 비교
git diff main feature-branch

# 특정 파일만 비교
git diff main feature-branch -- filename.txt

출력 형식​

# 통계만 보기
git diff --stat

# 단어 단위로 비교
git diff --word-diff

# 색상 강조
git diff --color-words

특정 커밋 정보 보기​

git show​

# 마지막 커밋 상세 정보
git show

# 특정 커밋
git show commit-hash

# 특정 파일의 특정 커밋
git show commit-hash:path/to/file.txt

# 특정 커밋의 통계만
git show --stat commit-hash

파일 이력 추적​

git blame​

파일의 각 라인을 마지막으로 수정한 사람과 커밋을 확인합니다.

# 전체 파일
git blame filename.txt

# 특정 라인 범위
git blame -L 10,20 filename.txt

# 이메일 주소 포함
git blame -e filename.txt

# 더 간단하게
git blame -s filename.txt

git log로 파일 이력 추적​

# 파일이 언제 삭제/이동되었는지 추적
git log --follow -- filename.txt

# 파일의 변경 내용 포함
git log -p -- filename.txt

# 파일을 수정한 커밋만
git log --all --full-history -- filename.txt

실습 예제​

예제 1: 최근 커밋 5개 보기​

git log --oneline -5

출력 예시:

a1b2c3d (HEAD -> main) feat: add login page
e4f5g6h docs: update README
i7j8k9l fix: resolve login bug
m0n1o2p refactor: improve code structure
q3r4s5t chore: update dependencies

예제 2: 오늘 작업한 내용 확인​

git log --since="midnight" --author="$(git config user.name)" --oneline

예제 3: 특정 기능 개발 이력 찾기​

# "auth" 관련 커밋 찾기
git log --grep="auth" --oneline

# src/auth/ 디렉토리 변경 이력
git log --oneline -- src/auth/

예제 4: 두 브랜치의 차이 확인​

# main과 feature 브랜치의 차이
git diff main..feature

# feature에는 있지만 main에 없는 커밋
git log main..feature --oneline

# main에는 있지만 feature에 없는 커밋
git log feature..main --oneline

유용한 Alias 설정​

# 보기 좋은 로그
git config --global alias.lg "log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

# 최근 5개 커밋
git config --global alias.last "log -5 --oneline"

# 오늘의 작업
git config --global alias.today "log --since=midnight --author='$(git config user.name)' --oneline"

사용 예시:

git lg
git last
git today

고급 검색​

파일이 언제 삭제되었는지 찾기​

git log --all --full-history -- "**/deleted-file.txt"

특정 함수를 변경한 커밋 찾기​

git log -L :functionName:path/to/file.js

병합 커밋만 보기​

git log --merges

병합 커밋 제외​

git log --no-merges

시각화 도구​

Git GUI 도구​

# Git 기본 GUI
gitk

# 모든 브랜치 보기
gitk --all

웹 기반 시각화​
