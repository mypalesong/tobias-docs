마이그레이션 히스토리

마이그레이션의 이력을 확인하고 관리하는 방법을 배워봅시다!

🎯 학습 목표​

  • history 명령어 사용법
  • 마이그레이션 체인 이해
  • 버전 관리 전략

📜 History 명령어​

기본 사용법​

# 전체 히스토리 보기
alembic history

# 출력 예시:
# 3a7c4e8f2b1d -> 5d9e2f7a3c4b (head), add comments table
# 2f8b9c3d1a5e -> 3a7c4e8f2b1d, create posts table
# <base> -> 2f8b9c3d1a5e, create users table

상세 정보 보기​

# 자세한 정보 (-v 또는 --verbose)
alembic history -v

# 출력 예시:
# Rev: 5d9e2f7a3c4b (head)
# Parent: 3a7c4e8f2b1d
# Path: /path/to/versions/5d9e2f7a3c4b_add_comments_table.py
#
#     add comments table
#
#     Revision ID: 5d9e2f7a3c4b
#     Revises: 3a7c4e8f2b1d
#     Create Date: 2024-01-17 14:20:30.123456

특정 범위 보기​

# 특정 범위의 히스토리
alembic history -r 2f8b9c3d1a5e:head

# 최근 N개만 보기
alembic history | head -n 5

# 현재 버전부터 base까지
alembic history -r current:base

역순으로 보기​

# 최신부터 과거 순으로 (-r 옵션)
alembic history -r :head

# 과거부터 최신 순으로
alembic history -r base:

🔗 마이그레이션 체인 이해​

리비전 체인 구조​

base
  ↓
[2f8b9c3d1a5e] create users table
  ↓
[3a7c4e8f2b1d] create posts table
  ↓
[5d9e2f7a3c4b] add comments table (head)

파일 내부 구조​

# versions/5d9e2f7a3c4b_add_comments_table.py

revision = '5d9e2f7a3c4b'      # 현재 리비전
down_revision = '3a7c4e8f2b1d'  # 부모 리비전 (이전)

📊 현재 상태와 히스토리​

current + history 조합​

# 현재 위치 확인
alembic current

# 히스토리에서 현재 위치 표시 (-i)
alembic history -i

# 출력 예시:
# 3a7c4e8f2b1d -> 5d9e2f7a3c4b (head), add comments table
# 2f8b9c3d1a5e -> 3a7c4e8f2b1d (current), create posts table
# <base> -> 2f8b9c3d1a5e, create users table

🎨 실전 예제: 프로젝트 히스토리 추적​

블로그 프로젝트 마이그레이션 만들기​

# 1. 사용자 테이블
alembic revision -m "create users table"

# 2. 카테고리 테이블
alembic revision -m "create categories table"

# 3. 포스트 테이블
alembic revision -m "create posts table"

# 4. 댓글 테이블
alembic revision -m "create comments table"

# 5. 태그 테이블
alembic revision -m "create tags table"

# 6. 포스트-태그 관계 테이블
alembic revision -m "create post_tags table"

히스토리 확인​

$ alembic history

# 6개의 마이그레이션이 순서대로 표시됨
aaa1111 -> bbb2222 (head), create post_tags table
999eee0 -> aaa1111, create tags table
888ddd9 -> 999eee0, create comments table
777ccc8 -> 888ddd9, create posts table
666bbb7 -> 777ccc8, create categories table
<base> -> 666bbb7, create users table

특정 시점의 스키마 확인​

# posts 테이블이 추가된 시점의 스키마
alembic upgrade 777ccc8

# 현재 상태 확인
alembic current
# 777ccc8

# 이 시점의 테이블 목록
sqlite3 blog.db ".tables"
# alembic_version  users  categories  posts

🔍 Heads와 Branches​

Head 확인​

# 현재 head(최신) 확인
alembic heads

# 출력:
# bbb2222 (head)

Multiple Heads (고급)​

여러 개발자가 동시에 작업할 때:

       ┌─ [aaa] feature A
base ──┤
       └─ [bbb] feature B
alembic heads
# aaa (head)
# bbb (head)

📝 Show 명령어​

특정 리비전의 상세 정보 보기:

# 특정 리비전 정보
alembic show 777ccc8

# head의 정보
alembic show head

# 현재 리비전 정보
alembic show current

출력 예시:

Rev: 777ccc8
Parent: 666bbb7
Path: /path/to/versions/777ccc8_create_posts_table.py

    create posts table

    Revision ID: 777ccc8
    Revises: 666bbb7
    Create Date: 2024-01-15 11:30:00.123456

🛠️ 실무 활용 예제​

1. 특정 기능의 모든 마이그레이션 찾기​

# grep으로 검색
alembic history -v | grep -i "user"

# 출력:
#     create users table
#     add email to users
#     add profile fields to users

2. 마이그레이션 날짜로 찾기​

# 특정 날짜 이후 마이그레이션
alembic history -v | grep "2024-01-15"

3. 변경 사항 요약 생성​

# 릴리즈 노트용 마이그레이션 목록
alembic history -r last_release:head > release_notes.txt

📊 히스토리 시각화​

간단한 트리 구조​

# 커스텀 스크립트로 시각화
alembic history | awk '{print $1, $3, $4, $5, $6, $7, $8, $9}' | column -t

출력:

aaa1111  ->  bbb2222  (head),  create  post_tags  table
999eee0  ->  aaa1111          create  tags       table
888ddd9  ->  999eee0          create  comments   table
777ccc8  ->  888ddd9          create  posts      table
666bbb7  ->  777ccc8          create  categories table
<base>   ->  666bbb7          create  users      table

Python 스크립트로 분석​

# analyze_migrations.py
from alembic.config import Config
from alembic.script import ScriptDirectory

config = Config("alembic.ini")
script = ScriptDirectory.from_config(config)

print("마이그레이션 분석")
print("=" * 50)

count = 0
for revision in script.walk_revisions():
    count += 1
    print(f"{count}. {revision.revision[:7]} - {revision.doc}")

print(f"\n총 {count}개의 마이그레이션")
python analyze_migrations.py

🎯 실전 시나리오​

시나리오 1: 특정 기능 롤백​

팀원이 새 기능을 3개의 마이그레이션으로 추가했는데, 모두 롤백해야 합니다.

# 1. 히스토리 확인
alembic history -i

# 출력:
# eee5555 -> fff6666 (current), add feature C
# ddd4444 -> eee5555, add feature B
# ccc3333 -> ddd4444, add feature A
# bbb2222 -> ccc3333, previous stable version
# ...

# 2. 안정 버전으로 롤백
alembic downgrade ccc3333

# 3. 확인
alembic current
# ccc3333

시나리오 2: 마이그레이션 충돌 해결​

# 동료가 먼저 push한 상황
$ git pull
$ alembic history

# 출력:
# yyy9999 -> zzz0000 (my changes)
# xxx8888 -> yyy9999 (teammate's changes)
# <base> -> xxx8888

# 내 마이그레이션의 down_revision 수정 필요
# versions/zzz0000_my_changes.py에서:
# down_revision = 'yyy9999'  # teammate's latest로 변경

시나리오 3: 프로덕션 배포 전 확인​

# 1. 현재 프로덕션 버전 확인
# (프로덕션 서버에서)
alembic current
# bbb2222

# 2. 로컬에서 배포할 마이그레이션 확인
alembic history -r bbb2222:head

# 출력:
# bbb2222 -> ccc3333, add user profiles
# ccc3333 -> ddd4444, add comments
# ddd4444 -> eee5555 (head), add tags

# 3. 각 마이그레이션 검토
alembic show ccc3333
alembic show ddd4444
alembic show eee5555

# 4. 문제없으면 배포

📋 히스토리 관리 베스트 프랙티스​

1. 명확한 메시지 작성​

# ❌ 나쁜 예
alembic revision -m "update"
alembic revision -m "fix"
alembic revision -m "changes"

# ✅ 좋은 예
alembic revision -m "add email column to users table"
alembic revision -m "create posts table with foreign key to users"
alembic revision -m "add index on posts.created_at for performance"

2. 논리적 순서 유지​

# ✅ 의존성 순서대로
alembic revision -m "create users table"
alembic revision -m "create posts table"  # users에 의존
alembic revision -m "create comments table"  # posts에 의존

3. 정기적인 히스토리 정리​

# squash: 여러 마이그레이션 합치기 (고급)
# 개발 중에는 작은 단위로, 릴리즈 전에는 합치기

🔍 트러블슈팅​

문제 1: 히스토리가 보이지 않음​

$ alembic history
# (출력 없음)

해결방법:

# versions 디렉토리 확인
ls alembic/versions/

# 파일이 있다면 설정 확인
cat alembic.ini | grep script_location

문제 2: Multiple heads​

$ alembic upgrade head
Error: Multiple heads are present

해결방법:

# heads 확인
alembic heads

# 병합 마이그레이션 생성 (고급편에서 자세히)
alembic merge -m "merge branches" head1 head2

✅ 핵심 정리​

명령어설명
alembic history전체 히스토리
alembic history -v상세 정보 포함
alembic history -i현재 위치 표시
alembic history -r A:B범위 지정
alembic current현재 버전
alembic show REV특정 리비전 정보
alembic heads최신 버전들
