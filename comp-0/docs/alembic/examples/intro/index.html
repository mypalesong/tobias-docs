실전 시나리오 소개

실제 프로젝트에서 마주치는 상황을 시나리오로 배워봅시다!

🎯 학습 방식​

각 시나리오는 다음 구조로 진행됩니다:

  1. 상황 설명: 실제 개발 현장의 문제
  2. 요구사항: 해결해야 할 과제
  3. 해결 과정: 단계별 해결 방법
  4. 베스트 프랙티스: 실무 팁

📚 시나리오 목록​

각 시나리오는 실제 개발 현장에서 마주칠 수 있는 상황을 다룹니다.

🌱 초급 시나리오​

시나리오 1: 신규 프로젝트 시작​

  • 주인공: 지우 (스타트업 신입 개발자)
  • 상황: 온라인 서점 프로젝트에 Alembic 처음 도입
  • 배울 내용:
    • Alembic 초기 설정
    • SQLAlchemy 모델 작성
    • autogenerate로 마이그레이션 생성
    • Git 연동

시나리오 2: 스키마 확장하기​

  • 주인공: 수진 (이커머스 백엔드 개발자)
  • 상황: 제품 리뷰 시스템 추가
  • 배울 내용:
    • 새 테이블 추가
    • 기존 테이블에 컬럼 추가
    • 관계(Relationship) 설정
    • 인덱스 최적화

🚀 중급 시나리오​

시나리오 3: 팀 협업 충돌 해결​

  • 주인공: 민호와 수진 (동료 개발자들)
  • 상황: 동시에 작업한 마이그레이션 충돌
  • 배울 내용:
    • Multiple heads 해결
    • Git rebase와 마이그레이션
    • merge 명령어 사용
    • 팀 워크플로우

시나리오 4: 대량 데이터 변환​

  • 주인공: 유진 (글로벌 서비스 개발자)
  • 상황: 가격 데이터를 KRW에서 USD로 변환 (10만 개 상품)
  • 배울 내용:
    • 안전한 데이터 변환 전략
    • 배치 처리
    • 제로 다운타임 마이그레이션
    • 데이터 검증

💎 고급 시나리오​

시나리오 5: 프로덕션 긴급 수정​

  • 주인공: 서연 (운영 중인 서비스 개발자)
  • 상황: 금요일 퇴근 직전, 프로덕션 장애 발생!
  • 배울 내용:
    • 긴급 상황 대응
    • 핫픽스 배포
    • 롤백 vs 핫픽스 판단
    • 사후 분석

시나리오 6: 레거시 데이터베이스​

  • 주인공: 태희 (레거시 시스템 관리자)
  • 상황: 5년 된 ERP 시스템에 Alembic 도입
  • 배울 내용:
    • 기존 DB에 Alembic 도입
    • sqlacodegen 활용
    • alembic stamp 사용
    • 점진적 마이그레이션

🎯 학습 로드맵​

시나리오 1 → 시나리오 2 → 시나리오 3
  (신규)      (확장)      (협업)
                            ↓
시나리오 6 ← 시나리오 5 ← 시나리오 4
 (레거시)     (긴급)      (데이터)

추천 순서:

  1. 처음이라면: 1 → 2 → 3
  2. 실무자라면: 4 → 5
  3. 레거시 담당자: 6

📊 난이도 및 소요 시간​

시나리오난이도시간핵심 개념
1. 신규 프로젝트⭐30분초기 설정, autogenerate
2. 스키마 확장⭐⭐40분테이블/컬럼 추가, 관계
3. 팀 협업 충돌⭐⭐⭐50분브랜치 병합, rebase
4. 데이터 변환⭐⭐⭐⭐60분배치 처리, 데이터 검증
5. 긴급 수정⭐⭐⭐30분핫픽스, 빠른 대응
6. 레거시 DB⭐⭐⭐⭐90분stamp, 점진적 도입
