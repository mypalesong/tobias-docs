시나리오 6: 태희의 레거시 정복

5년 된 레거시 데이터베이스에 Alembic을 도입하는 태희의 여정입니다.

📌 상황 설명​

태희는 오래된 ERP 시스템을 유지보수하는 개발자입니다. 5년 전에 만들어진 이 시스템은 데이터베이스 스키마 변경을 수동 SQL로 관리해왔습니다. 이제 팀에서 Alembic을 도입하기로 결정했습니다.

문제:

  • 프로덕션 DB에 이미 100개의 테이블
  • 수동 SQL 파일 200개
  • 어떤 SQL이 적용되었는지 불명확
  • 마이그레이션 히스토리 없음

🎯 목표​

  • 기존 스키마를 Alembic으로 관리
  • 현재 상태를 첫 마이그레이션으로 캡처
  • 팀원들이 안전하게 전환
  • 과거 히스토리 보존

💬 대화​

팀장 성민: "태희님, 이제 더 이상 수동 SQL은 안 됩니다. Alembic을 도입해주세요."

태희: "네! 근데 기존 데이터베이스가 있는데... 어떻게 시작하죠?"

시니어 지수: "걱정 마세요. 단계별로 안전하게 마이그레이션할 수 있어요."

3단계: SQLAlchemy 모델 작성​

지수: "먼저 현재 스키마를 SQLAlchemy 모델로 만들어야 해요. sqlacodegen이 도와줄 거예요."

# sqlacodegen 설치
pip install sqlacodegen

# 데이터베이스에서 모델 자동 생성
sqlacodegen postgresql://user:pass@production-db/erp_production > models_generated.py
# models_generated.py (자동 생성됨)
from sqlalchemy import Column, Integer, String, DateTime, ForeignKey, Text, Numeric
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import relationship

Base = declarative_base()

class User(Base):
    __tablename__ = 'users'

    id = Column(Integer, primary_key=True)
    username = Column(String(50), unique=True, nullable=False)
    email = Column(String(100), unique=True, nullable=False)
    created_at = Column(DateTime)
    # ... 50개 필드

class Employee(Base):
    __tablename__ = 'employees'

    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey('users.id'))
    department_id = Column(Integer, ForeignKey('departments.id'))
    position = Column(String(100))
    salary = Column(Numeric(12, 2))
    # ... 30개 필드

4단계: 베이스라인 마이그레이션 생성​

지수: "이제 중요한 부분이에요. 현재 상태를 '첫 마이그레이션'으로 만들어야 해요."

# env.py 설정
# alembic/env.py
from models import Base
target_metadata = Base.metadata

# autogenerate로 초기 마이그레이션 생성
alembic revision --autogenerate -m "baseline: capture existing schema"
# versions/abc123_baseline_capture_existing_schema.py
"""baseline: capture existing schema

Revision ID: abc123
Revises:
Create Date: 2024-01-26 10:00:00
"""

def upgrade():
    # Users 테이블
    op.create_table('users',
        sa.Column('id', sa.Integer(), nullable=False),
        sa.Column('username', sa.String(50), nullable=False),
        # ... 모든 컬럼
    )

    # Employees 테이블
    op.create_table('employees',
        # ... 모든 컬럼
    )

    # ... 100개 테이블 생성 코드

def downgrade():
    # 전체 삭제 (실제로는 사용 안 함)
    op.drop_table('employees')
    op.drop_table('users')
    # ...

태희: "잠깐! 이 마이그레이션을 실행하면 테이블을 만들려고 할 텐데, 이미 존재하잖아요?"

지수: "맞아요! 그래서 'stamp' 명령을 사용해요."

5단계: Stamp - 가짜 적용​

# 프로덕션에는 이미 테이블이 있으므로
# 마이그레이션을 '적용된 것으로 표시'만 함

# 개발 DB에서 먼저 테스트
psql -d erp_dev

# Alembic 버전 테이블 확인
SELECT * FROM alembic_version;
# (비어있음)

# Stamp: "abc123 마이그레이션이 이미 적용되었다"고 표시
alembic stamp head

# 확인
SELECT * FROM alembic_version;
#  version_num
# -------------
#  abc123

태희: "오! 실제로 테이블을 건드리지 않고, 히스토리만 기록하는 거군요!"

6단계: 프로덕션 적용​

# 프로덕션 DB
ssh production-server
cd /var/www/erp

# 코드 배포
git pull origin main

# Stamp (실제 스키마 변경 없음)
alembic stamp abc123

# 확인
alembic current
# abc123 (head)

alembic history
# <base> -> abc123 (head), baseline: capture existing schema

성민: "완벽해요! 이제부터는 모든 변경사항을 Alembic으로 관리할 수 있겠네요."

7단계: 첫 번째 실제 마이그레이션​

# models.py에 새 필드 추가
class User(Base):
    __tablename__ = 'users'
    # ... 기존 필드들
    last_login_at = Column(DateTime)  # 새 필드
    is_active = Column(Boolean, default=True)  # 새 필드
# 마이그레이션 생성
alembic revision --autogenerate -m "add last_login_at and is_active to users"
"""add last_login_at and is_active to users

Revision ID: def456
Revises: abc123  ← 베이스라인을 부모로!
"""

def upgrade():
    op.add_column('users', sa.Column('last_login_at', sa.DateTime()))
    op.add_column('users', sa.Column('is_active', sa.Boolean()))

def downgrade():
    op.drop_column('users', 'is_active')
    op.drop_column('users', 'last_login_at')
# 개발 환경 테스트
alembic upgrade head
# INFO  Running upgrade abc123 -> def456

# 프로덕션 배포
alembic upgrade head

태희: "드디어! 깔끔한 마이그레이션 시스템이 생겼어요!"

시나리오 A: 일부만 마이그레이션​

문제: 100개 테이블 중 핵심 20개만 먼저 Alembic으로 관리하고 싶어요.

# alembic/env.py
def include_object(object, name, type_, reflected, compare_to):
    # 관리할 테이블 목록
    managed_tables = [
        'users', 'employees', 'departments', 'projects',
        'tasks', 'time_entries', # ... 20개
    ]

    # 테이블이면 목록에 있는지 확인
    if type_ == "table":
        return name in managed_tables

    # 다른 객체는 무시
    return False

def run_migrations_online():
    context.configure(
        # ...
        include_object=include_object,
    )

시나리오 B: 히스토리 재구성​

문제: 과거 수동 SQL 파일을 마이그레이션으로 변환하고 싶어요.

# 과거 SQL 파일 분석
ls migrations_manual/*.sql | sort

# 각 SQL을 마이그레이션으로 변환
# versions/001_2020_01_15_initial_users.py
"""initial users table (historical)

Revision ID: 001
Revises:
Create Date: 2020-01-15 10:00:00
"""

def upgrade():
    # 원래 SQL 파일 내용을 op.execute로
    op.execute('''
        CREATE TABLE users (
            id SERIAL PRIMARY KEY,
            username VARCHAR(50) UNIQUE NOT NULL,
            ...
        )
    ''')

def downgrade():
    op.drop_table('users')

# 주의: 이 마이그레이션들은 실제 적용하지 않음!
# 히스토리 기록용
# 과거 마이그레이션을 "적용된 것으로" 표시
alembic stamp 001
alembic stamp 002
# ...
alembic stamp abc123  # 현재 상태

from sqlalchemy import create_engine, text
import sys

def check_status(db_url, env_name):
    engine = create_engine(db_url)

    with engine.connect() as conn:
        # Alembic 버전 확인
        result = conn.execute(text('SELECT version_num FROM alembic_version'))
        current = result.scalar()

        print(f"{env_name}: {current}")

        # 대기 중인 마이그레이션 확인
        # (실제로는 alembic history와 비교)

if __name__ == '__main__':
    check_status('postgresql://localhost/erp_dev', 'Development')
    check_status('postgresql://staging/erp_staging', 'Staging')
    check_status('postgresql://prod/erp_prod', 'Production')

기존 데이터베이스에 Alembic을 도입해보세요:

정답 보기
# 1. 레거시 DB 생성
psql -d postgres -c "CREATE DATABASE legacy_db"
psql -d legacy_db << EOF
CREATE TABLE users (id SERIAL PRIMARY KEY, name VARCHAR(50));
CREATE TABLE posts (id SERIAL PRIMARY KEY, user_id INT, title VARCHAR(200));
CREATE TABLE comments (id SERIAL PRIMARY KEY, post_id INT, content TEXT);
EOF

# 2. 모델 생성
sqlacodegen postgresql://localhost/legacy_db > models.py

# 3. Alembic 초기화
alembic init alembic
# env.py에 target_metadata 설정

# 4. 베이스라인 생성
alembic revision --autogenerate -m "baseline"

# 5. Stamp
alembic stamp head

# 6. 새 필드 추가
# models.py에서 User에 email 필드 추가

alembic upgrade head
