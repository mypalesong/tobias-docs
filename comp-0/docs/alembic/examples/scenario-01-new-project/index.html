시나리오 1: 지우의 첫 프로젝트

스타트업 '코드스톰'에 입사한 지우가 새 프로젝트에 Alembic을 도입하는 이야기입니다.

📌 상황 설명​

지우는 오늘 스타트업 '코드스톰'에 백엔드 개발자로 입사했습니다. 첫 프로젝트는 온라인 서점 플랫폼입니다. 팀장 민수는 지우에게 데이터베이스 마이그레이션 도구로 Alembic을 사용하라고 안내합니다.

🎯 목표​

  • Alembic 프로젝트 초기 설정
  • 초기 스키마 설계 및 마이그레이션 생성
  • 개발 환경에서 테스트
  • Git 저장소에 커밋

💬 대화​

민수 (팀장): "지우님, 환영합니다! 오늘은 프로젝트 셋업을 도와드릴게요."

지우: "감사합니다! 데이터베이스는 어떤 걸 사용하나요?"

민수: "PostgreSQL을 사용하고, 마이그레이션은 Alembic으로 관리합니다. SQLAlchemy도 함께 쓰고 있어요."

지우: "Alembic은 처음인데 어렵지 않을까요?"

민수: "걱정 마세요. 단계별로 알려드릴게요. 먼저 가상환경부터 만들어봅시다."

⚡ 실습​

1단계: 프로젝트 초기화​

민수: "먼저 프로젝트 디렉토리를 만들고 가상환경을 설정해봅시다."

# 프로젝트 디렉토리 생성
mkdir bookstore
cd bookstore

# Git 초기화
git init
git branch -M main

# 가상환경 생성 및 활성화
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 필요한 패키지 설치
pip install sqlalchemy alembic psycopg2-binary python-dotenv

지우: "설치 완료했어요!"

2단계: 프로젝트 구조 만들기​

민수: "좋아요. 이제 프로젝트 구조를 만들어봅시다."

mkdir bookstore
touch bookstore/__init__.py
touch bookstore/models.py
touch bookstore/database.py
# bookstore/database.py
from sqlalchemy import create_engine
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker
import os
from dotenv import load_dotenv

load_dotenv()

DATABASE_URL = os.getenv('DATABASE_URL', 'postgresql://localhost/bookstore_dev')

engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()

3단계: SQLAlchemy 모델 작성​

민수: "이제 핵심 모델들을 만들어볼까요? 책, 저자, 사용자, 주문 이렇게 4개 테이블이 필요해요."

지우: "네, 한번 작성해볼게요!"

# bookstore/models.py
from sqlalchemy import Column, Integer, String, Text, Numeric, DateTime, ForeignKey, Boolean
from sqlalchemy.orm import relationship
from datetime import datetime
from .database import Base

class User(Base):
    __tablename__ = 'users'

    id = Column(Integer, primary_key=True, index=True)
    email = Column(String(100), unique=True, nullable=False, index=True)
    username = Column(String(50), unique=True, nullable=False)
    password_hash = Column(String(255), nullable=False)
    is_active = Column(Boolean, default=True)
    created_at = Column(DateTime, default=datetime.utcnow)

    orders = relationship('Order', back_populates='user')

class Author(Base):
    __tablename__ = 'authors'

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String(100), nullable=False)
    bio = Column(Text)
    birth_year = Column(Integer)
    created_at = Column(DateTime, default=datetime.utcnow)

    books = relationship('Book', back_populates='author')

class Book(Base):
    __tablename__ = 'books'

    id = Column(Integer, primary_key=True, index=True)
    title = Column(String(200), nullable=False, index=True)
    isbn = Column(String(13), unique=True, nullable=False)
    description = Column(Text)
    price = Column(Numeric(10, 2), nullable=False)
    stock = Column(Integer, default=0)
    author_id = Column(Integer, ForeignKey('authors.id'), nullable=False)
    published_year = Column(Integer)
    created_at = Column(DateTime, default=datetime.utcnow)

    author = relationship('Author', back_populates='books')

class Order(Base):
    __tablename__ = 'orders'

    id = Column(Integer, primary_key=True, index=True)
    user_id = Column(Integer, ForeignKey('users.id'), nullable=False)
    book_id = Column(Integer, ForeignKey('books.id'), nullable=False)
    quantity = Column(Integer, nullable=False)
    total_price = Column(Numeric(12, 2), nullable=False)
    status = Column(String(20), default='pending')
    order_date = Column(DateTime, default=datetime.utcnow)

    user = relationship('User', back_populates='orders')

민수: "훌륭해요! 모델 구조가 깔끔하네요. 관계(relationship)도 잘 설정했어요."

5단계: 환경 설정​

민수: "환경변수 파일을 만들고 설정을 수정해야 해요."

# .env 파일 생성
cat > .env << EOF
DATABASE_URL=postgresql://bookstore_user:password@localhost/bookstore_dev

# alembic/env.py 수정
import os
import sys
from logging.config import fileConfig
from sqlalchemy import engine_from_config, pool
from alembic import context
from dotenv import load_dotenv

# 프로젝트 루트를 Python 경로에 추가
sys.path.append(os.path.dirname(os.path.dirname(__file__)))

# .env 로드
load_dotenv()

# Alembic Config 객체
config = context.config

# 환경변수에서 데이터베이스 URL 가져오기
config.set_main_option('sqlalchemy.url', os.getenv('DATABASE_URL'))

# 로깅 설정
if config.config_file_name is not None:
    fileConfig(config.config_file_name)

# 모델의 MetaData 가져오기
from bookstore.database import Base
from bookstore.models import User, Author, Book, Order

target_metadata = Base.metadata

지우: "설정이 좀 복잡하네요..."

6단계: 초기 마이그레이션 생성​

민수: "이제 재미있는 부분이에요. autogenerate로 마이그레이션을 자동 생성해봅시다!"

# 마이그레이션 자동 생성
alembic revision --autogenerate -m "create initial schema"

지우: "와! 파일이 자동으로 생성되었어요!"

# 생성된 파일 확인
cat alembic/versions/xxxxx_create_initial_schema.py

생성된 내용:

민수: "완벽해요! 보시다시피 테이블 생성 순서도 자동으로 올바르게 설정되었어요. 외래 키 의존성을 Alembic이 자동으로 파악했죠."

✅ 검증​

다음 항목들을 확인해보세요:

# 1. 현재 마이그레이션 버전 확인
alembic current
# 2a3b4c5d6e7f (head)

# 2. 히스토리 확인
alembic history
# <base> -> 2a3b4c5d6e7f (head), create initial schema

# 3. 다운그레이드 테스트
alembic downgrade base
alembic current
# (비어있음)

# 4. 다시 업그레이드
alembic upgrade head
alembic current
# 2a3b4c5d6e7f (head)

import os
from dotenv import load_dotenv

load_dotenv()

class Config:
    SQLALCHEMY_DATABASE_URI = os.getenv('DATABASE_URL')
    SQLALCHEMY_TRACK_MODIFICATIONS = False

class DevelopmentConfig(Config):
    DEBUG = True

class ProductionConfig(Config):
    DEBUG = False

config = {
    'development': DevelopmentConfig,
    'production': ProductionConfig,
    'default': DevelopmentConfig
}
