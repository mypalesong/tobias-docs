실전 시나리오 소개

실제 프로젝트에서 RabbitMQ를 어떻게 활용하는지 배워봅시다!

🎯 학습 방식​

각 시나리오는 실제 개발 현장에서 마주치는 문제를 해결합니다:

  1. 상황 설명: 실제 비즈니스 요구사항
  2. 아키텍처: 시스템 구조 설계
  3. 구현: Python 코드로 구현
  4. 테스트: pytest로 검증
  5. 베스트 프랙티스: 실무 팁

시나리오 1: 주문 처리 시스템​

  • 상황: 쇼핑몰 주문을 비동기로 처리
  • 패턴: Work Queue
  • 기술:
    • 주문 큐로 요청 분산
    • 여러 워커가 병렬 처리

시나리오 2: 알림 시스템​

  • 상황: 이메일, SMS, 푸시 알림 발송

  • 기술:
    • Fanout Exchange
    • 여러 알림 채널
    • 메시지 포맷팅
  • 난이도: ⭐⭐

시나리오 3: 이벤트 기반 아키텍처​

  • 상황: 서비스 간 느슨한 결합
  • 패턴: Topic Exchange
  • 기술:
    • 도메인 이벤트
    • 이벤트 소싱
    • CQRS 패턴
  • 난이도: ⭐⭐⭐⭐

시나리오 4: 마이크로서비스 통신​

  • 상황: 서비스 간 요청/응답
  • 패턴: RPC
  • 기술:
    • Correlation ID
    • Reply-To Queue
    • 타임아웃 처리
  • 난이도: ⭐⭐⭐

🛠️ 공통 유틸리티​

모든 시나리오에서 사용할 공통 코드:

# common/rabbitmq.py
import pika
import json
from typing import Optional, Dict, Any
from contextlib import contextmanager

class RabbitMQClient:
    """RabbitMQ 클라이언트 헬퍼"""

    def __init__(self, host='localhost', port=5672,
                 username='guest', password='guest'):
        self.host = host
        self.port = port
        self.username = username
        self.password = password

    @contextmanager
    def get_connection(self):
        """연결 컨텍스트 매니저"""
        credentials = pika.PlainCredentials(
            self.username, self.password
        )
        parameters = pika.ConnectionParameters(
            host=self.host,
            port=self.port,
            credentials=credentials,
            heartbeat=600
        )
        connection = pika.BlockingConnection(parameters)
        try:
            yield connection
        finally:
            connection.close()

    @contextmanager
    def get_channel(self):
        """채널 컨텍스트 매니저"""
        with self.get_connection() as connection:
            channel = connection.channel()
            yield channel

    def publish_json(self, exchange: str, routing_key: str,
                     data: Dict[str, Any], **kwargs):
        """JSON 메시지 발행"""
        with self.get_channel() as channel:
            message = json.dumps(data)
            channel.basic_publish(
                exchange=exchange,
                routing_key=routing_key,
                body=message,
                properties=pika.BasicProperties(
                    content_type='application/json',
                    **kwargs
                )
            )

    def consume(self, queue: str, callback, auto_ack=False):
        """메시지 소비"""
        with self.get_connection() as connection:
            channel = connection.channel()
            channel.queue_declare(queue=queue, durable=True)
            channel.basic_consume(
                queue=queue,
                on_message_callback=callback,
                auto_ack=auto_ack
            )
            print(f' [*] Waiting for messages on {queue}...')
            channel.start_consuming()
# common/logger.py
import logging
from datetime import datetime

def setup_logger(name: str) -> logging.Logger:
    """로거 설정"""
    logger = logging.getLogger(name)
    logger.setLevel(logging.INFO)

    handler = logging.StreamHandler()
    formatter = logging.Formatter(
        '[%(asctime)s] %(name)s - %(levelname)s - %(message)s'
    )
    handler.setFormatter(formatter)
    logger.addHandler(handler)

    return logger

🧪 테스트 환경​

모든 시나리오는 다음 환경에서 테스트됩니다:

# docker-compose.test.yml
version: '3.8'

services:
  rabbitmq:
    image: rabbitmq:3-management
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_DEFAULT_USER: test_user
      RABBITMQ_DEFAULT_PASS: test_pass
# 테스트 환경 시작
docker-compose -f docker-compose.test.yml up -d

# 테스트 실행
pytest scenarios/ -v

# 테스트 환경 종료
docker-compose -f docker-compose.test.yml down
